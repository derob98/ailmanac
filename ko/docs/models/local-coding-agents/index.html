로컬 코딩 에이전트 (그리고 Claude와의 조합 방법)

중급

로컬 코딩 에이전트는 당신 자신의 머신에서 실행되며, 레포지토리를 읽고, 당신을 위해 파일을 편집하는 프로그램입니다. 당신이 평범한 언어로 변경 사항을 설명하면, 에이전트가 이를 계획하고, 여러 파일에 걸쳐 코드를 작성하고, 테스트를 실행하고, 실패를 읽고, 다시 시도합니다 — 단일한 답변이 아니라 하나의 루프입니다. 브라우저 탭의 채팅 어시스턴트는 붙여넣을 스니펫을 건네주지만, 코딩 에이전트는 그 붙여넣기, 실행, 수정을 당신의 레포지토리 안에서 스스로 수행합니다.

이 페이지를 읽을 가치가 있게 만드는 핵심 통찰: 이 도구들 거의 전부가 모델 독립적(model-agnostic)이다. 에이전트(당신의 파일을 건드리는 루프)는 모델(무엇을 작성할지 결정하는 두뇌)과 분리되어 있습니다. 따라서 동일한 도구를 최고의 코드 품질을 위해 Claude로 향하게 하거나, 코드가 절대 노트북을 떠나지 않도록 Ollama를 통한 로컬 오픈 웨이트 모델로 향하게 할 수 있습니다. 워크플로우에 어떻게 들어맞는지에 따라 에이전트를 고르고, 품질 대 프라이버시의 트레이드오프에 따라 모델을 고릅니다 — 각각 독립적으로 말이죠.

What you'll learn
  • 코딩 에이전트의 원형을 이해하기: 레포지토리를 읽고, 계획하고, 파일을 편집하고, 테스트를 실행하고, 반복하는 에이전트 루프
  • 실제 오픈 도구들 — Aider, Cline, Continue, OpenHands — 과 Anthropic 자체의 Claude Code가 어디에 들어맞는지 알기
  • '모델 독립적'이 왜 중요한지 파악하기: 하나의 에이전트를 품질을 위한 Claude 또는 프라이버시를 위한 로컬 모델로 향하게 하기
  • 프라이버시 측면 이해하기 — 로컬 모델은 당신의 코드가 절대 머신을 떠나지 않음을 의미한다
  • 두 축을 따라 선택하기: IDE 통합형 vs CLI vs 자율형, 그리고 Claude 구동 vs 완전 로컬

원형: 채팅 상자가 아니라 에이전트 루프​

이 페이지의 모든 도구는 동일한 루프의 변형입니다. 일단 이 루프를 보고 나면, 각 도구는 그것을 둘러싼 다른 형태일 뿐입니다:

  1. 읽기(Read) — 에이전트가 컨텍스트를 수집합니다: 당신이 지목한 파일, 레포지토리 구조, 때로는 테스트 출력이나 오류 로그.
  2. 계획(Plan) — 무엇을 어떤 파일에서 바꿀지 결정하며, 무언가를 건드리기 전에 종종 계획을 제안합니다.
  3. 편집(Edit) — 디스크의 실제 파일에 변경 사항을 작성합니다(보통 당신이 승인하는 diff로 표시됨).
  4. 실행(Run) — 명령을 실행합니다: 당신의 테스트 스위트, 빌드, 린터, 스크립트.
  5. 관찰과 반복(Observe and iterate) — 무슨 일이 일어났는지 읽고(테스트가 통과했나? 스택 트레이스인가?), 작업이 완료되거나 통제권을 당신에게 넘길 때까지 2단계로 되돌아갑니다.

이는 에이전트형 도구 사용(tool use)과 동일한 형태입니다: 모델에 도구(파일 읽기, 파일 편집, 명령 실행)가 주어지고, 언제 그것들을 호출할지 결정합니다. 도구들 사이의 차이는 대부분 루프가 얼마나 많은 자율성을 갖는지와 어디에 사는지 — 당신의 터미널, 에디터, 또는 독립적으로 실행되는 샌드박스 환경 — 에 있습니다.

그 루프 중앙의 모델은 교체 가능합니다. 이것이 단일 에이전트가 매우 다른 두 부류의 사용자를 섬길 수 있는 이유의 전부입니다: 가능한 한 가장 강력한 코드를 원하는 사람(Claude로 향하게)과, 법적으로 코드를 네트워크 밖으로 내보낼 수 없는 사람(로컬 모델로 향하게).

왜 모델 독립성이 핵심 기능인가​

에이전트와 모델은 별개의 조각입니다. 에이전트는 루프를 소유합니다 — 파일 읽기, diff 적용, 테스트 실행. 모델은 판단을 소유합니다 — 무엇을, 왜 작성하고, 언제 멈출지. 이 둘을 분리하면 독립적인 두 개의 다이얼이 생깁니다:

  • 품질 다이얼 → Claude. 어려운 추론, 다중 파일 리팩터링, 장문맥 작업에 대해서는 Claude 같은 프런티어 모델이 눈에 띄게 더 나은 편집을 만들어내고 실패로부터 더 안정적으로 복구합니다. 이 도구들 대부분이 Claude를 권장 또는 기본 선택지로 문서화하고 있습니다. (어느 Claude 등급을 고를지는 그 자체로 별개의 결정입니다 — 모델 선택하기를 참고하세요.)
  • 프라이버시 다이얼 → 로컬 모델. 동일한 에이전트를 Ollama 하에서 실행되는 오픈 웨이트 모델로 향하게 하면 당신의 코드, 프롬프트, diff가 절대 머신을 떠나지 않습니다 — API 호출도, 제3자도, 통신선 위에 아무것도 없습니다. 이는 규제되거나, 에어갭이 걸려 있거나, 기밀인 코드베이스에서 결정적인 요인입니다. 솔직한 트레이드오프: 노트북에서 실행될 만큼 작은 모델은 가장 어려운 작업에서 프런티어 모델과 같은 급이 아닙니다. 당신은 완전한 데이터 통제를 위해 일부 역량을 맞바꾸는 것입니다.
What you'll learn
  • 모델 독립적이라는 것이 모든 모델이 똑같이 뛰어나다는 뜻은 아닙니다. 루프는 동일하게 돌아가지만, 약한 모델은 더 나쁜 편집을 하고, 테스트 실패를 잘못 읽고, 비생산적으로 루프를 돕니다. 프라이버시와 역량은 실제로 트레이드오프 관계에 있습니다 — 한 번 영원히가 아니라 작업마다 선택하세요.

실제 도구들 (검증됨)​

알아둘 가치가 있는 다섯 가지 도구. 처음 넷은 오픈 소스이며 모델 독립적입니다; 다섯 번째는 Anthropic 자체의 에이전트로, Claude 네이티브 경험이 어디에 위치하는지 볼 수 있도록 포함했습니다.

Aider — CLI 페어 프로그래머​

Aider는 당신의 터미널에서 실행되며 git을 중심으로 만들어졌습니다: 레포지토리의 파일을 편집하고 각 변경을 합당한 메시지와 함께 자동으로 커밋하므로, 모든 AI 편집은 당신이 검토하거나, 취소하거나, 되돌릴 수 있는 diff입니다. 모델 독립적입니다 — Claude 같은 프런티어 모델과 가장 잘 작동하며, Ollama를 통한 로컬 모델(aider --model ollama_chat/<model>)에도 연결됩니다. 터미널에서 살고 빈틈없는 git 위생을 원한다면, Aider가 표준적인 CLI 선택지입니다.

Cline — IDE 안의 에이전트​

Cline은 당신의 에디터(VS Code와 JetBrains)에 사는 오픈 소스 코딩 에이전트로, 터미널/CLI 표면도 제공합니다. 파일을 생성하고, 명령을 실행하며, 인간이 개입하는(human-in-the-loop) 승인으로 도구를 사용합니다 — 각 단계를 보고 승인합니다. 모델 독립적이며 Claude뿐 아니라 Ollama나 LM Studio를 통한 로컬 모델도 명시적으로 지원합니다. 에디터를 떠나지 않으면서 다소 자율적인 에이전트를 원할 때 Cline을 택하세요.

Roo Code는 커스텀 "모드"(예: 계획용 Architect 모드, 편집용 Coder 모드)를 추가한, Cline의 잘 알려진 포크입니다. 이것도 모델 독립적이었습니다. 아래에서 그 상태를 참고하세요 — 이제는 한때 그랬던 것처럼 안전한 기본값이 아닙니다.

최종 검증: 2026-06-28.

Roo Code의 상태는 변동성이 큽니다. 검증 시점 기준으로, Roo Code 팀은 VS Code 확장의 축소와 클라우드 에이전트로의 전환을 발표했습니다 — 따라서 이전 소개 글들이 설명하는, 유지보수되는 로컬 우선 기본값이 더 이상 아닙니다. 특별히 Cline 스타일의 포크를 원한다면, 채택하기 전에 오늘 무엇이 활발히 유지보수되는지 확인하세요. Cline 자체(상류 프로젝트)는 여전히 더 안전한 로컬 우선 선택지로 남아 있습니다. 현재 상태를 github.com/cline/cline에서 확인하세요.

다음에서 확인하세요: 공식 출처.

Continue — 오픈 IDE 어시스턴트​

Continue는 채팅, 편집, 자동완성, 에이전트 모드를 아우르는 VS Code와 JetBrains용(그리고 CLI) 오픈 소스 어시스턴트입니다. 철저히 모델 독립적입니다: 문서에는 Anthropic (Claude), 로컬 모델용 Ollama, 그리고 여러 다른 제공자가 나열되어 있으며, 서로 다른 모델을 서로 다른 역할(채팅, 편집, 자동완성)에 할당할 수 있습니다. 어느 모델이 무엇을 하는지 정확히 배선하는, 하나의 설정 가능한 어시스턴트를 원한다면 강력한 선택입니다.

최종 검증: 2026-06-28.

Continue의 유지보수 상태는 확인해 볼 가치가 있습니다. 검증 시점 무렵 이 프로젝트는 2.0 릴리스를 내놓았고 활발한 유지보수에 관한 신호들이 변하고 있었습니다. 이 도구와 그것의 Claude/Ollama 지원은 실재하지만, 팀 표준으로 삼기 전에 현재 릴리스와 유지보수 상태를 github.com/continuedev/continue에서 확인하세요.

다음에서 확인하세요: 공식 출처.

OpenHands — 자율형 에이전트​

OpenHands(이전 명칭 OpenDevin)는 넷 중 가장 자율적입니다: 샌드박스 환경에서 작동하고, 작업(예: GitHub 이슈)을 분해하고, 단계별 승인을 덜 받으면서 그것을 처음부터 끝까지 수행하는 AI 소프트웨어 개발 에이전트 플랫폼입니다. 모델 독립적입니다 — "어떤 LLM과도 사용," Claude 포함 — 그리고 자체 호스팅으로 실행할 수 있습니다. 각 편집을 승인하기보다 작업 전체를 넘겨주고 다시 확인하고 싶을 때 OpenHands를 택하세요.

최종 검증: 2026-06-28.

OpenHands는 OpenDevin에서 이름이 바뀌었고, GitHub 조직은 All-Hands-AI에서 OpenHands로 개명되었습니다(예전 All-Hands-AI/OpenHands 경로는 여전히 리디렉션됩니다). 문서는 역사적으로 docs.all-hands.dev에 있었습니다. 현재의 정식 레포지토리와 문서 URL을 github.com/OpenHands/OpenHands에서 확인하세요.

다음에서 확인하세요: 공식 출처.

Claude Code — Anthropic 자체의 에이전트​

Claude Code는 Anthropic의 공식 에이전트형 코딩 도구입니다: 당신의 코드베이스를 읽고, 파일을 편집하고, 명령과 테스트를 실행하며, 당신의 터미널, IDE, 데스크톱 앱, 웹에 걸쳐 작동합니다. 위의 넷과 달리 "아무 모델이나 향하게" 하는 것이 아닙니다 — Claude를 중심으로 만들어지고 그에 맞춰 튜닝되었으며, 바로 그 때문에 루프가 더 촘촘하게 느껴지고 어려운 작업에서 편집이 더 자주 성공합니다. 최대한의 모델 교체 가능성보다 최고의 Claude 구동 코딩 경험이 우선순위라면, 이것이 네이티브 경로입니다. 자세한 내용은 Claude Code란 무엇인가에 있습니다.

선택하는 방법​

두 개의 축이 거의 모든 것을 결정합니다:

축 1 — 에이전트가 사는 곳:

  • CLI / 터미널 (Aider, Claude Code) — 스크립트화 가능, git 친화적, 다른 도구로 파이핑 가능. 셸에서 산다면 최선.
  • IDE 통합형 (Cline, Continue, Claude Code의 에디터 표면) — diff와 승인이 에디터 안에서, 당신의 코드가 바로 거기 있음. 일상적인 편집에 최선.
  • 자율형 / 샌드박스형 (OpenHands) — 작업 전체를 넘겨주고 결과를 검토. 배치 작업이나 손 떼고 하는 작업에 최선.

축 2 — Claude 구동 vs 완전 로컬:

  • Claude 구동 — 최고 품질, 특히 어려운 다중 파일 또는 장문맥 작업에서. 당신의 코드는 모델 API로 전송됩니다. 품질이 가장 중요하고 코드를 제공자에게 보낼 수 있을 때 선택하세요.
  • 완전 로컬 (Ollama) — 코드가 절대 머신을 떠나지 않음; 실행 무료; 오프라인 작동. 프라이버시, 규제, 또는 에어갭이 타협 불가일 때 선택하세요 — 노트북 크기의 모델은 가장 어려운 작업에서 더 약하다는 점을 받아들이면서.

지속 가능한 방식: 워크플로우에 맞는 표면으로 에이전트를 고르고, 그다음 작업마다 모델을 골라라. 기밀 레포지토리에는 로컬 모델로 IDE 에이전트를 돌리고, 그 밖의 모든 것에는 동일한 에이전트를 Claude로 돌리는 것을 막을 것은 아무것도 없습니다.

코딩 에이전트 고르고 설정하기​

Guided walkthrough1 of 5
  1. CLI(Aider, Claude Code), IDE 통합형(Cline, Continue), 또는 자율형(OpenHands). 이미 일하는 곳에 맞추세요 — 터미널 vs 에디터 vs 넘겨주고-검토하기.
  2. 최고 품질을 위한 Claude(코드가 API로 전송됨) 또는 완전한 프라이버시를 위한 로컬 Ollama 모델(코드가 절대 머신을 떠나지 않음). 이는 Claude 네이티브인 Claude Code를 제외한 모든 도구에서 에이전트와 독립적입니다.
  3. 도구의 문서를 따르세요. Aider는 pip으로 설치됩니다; Cline과 Continue는 VS Code나 JetBrains 마켓플레이스에서 설치됩니다; OpenHands는 자체 호스팅됩니다; Claude Code는 공식 설치 프로그램으로 설치됩니다.
  4. Claude의 경우, Anthropic API 키를 설정하세요(또는 Claude Code라면 로그인). 로컬의 경우, 먼저 Ollama를 설치하고 모델을 pull 하세요 — Ollama로 로컬에서 모델 실행하기를 참고 — 그다음 에이전트를 그것으로 향하게 하세요.
  5. 경계가 있는 하나의 작업을 실행하세요(실패하는 테스트 고치기, 작은 함수 추가하기). 승인하기 전에 모든 diff를 검토하세요. 당신의 레포지토리에서 루프가 어떻게 행동하는지 본 뒤에야 자율성을 더 신뢰하세요.

Aider를 로컬 모델로 향하게 한 구체적인 첫 실행(비공개, 무료, 오프라인) — 먼저 Ollama로 모델을 pull 한 다음:

로컬 Ollama 모델로 Aider 실행하기 (코드가 머신에 머무름)

aider --model ollama_chat/qwen3

같은 도구를 대신 Claude로 향하게 하려면(최고 품질, 코드가 API로 전송됨), Anthropic 키를 설정하고 Claude 모델을 선택합니다 — 에이전트 루프는 동일하고, 두뇌만 바뀝니다:

export ANTHROPIC_API_KEY=sk-ant-...
aider --model sonnet

그 교체 — 플래그 하나, 완전히 다른 두 개의 프라이버시/품질 프로필, 같은 에이전트 — 가 이 페이지의 요점 전부입니다.

Key takeaways
  • 코딩 에이전트는 당신의 머신에서 루프를 돌린다 — 레포지토리 읽기, 계획, 파일 편집, 테스트 실행, 반복 — 붙여넣을 스니펫을 건네주는 대신에.
  • 이 도구들 거의 전부가 모델 독립적이다: 에이전트와 모델이 분리되어 있어서, 루프를 바꾸지 않고 두뇌를 교체한다.
  • 두 개의 다이얼: 최고의 코드 품질을 위한 Claude, 또는 완전한 프라이버시를 위한 Ollama를 통한 로컬 오픈 웨이트 모델 — 당신의 코드가 절대 머신을 떠나지 않는다.
  • Aider = CLI/git; Cline = IDE 안의 에이전트; Continue = 설정 가능한 IDE 어시스턴트; OpenHands = 자율형/샌드박스형; Claude Code = Anthropic 자체의 Claude 네이티브 에이전트.
  • 두 축을 따라 선택하라 — 에이전트가 사는 곳(CLI vs IDE vs 자율형)과 Claude 구동 vs 완전 로컬 — 그리고 한 번 영원히가 아니라 작업마다 모델을 골라라.
