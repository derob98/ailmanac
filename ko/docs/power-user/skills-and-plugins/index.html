가장 유용한 Skills와 Plugins

중급
What you'll learn
  • 스킬과 플러그인의 차이, 그리고 각각이 어떻게 설치되는지 알기
  • 공식, 커뮤니티, 데모 마켓플레이스를 추가하고 거기서 플러그인 설치하기
  • 소스 관리, 보안, LSP, git 워크플로에 맞는 올바른 공식 Anthropic 플러그인 고르기
  • 해야 할 일(job-to-be-done) 기준으로 스킬 큐레이션하기: 개발, 글쓰기, 리서치, 데이터, 운영
  • 유명한 커뮤니티 목록에서 더 많은 것을 찾되 신호가 약한 포크에 파묻히지 않기

Claude Code 실력을 가장 빠르게 끌어올리는 방법은 모든 것을 직접 손으로 쓰는 것을 멈추고 다른 사람들이 이미 완성해 둔 역량을 설치하는 것입니다. 스킬은 Claude에게 필요할 때 꺼내 쓰는 전문성을 주고, 플러그인은 스킬, 에이전트, 커맨드, 훅, MCP 서버를 하나의 버전 관리되는 설치 가능한 단위로 묶습니다. 이 페이지는 설치 준비가 끝난 큐레이션 목록입니다 — 각 항목이 무엇을 하는지, 언제 꺼내 쓸지, 그리고 추가하는 정확한 명령어까지 담았습니다.

작동 원리가 아직 헷갈린다면, 먼저 Skills와 Plugins & Marketplaces를 읽어 보세요 — 이 페이지는 여러분이 SKILL.md가 무엇인지 안다고 가정합니다.

최종 검증: 2026-06-24.

마켓플레이스 이름, 공식 플러그인 카탈로그, 커뮤니티 리포지토리의 스타 수는 자주 바뀝니다. 플러그인은 여러분의 권한으로 코드를 실행합니다 — 신뢰하는 출처에서만 설치하고, 의존하기 전에 공식 문서에서 현재 명령어를 확인하세요.

다음에서 확인하세요: 공식 출처.

스킬 vs 플러그인: 20초 모델​

스킬은 SKILL.md(frontmatter description + 지침)를 담은 폴더입니다. Claude는 관련이 있을 때만 로드하거나, 여러분이 /skill-name으로 호출합니다. 스킬은 ~/.claude/skills/(개인), 프로젝트의 .claude/skills/, 또는 플러그인 내부에 위치합니다. 공식 문서에 따르면 커스텀 슬래시 커맨드는 스킬로 통합되었습니다 — .claude/commands/deploy.md와 .claude/skills/deploy/SKILL.md는 둘 다 /deploy를 만듭니다.

플러그인은 스킬, 에이전트, 훅, MCP 서버, LSP 서버를 함께 담아 배포할 수 있는 번들이며, .claude-plugin/plugin.json 매니페스트를 가집니다. 플러그인은 마켓플레이스에서 얻습니다 — 한 번 추가한 뒤 거기서 설치하는 카탈로그입니다. 플러그인 스킬은 네임스페이스가 붙습니다. 예: /commit-commands:commit.

Enter 또는 스페이스 키를 눌러 카드를 뒤집습니다. 좌우 화살표 키로 카드를 이동할 수 있습니다.용어가 표시되었습니다.

마켓플레이스 추가, 플러그인 설치​

공식 Anthropic 마켓플레이스(claude-plugins-official)는 자동으로 사용할 수 있습니다. 수동으로 추가할 만한 것이 둘 더 있습니다: 큐레이션된 커뮤니티 마켓플레이스와 예제 플러그인의 데모 마켓플레이스입니다.

Guided walkthrough1 of 4
  1. /plugin marketplace add anthropics/claude-plugins-community 와 /plugin marketplace add anthropics/claude-code 를 실행하세요. 마켓플레이스를 추가하면 카탈로그만 등록될 뿐 — 아직 아무것도 설치되지 않습니다.
  2. /plugin 을 실행해 관리자를 엽니다(탭: Discover, Installed, Marketplaces, Errors). 상세 창은 컨텍스트 비용 추정치와 모든 커맨드, 스킬, 에이전트, 훅, 서버의 'Will install' 목록을 보여줍니다 — 설치하기 전에 검토하세요.
  3. CLI에서 /plugin install <name>@<marketplace> 로 설치하거나, UI에서 범위(User / Project / Local)를 고르세요. Project 범위는 .claude/settings.json 을 통해 리포지토리의 모두와 플러그인을 공유합니다.
  4. /reload-plugins 를 실행하면 재시작 없이 세션 중간에 새 플러그인을 인식합니다. 그런 다음 네임스페이스가 붙은 스킬을 호출하세요. 예: /commit-commands:commit.

Bootstrap your marketplaces (run inside Claude Code)

/plugin marketplace add anthropics/claude-plugins-community
/plugin marketplace add anthropics/claude-code
/plugin
Watch out
  • 플러그인과 마켓플레이스는 고도로 신뢰되는 구성 요소입니다 — 여러분의 권한으로 임의의 코드를 실행할 수 있습니다. 신뢰하는 출처에서만 마켓플레이스를 추가하고 플러그인을 설치하며, 먼저 'Will install' 목록을 검토하세요.
  • 플러그인 스킬은 /reload-plugins(또는 재시작) 후에만 나타납니다. 그래도 스킬이 보이지 않으면 캐시를 지우세요: rm -rf ~/.claude/plugins/cache, 그런 다음 재설치하세요.

가장 먼저 설치할 만한 공식 플러그인​

이것들은 Anthropic의 공식 마켓플레이스(claude-plugins-official)에 포함되어 있으며, 신호가 가장 강한 출발점입니다. /plugin install <name>@claude-plugins-official 로 설치하세요.

코드 인텔리전스(LSP) — Claude가 타입 오류를 즉시 보게 하기​

LSP 플러그인은 언어 서버를 Claude Code에 연결해, 편집할 때마다 Claude가 grep 추측 대신 자동 진단(타입 오류, 누락된 임포트)과 정밀한 탐색(정의로 이동, 참조 찾기)을 받도록 합니다. 이들은 여러분의 컴퓨터에 맞는 언어 서버 바이너리가 필요합니다.

  • 언제 사용할까: 타입 언어로 진행하는 사소하지 않은 모든 코딩 세션. 이것이 아마도 개발 작업에서 정확도를 가장 크게 끌어올리는 단일 업그레이드일 것입니다.
  • 트레이드오프: rust-analyzer나 pyright 같은 서버는 큰 리포지토리에서 많은 메모리를 사용할 수 있습니다. 문제가 되면 비활성화하세요.

Install LSP for your stack

/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls

Git & PR 워크플로​

  • commit-commands — git 워크플로: 스테이징, 커밋 메시지 생성, 푸시, PR 열기. 형식적인 절차를 타이핑하지 않고도 깔끔하고 일관된 커밋을 원할 때 사용하세요. 스킬에는 네임스페이스가 붙습니다. 예: /commit-commands:commit.
  • pr-review-toolkit — 풀 리퀘스트를 리뷰하는 전문 에이전트. 머지 전이나, 디프에 대해 다른 시각의 검토를 얻고 싶을 때 사용하세요.

Git/PR essentials

/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins

자동 보안 리뷰​

  • security-guidance — Claude가 하는 각 변경을 흔한 취약점에 대해 리뷰하고, 같은 세션에서 고치도록 Claude에게 지시합니다. 코드를 작성하는 모든 프로젝트에서 상시 안전망으로 사용하세요. 실제 리뷰를 대체하는 것이 아니라 보완합니다.

Always-on security pass

/plugin install security-guidance@claude-plugins-official

외부 통합(MCP, 사전 구성됨)​

이것들은 바로 쓸 수 있는 MCP 서버를 묶어 수동 설정을 건너뛰게 합니다. Claude가 해당 서비스에서 직접 동작하기를 원할 때 하나를 꺼내 쓰세요.

  • 소스 관리: github, gitlab
  • 프로젝트 관리: linear, notion, asana, atlassian (Jira/Confluence)
  • 인프라: vercel, firebase, supabase
  • 디자인 / 커뮤니케이션 / 모니터링: figma, slack, sentry

Connect Claude to your tools

/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official

직접 만드는 툴킷 & 출력 스타일​

  • plugin-dev — 여러분만의 플러그인을 만드는 툴킷. 여러분의 .claude/ 설정이 공유할 만한 가치가 생기는 순간 사용하세요.
  • agent-sdk-dev — Claude Agent SDK 위에서 빌드하기 위한 도구.
  • explanatory-output-style / learning-output-style — Claude가 응답하는 방식을 바꿉니다: 진행하며 가르치는 설명, 또는 인터랙티브 학습 모드. 온보딩이나 변경 사항을 실제로 이해하고 싶을 때 훌륭합니다.

해야 할 일(job-to-be-done)별 고가치 스킬​

스킬은 반복적이고 컨텍스트가 실린 작업에서 빛을 발합니다. Anthropic은 anthropics/skills 리포지토리에 공개 레퍼런스 세트를 게시합니다 — Claude의 문서 및 창작 역량을 구동하는 바로 그 패턴들입니다. 스킬 폴더를 ~/.claude/skills/(개인)나 .claude/skills/(프로젝트)에 복사하면 다음 세션에 로드됩니다.

JobSkill (from anthropics/skills)언제 꺼내 쓸까
문서docx, pdf, pptx, xlsx실제 Office/PDF 파일 생성 또는 편집 — 보고서, 덱, 스프레드시트, 입력 가능한 폼. (소스 공개 레퍼런스 구현.)
UI 빌드web-artifacts-builder / artifacts-builder자체 완결형 HTML/JS 아티팩트와 미니 앱 제작.
Claude 확장mcp-builder아직 통합이 없을 때 새 MCP 서버 스캐폴딩.
브랜드 & 커뮤니케이션brand-guidelines / internal-comms 스타일 스킬결과물 전반에서 톤, 네이밍, 포맷을 브랜드에 맞게 유지.
Pro tip
  • ~/.claude/skills 에 한 번에 30개의 스킬을 쏟아붓지 마세요. 스킬은 점진적 공개(progressive-disclosure)입니다 — 트리거될 때까지 이름 + description만 로드되지만 — 방대하고 모호하게 설명된 더미는 트리거를 불안정하게 만듭니다. 몇 개만 설치하고 날카로운 "Use when…" 설명을 쓰며 의도적으로 늘려 가세요.
  • AILmanac은 바로 넣어서 시도해 볼 수 있는 완성된 스킬 팩을 제공합니다 — /docs/templates/skills 를 보세요.

지금 바로 붙여 넣을 수 있는 자체 트리거 스킬​

리서치 중심 작업에서는, 매번 방법론을 다시 설명하기보다 Claude를 신선한 출처에 근거하게 하는 스킬이 낫습니다. 이것을 ~/.claude/skills/research-brief/SKILL.md 에 넣으세요:

~/.claude/skills/research-brief/SKILL.md

---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.

커뮤니티가 사는 곳(큐레이션된 목록을 쓰고, 노이즈는 건너뛰기)​

수십 개의 "awesome" 리포지토리가 있지만, 대부분은 자동 생성된 미러입니다. 몇 개는 진짜로 큐레이션되고 유지 관리됩니다 — 거기서 시작하세요.

  • hesreallyhim/awesome-claude-code — 스킬, 훅, 슬래시 커맨드, 에이전트 오케스트레이터, 애플리케이션, 플러그인을 사람이 직접 큐레이션한 가장 잘 알려진 인덱스. 수만 개의 스타; 활발히 재구성됩니다.
  • ComposioHQ/awesome-claude-skills — 기능별(문서, 개발, 데이터, 마케팅, 보안, SaaS 자동화)로 그룹화된 대규모 큐레이션 Claude Skills 카탈로그.
  • VoltAgent/awesome-agent-skills — 1,400개 이상의 커뮤니티 + 공식 스킬(Anthropic, Vercel, Stripe, Cloudflare, Sentry, Figma 등)로, Claude Code, Cursor, Gemini CLI, Codex 등 여러 환경에서 이식 가능합니다.
  • anthropics/claude-plugins-community — 공식 커뮤니티 플러그인 마켓플레이스; Anthropic의 자동 검증 및 안전 심사를 통과한 서드파티 플러그인.
Watch out
  • 이 리포지토리들의 스타 수와 '가장 포괄적'이라는 주장은 변동성이 크고 때로는 AI 생성 포크에 의해 부풀려집니다. 숫자는 대략적인 것으로 여기고, 유지 관리와 큐레이션으로 판단하며, 설치하기 전에 항상 스킬이나 플러그인의 실제 내용을 검토하세요.
Key takeaways
  • 스킬 = 필요할 때 꺼내 쓰는 하나의 역량(SKILL.md); 플러그인 = 마켓플레이스에서 설치하는 버전 관리된 번들(스킬 + 에이전트 + 훅 + MCP/LSP).
  • /plugin marketplace add <owner/repo> 로 마켓플레이스를 추가하고, /plugin install <name>@<marketplace> 로 설치한 뒤, /reload-plugins 하세요.
  • 첫날의 가장 큰 성과: 여러분 언어를 위한 LSP 플러그인, commit-commands, security-guidance, 그리고 여러분이 늘 머무는 서비스의 MCP 통합.
  • 스킬은 이식 가능하고 점진적 공개 방식입니다 — 수백 개를 쌓아 두기보다 날카로운 몇 개를 큐레이션하세요.
  • 신뢰는 선택이 아닙니다: 플러그인과 마켓플레이스는 여러분의 권한으로 코드를 실행합니다. 설치하기 전에 검토하세요.

스스로 점검하기

0/4
  1. 1다른 사람의 카탈로그에 있는 플러그인을 사용하는 올바른 2단계 흐름은 무엇인가요?
  2. 2방금 설치한 플러그인의 스킬이 나타나지 않는 이유는 무엇일까요?
  3. 3타입 언어 개발 작업에서 정확도를 가장 크게 끌어올리는 단일 업그레이드는 무엇인가요?
  4. 4공식 문서 기준으로 스킬에 관한 정확한 진술은 무엇인가요?
