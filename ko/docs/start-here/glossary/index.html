용어집

모든 레벨

사전 지식 없이도 이해할 수 있는 쉬운 정의입니다. 훑어보거나, 다른 곳에서 마주친 용어를 검색해 보세요.

핵심 용어를 플래시카드로 복습하세요 — 탭하면 뒤집힙니다:

핵심 용어
Enter 또는 스페이스 키를 눌러 카드를 뒤집습니다. 좌우 화살표 키로 카드를 이동할 수 있습니다.용어가 표시되었습니다.
1 / 9

에이전트 (Agent)​

루프 안에서 행동을 수행하도록 설정된 AI — 목표를 읽고, 도구를 호출하고, 결과를 보고, 완료될 때까지 반복합니다. Claude Code는 여러분의 코드베이스를 위한 에이전트입니다.

Anthropic​

Claude를 만드는 회사입니다.

API (Application Programming Interface)​

채팅 창에 입력하는 대신 여러분의 소프트웨어가 Claude와 프로그래밍 방식으로 통신하는 방법입니다. 첫 API 호출을 참고하세요.

아티팩트 (Artifact)​

Claude가 claude.ai에서 채팅과 분리된 사이드 패널에 렌더링하는, 실행 가능한 출력물(앱, 차트, 문서)입니다.

사고 연쇄 (Chain-of-thought, CoT)​

답하기 전에 단계별로 추론하도록 모델에 요청하는 것으로, 어려운 문제에서 정확도를 높여 줍니다.

Claude​

Anthropic이 만든 AI 모델(및 앱)의 계열입니다. "Claude"는 맥락에 따라 채팅 앱, 모델, 또는 브랜드를 의미할 수 있습니다.

Claude Code​

여러분의 코드베이스를 읽고 편집하고, 명령을 실행하고, 도구를 사용할 수 있는 Anthropic의 에이전트형 명령줄/IDE 도구입니다. Claude Code란 무엇인가를 참고하세요.

CLAUDE.md​

프로젝트에 대한 지속적인 지시를 Claude Code에 제공하는 일반 텍스트 파일 — 가장 효과가 큰 커스터마이징입니다.

컴팩션 (Compaction)​

컨텍스트 윈도우가 가득 차면 오래된 대화 내용을 짧은 요약으로 자동 압축하여, 긴 세션이 흐름을 잃지 않고 계속되도록 하는 것입니다. 컨텍스트 관리를 참고하세요.

컨텍스트 엔지니어링 (Context engineering)​

모델의 컨텍스트 윈도우에 무엇을 넣을지 — 지시, 예시, 검색된 데이터 — 의도적으로 결정하여 최상의 결과를 얻는 것입니다. 프롬프팅, RAG, 메모리의 밑바탕이 되는 분야입니다.

컨텍스트 윈도우 (Context window)​

모델이 한 번에 고려할 수 있는 최대 텍스트 양(토큰 단위로 측정) — 프롬프트에 응답을 더한 것입니다. 이를 초과하면 가장 오래된 내용이 밀려납니다.

에포트 (Effort)​

답하기 전에 모델이 대략 얼마나 생각할지를 조절하는 최신 Claude 모델의 제어 값입니다. 무조건 최대로 두지 말고 작업에 맞게 등급을 맞추세요 — 생각에는 지연과 비용이 듭니다. 확장 사고 & 에포트를 참고하세요.

임베딩 (Embedding)​

텍스트 조각의 의미를 나타내는 숫자 목록으로, 소프트웨어가 (키워드 일치가 아니라) 의미가 유사한 것을 찾을 수 있게 합니다. 시맨틱 검색과 RAG를 뒷받침합니다.

평가 (Eval)​

AI 출력 품질에 대한 반복 가능한 테스트로, 어떤 변경이 상황을 나아지게 했는지 나빠지게 했는지 판별할 수 있게 합니다.

퓨샷 (Few-shot)​

원하는 패턴을 모델에 보여 주기 위해 프롬프트에 몇 개의 완성된 예시를 포함하는 것입니다.

파인튜닝 (Fine-tuning)​

자체 예시로 모델을 추가 학습시켜 특화하는 것입니다. 보통 최후의 수단이며, 먼저 프롬프팅과 RAG를 시도하세요.

그라운딩 (Grounding)​

제공한 컨텍스트에서만 답하고, 답이 거기에 없으면 "모르겠다"고 말하도록 모델에 지시하는 것입니다. 환각에 대한 주된 프롬프트 수준 방어책입니다. 패턴 라이브러리를 참고하세요.

가드레일 (Guardrails)​

AI의 동작을 안전하고 과업에 맞게 유지하는 규칙과 한계 — 예: 권한 규칙, "완료 전 테스트 실행", 위험한 행동에 대한 사람의 승인 등. 에이전트 보안을 참고하세요.

환각 (Hallucination)​

모델이 거짓이지만 확신에 찬 내용을 진술하는 것입니다. 핵심 역량은 검증입니다. 환각을 참고하세요.

훅 (Hook)​

Claude Code에서 라이프사이클 이벤트(예: 모든 파일 편집 후)에 자동으로 실행되어 서식, 게이트, 안전을 강제하는 명령입니다.

추론(실행) (Inference)​

모델을 (학습하는 것이 아니라) 실행하여 출력을 얻는 행위입니다.

지식 마감일 (Knowledge cutoff)​

모델에 학습 데이터가 없는 기준 날짜 — 프롬프트에 사실을 제공하지 않는 한, 그 이후에 일어난 일을 정말로 알지 못합니다. Claude의 마감일은 각 모델의 릴리스 노트에 공개됩니다. 이것이 매우 최근 사건을 맥락 없이 물으면 오래되거나 지어낸 답이 나올 수 있는 이유이며, 해결책은 관련 사실을 붙여 넣거나 실시간 정보를 검색하는 도구를 쓰는 것입니다.

지연 시간 (Latency)​

응답을 기다리는 시간입니다. 더 큰 모델은 보통 더 느리며, 스트리밍은 출력을 생성되는 대로 보여 주어 체감 지연을 개선합니다.

LLM (대규모 언어 모델)​

Claude의 바탕이 되는 AI의 종류로, 다음 텍스트 조각을 예측하도록 학습되었으며, 이것만으로도 글쓰기, 코딩, 요약, 추론에 충분합니다. LLM이란?을 참고하세요.

MCP (Model Context Protocol)​

AI를 외부 도구 및 데이터(데이터베이스, GitHub, 브라우저)에 연결하는 개방형 표준입니다. "MCP 서버"가 도구를 노출하면 Claude가 이를 호출할 수 있습니다. MCP를 참고하세요.

멀티모달 (Multimodal)​

텍스트 이상을 다루는 모델 — 예: 이미지와 PDF를 입력으로 받는 모델입니다.

출력 스타일 (Output style)​

Claude Code에서 할 수 있는 일은 바꾸지 않으면서 Claude가 소통하는 방식(예: 간결, 설명형, 학습형)을 바꾸는 프리셋입니다. 출력 스타일을 참고하세요.

권한 모드 (Permission mode)​

Claude Code에서 묻지 않고 얼마나 많은 일을 할 수 있는지 — 모든 행동마다 확인하는 것부터 편집을 자동 수락하는 것까지입니다. 속도 대 안전의 주요 다이얼입니다. 권한을 참고하세요.

플랜 모드 (Plan mode)​

파일을 건드리기 전에 조사하고 계획을 제안하는 읽기 전용 Claude Code 모드입니다 — 실행 전에 여러분이 승인합니다. 플랜 모드를 참고하세요.

프롬프트 (Prompt)​

모델에 보내는 것: 지시, 질문, 그리고 모든 컨텍스트입니다.

프롬프트 캐싱 (Prompt caching)​

여러 호출에 걸쳐 변하지 않는 프롬프트 앞부분을 재사용하여 비용과 지연을 줄이는 것입니다. 프롬프트 캐싱을 참고하세요.

프롬프트 인젝션 (Prompt injection)​

AI가 읽는 콘텐츠(웹 페이지, 문서)에 숨겨진 악의적 지시가 AI를 탈취하려는 공격입니다. 프롬프트 인젝션을 참고하세요.

RAG (Retrieval-Augmented Generation)​

여러분의 데이터에서 관련 조각을 가져와 프롬프트에 추가하여, 모델이 그에 관해 답하도록 하는 것입니다. RAG를 참고하세요.

추론 / 확장 사고 (Reasoning / extended thinking)​

더 어려운 작업을 위해 모델이 답하기 전에 추가 연산을 들여 "생각"하는 모드입니다. 최신 모델에서는 에포트로 이를 조정합니다.

스킬 (Skill)​

관련이 있을 때 Claude가 필요 시 로드하는 패키지화된 기능(지시 + 선택적 스크립트)입니다. 스킬을 참고하세요.

슬래시 명령 (Slash command)​

Claude Code에서 /로 시작하는 단축키 — 기본 제공(예: /clear)이든 여러분이 만든 재사용 프롬프트든 — 정의된 동작을 실행합니다. 슬래시 명령을 참고하세요.

스트리밍 (Streaming)​

전체 응답을 기다리는 대신, 모델의 출력을 생성되는 대로 토큰 단위로 전송하는 것입니다. 사용자에게는 모델이 실시간으로 타이핑하는 것처럼 보이고, 개발자에게는 하나의 응답 객체가 아니라 청크의 스트림을 처리하는 것을 뜻합니다. 긴 출력에서 체감 지연을 크게 개선하고, 모델이 끝나기 전에 렌더링을 시작할 수 있게 합니다. 스트리밍을 참고하세요.

구조화된 출력 (Structured output)​

자유 형식 텍스트 대신, 엄격하고 기계가 읽을 수 있는 형태(예: 스키마에 맞는 JSON)로 결과를 얻는 것 — 다른 소프트웨어가 이를 신뢰할 수 있게 합니다. 구조화된 출력을 참고하세요.

서브에이전트 (Subagent)​

자체 컨텍스트와 도구를 가진 별도의 Claude 인스턴스로, 작업의 일부(예: 조사)를 위임하여 메인 세션이 집중을 유지하도록 합니다.

시스템 프롬프트 (System prompt)​

대화 전체에 대해 모델의 역할과 규칙을 설정하는 상위 수준 지시 — 여러분의 가장 강력한 조종 레버입니다.

온도 (Temperature)​

무작위성을 제어하는 (일부 모델의) 설정: 낮음 = 집중되고 반복 가능, 높음 = 다양하고 창의적입니다.

토큰 (Token)​

모델이 읽고 세는 단위 — 영어에서 대략 단어의 ¾ 정도입니다. 가격과 컨텍스트 한도는 토큰 단위로 측정됩니다. 토큰 & 컨텍스트를 참고하세요.

도구 사용 / 함수 호출 (Tool use / function calling)​

모델이 여러분이 정의한 함수(검색, 계산기, 여러분의 API)를 호출하고 그 결과를 사용하게 하는 것입니다. 도구 사용을 참고하세요.

제로샷 (Zero-shot)​

예시 없이 지시만으로 모델에게 작업을 요청하는 것입니다. 일상적인 프롬프트 대부분이 제로샷입니다. 형식이나 스타일을 안내하는 완성된 예시를 포함하는 **퓨샷(few-shot)**과 대비됩니다. 제로샷은 간단한 작업에 충분하며, 모델이 원하는 출력 형태를 자꾸 놓칠 때 예시를 추가하세요.

빠진 용어가 있나요? 추가해 주세요 — 용어집 항목은 완벽한 첫 기여입니다.
