컨텍스트 관리

🟡중급
✅
최종 검증: 2026-06-20.

/compact, /clear 같은 명령 이름과 자동 압축 동작은 바뀔 수 있습니다 — 공식 문서에서 확인하세요.

다음에서 확인하세요: 공식 출처.

모든 세션에는 유한한 컨텍스트 윈도우가 있습니다. 그것이 차오르면 오래된 내용이 밀려나고 Claude가 "표류"하기 시작할 수 있습니다. 컨텍스트를 의도적으로 관리하는 것이 긴 세션을 날카롭게 유지하는 비결입니다.

두 가지 필수 명령​

  • /clear — 대화를 지우고 새로 시작합니다. 관련 없는 작업으로 전환할 때 사용하세요. 깨끗하고 집중된 컨텍스트를 되찾는 가장 저렴한 방법입니다.
  • /compact — 지금까지의 대화를 압축된 형태로 요약해, 요지를 유지하면서 공간을 확보합니다. 같은 작업을 계속하고 싶지만 윈도우가 가득 차고 있을 때 사용하세요.

자동 압축​

컨텍스트가 한계에 가까워지면 Claude Code는 작업이 갑자기 멈추지 않도록 자동 압축할 수 있습니다. 이는 안전망입니다 — 하지만 그것에 의존하기보다 작업 사이사이에 능동적으로 /clear 하는 편이 더 나은 결과를 줍니다.

효과를 유지하는 기법​

  • 세션당 작업 하나. 오후 내내의 관련 없는 작업이 하나의 컨텍스트에 쌓이지 않게 하세요.
  • 무거운 읽기는 위임하세요 서브에이전트에게 — 그들은 자신의 컨텍스트를 소모하고 결론만 반환합니다.
  • 붙여넣지 말고 참조하세요. 큰 덩어리를 붙여넣는 대신 Claude를 파일(@path)로 가리키세요. CLAUDE.md는 군더더기 없이 유지하세요.
  • 예산을 주시하세요. 컨텍스트 사용량을 보여주는 상태줄은 언제 압축할지 알려줍니다.

:::tip 증상 → 해결 Claude가 앞서 받은 지침을 잊거나 같은 말을 반복하기 시작하면 윈도우가 가득 찼을 가능성이 큽니다. /compact(같은 작업) 또는 /clear(새 작업)로 대개 즉시 해결됩니다. 자세한 내용은 문제 해결을 참고하세요. :::
