AILmanac이 "완전하고 항상 최신"인 이유는 여러분 같은 사람들이 채워주기 때문입니다. 개발자일 **필요는 없습니다** — 오타 수정, 용어집 항목 추가, 템플릿 공유는 모두 진짜 기여입니다.

## 빠른 경로

1. 사이트에서 **페이지를 찾아** 하단의 **"Edit this page"**를 클릭하세요(GitHub에서 파일이 열립니다).
2. ✏️ 연필 아이콘을 클릭하여 브라우저에서 편집합니다. GitHub이 자동으로 저장소를 포크해 줍니다.
3. Markdown으로 변경 사항을 작성합니다.
4. 하단에 짧은 설명을 작성하고 **propose changes** → **open a pull request**를 클릭합니다.
5. 메인테이너가 리뷰합니다. 그게 전부입니다. 🎉

로컬을 선호하시나요? 저장소를 클론한 다음:

```bash
npm install
npm run start      # live preview at http://localhost:3000
npm run build      # must pass: it fails on broken links
```

## 모든 페이지가 따르는 4가지 규칙

이 규칙들을 지키면 리뷰가 빨라지고 품질이 높게 유지됩니다:

1. **레벨을 태그하세요.** 본문을 `<LevelBadge level="beginner" />`(또는 `intermediate` / `advanced` / `all`)로 시작합니다.
2. **변동성 있는 사실은 출처를 링크하세요.** 모델 이름, 가격, 한도, 또는 변하는 기능에 관한 것은 모두 공식 페이지를 인용해야 하며 — `<VerifyNote lastVerified="YYYY-MM-DD" source="..." />`를 사용합니다. 모델 관련 사실은 절대 하드코딩하지 말고 [모델 표](/docs/whats-new/models-and-pricing)로 링크하세요.
3. **소신을 가지되 정직하세요.** 권장하는 방법을 먼저 제시하고, 대안을 언급합니다. 공식 문서를 복제하지 말고 — 그곳으로 링크하세요.
4. **쉬운 영어, 이론보다 예시.** 짧은 문장. 말로만 하지 말고 보여주세요.

## 페이지 프런트매터 템플릿

```markdown
---
sidebar_position: 2
title: Your Page Title
description: One sentence shown in search and previews.
keywords: [relevant, search, terms]
---

<LevelBadge level="beginner" />

Your first paragraph...
```

## 훌륭한 첫 기여란

- 누락된 **용어집** 항목을 추가하세요.
- 이미 사용 중인 **템플릿**이나 프롬프트를 추가하세요.
- **새로 출시된** 내용 때문에 페이지가 지금 잘못된 부분을 수정하세요(그리고 그 `lastVerified` 날짜를 갱신하세요).
- 페이지를 번역하세요 — [번역 플레이북](/docs/contribute)을 참조하세요.

## 라이선스 및 서명

기여함으로써 여러분의 산문은 **CC BY 4.0**으로, 코드는 **MIT**로 공유되는 데 동의하며, 커밋에 `Signed-off-by` 줄(DCO)을 추가합니다(`git commit -s`). 자세한 내용은 [거버넌스 페이지](/docs/contribute)에 있습니다.

친절하게 — 우리는 Contributor Covenant [행동 강령](/docs/contribute)을 따릅니다.
