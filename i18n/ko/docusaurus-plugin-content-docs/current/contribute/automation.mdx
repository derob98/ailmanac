투명성 페이지: 최소한의 메인테이너 노력으로 AILmanac을 신선하고 고품질로 유지하도록 돕는 장치들을 소개합니다. 전반에 걸친 원칙: **자동화는 사람이 승인하도록 작업을 드러낼 뿐 — 검증되지 않은 주장을 말없이 게시하지 않습니다.**

## 빌드 게이트(활성화됨)

모든 변경은 `onBrokenLinks: 'throw'`로 빌드되므로, 죽은 내부 링크나 깨진 MDX는 **병합 전에 빌드를 실패시킵니다**. 이것이 가장 가치 있는 단일 검사입니다 — 내비게이션을 신뢰할 수 있게 유지합니다.

## CI 워크플로

> 워크플로 파일은 `ci-workflows` 브랜치에 있으며, 저장소에 `workflow` 스코프를 가진 토큰이 생기면 활성화됩니다(README 참조). 그때까지는 사이트가 `gh-pages` 브랜치에서 배포됩니다.

- **`build-check`** — 모든 PR을 빌드합니다(깨진 링크는 실패). 필수 검사입니다.
- **`deploy`** — `main`에 푸시되면 빌드하여 GitHub Pages에 게시합니다.
- **`link-check`** — `lychee`가 매주 그리고 PR에서 실행됩니다. 깨진 외부 링크가 있으면 단순히 빨간색으로 표시하는 대신 **추적 이슈를 엽니다**(라벨 `link-rot`).

## 최신성 자동화(계획됨 / 단계적)

- **릴리스 감시기** — Anthropic의 문서/체인지로그 **와** 가장 빠르게 변하는 소비자 표면(앱, 음성, 모바일, 마켓플레이스)을 폴링하는 예약 작업입니다. 변경이 감지되면 영향을 받을 가능성이 있는 페이지를 가리키며 `upstream-change` + `needs-verification` 라벨이 붙은 이슈/PR을 엽니다. 사람이 업데이트를 작성합니다.
- **모델 표 검증기** — `data/models.json`을 공식 모델 목록과 비교(diff)하여, 차이가 발생하면 PR을 엽니다. [모델 표](/docs/whats-new/models-and-pricing)에는 "마지막 검증" 날짜가 표시됩니다.
- **오래됨 점검(Staleness sweep)** — `lastVerified`가 임계값보다 오래된 변동성 있는 페이지를 표시합니다.

## 왜 사람이 개입하는가(human-in-the-loop)

:::tip 자동화되고 정직한 것 > 자동화되고 틀린 것
레퍼런스 사이트에 "뉴스"를 자동 커밋하는 AI는 환각을 게시할 위험이 있습니다 — 신뢰할 수 있다는 것의 정반대입니다. 그래서 우리의 자동화는 **제안**(이슈/PR)하고, 사람이 **검증하고 병합**합니다. [사실 검증](/docs/contribute/fact-verification)을 참조하세요.
:::

## 자동화를 돕고 싶으신가요?

이 워크플로를 개선하는 것은 훌륭한 고급 기여입니다 — `.github/workflows`(`ci-workflows` 브랜치에 있음)와 [기여하기](/docs/contribute/contribute-in-10-minutes)를 참조하세요.
