가장 여러분 같은 트랙을 고르세요. 각각은 순서가 정해진 경로입니다 — 단계를 순서대로 하되 이미 아는 것은 건너뛰세요. 시간은 대략적입니다.

<Callout type="objectives" items={["여러분의 목표에 맞는 단 하나의 트랙 찾기 — 입문자, 지식 노동자, 빌더, 팀 리드", "어디서 시작할지 추측하는 대신 명확하고 순서 있는 읽기 경로 따라가기", "각 트랙이 대략 얼마나 걸리는지 알기 (~45분 ~ ~반나절)", "어떤 트랙도 딱 맞지 않을 때 어디서 시작할지"]} />

:::tip 어떤 트랙이 내 것인지 모르겠나요?
직함이 아니라 **목표**로 맞추세요. 코딩 없이 결과를 원하나요? 입문자. 실제 업무를 더 빠르게? 지식 노동자. Claude Code나 API를 커스터마이징? 빌더. AI를 다른 사람들에게 배포? 팀 리드.
:::

## 트랙 한눈에 보기

| 트랙 | 여러분은… | 시간 | 첫 단계 |
| --- | --- | --- | --- |
| 🌱 호기심 많은 입문자 | 코딩 없이 AI를 잘 쓰기 | ~45분 | [첫 5분](/docs/start-here/your-first-5-minutes) |
| 💼 지식 노동자 | 실제 업무 속도 높이기 | ~2시간 | [프롬프트 패턴 라이브러리](/docs/prompting) |
| 🛠️ 빌더 / 개발자 | Claude Code + API 커스터마이징 | ~반나절 | [Claude Code란 무엇인가](/docs/claude-code/what-is-claude-code) |
| 👥 팀 리드 | AI를 팀에 배포 | ~3시간 | [플러그인 & 마켓플레이스](/docs/claude-code) |

:::note 트랙은 서로 쌓입니다
지식 노동자 트랙은 입문자 트랙 전체로 시작합니다. 팀 리드 트랙은 빌더 필수(1~3단계)로 시작합니다. 한 트랙을 끝내면 그 다음 트랙은 대부분 이미 끝난 셈입니다.
:::
