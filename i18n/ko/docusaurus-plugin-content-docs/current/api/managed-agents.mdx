[직접 에이전트 루프를 구축하는 것](/docs/api/building-agents)이 당신이 떠안고 싶은 것보다 더 많은 인프라라면, **관리형**(Anthropic 호스팅) 에이전트 옵션이 루프를 대신 실행해 줍니다 — 그래서 세션 배관, 재시도, 상태 관리가 아니라 에이전트의 *일*에 집중할 수 있습니다.

## "관리형"이 가져다주는 것

루프를 직접 짜고 호스팅하는 대신, 다음과 같은 호스팅 빌딩 블록을 얻습니다:

- **세션** — 재개할 수 있는 영속적 에이전트 실행.
- **환경** — 에이전트가 자신의 도구로 작동할 수 있는 공간.
- **메모리** — 데이터베이스를 직접 연결하지 않고도 턴/실행 전반에 걸쳐 유지되는 상태.
- **스케줄링** — 무인 상태로 일정에 따라 실행되는 에이전트.

(정확한 이름과 표면적은 발전합니다 — [공식 문서](https://docs.anthropic.com/en/docs/agents-and-tools)를 권위 있는 출처로 삼으세요.)

## 관리형 vs 커스텀, 언제 고를까

| **관리형**을 고를 때… | **커스텀 루프 / SDK**를 고를 때… |
|---|---|
| 호스팅, 상태, 스케줄링이 처리되기를 원할 때 | 루프와 도구에 대한 완전한 제어가 필요할 때 |
| 빠르게 프로토타이핑할 때 | 엄격한 커스텀 인프라/컴플라이언스 요구가 있을 때 |
| 제어보다 운영 단순성이 더 중요할 때 | 자체 스택에 깊이 임베딩할 때 |

이것은 스펙트럼입니다 — 단일 호출 → 워크플로 → 커스텀 에이전트(SDK) → 관리형. 작업이 허용하는 만큼 단순하게 시작하고, 필요할 때만 위로 올라가세요.

## 동일한 가드레일이 적용됩니다

호스팅이든 아니든, 자율 에이전트는 여전히 작업을 수행합니다. **최소 권한**, **제한된 비용/반복 횟수**, **위험한 단계에 대한 사람의 승인**을 유지하세요 — [에이전트 보안](/docs/security/securing-agents)과 [자율 실행 강화](/docs/security/hardening-autonomous-runs) 참고.
