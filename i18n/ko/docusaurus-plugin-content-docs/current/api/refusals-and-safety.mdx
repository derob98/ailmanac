## 두 가지 서로 다른 것

- **모델 거부** — Claude가 요청을 거절합니다(예: 유해하다고 판단). 응답이 이를 신호로 보냅니다(일반적으로 refusal `stop_reason`/콘텐츠). 오류가 아닌 정상 결과로 취급하세요.
- **분류기/안전 차단** — 별도의 안전 계층이 콘텐츠를 차단할 수 있습니다. 이는 모델 거부와 다르게 보일 수 있습니다.

어느 쪽을 받았는지 아는 것이 맹목적으로 재시도하는 대신 적절히 대응하게 해줍니다.

## 우아하게 처리하기

```python
resp = client.messages.create(...)
if getattr(resp, "stop_reason", None) == "refusal":
    # Don't show a raw/empty result. Offer a safe fallback or a clarifying ask.
    show_user("I can't help with that as asked. Here's what I can do instead…")
else:
    render(resp)
```

## 원하지 않은 거부 줄이기

- **정당한 맥락을 추가하세요.** 요청이 민감한 무언가에 패턴 매칭될 수 있지만 의도가 무해할 때, 실제 정당한 목적을 진술하는 것이 도움이 됩니다.
- **구체적으로.** 모호하거나 도발적인 표현은 조심을 부릅니다.
- **싸우지 마세요.** 요청이 정말로 허용되지 않는다면 거부는 옳습니다 — 우아한 경로를 설계하되, 탈옥하지 마세요.

## 폴백 패턴

- **명확화 질문** 대신 막다른 골목.
- **안전한 대안**("대신 공개 정보를 요약할 수 있습니다").
- 파이프라인에서는, 신뢰도/자격이 낮을 때 **사람에게 라우팅**.
