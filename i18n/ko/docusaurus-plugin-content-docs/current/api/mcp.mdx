API 요청/에이전트에서 MCP를 선언하는 방식과 인증 옵션은 발전합니다 — 공식 문서와 modelcontextprotocol.io에서 확인하세요.
</VerifyNote>

**Model Context Protocol(MCP)**은 AI를 외부 도구 및 데이터에 연결하기 위한 개방형 표준입니다. API에서 MCP를 사용하면 각 도구를 일일이 정의하는 대신, 참조만으로 Claude에게 **서버 단위 도구 묶음**에 대한 접근 권한을 줄 수 있습니다.

## MCP vs 직접 정의한 도구

| | [도구 사용](/docs/api/tool-use) (커스텀) | MCP |
|---|---|---|
| 당신이 정의하는 것 | 각 도구의 스키마, 그리고 직접 실행 | 도구를 *게시하는* 서버로의 연결 |
| 적합한 경우 | 앱 내의 소수의 맞춤 함수 | 기존 통합 재사용(GitHub, DB, 브라우저, SaaS) |
| 인증 | 당신의 코드 | 서버에 대한 OAuth 또는 네이티브 키 |

둘은 공존합니다: 앱 전용 도구는 직접 정의하고, 기성 기능은 MCP를 통해 끌어오세요.

## 서버 선언하기

요청/에이전트에 MCP 서버를 연결합니다(URL 또는 명령어, 그리고 인증). 그러면 Claude는 [에이전트 루프](/docs/api/building-agents) 안에서 다른 도구와 마찬가지로 서버의 도구를 호출할 수 있습니다. 정확한 요청 필드와 인증 흐름은 [공식 문서](https://platform.claude.com/docs/en/docs/build-with-claude/mcp)에 있습니다.

## 같은 표준, 세 가지 표면

- **API** (이 페이지) — 요청/에이전트에 붙는 MCP 서버.
- **[Claude Code](/docs/claude-code/mcp)** — 개발 세션 안의 MCP 서버.
- **[앱](/docs/claude-app/connectors)** — MCP가 커넥터(Connectors)를 구동합니다.

한 번 익히면 그대로 전이됩니다.

## 신뢰

:::warning MCP 서버는 코드 + 접근 권한입니다
신뢰하는 서버만 연결하고, 최소 권한으로 범위를 좁히며, 서버가 반환하는 외부 콘텐츠가 [프롬프트 인젝션](/docs/security/prompt-injection)을 품을 수 있음을 기억하세요. 서드파티 서버는 먼저 검토하세요 — [서드파티 코드 검토하기](/docs/security/reviewing-third-party-code).
:::
