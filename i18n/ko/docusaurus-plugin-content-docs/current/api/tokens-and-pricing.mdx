토큰당 가격과 컨텍스트 한도는 변합니다 — 항상 공식 가격/모델 페이지에서 현재 숫자를 읽으세요. [모델 표](/docs/whats-new/models-and-pricing)를 참고하세요.
</VerifyNote>

API의 비용과 한도는 모두 **[토큰](/docs/foundations/tokens-and-context)**(단어의 약 ¾)으로 측정됩니다. 제대로 잡아야 할 세 가지가 있습니다.

## 1. 토큰을 정확히 세기

추측하지 말고, **다른 모델의 토크나이저를 쓰지 마세요**(예: `tiktoken`) — 토큰 수는 모델 계열마다 다릅니다. 요청을 보내기 *전에* 측정하려면 Anthropic의 **토큰 세기** 엔드포인트/SDK 헬퍼를 사용하세요.

<Callout type="tip" items={["대략적인 계획 규칙: ~750단어 ≈ ~1,000토큰(토큰은 대략 단어의 ¾)."]} />

## 2. `max_tokens` ≠ 컨텍스트 윈도우

이 두 한도는 헷갈리기 쉽지만 서로 다른 것을 관리합니다.

| 한도 | 무엇을 제한하나 | 언제 바꾸나 |
| --- | --- | --- |
| **`max_tokens`** | *응답*의 길이(출력만) | 출력이 잘리면 올리세요 |
| **컨텍스트 윈도우** | **입력 + 출력**의 전체 예산 | 큰 입력은 출력 여지를 줄입니다 |

<Callout type="warning" items={["max_tokens는 작업에 필요한 만큼 설정하세요. 너무 낮으면 응답이 잘립니다. 불필요하게 높아도 비용이 더 들지는 않지만 — 실제 생성된 토큰만큼만 지불합니다 — 응답이 장황해질 수 있습니다."]} />

## 3. 비용 추정

**입력 토큰 + 출력 토큰**에 대해 모델별 요율(Opus > Sonnet > Haiku)로 청구됩니다. 빠른 추정:

```
cost ≈ (input_tokens × input_rate) + (output_tokens × output_rate)
```

현재 요율은 [공식 가격 페이지](https://platform.claude.com/docs/en/docs/about-claude/pricing)에서 확인하세요 — 여기서는 일부러 하드코딩하지 않습니다.
