좋은 소식: **프롬프팅의 기본 원칙은 모든 모델에 걸쳐 그대로 옮겨집니다.** Claude를 잘 프롬프팅할 수 있다면 ChatGPT, Gemini, Llama도 잘 프롬프팅할 수 있습니다. 표면적인 세부 사항 몇 가지가 다를 뿐입니다 — 여기 그 로제타석이 있습니다.

## 어디서나 그대로 옮겨지는 것

이것들은 모든 모델에서 작동합니다 — 특정 벤더가 아니라 *소통*에 관한 것이기 때문입니다:

- **맥락 제공하기**: 목표, 대상, 형식, 톤 ([프롬프팅 기초](/docs/prompting/basics)).
- **예시** ([퓨샷](/docs/prompting/few-shot)).
- 어려운 작업에서 **추론 요청하기**.
- **출력 제약하기** (길이, 형식, "JSON만").
- 평이한 언어로 **반복하기**.
- **역할 / 시스템 프롬프트** — 모든 주요 모델이 이 개념을 가지고 있습니다 ([역할](/docs/foundations/roles)).

이것들을 한 번 익히면 어디서나 이식할 수 있습니다.

## 다른 점 (그리고 무엇을 조정해야 하는가)

| 측면 | 모델을 바꿀 때 유의할 점 |
|---|---|
| **구조 단서** | Claude는 [XML 태그](/docs/prompting/xml-tags)를 매우 좋아합니다. 다른 모델도 받아들이지만 Markdown 제목이나 JSON을 선호할 수 있습니다. |
| **톤 강도** | 구형 모델은 강압적이거나 대문자로 된 프롬프트를 견뎠습니다. 곧이곧대로 따르는 신형 모델은 차분하고 구체적인 지시를 선호합니다. 모델마다 다시 조율하세요. |
| **추론** | "단계별로 생각해줘"는 일부 모델에 도움이 됩니다. 전용 추론 모델은 내부적으로 이를 수행하므로 — 중복하지 마세요. |
| **프리필링** | 어시스턴트의 입에 말을 넣어주는 것은 Claude 스타일의 지렛대입니다. 다른 곳에서는 지원 여부가 다릅니다. |
| **시스템 프롬프트 비중** | 모두 가지고 있지만, 얼마나 강하게 따르는지는 다릅니다 — 테스트하세요. |
| **도구/함수 호출** | 같은 개념이지만 제공자마다 요청 스키마가 다릅니다. |

*표면*만 바뀌기 때문에, 잘 구조화된 하나의 프롬프트는 대개 한 줄 수정으로 이식됩니다. 여기 보편적인 방식으로 작성된 예시가 있습니다 — 맥락, 명시적 형식, 명확한 작업으로 이루어져 있고 제거해야 할 벤더별 구문이 없습니다:

<PromptCard title="이식 가능하고 벤더 중립적인 프롬프트">{`You are a technical editor. Rewrite the text below to be clearer and more concise.

Constraints:
- Keep every fact unchanged.
- Target a general professional audience.
- Return only the rewritten text, no preamble.

Text:
"""
[paste the draft here]
"""`}</PromptCard>

그 프롬프트는 Claude, GPT, Gemini, 또는 오픈 모델에서 그대로 실행됩니다. 어떤 모델이 "다시 쓴 텍스트만 반환하라"를 무시한다면, 그것이 바로 당신의 한 가지 표면 수정입니다 — 제약을 더 조이거나, Claude에서는 답변을 [프리필](/docs/prompting/prompting-claude)하세요.

:::tip 처음부터 다시 쓰지 마세요
새 모델로 옮길 때 새 프롬프트가 필요한 경우는 드뭅니다 — 보통은 톤/형식 조정만 하면 됩니다. 구조와 예시는 그대로 이어집니다.
:::
