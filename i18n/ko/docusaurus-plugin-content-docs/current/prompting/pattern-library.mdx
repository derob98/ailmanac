재사용 가능한 프롬프팅 패턴 도구 모음입니다. 각 카드에는 그것이 무엇인지, 언제 꺼내 쓸지, 그리고 복사-붙여넣기 템플릿이 담겨 있습니다. 이들은 쌓을 수 있습니다 — 자유롭게 조합하세요. 여기가 처음이신가요? 먼저 [프롬프팅 기초](/docs/prompting/basics)부터 시작하세요.

<Callout type="objectives" items={["11가지 핵심 프롬프팅 패턴과 각각이 언제 적용되는지 알아보기","어떤 작업에든 복사-붙여넣기 템플릿 가져오기 — 제로샷, 퓨샷, 사고 연쇄 등","패턴을 조합하기(역할 + 예시 + 출력 제약 + 답하지 않을 권한)해 하나의 강력한 프롬프트로","그라운딩으로 환각을 줄이고 메타 프롬프팅으로 약한 프롬프트 고치기"]} />

## 제로샷 (명확하게, 그냥 요청하기)

**언제:** 단순하고 흔한 작업. **이유:** 가장 적은 노력, 종종 충분합니다.

<PromptCard title="제로샷 요약">{`Summarize the text below in 5 bullets for a busy manager.
TEXT: """{paste}"""`}</PromptCard>

## 퓨샷 (예시 보여주기)

**언제:** 특정 형식이나 스타일이 필요할 때. **이유:** 예시는 설명보다 빠르게 가르칩니다. [퓨샷 제대로 하기](/docs/prompting/few-shot)를 참고하세요.

<PromptCard title="퓨샷 감성 분류기">{`Classify the sentiment as positive/negative/neutral.
Review: "Loved it, shipped fast" → positive
Review: "Broke on day one" → negative
Review: "{new review}" →`}</PromptCard>

## 사고 연쇄 (단계별로 생각하기)

**언제:** 수학, 논리, 여러 단계의 추론. **이유:** 답하기 전에 추론하면 정확도가 향상됩니다.

<PromptCard title="사고 연쇄">{`Work through this step by step, then give the final answer on its own line.
PROBLEM: {problem}`}</PromptCard>

## 분해 (잘게 나누기)

**언제:** 크고 모호한 작업. **이유:** 더 작은 하위 작업이 더 쉽고 더 신뢰할 수 있습니다.

<PromptCard title="분해">{`First, list the sub-tasks needed to {goal}. Then do them one at a time,
showing each result before moving on.`}</PromptCard>

## 역할 프롬프팅 (페르소나 부여하기)

**언제:** 특정한 관점이나 기준을 원할 때. **이유:** 관련된 행동을 유도합니다.

<PromptCard title="역할 프롬프팅">{`You are a meticulous senior copy editor. Edit the text for clarity and concision
without changing meaning. Show a bullet list of the changes you made.`}</PromptCard>

## 출력 형식 제약

**언제:** 결과가 다른 도구로 들어가거나 일관성을 원할 때. **이유:** 예측 가능한 모양은 사용 가능합니다.

<PromptCard title="출력 형식 제약">{`Reply with ONLY valid JSON matching: {"title": string, "tags": string[]}.
No prose, no markdown fences.`}</PromptCard>

## 자기 일관성 / 자기 비평

**언제:** 정확성이 중요할 때. **이유:** 두 번째 점검이 첫 번째 실수를 잡아냅니다.

<PromptCard title="자기 비평">{`Answer the question. Then, in a section called "Check", critique your own answer
for errors or missing cases and revise if needed.`}</PromptCard>

## 그라운딩 (제공된 맥락에서만 답하기)

**언제:** 사실이 중요할 때 — 문서 Q&A, RAG, 틀린 답이 답이 없는 것보다 나쁜 모든 경우. **이유:** [환각](/docs/foundations/hallucinations)에 맞서는 가장 큰 단 하나의 지렛대는 모델이 지어내는 것을 금지하고 답하지 않아도 된다는 명시적 허가를 주는 것입니다.

<PromptCard title="그라운딩된 Q&A">{`Answer the QUESTION using ONLY the CONTEXT below. Quote the exact sentence(s)
you relied on. If the context does not contain the answer, reply exactly:
"Not found in the provided context." Do not use outside knowledge.

CONTEXT: """{paste your sources}"""
QUESTION: {question}`}</PromptCard>

## 메타 프롬프팅 (AI가 당신의 프롬프트를 개선하게 하기)

**언제:** 당신의 프롬프트가 기대에 못 미칠 때. **이유:** 모델은 무엇이 빠졌는지 잘 찾아냅니다.

<PromptCard title="메타 프롬프팅">{`Here's my prompt: "{prompt}". Ask me the 3 questions that would most improve it,
then rewrite it incorporating sensible defaults.`}</PromptCard>

## 반복적 개선

**언제:** 거의 항상. **이유:** 첫 출력은 초안일 뿐, 종착지가 아닙니다.

<PromptCard title="반복적 개선">{`{after a result} Shorter and more concrete. Keep the second paragraph. Add one
example. Drop the marketing tone.`}</PromptCard>

:::tip 조합하세요
강력한 실전 프롬프트는 종종 **역할** + **예시** + **출력 제약** + "모르겠다"고 말할 **권한**으로 이루어집니다. 카드를 겹쳐 쌓으세요.
:::
