권한은 **Claude Code가 당신에게 묻기 위해 멈추지 않고 할 수 있는 것**을 결정합니다. 잘 조정하면 제어를 잃지 않고 흐름을 얻고, 잘못 조정하면 모든 것을 고무도장 찍거나 프롬프트에 빠져 죽습니다.

## 세 가지 판정

모든 잠재적 동작은 다음 중 하나로 귀결됩니다:

<Flashcards cards={[{front: "allow", back: "실행, 프롬프트 없음."}, {front: "ask", back: "멈추고 당신에게 물음 — 명시적으로 허용되거나 거부되지 않은 모든 것의 기본값."}, {front: "deny", back: "절대 안 됨, 물어봐도 안 됨."}]} />

규칙은 보통 도구와 패턴을 매칭합니다, 예: allow `Bash(npm run test:*)` 또는 deny `Read(./.env)`.

<PromptCard title="도구 + 패턴 허용">{`Bash(npm run test:*)`}</PromptCard>

<PromptCard title="도구 + 패턴 거부">{`Read(./.env)`}</PromptCard>

## 권한 모드

모드는 세션 전체의 자세를 설정합니다:

| 모드 | 동작 | 사용 시점 |
|---|---|---|
| **default / ask** | 미리 허용되지 않은 모든 것에 프롬프트 | 일상 작업 |
| **plan** | 읽기 전용; 제안하되 절대 행동 안 함 | 크거나 위험한 작업 — [플랜 모드](/docs/claude-code/plan-mode) 참조 |
| **acceptEdits** | 파일 편집 자동 수락 | 신뢰되고 범위가 잘 잡힌 편집 세션 |
| **bypass / yolo** | 프롬프트 완전 건너뜀 | 샌드박스/CI만 — 시크릿이 있는 머신에서는 **절대** 안 됨 |

<Callout type="warning" items={["Bypass는 샌드박스에 속합니다. 실제 머신에서 모든 프롬프트를 끄고 실행하는 것은 에이전트가 건드리면 안 되는 것을 건드리게 되는 방법입니다. 일회용 환경에만 남겨두세요. /docs/security/hardening-autonomous-runs의 자율 실행 강화하기를 참조하세요."]} />

## 합리적 시작 허용목록

목표: 안전하고 반복적인 것은 미리 허용하고; 파괴적인 것은 **ask** 또는 **deny**에 두기.

<Steps items={[{title: "안전 + 반복적인 것 허용", body: "파일 읽기, 테스트/린트/빌드 명령 실행, git status/diff."}, {title: "중간 위험 전에 묻기", body: "의존성 설치, 프로젝트 외부 파일 쓰기, 네트워크 호출."}, {title: "파괴적인 것 거부", body: "시크릿 파일 읽기(.env, 키 파일), force-push, rm -rf."}]} />

프로젝트 규칙은 [`settings.json`](/docs/claude-code/settings)(공유)에, 개인 오버라이드는 `settings.local.json`에 저장하세요.
