지원되는 IDE와 설정 단계는 바뀝니다 — 공식 IDE 통합 문서에서 확인하세요.
</VerifyNote>

터미널에서만 지내야 하는 것은 아닙니다. Claude Code는 에디터와 통합되어, **코드를 작성하는 바로 그곳에서 변경 사항을 diff로** 검토할 수 있게 합니다.

## 무엇을 얻는가

- **인라인 diff.** 제안된 편집이 터미널의 텍스트가 아니라 에디터 안에서 검토 가능한 diff로 나타납니다.
- **에디터 컨텍스트.** 열려 있는 파일, 선택 영역, 프로젝트가 Claude에게 제공됩니다.
- **동일한 두뇌.** 같은 Claude Code입니다 — [CLAUDE.md](/docs/claude-code/claude-md), [권한](/docs/claude-code/permissions), [훅](/docs/claude-code/hooks), [설정](/docs/claude-code/settings)이 CLI와 IDE에서 동일하게 적용됩니다. 한 번 설정하면 어디서나 사용합니다.

## 지원되는 에디터

- 확장을 통한 **VS Code** (및 VS Code 기반 에디터).
- 플러그인을 통한 **JetBrains IDE** (IntelliJ, PyCharm, WebStorm 등).

에디터 마켓플레이스에서 설치하고 인증하세요. 현재 단계는 [공식 문서](https://docs.anthropic.com/en/docs/claude-code/ide-integrations)를 참고하세요.

## CLI냐 IDE냐 — 무엇을?

둘은 같은 도구입니다. 선호와 작업에 따라 고르세요:

| IDE를 선호할 때… | CLI를 선호할 때… |
|---|---|
| diff를 시각적으로 검토하고 싶을 때 | 스크립트를 짜거나 [헤드리스](/docs/claude-code/headless-and-agent-sdk) 모드일 때 |
| 이미 에디터에서 코딩 중일 때 | 서버 / SSH 환경일 때 |
| 에디터 컨텍스트가 내장된 것이 좋을 때 | 최대한의 제어와 파이프를 원할 때 |

:::tip 설정은 공유됩니다
CLI와 IDE가 같은 설정을 읽기 때문에, [CLAUDE.md](/docs/claude-code/claude-md)와 [권한](/docs/claude-code/permissions)을 한 번만 설정하면 둘 사이에서 따라다닙니다.
:::
