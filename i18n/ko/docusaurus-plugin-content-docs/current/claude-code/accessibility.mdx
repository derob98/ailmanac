## 스크린 리더 모드

스크린 리더 모드는 장식적인 TUI를 보조 기술이 깔끔하게 읽을 수 있는 평문 렌더링으로 바꿉니다. 세 가지 방법 중 아무거나로 켜세요 — 일회성 플래그, 모든 세션에 적용되는 환경 변수, 또는 영구 설정.

<Steps items={[
  { title: "한 세션만 시도하기", body: "플래그로 실행하세요: claude --ax-screen-reader. 다른 것은 바뀌지 않습니다 — 같은 세션, 더 평이한 출력." },
  { title: "모든 세션에 켜기", body: "셸 프로필에서 환경 변수 CLAUDE_AX_SCREEN_READER=1을 export하면 어디서나 적용됩니다." },
  { title: "설정에 영구 반영하기", body: "settings.json 파일에 \"axScreenReader\": true를 설정하세요 (사용자 수준 ~/.claude/settings.json은 모든 프로젝트에 적용됩니다)." }
]} />

<PromptCard title="영구 스크린 리더 모드 (~/.claude/settings.json)">{`{
  "axScreenReader": true
}`}</PromptCard>

  "중첩된 표는 Header: value. 형태의 줄로 읽힙니다 — 말할 수 있는 행으로 평탄화됩니다.",
  "Shift+Tab으로 모드를 순환할 때 권한 모드 변경이 소리로 안내됩니다."

마지막 항목이 중요합니다. **Shift+Tab**으로 권한 모드를 순환하는 것은 Claude Code의 핵심 상호작용이며([권한](/docs/claude-code/permissions) 참고), 스크린 리더 모드는 새 모드를 소리로 말해 주므로 상태 표시줄을 읽지 않고도 plan, accept-edits, auto 중 어디에 있는지 항상 알 수 있습니다.

## vim 인체공학: 입력 모드 키 다시 매핑하기

vim 모드를 쓰는데 `Escape`에 손을 뻗는 것이 부담스럽다면, 두 개 키로 이루어진 입력 모드 시퀀스를 대신 매핑하세요. `vimInsertModeRemaps` 설정은 `jj` 같은 시퀀스를 `Escape`로 바꿔 주므로 홈 로우를 벗어날 일이 없습니다.

<PromptCard title="vim 모드에서 jj를 Escape로 매핑 (settings.json)">{`{
  "editorMode": "vim",
  "vimInsertModeRemaps": {
    "jj": "Escape"
  }
}`}</PromptCard>

<Callout type="warning" items={[
  "리매핑은 입력 모드에서만 발동하므로 jj를 문자 그대로 입력하려면 키 사이에 짧은 멈춤이 필요합니다 — 빠르게 칠 일이 드문 시퀀스를 고르세요 (jj와 kj가 흔한 선택입니다).",
  "editorMode: vim은 vim 키바인딩을 활성화합니다. vimInsertModeRemaps는 vim 모드가 켜져 있을 때만 효과가 있습니다."
]} />

## 마우스와 스크롤 제어

전체 화면 메뉴는 이제 **마우스 클릭**을 받습니다 — 다중 선택 메뉴와 "Other" 입력 행 포함 — 키보드 전용 탐색이 힘들다면 도움이 됩니다. 두 가지 설정으로 움직임 민감도나 정밀도에 맞춰 포인터 동작을 조정할 수 있습니다.

| 설정 | 하는 일 |
|---|---|
| `CLAUDE_CODE_DISABLE_MOUSE_CLICKS` | 휠 스크롤은 유지한 채 마우스 클릭, 드래그, 호버를 비활성화합니다 — 의도치 않은 클릭이 방해될 때 유용합니다. |
| `wheelScrollAccelerationEnabled` | `false`로 설정하면 전체 화면에서 마우스 휠 스크롤 가속이 꺼져 더 안정적이고 예측 가능한 스크롤이 됩니다. |

<PromptCard title="더 안정적인 스크롤, 실수 클릭 없음 (settings.json)">{`{
  "wheelScrollAccelerationEnabled": false,
  "env": {
    "CLAUDE_CODE_DISABLE_MOUSE_CLICKS": "1"
  }
}`}</PromptCard>

## 대비와 테마

저시력 사용자에게는 터미널 대비가 첫 번째 레버입니다. Claude Code는 라이트와 다크 테마를 제공하며 `/theme`에서 만들거나 플러그인으로 배포하는 **커스텀 테마**를 지원하므로, 필요에 맞는 고대비 팔레트를 맞출 수 있습니다. 테마 선택이 어디에 저장되는지는 [설정](/docs/claude-code/settings)을, 더 평이하고 장식이 적은 *산문*도 원한다면 [출력 스타일](/docs/claude-code/output-styles)을 참고하세요 (Explanatory와 Learning 스타일은 더 간결한 게 아니라 더 장황합니다 — 간결한 출력을 원하면 Default를 유지하세요).

:::tip CLI 너머
스크린 리더 모드는 터미널을 다룹니다. Claude를 더 넓게 손 없이 쓰려면 앱의 [보이스 모드](/docs/claude-app/voice-mode)에서 말하고 들을 수 있고, [접근성 플레이북](/docs/playbooks/accessibility)은 Claude 자체를 보조 도구로 쓰는 법 — 텍스트 단순화, 체크리스트 작성, 읽는 방식에 맞춘 정보 조정 — 을 다룹니다.
:::
