**출력 스타일**은 Claude Code가 무엇을 할 수 있는지를 바꾸지 않으면서 *어떻게* 소통하는지 — 어조, 장황함, 그리고 추론을 설명할지 여부 — 를 바꿉니다. 응답에 대한 성격/형식 프리셋이라고 생각하세요.

## 왜 사용하는가

- **워크플로에 맞추기.** 빠르게 움직일 때는 간결하고 행동 우선으로, 배우는 중일 때는 설명형으로.
- **교육 모드.** 일부 스타일은 교육적 해설을 덧붙입니다 — 코드베이스나 기법을 익히는 동안 좋습니다.
- **일관성.** 선호하는 스타일을 설정해 모든 세션이 원하는 방식으로 읽히게 하세요.

## 흔한 스타일

대표적인 내장 스타일은 간결한 기본값에서, "왜"를 짚어가는 더 **설명적인** 또는 **학습 지향** 모드까지 다양합니다. 활성 스타일은 해당 명령/설정으로 지정하세요(현재 목록과 정확한 메커니즘은 [공식 문서](https://code.claude.com/docs/en/output-styles)를 참고하세요).

## 출력 스타일 대 CLAUDE.md 대 응답 스타일

- **출력 스타일** = Claude Code 응답의 표현 프리셋(어조/장황함).
- **[CLAUDE.md](/docs/claude-code/claude-md)** = 지속적인 *프로젝트* 사실과 규칙(무엇을 할지, 관례, 가드레일).
- **[커스텀 지침 / 응답 스타일](/docs/claude-app/custom-instructions)** = Claude.ai 소비자 앱 버전의 사촌 격.

:::tip 올바른 레버를 사용하세요
Claude가 "끝내기 전에 항상 테스트를 실행"하게 하고 싶나요? 그것은 스타일이 아니라 **CLAUDE.md 가드레일**입니다. "진행하면서 각 단계를 설명"하게 하고 싶나요? 그것은 **출력 스타일**입니다. 동작 규칙은 CLAUDE.md에, 표현은 스타일에 두세요.
:::
