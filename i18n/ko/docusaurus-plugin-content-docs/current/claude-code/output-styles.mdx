내장 스타일, 정확한 전환 방식, 그리고 사용자 정의 스타일 파일 형식은 릴리스마다 바뀔 수 있습니다. 공식 문서에서 확인하세요(Claude Code 문서는 `docs.anthropic.com`에서 `code.claude.com`으로 이전되었습니다).
</VerifyNote>

**출력 스타일**은 시스템 프롬프트를 직접 수정하여 Claude Code가 소통하는 *방식* — 역할, 어조, 장황함, 그리고 추론 과정을 설명하는지 여부 — 을 바꿉니다. 이는 Claude가 무엇을 *아는지*가 아니라 *어떻게* 응답하는지를 바꿉니다. 매 차례마다 같은 어조나 형식을 얻으려고 계속 다시 프롬프트하고 있다면 이 기능을 활용하세요.

## 왜 사용하나

- **워크플로에 맞추기.** 빠르게 움직일 때는 간결하고 행동 우선; 배우는 중일 때는 설명적으로.
- **교육 모드.** 일부 스타일은 교육적인 해설을 추가합니다 — 코드베이스나 기법을 익히는 동안 아주 유용합니다.
- **일관성.** 선호하는 스타일을 한 번 설정해 두면 모든 세션이 원하는 방식대로 읽힙니다.

## 내장 스타일

Claude Code에는 네 가지가 기본 제공됩니다.

| 스타일 | 하는 일 |
|---|---|
| **Default** | 표준 소프트웨어 엔지니어링 시스템 프롬프트 — 효율적이고 행동 우선. |
| **Proactive** | 일상적인 결정을 위해 멈추는 대신 즉시 실행하고 합리적인 가정을 합니다. 자동 모드보다 강한 자율 실행 지침을 제공하지만, 도구가 실행되기 전에는 여전히 권한 프롬프트가 표시됩니다. |
| **Explanatory** | 단계 사이에 교육적인 "Insights"를 추가하여 구현 선택과 코드베이스 패턴을 설명합니다. |
| **Learning** | 협업적이고 실습 위주: Insights를 공유*하며* 작은 부분은 직접 작성하도록 요청하여 코드에 `[TODO(human)]` 마커를 남깁니다. |

Explanatory와 Learning은 설계상 더 긴 응답을 생성하므로 Default보다 출력 토큰 비용이 더 듭니다.

## 전환하는 방법

`/config`를 실행하고 **Output style** 아래에서 스타일을 고릅니다. 선택한 값은 프로젝트 수준의 `.claude/settings.local.json`에 저장됩니다. 메뉴 없이 설정하려면 아무 설정 파일에서나 `outputStyle` 필드를 편집하세요:

```json
{
  "outputStyle": "Explanatory"
}
```

:::warning 변경에는 새 세션이 필요합니다
출력 스타일은 시스템 프롬프트의 일부이며, Claude Code는 이를 **세션 시작 시 한 번** 읽습니다. 변경은 `/clear` 이후 또는 새 세션에서만 적용됩니다. (독립 실행형 `/output-style` 명령은 v2.1.91에서 제거되었습니다 — `/config`나 위의 설정을 사용하세요.)
:::

## 사용자 정의 스타일 만들기

사용자 정의 스타일은 Markdown 파일입니다: 프런트매터, 그리고 시스템 프롬프트에 덧붙일 지침. 사용자 수준(`~/.claude/output-styles/`)이나 프로젝트 수준(`.claude/output-styles/`)에 저장하세요; `name`을 설정하지 않으면 파일 이름이 스타일 이름이 됩니다.

```markdown
---
name: Diagrams first
description: Lead every explanation with a diagram
keep-coding-instructions: true
---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.
```

Claude가 소통하는 *방식*만 바꾸고 코딩은 평소대로 계속하게 하려면 `keep-coding-instructions: true`를 설정하세요. Claude가 소프트웨어 엔지니어링을 전혀 하지 않을 때 — 가령 순수 글쓰기나 데이터 분석 어시스턴트일 때 — 는 이를 생략(기본값)하여 내장 코딩 지침이 제외되도록 하세요.

## 출력 스타일 대 CLAUDE.md 대 그 밖의 수단들

- **출력 스타일** = 역할, 어조, 형식에 대한 시스템 프롬프트 프리셋 — *모든* 응답에 적용됩니다.
- **[CLAUDE.md](/docs/claude-code/claude-md)** = 영속적인 *프로젝트* 사실 및 규칙(관례, 가드레일)으로, 시스템 프롬프트 뒤에 메시지로 추가됩니다.
- **[사용자 정의 지침 / 응답 스타일](/docs/claude-app/custom-instructions)** = Claude.ai 소비자 앱의 사촌 격입니다.

:::tip 올바른 수단 사용하기
Claude가 "끝내기 전에 항상 테스트를 실행"하게 하고 싶나요? 그건 스타일이 아니라 **CLAUDE.md 가드레일**입니다. "진행하면서 각 단계를 설명"하게 하고 싶나요? 그건 **출력 스타일**입니다. 동작 규칙은 CLAUDE.md에, 표현은 스타일에 두세요.
:::
