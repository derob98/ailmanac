명령 파일 위치와 프런트매터 옵션은 바뀔 수 있습니다 — 공식 슬래시 명령 문서에서 확인하세요.
</VerifyNote>

커스텀 슬래시 명령은 계속 다시 입력하는 프롬프트를 단일 `/word`로 패키징합니다. 한 번 정의하면 영원히 재사용하세요(그리고 팀과 공유하세요).

## 작동 방식

명령은 그저 Markdown 파일입니다. 명령 디렉터리에 넣으면 그 파일명이 명령이 됩니다:

- `.claude/commands/changelog.md` → `/changelog` (프로젝트 명령, 공유 가능)
- `~/.claude/commands/changelog.md` → `/changelog` (개인, 모든 프로젝트)

파일의 본문이 호출 시 Claude가 실행하는 프롬프트입니다.

## 최소 예시

```markdown
---
description: Draft a Conventional Commit message from the staged diff
---

Look at the staged changes with `git diff --cached` and write a single
Conventional Commit message (feat/fix/docs/refactor/chore). Output only the
message, no preamble.
```

이제 `/changelog`… 아니, `/commit`(파일 이름이 무엇이든 그대로 명명하세요)이 매번 그 작업을 합니다.

## 유용한 구성 요소

- **`$ARGUMENTS`** — 명령 뒤에 사용자가 입력한 내용을 주입합니다. `/review src/auth.ts` → 그 경로가 프롬프트에 들어옵니다.
- **bash를 실행하고 출력을 임베드** — 명령은 셸 출력(예: 현재 diff)을 포함할 수 있어 Claude가 실시간 상태에 따라 행동합니다.
- **파일 참조** — 파일을 가리켜 그 내용을 포함합니다.
- **프런트매터** — `description`(메뉴에 표시됨)과 기타 옵션.

인수, bash, 파일 참조의 정확한 구문은 [공식 문서](https://docs.anthropic.com/en/docs/claude-code/slash-commands)를 참고하세요.

## 훔쳐 쓸 만한 명령들

시작용 세트가 우리의 [슬래시 명령 라이브러리](/docs/templates/slash-commands)에 있습니다: 커밋 메시지, PR 설명, 코드 리뷰, 체인지로그, 컴포넌트 스캐폴딩.

## 명령 대 스킬 대 서브에이전트

- **슬래시 명령** — *당신*이 알려진 워크플로를 필요할 때 트리거.
- **[스킬](/docs/claude-code/skills)** — Claude가 관련 있을 때 *자동으로* 전문성을 로드.
- **[서브에이전트](/docs/claude-code/subagents)** — 작업의 한 덩어리를 격리된 에이전트에게 위임.
