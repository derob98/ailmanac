## 작동 방식

명령어는 그냥 Markdown 파일입니다. commands 디렉터리에 넣으면 파일명이 명령어가 됩니다. 파일 본문은 호출했을 때 Claude가 실행하는 프롬프트입니다.

파일의 *위치*가 범위를 결정합니다:

## 최소 예제

완전한 명령어 파일 하나입니다. frontmatter는 `description`을 담고, 그 아래 모든 것이 프롬프트입니다.

<PromptCard title="commit.md — Conventional Commit 초안 작성">{`---
description: Draft a Conventional Commit message from the staged diff
---

Look at the staged changes with \`git diff --cached\` and write a single
Conventional Commit message (feat/fix/docs/refactor/chore). Output only the
message, no preamble.`}</PromptCard>

이것을 `commit.md`로 저장하면 `/commit`이 매번 그 작업을 합니다. (파일명이 무엇이든 그것이 이름이 됩니다 — 같은 본문을 `changelog.md`로 저장하면 `/changelog`가 됩니다.)

## 유용한 구성 요소

네 가지 재료가 정적인 프롬프트를 유연한 명령어로 바꿉니다:

| 구성 요소 | 하는 일 |
| --- | --- |
| **`$ARGUMENTS`** | 사용자가 명령어 뒤에 입력한 것을 주입합니다. `/review src/auth.ts` → 그 경로가 프롬프트에 들어갑니다. |
| **bash 실행 후 출력 임베드** | 명령어에 셸 출력(예: 현재 diff)을 포함시켜 Claude가 실시간 상태에 따라 동작하게 합니다. |
| **파일 참조** | 파일을 가리켜 그 내용을 포함시킵니다. |
| **Frontmatter** | `description`(메뉴에 표시)와 기타 옵션. |

인자, bash, 파일 참조의 정확한 문법은 [공식 문서](https://code.claude.com/docs/en/slash-commands)를 참고하세요.

## 훔쳐 쓸 만한 명령어

시작용 세트가 [Slash Command Library](/docs/templates/slash-commands)에 있습니다: 커밋 메시지, PR 설명, 코드 리뷰, changelog, 컴포넌트 스캐폴딩.

## 명령어 vs 스킬 vs 서브에이전트

작업을 묶는 세 가지 방법 — 차이는 *누가 트리거하는가*와 *얼마나 격리되어 실행되는가*입니다:

- **슬래시 명령어** — *당신*이 아는 워크플로를 필요할 때 트리거합니다.
- **[스킬](/docs/claude-code/skills)** — Claude가 관련 있을 때 *자동으로* 전문성을 로드합니다.
- **[서브에이전트](/docs/claude-code/subagents)** — 작업 덩어리를 격리된 에이전트에 위임합니다.
