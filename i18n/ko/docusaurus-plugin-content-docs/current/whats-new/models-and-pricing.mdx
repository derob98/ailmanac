## 가격과 컨텍스트 한도가 표에 없는 이유

이들은 안전하게 복사하기에는 너무 자주 바뀝니다 — 오래된 가격은 가격이 없는 것보다 더 나쁩니다. 우리는 각 모델의 **정체성**(이름, API ID, 용도)을 나열하고, 항상 권위 있는 정보인 **가격, 컨텍스트 윈도우, 속도 제한**에 대해서는 공식 페이지로 안내합니다.

## 이 페이지를 최신으로 유지하는 방법

- 이 표는 **"마지막 검증" 날짜**를 담은 단일 데이터 파일(`data/models.json`)로부터 렌더링됩니다.
- 이를 업데이트하는 것은 한 줄만 바꾸면 되는 작업으로, 완벽한 [첫 기여](/docs/contribute/contribute-in-10-minutes)입니다.
- 이후 단계에서는 자동화가 이 파일을 공식 모델 엔드포인트와 비교하여 차이가 발생하면 풀 리퀘스트를 엽니다 — 하지만 배포되기 전에는 항상 사람이 검토합니다. ([자동화가 작동하는 방식](/docs/contribute) 참고.)

## 모델 선택하기

대략적인 기준(전체 가이드는 [모델 선택하기](/docs/api/choosing-a-model)에 있습니다):

- **Sonnet으로 시작하세요.** 대부분의 작업에서 품질, 속도, 비용의 균형이 가장 좋습니다.
- 어려운 문제에서 비용보다 품질이 더 중요할 때는 **Opus를 선택하세요.**
- 대량 처리, 지연 시간에 민감한 작업, 또는 단순한 작업에는 **Haiku로 낮추세요.**
