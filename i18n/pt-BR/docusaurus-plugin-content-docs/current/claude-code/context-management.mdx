Toda sessão tem uma [janela de contexto](/docs/foundations/tokens-and-context) finita. À medida que ela enche, o conteúdo mais antigo é comprimido e o Claude pode começar a "desviar". Gerenciar o contexto de forma deliberada é o que mantém sessões longas afiadas.

## Os dois comandos essenciais

- **`/clear`** — apaga a conversa e começa do zero. Use quando mudar para uma **tarefa não relacionada**. A maneira mais barata de recuperar um contexto limpo e focado.
- **`/compact`** — resume a conversa até o momento em uma forma compacta, mantendo a essência enquanto libera espaço. Use quando quiser **continuar a mesma tarefa** mas a janela estiver ficando cheia.

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]
  A -->|Same task, context full| C[/compact]
```

## Compactação automática

Quando o contexto se aproxima do limite, o Claude Code pode fazer **compactação automática** para que o trabalho continue sem uma parada brusca. É uma rede de segurança — mas usar `/clear` proativamente entre as tarefas dá resultados melhores do que depender dela.

## Técnicas para permanecer eficaz

- **Uma tarefa por sessão.** Não deixe uma tarde de trabalhos não relacionados se acumular em um único contexto.
- **Delegue leituras pesadas** para [subagentes](/docs/claude-code/subagents) — eles queimam o *próprio* contexto e retornam apenas a conclusão.
- **Referencie, não cole.** Aponte o Claude para arquivos (`@caminho`) em vez de colar grandes blocos de texto; mantenha o [CLAUDE.md](/docs/claude-code/claude-md) enxuto.
- **Observe o orçamento.** Uma [statusline](/docs/claude-code/statusline) mostrando o uso de contexto indica quando compactar.

:::tip Sintoma → solução
Se o Claude começar a esquecer uma instrução anterior ou a se repetir, a janela provavelmente está cheia. `/compact` (mesma tarefa) ou `/clear` (nova tarefa) costuma resolver na hora. Mais em [Solução de Problemas](/docs/contribute/troubleshooting).
:::
