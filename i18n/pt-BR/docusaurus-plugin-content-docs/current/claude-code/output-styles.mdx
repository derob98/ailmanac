Os **estilos de saída** mudam *como* o Claude Code se comunica — seu tom, sua verbosidade e se ele explica seu raciocínio — sem mudar *o que* ele pode fazer. Pense neles como uma predefinição de personalidade/formato para as respostas.

## Por que usá-los

- **Combine com seu fluxo de trabalho.** Conciso e voltado à ação quando você está se movendo rápido; explicativo quando você está aprendendo.
- **Modo didático.** Alguns estilos adicionam comentários educativos — ótimo enquanto você está conhecendo uma base de código ou uma técnica.
- **Consistência.** Defina um estilo preferido para que cada sessão seja lida do jeito que você gosta.

## Estilos comuns

Os estilos integrados típicos vão de um padrão conciso a modos mais **explicativos** ou **voltados ao aprendizado** que detalham o "porquê". Defina o estilo ativo via o comando/configuração relevante (veja a [documentação oficial](https://code.claude.com/docs/en/output-styles) para a lista atual e o mecanismo exato).

## Estilos de saída vs CLAUDE.md vs estilos de resposta

- **Estilo de saída** = predefinição de apresentação para as respostas do Claude Code (tom/verbosidade).
- **[CLAUDE.md](/docs/claude-code/claude-md)** = fatos e regras persistentes do *projeto* (o que fazer, convenções, proteções).
- **[Instruções personalizadas / estilos de resposta](/docs/claude-app/custom-instructions)** = o primo no app de consumo Claude.ai.

:::tip Use a alavanca certa
Quer que o Claude "sempre execute os testes antes de finalizar"? Isso é uma **proteção do CLAUDE.md**, não um estilo. Quer que ele "explique cada passo conforme avança"? Isso é um **estilo de saída**. Mantenha regras de comportamento no CLAUDE.md e apresentação nos estilos.
:::
