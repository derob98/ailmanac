Você não precisa viver no terminal. O Claude Code se integra ao seu editor para que você revise as mudanças **como diffs, exatamente onde você programa**.

## O que você ganha

- **Diffs inline.** As edições propostas aparecem como um diff revisável no editor, não apenas como texto em um terminal.
- **Contexto do editor.** Seus arquivos abertos, sua seleção e seu projeto ficam disponíveis para o Claude.
- **O mesmo cérebro.** É o mesmo Claude Code — seu [CLAUDE.md](/docs/claude-code/claude-md), [permissões](/docs/claude-code/permissions), [hooks](/docs/claude-code/hooks) e [configurações](/docs/claude-code/settings) se aplicam de forma idêntica entre a CLI e a IDE. Configure uma vez, use em todo lugar.

## Editores suportados

- **VS Code** (e editores baseados em VS Code) via a extensão.
- **IDEs JetBrains** (IntelliJ, PyCharm, WebStorm, etc.) via o plugin.

Instale a partir do marketplace do editor e autentique-se; veja a [documentação oficial](https://docs.anthropic.com/en/docs/claude-code/ide-integrations) para os passos atuais.

## CLI ou IDE — qual?

São a mesma ferramenta; escolha por preferência e tarefa:

| Prefira a IDE quando… | Prefira a CLI quando… |
|---|---|
| Você quer revisar diffs visualmente | Você está automatizando com scripts ou em modo [headless](/docs/claude-code/headless-and-agent-sdk) |
| Você já está programando no editor | Você está em um servidor / via SSH |
| Você gosta do contexto do editor integrado | Você quer controle máximo e pipes |

:::tip As configurações são compartilhadas
Como a CLI e a IDE leem a mesma configuração, configure seu [CLAUDE.md](/docs/claude-code/claude-md) e suas [permissões](/docs/claude-code/permissions) uma vez e eles o acompanham entre elas.
:::
