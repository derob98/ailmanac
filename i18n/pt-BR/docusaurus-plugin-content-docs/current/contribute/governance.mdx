Como o AILmanac é administrado — mantido leve para que uma equipe pequena mais voluntários consigam sustentá-lo.

## Independência & marcas registradas

O AILmanac é um **recurso comunitário independente**. Ele **não é afiliado nem endossado pela Anthropic.** "Claude" e "Anthropic" são marcas registradas da Anthropic, usadas de forma nominativa. A [documentação oficial](https://docs.anthropic.com) é a fonte da verdade.

## Licenciamento (duplo)

Para que tanto as palavras quanto o código sejam reutilizáveis:

- **Prosa / documentação → CC BY 4.0** (reutilize com atribuição).
- **Código, exemplos, configuração → MIT.**

Veja `LICENSE-CONTENT`, `LICENSE-CODE` e `NOTICE` no repositório. Ao contribuir, você concorda em licenciar sua prosa sob a CC BY 4.0 e o código sob a MIT.

## Código de Conduta

Seguimos o **Contributor Covenant** (veja `CODE_OF_CONDUCT.md`). Seja gentil, seja construtivo, presuma boa-fé. Comportamento inaceitável pode ser reportado de forma privada aos mantenedores.

## Modelo de contribuição

- **Assinatura DCO**, não um CLA — `git commit -s` certifica que você pode submeter o trabalho. Barreira baixa, procedência clara.
- **Pull requests** para tudo; o [portão de build](/docs/contribute/automation) e a revisão de um mantenedor mantêm a qualidade alta.
- **CODEOWNERS** direciona áreas para os revisores conforme o projeto cresce.

## Rótulos & triagem

Rótulos como código (`.github/labels.yml`) mantêm o backlog legível:

- **Tipo:** `content`, `code`, `i18n`, `automation`
- **Área:** `core-claude`, `ring-general`, `infra`
- **Workflow:** `good first issue`, `help wanted`, `upstream-change`, `needs-verification`, `link-rot`, `stale`

Novos contribuidores: procure por **`good first issue`** — adicionar um termo ao glossário, preencher uma página planejada ou corrigir um fato desatualizado são começos perfeitos.

## Decisões & discussão

- **GitHub Discussions** para perguntas e propostas; **Issues** para trabalho acionável.
- Os contribuidores são reconhecidos no repositório.
