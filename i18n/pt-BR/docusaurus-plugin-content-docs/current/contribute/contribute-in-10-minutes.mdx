O AILmanac só é "completo e sempre atualizado" porque pessoas como você contribuem com ele. Você **não** precisa ser desenvolvedor — corrigir um erro de digitação, adicionar um termo ao glossário ou compartilhar um template são todas contribuições reais.

## O caminho rápido

1. **Encontre a página** no site e clique em **"Edit this page"** no rodapé (isso abre o arquivo no GitHub).
2. Clique no lápis ✏️ para editar no seu navegador. O GitHub faz um fork do repositório para você automaticamente.
3. Faça sua alteração em Markdown.
4. No rodapé, escreva uma breve descrição e **propose changes** → **open a pull request**.
5. Um mantenedor revisa. É isso. 🎉

Prefere fazer localmente? Clone o repositório e então:

```bash
npm install
npm run start      # live preview at http://localhost:3000
npm run build      # must pass: it fails on broken links
```

## As 4 regras que toda página segue

Seguir estas regras torna a revisão rápida e mantém a qualidade alta:

1. **Marque o nível.** Comece o corpo com `<LevelBadge level="beginner" />` (ou `intermediate` / `advanced` / `all`).
2. **Cite a fonte para fatos voláteis.** Qualquer coisa sobre nomes de modelos, preços, limites ou funcionalidades que mude deve citar a página oficial — e usar `<VerifyNote lastVerified="YYYY-MM-DD" source="..." />`. Nunca fixe um fato de modelo no código; faça link para a [tabela de modelos](/docs/whats-new/models-and-pricing).
3. **Seja opinativo, depois honesto.** Apresente primeiro a forma recomendada; anote as alternativas. Não duplique a documentação oficial — faça link para ela.
4. **Inglês simples, exemplos em vez de teoria.** Frases curtas. Mostre, não apenas conte.

## Template de front-matter da página

```markdown
---
sidebar_position: 2
title: Your Page Title
description: One sentence shown in search and previews.
keywords: [relevant, search, terms]
---

<LevelBadge level="beginner" />

Your first paragraph...
```

## O que faz uma ótima primeira contribuição

- Adicione um termo de **glossário** que esteja faltando.
- Adicione um **template** ou prompt que você já usa.
- Corrija algo **recém-lançado** que uma página agora descreve errado (e atualize a data de `lastVerified`).
- Traduza uma página — veja o [Manual de Tradução](/docs/contribute).

## Licenciamento & assinatura

Ao contribuir, você concorda que sua prosa é compartilhada sob a licença **CC BY 4.0** e o código sob a **MIT**, e adiciona uma linha `Signed-off-by` (DCO) aos seus commits (`git commit -s`). Detalhes na [página de governança](/docs/contribute).

Seja gentil — seguimos o Contributor Covenant [Código de Conduta](/docs/contribute).
