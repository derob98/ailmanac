O **Model Context Protocol (MCP)** é o padrão aberto para conectar IA a ferramentas e dados externos. Na API, o MCP permite que você dê ao Claude acesso a um **servidor inteiro de ferramentas** por referência — em vez de definir cada ferramenta manualmente.

## MCP vs. ferramentas definidas manualmente

| | [Uso de ferramentas](/docs/api/tool-use) (personalizado) | MCP |
|---|---|---|
| Você define | O schema de cada ferramenta, e a executa você mesmo | Uma conexão com um servidor que *publica* ferramentas |
| Melhor para | Algumas funções sob medida no seu app | Reutilizar integrações existentes (GitHub, BDs, navegadores, SaaS) |
| Autenticação | Seu código | OAuth ou chaves nativas do servidor |

Eles coexistem: defina suas ferramentas específicas do app diretamente e incorpore capacidades prontas via MCP.

## Declarando um servidor

Você anexa servidor(es) MCP a uma requisição/agente (URL ou comando, mais autenticação). O Claude pode então chamar as ferramentas do servidor dentro do [loop do agente](/docs/api/building-agents), como qualquer ferramenta. O campo exato da requisição e o fluxo de autenticação estão na [documentação oficial](https://platform.claude.com/docs/en/build-with-claude/mcp).

## Mesmo padrão, três superfícies

- **API** (esta página) — servidores MCP nas suas requisições/agentes.
- **[Claude Code](/docs/claude-code/mcp)** — servidores MCP nas suas sessões de desenvolvimento.
- **[Os apps](/docs/claude-app/connectors)** — o MCP alimenta os Conectores.

Aprenda uma vez; o conhecimento é transferível.

## Confiança

:::warning Um servidor MCP é código + acesso
Conecte apenas servidores em que você confia, restrinja-os ao privilégio mínimo e lembre-se de que o conteúdo externo que eles retornam pode carregar [prompt injection](/docs/security/prompt-injection). Revise primeiro servidores de terceiros — [Revisando Código de Terceiros](/docs/security/reviewing-third-party-code).
:::
