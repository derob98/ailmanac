A maior parte do [prompting](/docs/prompting/basics) é universal. Mas algumas técnicas são especialmente eficazes com o Claude — e alguns hábitos de modelos mais antigos são agora contraproducentes. Esta lição separa o sinal do ruído: o que é de fato específico do Claude.

## Aposte nestes com o Claude

Estes são os movimentos que mais compensam com o Claude. O primeiro é o hábito de maior valor que você pode construir.

- **[Tags XML](/docs/prompting/xml-tags).** O Claude é bem ajustado para prompts estruturados com tags nomeadas para entradas, instruções e exemplos. Este é o hábito de maior valor com o Claude.
- **Seja claro e direto.** Declare o objetivo, as restrições e como é "estar pronto". O Claude segue de perto as instruções explícitas.
- **Atribua um papel** via [system prompt](/docs/foundations/roles).
- **Deixe-o pensar em problemas difíceis** — peça raciocínio passo a passo, ou use [extended thinking/effort](/docs/api/thinking-and-effort) em modelos de raciocínio (onde muitas vezes você *não* precisa pedir CoT explicitamente).
- **Use exemplos** ([few-shot](/docs/prompting/few-shot)) para formato e casos extremos.
- **Faça prefill** do início da resposta do Claude (na API) para direcionar o formato.

### Experimente o prefilling

Fazer prefill significa começar o turno do assistente você mesmo, para que o Claude continue no formato que você quer. Na API, comece o turno do assistente com um caractere de abertura ou um título:

<PromptCard title="Prefill para forçar saída em JSON">{`{`}</PromptCard>

<PromptCard title="Prefill de um título para forçar estrutura">{`## Summary`}</PromptCard>

<Callout type="tip" items={["Comece o turno do assistente com { para forçar JSON, ou com um título para forçar saída estruturada. O prefilling é uma alavanca exclusiva da API — direciona o formato sem gastar palavras em instruções."]} />

## Abandone estes hábitos antigos

Modelos Claude mais novos, que seguem instruções de forma literal, mudam as regras. Truques que ajudavam em modelos mais antigos agora atrapalham.

<Callout type="warning" items={["Vá com calma no CAIXA-ALTA e em prompts em tom de ameaça (\"VOCÊ DEVE… SENÃO\"). Os modelos Claude mais novos seguem instruções literalmente, e instruções específicas e formuladas com polidez funcionam melhor do que gritar.", "Não exagere na engenharia com andaimes elaborados em estilo \"jailbreak\" — instruções claras e honestas superam truques.", "Menos costuma ser mais — um prompt enxuto vence um prolixo que esconde o pedido."]} />

## Claude vs. outros modelos

Os *princípios* se transferem (veja [Tradução de Prompts entre IAs](/docs/prompting/cross-ai-translation)), mas duas coisas são as mais dignas de lembrar como específicas do Claude.
