Boa notícia: **os fundamentos do prompting se transferem para todos os modelos.** Se você sabe fazer prompts bem com o Claude, sabe fazer prompts bem com o ChatGPT, o Gemini ou o Llama. Alguns detalhes de superfície diferem — aqui está a pedra de Roseta.

## O que se transfere em todo lugar

Estes funcionam em todos eles — têm a ver com *comunicação*, não com um fornecedor:

- **Dê contexto**: objetivo, público, formato, tom ([Fundamentos do Prompting](/docs/prompting/basics)).
- **Exemplos** ([few-shot](/docs/prompting/few-shot)).
- **Peça raciocínio** em tarefas difíceis.
- **Restrinja a saída** (tamanho, formato, "apenas JSON").
- **Itere** em linguagem comum.
- **Papéis / system prompts** — todo modelo importante tem o conceito ([Papéis](/docs/foundations/roles)).

Aprenda isso uma vez e você se torna portável.

## O que difere (e o que ajustar)

| Aspecto | Observe ao trocar |
|---|---|
| **Pistas de estrutura** | O Claude adora [tags XML](/docs/prompting/xml-tags); outros modelos as aceitam, mas podem preferir títulos em Markdown ou JSON. |
| **Intensidade do tom** | Modelos mais antigos toleravam prompts agressivos/em CAIXA-ALTA; os mais novos, que seguem literalmente, preferem instruções calmas e específicas. Reajuste por modelo. |
| **Raciocínio** | "Pense passo a passo" ajuda alguns; modelos de raciocínio dedicados fazem isso internamente — não duplique. |
| **Prefilling** | Colocar palavras na boca do assistente é uma alavanca no estilo Claude; o suporte varia em outros lugares. |
| **Peso do system prompt** | Todos têm um, mas o quão fortemente ele é seguido varia — teste. |
| **Chamada de ferramentas/funções** | Mesmo conceito, esquemas de requisição diferentes por fornecedor. |

## Um fluxo de trabalho portável

1. Escreva o prompt com estrutura **universal** (contexto + formato + exemplos).
2. **Teste no seu modelo-alvo** com algumas entradas reais ([eval](/docs/foundations/evals)).
3. Ajuste apenas os **detalhes de superfície** acima, se necessário.
4. Mantenha os prompts em controle de versão para que trocar de modelo seja barato.

:::tip Não reescreva do zero
Mudar para um novo modelo raramente exige um novo prompt — geralmente só um ajuste de tom/formato. Sua estrutura e seus exemplos se mantêm.
:::
