Boa notícia: **os fundamentos de prompting se transferem para todos os modelos.** Se você sabe fazer bons prompts para o Claude, sabe fazer bons prompts para o ChatGPT, o Gemini ou o Llama. Alguns detalhes de superfície diferem — aqui está a pedra de Roseta.

## O que se transfere em todo lugar

Estes funcionam em todos eles — têm a ver com *comunicação*, não com um fornecedor:

- **Dê contexto**: objetivo, público, formato, tom ([Fundamentos de Prompting](/docs/prompting/basics)).
- **Exemplos** ([few-shot](/docs/prompting/few-shot)).
- **Peça raciocínio** em tarefas difíceis.
- **Restrinja a saída** (tamanho, formato, "apenas JSON").
- **Itere** em linguagem natural.
- **Papéis / prompts de sistema** — todo modelo importante tem o conceito ([Papéis](/docs/foundations/roles)).

Aprenda isto uma vez e você se torna portátil.

## O que difere (e o que ajustar)

| Aspecto | Observação ao trocar |
|---|---|
| **Sinais de estrutura** | O Claude adora [tags XML](/docs/prompting/xml-tags); outros modelos as aceitam, mas podem preferir títulos em Markdown ou JSON. |
| **Intensidade do tom** | Modelos mais antigos toleravam prompts enfáticos/em CAIXA-ALTA; os mais novos, que seguem literalmente, preferem instruções calmas e específicas. Reajuste por modelo. |
| **Raciocínio** | "Pense passo a passo" ajuda alguns; modelos de raciocínio dedicados fazem isso internamente — não duplique. |
| **Prefilling** | Colocar palavras na boca do assistente é uma alavanca no estilo Claude; o suporte varia em outros lugares. |
| **Peso do prompt de sistema** | Todos têm um, mas o quanto ele é seguido varia — teste. |
| **Chamada de ferramentas/funções** | Mesmo conceito, esquemas de requisição diferentes por fornecedor. |

## Um fluxo de trabalho portátil

<Steps items={[
  {title: "Escreva com estrutura universal", body: "Rascunhe o prompt com contexto + formato + exemplos — as partes que se transferem para qualquer modelo. Ainda não embuta sintaxe específica de fornecedor."},
  {title: "Teste no seu modelo-alvo", body: "Rode algumas entradas reais pelo modelo em que você realmente vai publicar (/docs/foundations/evals). Julgue as saídas, não sua intuição sobre o modelo."},
  {title: "Ajuste apenas os detalhes de superfície", body: "Se algo estiver errado, mude uma alavanca de superfície por vez — estilo de delimitador, intensidade do tom, se você pede raciocínio — não o prompt inteiro."},
  {title: "Mantenha os prompts em controle de versão", body: "Guarde os prompts como arquivos para que trocar de modelo seja um diff pequeno e revisável em vez de uma reescrita."}
]} />

Como só a *superfície* muda, um único prompt bem estruturado normalmente migra com um ajuste de uma linha. Aqui está um escrito da forma universal — contexto, formato explícito e uma tarefa clara, sem sintaxe específica de fornecedor para remover:

<PromptCard title="Um prompt portátil e neutro em relação a fornecedores">{`You are a technical editor. Rewrite the text below to be clearer and more concise.

Constraints:
- Keep every fact unchanged.
- Target a general professional audience.
- Return only the rewritten text, no preamble.

Text:
"""
[paste the draft here]
"""`}</PromptCard>

Esse prompt roda como está no Claude, no GPT, no Gemini ou em um modelo aberto. Se um modelo ignorar "retorne apenas o texto reescrito", esse é o seu único ajuste de superfície — aperte a restrição ou, no Claude, faça o [prefill](/docs/prompting/prompting-claude) da resposta.

:::tip Não reescreva do zero
Mudar para um novo modelo raramente exige um novo prompt — geralmente só um ajuste de tom/formato. Sua estrutura e seus exemplos se mantêm.
:::
