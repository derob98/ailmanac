Por padrão, cada conversa é uma [folha em branco](/docs/foundations/tokens-and-context). Os recursos de **memória** permitem que o Claude carregue fatos úteis sobre você entre as conversas — para que você não precise se reapresentar toda vez.

## O que "memória" significa aqui

Há algumas coisas distintas que as pessoas chamam de "memória" — não as confunda:

| Tipo | O que é |
|---|---|
| **Janela de contexto** | Memória de trabalho *dentro* de uma conversa (esquecida quando ela termina) — veja [Tokens e Contexto](/docs/foundations/tokens-and-context) |
| **Memória entre conversas** | O Claude lembra fatos sobre você *entre* conversas (o tema desta página) |
| **[Instruções personalizadas](/docs/claude-app/custom-instructions)** | Fatos permanentes que *você* define explicitamente |
| **[Projetos](/docs/claude-app/projects)** | Conhecimento com escopo em um espaço de trabalho |

## Como a memória entre conversas ajuda

Quando habilitada, o Claude pode lembrar coisas como suas preferências, projetos em andamento ou detalhes que você compartilhou — fazendo com que pareça mais um assistente contínuo do que um estranho a cada sessão. A disponibilidade e os controles dependem do seu plano.

## Mantenha o controle

- **Revise e edite** o que é lembrado (onde o aplicativo oferece controles).
- **Limpe ou desative** se quiser uma folha em branco ou mais privacidade.
- **Não dependa dela para a exatidão** de fatos importantes — confirme detalhes críticos em vez de confiar na lembrança.
- **Atenção à privacidade** — a memória persiste informações; não armazene dados sensíveis que você não gostaria que fossem mantidos. Veja [Privacidade](/docs/foundations/privacy).

:::tip O explícito vence o implícito
Para fatos que *precisam* ser aplicados (seu papel, unidades, regras rígidas), defina [instruções personalizadas](/docs/claude-app/custom-instructions) ou use um [Projeto](/docs/claude-app/projects) — isso é mais confiável do que torcer para que a memória os recupere.
:::
