As políticas de tratamento de dados e de treinamento diferem por provedor, produto e plano (consumidor vs. API vs. enterprise) e mudam ao longo do tempo. Sempre confirme os termos atuais com o seu provedor.
</VerifyNote>

Uma pergunta simples com consequências reais: **o que acontece com o texto que você envia a uma IA?** A resposta depende do provedor, do produto e do seu plano — então o hábito seguro é *saber antes de colar*.

## As perguntas a responder para a sua ferramenta

- **Minha entrada é usada para treinar modelos?** Isso varia muito — os planos de chat de consumidor, API e enterprise costumam diferir. Muitas ofertas de **API/enterprise** **não** treinam com os seus dados por padrão; algumas configurações de consumidor podem treinar, dependendo das suas escolhas. Verifique.
- **Por quanto tempo os dados são retidos** e quem pode acessá-los?
- **Onde eles são processados** (região/residência) e são criptografados?
- **Existe uma opção de zero retenção ou business/enterprise** para trabalho sensível?

## Regras práticas de bolso

- **Não cole segredos** — chaves de API, senhas, tokens — em nenhuma ferramenta de IA. Nunca.
- **Minimize dados pessoais.** Envie só o que a tarefa precisa; censure nomes, IDs, números de conta quando puder.
- **Os dados de outras pessoas são delas.** Tenha cuidado com PII de clientes, dados de saúde ou financeiros — pode haver obrigações legais (LGPD, GDPR, HIPAA, etc.).
- **Use o plano certo.** Para trabalho regulado ou confidencial, use uma opção business/enterprise/zero-retenção, não uma conta pessoal.
- **Considere modelos locais/self-hosted** quando os dados realmente não puderem sair do seu ambiente — você troca alguma capacidade por controle total.

## On-device vs. nuvem

| | Nuvem (hospedado) | Local / self-hosted |
|---|---|---|
| Capacidade | A mais alta | Menor (mas melhorando) |
| Dados saem da sua máquina | Sim (conforme os termos do provedor) | Não |
| Configuração/custo | Fácil, pague por uso | Mais esforço, seu hardware |
