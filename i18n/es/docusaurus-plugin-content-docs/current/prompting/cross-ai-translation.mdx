Buenas noticias: **los fundamentos del prompting se transfieren a todos los modelos.** Si sabes dar buenos prompts a Claude, también puedes dárselos bien a ChatGPT, Gemini o Llama. Algunos detalles de superficie difieren — aquí tienes la piedra de Rosetta.

## Lo que se transfiere a todas partes

Esto funciona en todos ellos — tiene que ver con la *comunicación*, no con un proveedor:

- **Da contexto**: objetivo, audiencia, formato, tono ([Fundamentos del prompting](/docs/prompting/basics)).
- **Ejemplos** ([few-shot](/docs/prompting/few-shot)).
- **Pide razonamiento** en tareas difíciles.
- **Restringe la salida** (longitud, formato, "solo JSON").
- **Itera** en lenguaje sencillo.
- **Roles / system prompts** — todos los modelos importantes tienen el concepto ([Roles](/docs/foundations/roles)).

Apréndelos una vez y serás portable.

## Lo que difiere (y qué ajustar)

| Aspecto | Nota al cambiar |
|---|---|
| **Pistas de estructura** | A Claude le encantan las [etiquetas XML](/docs/prompting/xml-tags); otros modelos las aceptan pero pueden preferir encabezados Markdown o JSON. |
| **Intensidad del tono** | Los modelos antiguos toleraban prompts contundentes o en MAYÚSCULAS; los nuevos, que siguen las instrucciones de forma literal, prefieren instrucciones calmadas y específicas. Reajústalo según el modelo. |
| **Razonamiento** | "Piensa paso a paso" ayuda a algunos; los modelos de razonamiento dedicados lo hacen internamente — no lo dupliques. |
| **Prefilling** | Poner palabras en boca del asistente es una palanca al estilo de Claude; el soporte varía en otros sitios. |
| **Peso del system prompt** | Todos tienen uno, pero cuán estrictamente se sigue varía — pruébalo. |
| **Llamada a herramientas/funciones** | Mismo concepto, distintos esquemas de solicitud según el proveedor. |

## Un flujo de trabajo portable

1. Escribe el prompt con una estructura **universal** (contexto + formato + ejemplos).
2. **Pruébalo en tu modelo objetivo** con algunas entradas reales ([eval](/docs/foundations/evals)).
3. Ajusta solo los **detalles de superficie** de arriba si es necesario.
4. Mantén los prompts bajo control de versiones para que cambiar de modelo salga barato.

:::tip No lo reescribas desde cero
Pasar a un modelo nuevo rara vez requiere un prompt nuevo — normalmente solo un retoque de tono o formato. Tu estructura y tus ejemplos se transfieren.
:::
