Las políticas de manejo de datos y de entrenamiento difieren según el proveedor, el producto y el plan (consumidor vs. API vs. empresa) y cambian con el tiempo. Confirma siempre los términos vigentes con tu proveedor.
</VerifyNote>

Una pregunta sencilla con consecuencias reales: **¿qué le pasa al texto que envías a una IA?** La respuesta depende del proveedor, del producto y de tu plan — así que el hábito seguro es *saberlo antes de pegar*.

## Las preguntas que debes responder para tu herramienta

- **¿Mi entrada se usa para entrenar modelos?** Esto varía mucho — los niveles de chat de consumidor, API y empresa a menudo difieren. Muchas ofertas de **API/empresa** **no** entrenan con tus datos por defecto; algunas configuraciones de consumidor podrían hacerlo, según tus elecciones. Compruébalo.
- **¿Cuánto tiempo se retienen los datos** y quién puede acceder a ellos?
- **¿Dónde se procesan** (región/residencia) y están cifrados?
- **¿Existe una opción de retención cero o de empresa/negocio** para el trabajo sensible?

## Reglas prácticas

- **No pegues secretos** — claves de API, contraseñas, tokens — en ninguna herramienta de IA. Nunca.
- **Minimiza los datos personales.** Envía solo lo que la tarea necesita; redacta nombres, identificadores, números de cuenta cuando puedas.
- **Los datos de otras personas son sus datos.** Ten cuidado con la PII de clientes, información de salud o financiera — puede haber obligaciones legales (RGPD, HIPAA, etc.).
- **Usa el nivel adecuado.** Para trabajo regulado o confidencial, usa una opción de empresa/negocio/retención cero, no una cuenta personal.
- **Considera modelos locales/autoalojados** cuando los datos realmente no puedan salir de tu entorno — sacrificas algo de capacidad a cambio de control total.

## En el dispositivo vs. en la nube

| | Nube (alojado) | Local / autoalojado |
|---|---|---|
| Capacidad | La más alta | Menor (pero mejorando) |
| Los datos salen de tu máquina | Sí (según los términos del proveedor) | No |
| Configuración/coste | Fácil, pago por uso | Más esfuerzo, tu hardware |
