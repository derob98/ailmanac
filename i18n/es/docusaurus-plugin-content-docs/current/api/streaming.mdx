Crear experiencias tipo chat con la API se reduce a dos realidades prácticas: usa **streaming** para que los usuarios vean la salida de inmediato, y **gestiona tú mismo el historial** porque la API es sin estado. Domina ambas y tu UX de chat se sentirá rápida y lo recordará todo.

## Streaming

Sin streaming, el usuario espera a la respuesta completa. Con streaming, los tokens llegan a medida que se generan, lo que da una velocidad percibida mucho mejor.

<Callout type="tip" items={["Usa el ayudante de streaming del SDK en lugar de analizar los eventos crudos a mano: gestiona por ti el ciclo de vida de los eventos."]} />

<Tabs groupId="lang">
<TabItem value="python" label="Python">

```python
with client.messages.stream(
    model="claude-sonnet-5", max_tokens=1024,
    messages=[{"role": "user", "content": "Explain RAG in two sentences."}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
```

</TabItem>
<TabItem value="ts" label="TypeScript">

```ts
const stream = client.messages.stream({
  model: "claude-sonnet-5", max_tokens: 1024,
  messages: [{ role: "user", content: "Explain RAG in two sentences." }],
});
for await (const event of stream) {
  if (event.type === "content_block_delta") process.stdout.write(event.delta.text ?? "");
}
```

## Multiturno: tú gestionas el historial

La API **no tiene memoria** entre llamadas ([por qué](/docs/foundations/tokens-and-context)). Para continuar una conversación, vuelve a enviar **todo el intercambio previo** cada vez.

<Steps items={[{title: "Inicia la conversación", body: "Envía el primer mensaje del usuario en la lista de mensajes."}, {title: "Captura la respuesta", body: "Lee el texto de la respuesta del asistente desde el resultado."}, {title: "Añade ambos turnos", body: "Agrega la respuesta del asistente y el siguiente mensaje del usuario a la misma lista de mensajes."}, {title: "Reenvía la lista completa", body: "Envía de nuevo la lista de mensajes completa: así es como Claude 'recuerda' el intercambio."}]} />

```python
# ... get assistant reply, then append both turns:
messages.append({"role": "assistant", "content": assistant_text})
messages.append({"role": "user", "content": "Make it 3 days."})

```

## Las conversaciones largas llenan la ventana

A medida que crece el historial, consume la [ventana de contexto](/docs/foundations/tokens-and-context) y el coste aumenta. Estrategias para mantenerlo bajo control:

- **Resume/compacta** los turnos más antiguos en un breve recordatorio que arrastres hacia adelante.
- **Recorta** los turnos anteriores irrelevantes.
- Combínalo con la **[caché de prompts](/docs/api/prompt-caching)** para evitar volver a pagar por un prefijo estable.

<Callout type="warning" items={["Cada llamada reenvía todo el historial: las conversaciones largas cuestan más y, con el tiempo, pueden superar la ventana de contexto si nunca compactas ni recortas."]} />
