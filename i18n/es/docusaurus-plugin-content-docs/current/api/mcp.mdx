El **Model Context Protocol (MCP)** es el estándar abierto para conectar la IA con herramientas y datos externos. En la API, MCP te permite dar a Claude acceso a todas las **herramientas de un servidor** por referencia, en lugar de definir cada herramienta a mano.

## MCP vs. herramientas definidas a mano

| | [Uso de herramientas](/docs/api/tool-use) (personalizado) | MCP |
|---|---|---|
| Tú defines | El esquema de cada herramienta, y la ejecutas tú mismo | Una conexión a un servidor que *publica* herramientas |
| Ideal para | Unas pocas funciones a medida en tu aplicación | Reutilizar integraciones existentes (GitHub, bases de datos, navegadores, SaaS) |
| Autenticación | Tu código | OAuth o claves nativas del servidor |

Coexisten: define directamente tus herramientas específicas de la aplicación e incorpora capacidades ya hechas vía MCP.

## Declarar un servidor

Adjuntas uno o varios servidores MCP a una solicitud/agente (URL o comando, más la autenticación). Claude puede entonces llamar a las herramientas del servidor dentro del [bucle del agente](/docs/api/building-agents), igual que cualquier herramienta. El campo exacto de la solicitud y el flujo de autenticación están en la [documentación oficial](https://platform.claude.com/docs/en/docs/build-with-claude/mcp).

## El mismo estándar, tres superficies

- **API** (esta página) — servidores MCP en tus solicitudes/agentes.
- **[Claude Code](/docs/claude-code/mcp)** — servidores MCP en tus sesiones de desarrollo.
- **[Las aplicaciones](/docs/claude-app/connectors)** — MCP impulsa los conectores.

Apréndelo una vez; se transfiere.

## Confianza

:::warning Un servidor MCP es código + acceso
Conecta solo servidores en los que confíes, restríngelos al mínimo privilegio y recuerda que el contenido externo que devuelven puede contener [inyección de prompts](/docs/security/prompt-injection). Revisa primero los servidores de terceros: [Revisar código de terceros](/docs/security/reviewing-third-party-code).
:::
