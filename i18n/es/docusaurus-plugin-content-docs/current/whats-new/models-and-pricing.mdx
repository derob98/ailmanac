## Por qué los precios y los límites de contexto no están en la tabla

Cambian con demasiada frecuencia para copiarlos de forma segura — un precio obsoleto es peor que ningún precio. Listamos la **identidad** de cada modelo (nombre, ID de API, para qué sirve) y te remitimos a la página oficial para **precios, ventanas de contexto y límites de tasa**, que siempre es la fuente autorizada.

## Cómo lo mantenemos al día

- La tabla se genera a partir de un único archivo de datos (`data/models.json`) que lleva una **fecha de "última verificación"**.
- Actualizarla es un cambio de una sola línea — una [primera contribución](/docs/contribute/contribute-in-10-minutes) perfecta.
- En una fase posterior, la automatización comparará este archivo con el endpoint oficial de modelos y abrirá una pull request cuando algo se desvíe — pero siempre revisa un humano antes de que se publique. (Consulta [Cómo funciona nuestra automatización](/docs/contribute).)

## Cómo elegir entre ellos

Una regla rápida (la guía completa está en [Elegir un modelo](/docs/api/choosing-a-model)):

- **Empieza con Sonnet.** El mejor equilibrio entre calidad, velocidad y coste para la mayoría del trabajo.
- **Recurre a Opus** cuando la calidad en un problema difícil importe más que el coste.
- **Baja a Haiku** para tareas de alto volumen, sensibles a la latencia o sencillas.
