Cómo se gestiona AILmanac, manteniéndolo ligero para que un equipo pequeño más voluntarios puedan sostenerlo.

## Independencia y marcas registradas

AILmanac es un **recurso comunitario independiente**. **No está afiliado ni respaldado por Anthropic.** "Claude" y "Anthropic" son marcas registradas de Anthropic, usadas de forma nominativa. La [documentación oficial](https://platform.claude.com/docs) es la fuente de la verdad.

## Licencias (duales)

Para que tanto las palabras como el código sean reutilizables:

- **Prosa / documentación → CC BY 4.0** (reutilización con atribución).
- **Código, ejemplos, configuración → MIT.**

Consulta `LICENSE-CONTENT`, `LICENSE-CODE` y `NOTICE` en el repositorio. Al contribuir, aceptas licenciar tu prosa bajo CC BY 4.0 y tu código bajo MIT.

## Código de conducta

Seguimos el **Contributor Covenant** (consulta `CODE_OF_CONDUCT.md`). Sé amable, sé constructivo, presume buena fe. Los comportamientos inaceptables pueden reportarse de forma privada a los responsables del mantenimiento.

## Modelo de contribución

- **Firma DCO**, no un CLA — `git commit -s` certifica que puedes enviar el trabajo. Barrera baja, procedencia clara.
- **Pull requests** para todo; la [puerta de compilación](/docs/contribute/automation) y la revisión de un responsable mantienen la calidad alta.
- **CODEOWNERS** dirige cada área a los revisores a medida que el proyecto crece.

## Etiquetas y clasificación

Las etiquetas como código (`.github/labels.yml`) mantienen el backlog legible:

- **Tipo:** `content`, `code`, `i18n`, `automation`
- **Área:** `core-claude`, `ring-general`, `infra`
- **Flujo de trabajo:** `good first issue`, `help wanted`, `upstream-change`, `needs-verification`, `link-rot`, `stale`

Nuevos colaboradores: busca **`good first issue`** — añadir un término al glosario, completar una página planificada o corregir un dato desactualizado son comienzos perfectos.

## Decisiones y debate

- **GitHub Discussions** para preguntas y propuestas; **Issues** para el trabajo accionable.
- A los colaboradores se les reconoce en el repositorio.
