AILmanac solo está "completo y siempre actualizado" porque personas como tú aportan a él. **No** necesitas ser desarrollador — corregir una errata, añadir un término al glosario o compartir una plantilla son todas contribuciones reales.

## La vía rápida

1. **Encuentra la página** en el sitio y haz clic en **"Edit this page"** al final (abre el archivo en GitHub).
2. Haz clic en el lápiz ✏️ para editar en tu navegador. GitHub bifurca el repositorio por ti automáticamente.
3. Haz tu cambio en Markdown.
4. Al final, escribe una breve descripción y **propose changes** → **open a pull request**.
5. Un responsable lo revisa. Eso es todo. 🎉

¿Prefieres trabajar en local? Clona el repositorio y luego:

```bash
npm install
npm run start      # live preview at http://localhost:3000
npm run build      # must pass: it fails on broken links
```

## Las 4 reglas que sigue cada página

Cumplirlas hace que la revisión sea rápida y mantiene la calidad alta:

1. **Etiqueta el nivel.** Empieza el cuerpo con `<LevelBadge level="beginner" />` (o `intermediate` / `advanced` / `all`).
2. **Enlaza la fuente de los datos volátiles.** Cualquier cosa sobre nombres de modelos, precios, límites o funciones que cambie debe citar la página oficial — y usar `<VerifyNote lastVerified="YYYY-MM-DD" source="..." />`. Nunca codifiques un dato de un modelo a mano; enlaza a la [tabla de modelos](/docs/whats-new/models-and-pricing).
3. **Sé categórico y luego honesto.** Da primero la forma recomendada; señala las alternativas. No dupliques la documentación oficial — enlaza a ella.
4. **Inglés sencillo, ejemplos antes que teoría.** Frases cortas. Muestra, no solo cuentes.

## Plantilla de front-matter de una página

```markdown
---
sidebar_position: 2
title: Your Page Title
description: One sentence shown in search and previews.
keywords: [relevant, search, terms]
---

<LevelBadge level="beginner" />

Your first paragraph...
```

## Qué hace que una primera contribución sea estupenda

- Añade un término que falte en el **glosario**.
- Añade una **plantilla** o un prompt que ya uses.
- Corrige algo **recién publicado** que una página ahora describe mal (y actualiza su fecha `lastVerified`).
- Traduce una página — consulta el [Manual de traducción](/docs/contribute).

## Licencias y firma

Al contribuir aceptas que tu prosa se comparta bajo **CC BY 4.0** y tu código bajo **MIT**, y añades una línea `Signed-off-by` (DCO) a tus commits (`git commit -s`). Detalles en la [página de gobernanza](/docs/contribute).

Sé amable — seguimos el [Código de conducta](/docs/contribute) del Contributor Covenant.
