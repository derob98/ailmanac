Los **estilos de salida** cambian *cómo* se comunica Claude Code — su tono, su verbosidad y si explica su razonamiento — sin cambiar *lo que* puede hacer. Piénsalo como un preset de personalidad/formato para las respuestas.

## Por qué usarlos

- **Adáptalo a tu flujo de trabajo.** Escueto y orientado a la acción cuando vas rápido; explicativo cuando estás aprendiendo.
- **Modo de enseñanza.** Algunos estilos añaden comentarios didácticos — genial mientras te familiarizas con un código o una técnica.
- **Consistencia.** Configura un estilo preferido para que cada sesión se lea como te gusta.

## Estilos comunes

Los estilos integrados habituales van desde uno conciso por defecto hasta modos más **explicativos** u **orientados al aprendizaje** que recorren el "porqué". Configura el estilo activo mediante el comando/ajuste correspondiente (consulta la [documentación oficial](https://docs.anthropic.com/en/docs/claude-code/output-styles) para la lista actual y el mecanismo exacto).

## Estilos de salida frente a CLAUDE.md frente a estilos de respuesta

- **Estilo de salida** = preset de presentación para las respuestas de Claude Code (tono/verbosidad).
- **[CLAUDE.md](/docs/claude-code/claude-md)** = hechos y reglas persistentes del *proyecto* (qué hacer, convenciones, salvaguardas).
- **[Instrucciones personalizadas / estilos de respuesta](/docs/claude-app/custom-instructions)** = el primo en la app de consumo Claude.ai.

:::tip Usa la palanca adecuada
¿Quieres que Claude "siempre ejecute los tests antes de terminar"? Eso es una **salvaguarda de CLAUDE.md**, no un estilo. ¿Quieres que "explique cada paso a medida que avanza"? Eso es un **estilo de salida**. Mantén las reglas de comportamiento en CLAUDE.md y la presentación en los estilos.
:::
