Cada sesión tiene una [ventana de contexto](/docs/foundations/tokens-and-context) finita. A medida que se llena, el contenido más antiguo queda comprimido y Claude puede empezar a "derivar". Gestionar el contexto de forma deliberada es lo que mantiene las sesiones largas afiladas.

## Los dos comandos esenciales

- **`/clear`** — borra la conversación y empieza de cero. Úsalo cuando cambias a una **tarea no relacionada**. La forma más barata de recuperar un contexto limpio y enfocado.
- **`/compact`** — resume la conversación hasta el momento en una forma compacta, conservando la esencia mientras liberas espacio. Úsalo cuando quieres **continuar la misma tarea** pero la ventana se está llenando.

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]
  A -->|Same task, context full| C[/compact]
```

## Autocompactación

Cuando el contexto se acerca al límite, Claude Code puede **autocompactar** para que el trabajo continúe sin una parada en seco. Es una red de seguridad — pero hacer `/clear` de forma proactiva entre tareas da mejores resultados que depender de ella.

## Técnicas para mantenerse efectivo

- **Una tarea por sesión.** No dejes que una tarde de trabajo no relacionado se acumule en un mismo contexto.
- **Delega la lectura pesada** a [subagentes](/docs/claude-code/subagents) — ellos queman *su* contexto y devuelven solo la conclusión.
- **Referencia, no pegues.** Apunta a Claude hacia archivos (`@path`) en lugar de pegar bloques grandes; mantén [CLAUDE.md](/docs/claude-code/claude-md) ligero.
- **Vigila el presupuesto.** Una [statusline](/docs/claude-code/statusline) que muestre el uso del contexto te indica cuándo compactar.

:::tip Síntoma → solución
Si Claude empieza a olvidar una instrucción anterior o a repetirse, lo más probable es que la ventana esté llena. `/compact` (misma tarea) o `/clear` (tarea nueva) suele arreglarlo al instante. Más en [Resolución de problemas](/docs/contribute/troubleshooting).
:::
