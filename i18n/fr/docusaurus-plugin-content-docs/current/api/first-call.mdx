## 1. Obtenir une clé API

Créez-en une dans la [Console Anthropic](https://console.anthropic.com). Puis définissez-la comme variable d'environnement pour qu'elle ne se retrouve jamais dans votre code :

<PromptCard title="Exporter votre clé API">{`export ANTHROPIC_API_KEY="sk-ant-..."`}</PromptCard>

<Callout type="warning" items={["Ne commitez jamais votre clé. Gardez les clés dans des variables d'environnement ou un gestionnaire de secrets, jamais dans le contrôle de version. Voir la page Sécurité (/docs/security)."]} />

## 2. Installer le SDK

Choisissez votre langage. Le SDK lit automatiquement la clé depuis la variable d'environnement que vous venez de définir.

## 3. Effectuer l'appel

Chaque requête est une liste de `messages`. Le modèle répond avec du `content`. Exécutez l'extrait pour votre langage et vous verrez Claude répondre à la question.

<Tabs groupId="lang">
<TabItem value="python" label="Python">

```python
import anthropic

client = anthropic.Anthropic()  # reads ANTHROPIC_API_KEY from the environment

message = client.messages.create(
    model="claude-sonnet-5",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "In one sentence, what is an API?"}
    ],
)

print(message.content[0].text)
```

</TabItem>
<TabItem value="ts" label="TypeScript">

```ts
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic(); // reads ANTHROPIC_API_KEY from the environment

const message = await client.messages.create({
  model: "claude-sonnet-5",
  max_tokens: 1024,
  messages: [
    { role: "user", content: "In one sentence, what is an API?" },
  ],
});

console.log(message.content[0].text);
```

</TabItem>
<TabItem value="curl" label="cURL">

```bash
curl https://api.anthropic.com/v1/messages \
  --header "x-api-key: $ANTHROPIC_API_KEY" \
  --header "anthropic-version: 2023-06-01" \
  --header "content-type: application/json" \
  --data '{
    "model": "claude-sonnet-5",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "In one sentence, what is an API?"}
    ]
  }'
```

</TabItem>
</Tabs>

## Ce qui vient de se passer

Vous avez envoyé quatre informations. Voici ce que chacune contrôle :

- **`model`** — quel Claude utiliser. Ne le codez pas en dur à l'aveugle ; voir [Choisir un modèle](/docs/api/choosing-a-model).
- **`max_tokens`** — un plafond sur la longueur de la réponse (en [tokens](/docs/foundations)). Il ne définit *pas* la fenêtre de contexte.
- **`messages`** — la conversation jusqu'ici. L'API est **sans état** : pour poursuivre une conversation, renvoyez tout l'historique à chaque fois.
- **`system`** (optionnel) — une instruction de haut niveau qui définit le rôle de Claude pour l'appel.
