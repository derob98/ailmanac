Le **Model Context Protocol (MCP)** est le standard ouvert pour connecter l'IA à des outils et données externes. Sur l'API, MCP vous permet de donner à Claude l'accès à toute la **panoplie d'outils d'un serveur** par référence — au lieu de définir chaque outil à la main.

## MCP vs outils définis à la main

| | [Utilisation des outils](/docs/api/tool-use) (personnalisée) | MCP |
|---|---|---|
| Vous définissez | Le schéma de chaque outil, et vous l'exécutez vous-même | Une connexion à un serveur qui *publie* des outils |
| Idéal pour | Quelques fonctions sur mesure dans votre application | Réutiliser des intégrations existantes (GitHub, BD, navigateurs, SaaS) |
| Authentification | Votre code | OAuth ou clés natives vers le serveur |

Ils coexistent : définissez directement vos outils propres à l'application, et intégrez des capacités prêtes à l'emploi via MCP.

## Déclarer un serveur

Vous attachez un ou plusieurs serveurs MCP à une requête/un agent (URL ou commande, plus l'authentification). Claude peut alors appeler les outils du serveur au sein de la [boucle d'agent](/docs/api/building-agents), comme n'importe quel outil. Le champ de requête exact et le flux d'authentification figurent dans la [documentation officielle](https://docs.anthropic.com/en/docs/build-with-claude/mcp).

## Un même standard, trois surfaces

- **API** (cette page) — des serveurs MCP sur vos requêtes/agents.
- **[Claude Code](/docs/claude-code/mcp)** — des serveurs MCP dans vos sessions de développement.
- **[Les applications](/docs/claude-app/connectors)** — MCP alimente les Connecteurs.

Apprenez-le une fois ; cela se transpose.

## Confiance

:::warning Un serveur MCP, c'est du code + des accès
Ne connectez que des serveurs auxquels vous faites confiance, restreignez-les au moindre privilège, et rappelez-vous que le contenu externe qu'ils renvoient peut véhiculer de l'[injection de prompt](/docs/security/prompt-injection). Examinez d'abord les serveurs tiers — [Examiner le code tiers](/docs/security/reviewing-third-party-code).
:::
