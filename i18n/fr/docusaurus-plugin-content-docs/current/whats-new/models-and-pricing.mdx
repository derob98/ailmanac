## Pourquoi les tarifs et les limites de contexte ne figurent pas dans le tableau

Ils changent trop souvent pour être copiés sans risque — un tarif obsolète est pire que pas de tarif du tout. Nous indiquons l'**identité** de chaque modèle (nom, ID d'API, usage prévu) et nous vous renvoyons à la page officielle pour les **tarifs, les fenêtres de contexte et les limites de débit**, qui fait toujours autorité.

## Comment nous gardons cette page à jour

- Le tableau est généré à partir d'un fichier de données unique (`data/models.json`) qui porte une **date de « dernière vérification »**.
- Le mettre à jour est une modification d'une seule ligne — une [première contribution](/docs/contribute/contribute-in-10-minutes) idéale.
- Dans une phase ultérieure, l'automatisation comparera ce fichier avec le point de terminaison officiel des modèles et ouvrira une pull request lorsque quelque chose dérive — mais un humain effectue toujours une relecture avant la mise en production. (Voir [Comment fonctionne notre automatisation](/docs/contribute).)

## Comment choisir entre eux

Une règle empirique rapide (les conseils complets se trouvent dans [Choisir un modèle](/docs/api/choosing-a-model)) :

- **Commencez par Sonnet.** Le meilleur équilibre entre qualité, vitesse et coût pour la plupart des tâches.
- **Optez pour Opus** lorsque la qualité sur un problème difficile compte plus que le coût.
- **Descendez vers Haiku** pour les tâches à fort volume, sensibles à la latence ou simples.
