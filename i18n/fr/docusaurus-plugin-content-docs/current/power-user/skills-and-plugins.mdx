Le moyen le plus rapide de monter en niveau avec Claude Code est d'arrêter de tout écrire à la main et d'**installer des capacités que d'autres ont déjà perfectionnées**. Les skills donnent à Claude une expertise à la demande ; les plugins regroupent skills, agents, commandes, hooks et serveurs MCP en une seule unité versionnée et installable. Cette page est une liste courte, soigneusement sélectionnée et prête à installer : ce que fait chaque chose, quand y recourir, et la commande exacte pour l'ajouter.

Si les mécanismes sont encore flous pour vous, lisez d'abord [Skills](/docs/claude-code/skills) et [Plugins et marketplaces](/docs/claude-code/plugins-marketplaces) — cette page suppose que vous savez ce qu'est un `SKILL.md`.

<VerifyNote lastVerified="2026-06-24" source="https://code.claude.com/docs/en/discover-plugins">
Les noms des marketplaces, le catalogue de plugins officiels et le nombre d'étoiles des dépôts communautaires changent fréquemment. Les plugins exécutent du code avec vos privilèges — n'installez qu'à partir de sources auxquelles vous faites confiance, et confirmez les commandes actuelles dans la documentation officielle avant de vous y fier.
</VerifyNote>

## Skill ou plugin : le modèle en 20 secondes

Un **skill** est un dossier contenant un `SKILL.md` (frontmatter `description` + instructions). Claude ne le charge que lorsqu'il est pertinent, ou vous l'invoquez avec `/skill-name`. Il réside dans `~/.claude/skills/` (personnel), le `.claude/skills/` d'un projet, ou à l'intérieur d'un plugin. D'après la documentation officielle, les commandes slash personnalisées ont été fusionnées dans les skills — un `.claude/commands/deploy.md` et un `.claude/skills/deploy/SKILL.md` créent tous deux `/deploy`.

Un **plugin** est un ensemble distribuable qui peut livrer skills, agents, hooks, serveurs MCP et serveurs LSP ensemble, avec un manifeste `.claude-plugin/plugin.json`. Vous obtenez les plugins depuis une **marketplace** — un catalogue que vous ajoutez une fois, puis depuis lequel vous installez. Les skills de plugin ont un espace de noms, par exemple `/commit-commands:commit`.

## Ajouter une marketplace, installer un plugin

La marketplace officielle Anthropic (`claude-plugins-official`) est disponible automatiquement. Deux autres méritent d'être ajoutées manuellement : la marketplace **communautaire** soigneusement sélectionnée et la marketplace de **démonstration** avec des plugins d'exemple.

## Plugins officiels à installer en premier

Ceux-ci sont livrés dans la marketplace officielle d'Anthropic (`claude-plugins-official`) et constituent le point de départ à la plus forte valeur. Installez avec `/plugin install <name>@claude-plugins-official`.

### Intelligence de code (LSP) — faites voir instantanément les erreurs de type à Claude

Les plugins LSP relient un serveur de langage à Claude Code : après chaque modification, Claude reçoit automatiquement des diagnostics (erreurs de type, imports manquants) et une navigation précise (aller à la définition, trouver les références) au lieu de deviner via grep. Ils nécessitent le binaire de serveur de langage correspondant sur votre machine.

- **Quand l'utiliser :** toute session de code non triviale dans un langage typé. C'est probablement le plus grand gain de précision pour le travail de dev.
- **Compromis :** des serveurs comme `rust-analyzer` et `pyright` peuvent consommer beaucoup de mémoire sur les gros dépôts ; désactivez-les si cela pose problème.

<PromptCard title="Installer le LSP pour votre stack">
{`/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls`}
</PromptCard>

### Workflow Git et PR

- **`commit-commands`** — workflows git : indexer, générer un message de commit, pousser, ouvrir une PR. À utiliser quand vous voulez des commits propres et cohérents sans taper le cérémonial. Les skills ont un espace de noms, par exemple `/commit-commands:commit`.
- **`pr-review-toolkit`** — des agents spécialisés pour relire les pull requests. À utiliser avant de merger, ou pour avoir un second regard sur un diff.

<PromptCard title="Les essentiels Git/PR">
{`/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins`}
</PromptCard>

### Revue de sécurité automatique

- **`security-guidance`** — examine chaque modification que fait Claude pour détecter les vulnérabilités courantes et lui demande de les corriger dans la même session. À utiliser comme filet de sécurité permanent sur tout projet d'écriture de code ; il complète (ne remplace pas) une vraie revue.

<PromptCard title="Passe de sécurité permanente">
{`/plugin install security-guidance@claude-plugins-official`}
</PromptCard>

### Intégrations externes (MCP, préconfigurées)

Celles-ci regroupent un [serveur MCP](/docs/claude-code/mcp) prêt à l'emploi pour vous éviter la configuration manuelle. Recourez-y quand vous voulez que Claude agisse directement dans ce service.

- **Contrôle de source :** `github`, `gitlab`
- **Gestion de projet :** `linear`, `notion`, `asana`, `atlassian` (Jira/Confluence)
- **Infra :** `vercel`, `firebase`, `supabase`
- **Design / comms / monitoring :** `figma`, `slack`, `sentry`

<PromptCard title="Connectez Claude à vos outils">
{`/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official`}
</PromptCard>

### Boîtes à outils maison et styles de sortie

- **`plugin-dev`** — une boîte à outils pour créer vos propres plugins. À utiliser dès que votre configuration `.claude/` mérite d'être partagée.
- **`agent-sdk-dev`** — des outils pour développer sur le Claude Agent SDK.
- **`explanatory-output-style` / `learning-output-style`** — changent *la façon* dont Claude répond : explications pédagogiques au fil de l'eau, ou un mode d'apprentissage interactif. Parfaits pour l'onboarding ou quand vous voulez vraiment comprendre les changements.

## Skills à forte valeur, par tâche à accomplir

Les skills brillent pour les tâches répétables et riches en contexte. Anthropic publie un ensemble de référence public dans le dépôt [`anthropics/skills`](https://github.com/anthropics/skills) — les mêmes patterns qui alimentent les capacités documentaires et créatives de Claude. Vous pouvez copier un dossier de skill dans `~/.claude/skills/` (personnel) ou `.claude/skills/` (projet) et il se charge à la session suivante.

| Tâche | Skill (depuis anthropics/skills) | Quand y recourir |
| :-- | :-- | :-- |
| **Documents** | `docx`, `pdf`, `pptx`, `xlsx` | Générer ou modifier de vrais fichiers Office/PDF — rapports, présentations, feuilles de calcul, formulaires à remplir. (Implémentations de référence à source disponible.) |
| **Construire des UI** | `web-artifacts-builder` / `artifacts-builder` | Produire des artefacts HTML/JS autonomes et des mini-applications. |
| **Étendre Claude** | `mcp-builder` | Échafauder un nouveau serveur MCP quand aucune intégration n'existe encore. |
| **Marque et comms** | skills de style brand-guidelines / internal-comms | Garder le ton, le nommage et le formatage fidèles à la marque dans toutes les sorties. |

<Callout type="tip" items={["Ne balancez pas 30 skills dans ~/.claude/skills d'un coup. Les skills fonctionnent par divulgation progressive — seuls le nom + la description se chargent jusqu'au déclenchement — mais un tas surchargé et vaguement décrit rend le déclenchement peu fiable. Installez-en quelques-uns, rédigez des descriptions nettes « Utiliser lorsque… », et développez délibérément.", "AILmanac fournit des packs de skills prêts à l'emploi que vous pouvez déposer pour essayer — voir /docs/templates/skills."]} />

### Un skill auto-déclenchant que vous pouvez coller tout de suite

Pour le travail à forte composante de recherche, un skill qui ancre Claude dans des sources fraîches vaut mieux que de réexpliquer votre méthode à chaque fois. Déposez ceci dans `~/.claude/skills/research-brief/SKILL.md` :

<PromptCard title="~/.claude/skills/research-brief/SKILL.md">
{`---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.`}
</PromptCard>

## Où vit la communauté (utilisez les listes sélectionnées, évitez le bruit)

Il existe des dizaines de dépôts « awesome » ; la plupart sont des miroirs générés automatiquement. Quelques-uns sont réellement sélectionnés et maintenus — commencez par ceux-là.

- **[hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code)** — l'index le plus connu, sélectionné à la main, de skills, hooks, commandes slash, orchestrateurs d'agents, applications et plugins. Des dizaines de milliers d'étoiles ; réorganisé activement.
- **[ComposioHQ/awesome-claude-skills](https://github.com/ComposioHQ/awesome-claude-skills)** — un vaste catalogue sélectionné de Claude Skills regroupés par fonction (documents, dev, données, marketing, sécurité, automatisation SaaS).
- **[VoltAgent/awesome-agent-skills](https://github.com/VoltAgent/awesome-agent-skills)** — plus de 1 400 skills communautaires + officiels (Anthropic, Vercel, Stripe, Cloudflare, Sentry, Figma, et plus), portables entre Claude Code, Cursor, Gemini CLI, Codex, et d'autres.
- **[anthropics/claude-plugins-community](https://github.com/anthropics/claude-plugins-community)** — la marketplace officielle communautaire de **plugins** ; des plugins tiers ayant passé la validation automatisée et le contrôle de sécurité d'Anthropic.
