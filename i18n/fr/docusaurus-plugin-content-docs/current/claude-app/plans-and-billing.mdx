De quelle formule Claude avez-vous besoin ? Voici comment y réfléchir — sans citer des prix qui seraient périmés le mois prochain (consultez la [tarification officielle](https://www.anthropic.com/pricing) pour ceux-là).

## Les niveaux, par objectif

| Niveau | Plutôt pour |
|---|---|
| **Free** | Essayer Claude, un usage personnel léger |
| **Pro** | Un usage individuel régulier — plus d'utilisation, plus de capacités |
| **Max** | Les utilisateurs individuels intensifs qui atteignent les limites de Pro |
| **Team** | Les petites équipes — espace de travail partagé et gestion |
| **Enterprise** | Les organisations ayant besoin d'administration, de sécurité, de mise à l'échelle et de contrôles |

(Les noms / la structure exacts peuvent changer — confirmez sur la page de tarification.)

## Ce que représentent les « limites »

Les formules grand public plafonnent **la quantité que vous pouvez utiliser sur une période** (messages / utilisation qui se réinitialisent au fil du temps), et les niveaux supérieurs relèvent ces plafonds. Lorsque vous atteignez une limite, on vous demande généralement d'attendre la réinitialisation ou de passer à un niveau supérieur — votre travail n'est pas perdu, vous faites simplement une pause.

Les modèles plus lourds (et les conversations plus longues) consomment votre quota plus vite. Si vous atteignez les limites, une solution rapide consiste à **démarrer de nouvelles conversations** pour de nouveaux sujets et à garder des prompts ciblés (même idée que la [gestion du contexte](/docs/foundations/tokens-and-context)).

## Comment choisir

1. **Commencez avec Free.** Découvrez à quoi vous l'utilisez réellement.
2. **Passez à Pro** quand les limites interrompent un vrai travail.
3. **Optez pour Max** si vous êtes un utilisateur intensif qui atteint encore les plafonds.
4. **Team / Enterprise** quand des collègues ont besoin d'un accès partagé, d'administration ou de conformité.

## Facturation des applications vs API — des choses différentes

- **Les formules d'application** = un abonnement forfaitaire avec des limites d'utilisation (cette page).
- **La facturation de l'API** = paiement au [token](/docs/api/tokens-and-pricing) pour ce que vous construisez. Modèle différent, page différente.
