Les **styles de sortie** changent *comment* Claude Code communique — son rôle, son ton, sa verbosité, et s'il explique son raisonnement — en modifiant directement le prompt système. Ils changent *comment* Claude répond, pas *ce qu'il* sait. Recourez-y quand vous re-promptez sans cesse pour obtenir la même voix ou le même format à chaque tour.

## Pourquoi les utiliser

- **Adaptez-les à votre flux de travail.** Concis et orienté action quand vous allez vite ; explicatif quand vous apprenez.
- **Mode pédagogique.** Certains styles ajoutent un commentaire éducatif — parfait pendant que vous découvrez une base de code ou une technique.
- **Cohérence.** Définissez un style préféré une fois pour que chaque session se lise comme vous l'aimez.

## Les styles intégrés

Claude Code est livré avec quatre :

| Style | Ce qu'il fait |
|---|---|
| **Default** | Le prompt système standard d'ingénierie logicielle — efficace, orienté action. |
| **Proactive** | Exécute immédiatement et fait des hypothèses raisonnables au lieu de s'arrêter pour des décisions de routine. Guidage d'exécution autonome plus fort que le mode auto, mais vous voyez toujours les invites de permission avant l'exécution des outils. |
| **Explanatory** | Ajoute des « Insights » pédagogiques entre les étapes, expliquant les choix d'implémentation et les patterns de la base de code. |
| **Learning** | Collaboratif, apprentissage par la pratique : partage des Insights *et* vous demande d'écrire vous-même de petits morceaux, laissant des marqueurs `[TODO(human)]` dans votre code. |

<Callout type="warning" items={[
  "Explanatory et Learning produisent des réponses plus longues par conception, ils coûtent donc plus de tokens de sortie que Default."
]} />

## Comment changer

Exécutez `/config` et choisissez un style sous **Output style**. Votre choix est enregistré dans `.claude/settings.local.json` au niveau du projet. Pour le définir sans le menu, modifiez le champ `outputStyle` dans n'importe quel fichier de paramètres.

  { title: "Ou le définir directement", body: "Contournez le menu en modifiant le champ outputStyle dans n'importe quel fichier de paramètres (voir le JSON ci-dessous)." },
  { title: "Démarrer une nouvelle session", body: "Le style fait partie du prompt système, lu une seule fois au démarrage de la session — exécutez /clear ou ouvrez une nouvelle session pour qu'il prenne effet." }
]} />

```json
{
  "outputStyle": "Explanatory"
}
```

:::warning Les changements nécessitent une nouvelle session
Le style de sortie fait partie du prompt système, que Claude Code lit **une seule fois au démarrage de la session**. Un changement ne prend effet qu'après `/clear` ou une nouvelle session. (La commande autonome `/output-style` a été supprimée en v2.1.91 — utilisez `/config` ou le paramètre ci-dessus.)
:::

## Créer un style personnalisé

Un style personnalisé est un fichier Markdown : frontmatter, puis les instructions à ajouter au prompt système. Enregistrez-le au niveau utilisateur (`~/.claude/output-styles/`) ou projet (`.claude/output-styles/`) ; le nom du fichier devient le nom du style à moins que vous ne définissiez `name`.

<PromptCard title="Exemple de style personnalisé — Diagrammes d'abord">{`---
name: Diagrams first
description: Lead every explanation with a diagram
keep-coding-instructions: true
---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.`}</PromptCard>

## Styles de sortie vs CLAUDE.md vs autres leviers

- **Style de sortie** = un préréglage du prompt système pour le rôle, le ton et le format — s'applique à *chaque* réponse.
- **[CLAUDE.md](/docs/claude-code/claude-md)** = faits et règles *projet* persistants (conventions, garde-fous), ajoutés en tant que message après le prompt système.
- **[Instructions personnalisées / styles de réponse](/docs/claude-app/custom-instructions)** = le cousin de l'application grand public Claude.ai.

:::tip Utilisez le bon levier
Vous voulez que Claude « lance toujours les tests avant de terminer » ? C'est un **garde-fou CLAUDE.md**, pas un style. Vous voulez qu'il « explique chaque étape au fur et à mesure » ? C'est un **style de sortie**. Gardez les règles de comportement dans CLAUDE.md et la présentation dans les styles.
:::
