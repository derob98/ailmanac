Les **styles de sortie** changent *la manière dont* Claude Code communique — son ton, sa verbosité, et s'il explique son raisonnement — sans changer *ce qu'il* peut faire. Voyez-le comme un préréglage de personnalité/format pour les réponses.

## Pourquoi les utiliser

- **S'adapter à votre workflow.** Concis et orienté action quand vous allez vite ; explicatif quand vous apprenez.
- **Mode pédagogique.** Certains styles ajoutent des commentaires didactiques — parfait pendant que vous découvrez une base de code ou une technique.
- **Cohérence.** Définissez un style préféré pour que chaque session se lise comme vous l'aimez.

## Styles courants

Les styles intégrés typiques vont d'un défaut concis à des modes plus **explicatifs** ou **orientés apprentissage** qui détaillent le « pourquoi ». Définissez le style actif via la commande/le réglage approprié (voir la [documentation officielle](https://docs.anthropic.com/en/docs/claude-code/output-styles) pour la liste actuelle et le mécanisme exact).

## Styles de sortie vs CLAUDE.md vs styles de réponse

- **Style de sortie** = préréglage de présentation des réponses de Claude Code (ton/verbosité).
- **[CLAUDE.md](/docs/claude-code/claude-md)** = faits et règles *de projet* persistants (quoi faire, conventions, garde-fous).
- **[Instructions personnalisées / styles de réponse](/docs/claude-app/custom-instructions)** = le cousin de l'application grand public Claude.ai.

:::tip Utilisez le bon levier
Vous voulez que Claude « exécute toujours les tests avant de finir » ? C'est un **garde-fou de CLAUDE.md**, pas un style. Vous voulez qu'il « explique chaque étape au fur et à mesure » ? C'est un **style de sortie**. Gardez les règles de comportement dans CLAUDE.md et la présentation dans les styles.
:::
