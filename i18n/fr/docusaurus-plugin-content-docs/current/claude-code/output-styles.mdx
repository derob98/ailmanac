Les **styles de sortie** changent *la manière dont* Claude Code communique — son rôle, son ton, sa verbosité et le fait qu'il explique ou non son raisonnement — en modifiant directement l'invite système. Ils changent *la manière dont* Claude répond, pas *ce qu'il* sait. Adoptez-en un lorsque vous vous retrouvez à redemander à chaque tour la même voix ou le même format.

## Pourquoi les utiliser

- **S'adapter à votre flux de travail.** Concis et orienté action quand vous allez vite ; explicatif quand vous apprenez.
- **Mode pédagogique.** Certains styles ajoutent un commentaire éducatif — idéal pendant que vous découvrez une base de code ou une technique.
- **Cohérence.** Définissez une fois un style préféré pour que chaque session se lise comme vous l'aimez.

## Les styles intégrés

Claude Code est livré avec quatre styles :

| Style | Ce qu'il fait |
|---|---|
| **Default** | L'invite système standard d'ingénierie logicielle — efficace, orientée action. |
| **Proactive** | S'exécute immédiatement et fait des hypothèses raisonnables au lieu de s'arrêter pour des décisions de routine. Des directives d'exécution autonome plus fortes que le mode auto, mais vous voyez toujours les invites d'autorisation avant l'exécution des outils. |
| **Explanatory** | Ajoute des « Insights » éducatifs entre les étapes, expliquant les choix d'implémentation et les motifs de la base de code. |
| **Learning** | Collaboratif, apprentissage par la pratique : partage des Insights *et* vous demande d'écrire vous-même de petits morceaux, en laissant des marqueurs `[TODO(human)]` dans votre code. |

Explanatory et Learning produisent des réponses plus longues par conception, ils coûtent donc plus de tokens de sortie que Default.

## Comment basculer

Exécutez `/config` et choisissez un style sous **Output style**. Votre choix est enregistré dans `.claude/settings.local.json` au niveau du projet. Pour le définir sans le menu, modifiez le champ `outputStyle` dans n'importe quel fichier de paramètres :

```json
{
  "outputStyle": "Explanatory"
}
```

:::warning Les changements nécessitent une nouvelle session
Le style de sortie fait partie de l'invite système, que Claude Code lit **une seule fois au démarrage de la session**. Un changement ne prend effet qu'après `/clear` ou une nouvelle session. (La commande autonome `/output-style` a été supprimée dans v2.1.91 — utilisez `/config` ou le paramètre ci-dessus.)
:::

## Créer un style personnalisé

Un style personnalisé est un fichier Markdown : un frontmatter, puis les instructions à ajouter à l'invite système. Enregistrez-le au niveau utilisateur (`~/.claude/output-styles/`) ou au niveau projet (`.claude/output-styles/`) ; le nom du fichier devient le nom du style sauf si vous définissez `name`.

```markdown
---
name: Diagrams first
description: Lead every explanation with a diagram
keep-coding-instructions: true
---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.
```

Définissez `keep-coding-instructions: true` lorsque vous changez seulement *la manière dont* Claude communique tout en voulant qu'il continue à coder normalement. Omettez-le (la valeur par défaut) lorsque Claude ne fait pas du tout d'ingénierie logicielle — par exemple un assistant purement dédié à la rédaction ou à l'analyse de données — afin que les instructions de codage intégrées soient abandonnées.

## Styles de sortie vs CLAUDE.md vs autres leviers

- **Style de sortie** = un préréglage d'invite système pour le rôle, le ton et le format — s'applique à *chaque* réponse.
- **[CLAUDE.md](/docs/claude-code/claude-md)** = des faits et règles *de projet* persistants (conventions, garde-fous), ajoutés sous forme de message après l'invite système.
- **[Instructions personnalisées / styles de réponse](/docs/claude-app/custom-instructions)** = le cousin de l'application grand public Claude.ai.

:::tip Utilisez le bon levier
Vous voulez que Claude « exécute toujours les tests avant de terminer » ? C'est un **garde-fou CLAUDE.md**, pas un style. Vous voulez qu'il « explique chaque étape au fur et à mesure » ? C'est un **style de sortie**. Gardez les règles de comportement dans CLAUDE.md et la présentation dans les styles.
:::
