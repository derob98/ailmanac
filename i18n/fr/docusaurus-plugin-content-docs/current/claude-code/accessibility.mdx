Claude Code s'exécute dans le terminal — une interface textuelle, ce qui est un bon point de départ pour l'accessibilité, mais le rendu par défaut s'appuie sur la couleur, les glyphes de tracé de cadres et les indicateurs animés que les lecteurs d'écran et certaines configurations motrices gèrent mal. Depuis juillet 2026, il existe un **mode lecteur d'écran** dédié, ainsi que des réglages pour l'ergonomie vim et le comportement souris/défilement. Cette page rassemble les leviers qui rendent le CLI utilisable avec les technologies d'assistance.

## Mode lecteur d'écran

Le mode lecteur d'écran remplace l'interface TUI décorative par un rendu en texte brut que les technologies d'assistance peuvent lire proprement. Activez-le de trois façons — un flag ponctuel, une variable d'environnement pour chaque session, ou un réglage persistant.

<Steps items={[
  { title: "Essayez-le pour une session", body: "Lancez avec le flag : claude --ax-screen-reader. Rien d'autre ne change — même session, sortie plus simple." },
  { title: "Activez-le pour chaque session", body: "Exportez la variable d'environnement CLAUDE_AX_SCREEN_READER=1 dans votre profil shell pour qu'elle s'applique partout." },
  { title: "Rendez-le permanent dans les réglages", body: "Définissez \"axScreenReader\": true dans un fichier settings.json (au niveau utilisateur ~/.claude/settings.json, cela s'applique à tous les projets)." }
]} />

<PromptCard title="Mode lecteur d'écran persistant (~/.claude/settings.json)">{`{
  "axScreenReader": true
}`}</PromptCard>

### Ce que cela change

<Callout type="tip" items={[
  "Un rendu en texte brut remplace le TUI stylisé pour que les lecteurs d'écran reçoivent un texte propre et linéaire.",
  "Les glyphes décoratifs sont masqués aux lecteurs d'écran au lieu d'être lus comme du bruit.",
  "Les symboles de la transcription sont annoncés sous forme d'étiquettes courtes plutôt que de caractères bruts.",
  "Les tableaux imbriqués sont lus comme des lignes En-tête : valeur. — aplatis en rangées prononçables.",
  "Les changements de mode de permission sont annoncés à voix haute lorsque vous alternez les modes avec Maj+Tab."
]} />

Ce dernier point compte : alterner les modes de permission avec **Maj+Tab** est une interaction essentielle de Claude Code (voir [Permissions](/docs/claude-code/permissions)), et le mode lecteur d'écran énonce le nouveau mode à voix haute pour que vous sachiez toujours si vous êtes en mode plan, accept-edits ou auto sans lire la ligne d'état.

## Ergonomie vim : remapper les touches du mode insertion

Si vous utilisez le mode vim et qu'atteindre `Échap` fatigue votre main, mappez plutôt une séquence de deux touches en mode insertion. Le réglage `vimInsertModeRemaps` transforme des séquences comme `jj` en `Escape`, pour que vous ne quittiez jamais la rangée de repos.

<PromptCard title="Mapper jj vers Échap en mode vim (settings.json)">{`{
  "editorMode": "vim",
  "vimInsertModeRemaps": {
    "jj": "Escape"
  }
}`}</PromptCard>

<Callout type="warning" items={[
  "Le remap ne se déclenche qu'en mode insertion, donc taper jj comme texte littéral nécessite une brève pause entre les touches — choisissez une séquence que vous tapez rarement vite (jj et kj sont des choix courants).",
  "editorMode: vim active les raccourcis vim ; vimInsertModeRemaps ne prend effet que lorsque le mode vim est activé."
]} />

## Contrôles souris et défilement

Les menus en plein écran acceptent désormais les **clics souris** — y compris les menus à sélection multiple et les lignes de saisie « Autre » — ce qui aide si la navigation au clavier uniquement est pénible. Deux réglages ajustent le comportement du pointeur pour la sensibilité au mouvement ou la précision :

| Réglage | Ce qu'il fait |
|---|---|
| `CLAUDE_CODE_DISABLE_MOUSE_CLICKS` | Désactive le clic, le glisser et le survol de la souris tout en conservant le défilement à la molette — utile si des clics parasites vous dérangent. |
| `wheelScrollAccelerationEnabled` | Réglez sur `false` pour désactiver l'accélération du défilement à la molette en plein écran, pour un défilement plus stable et prévisible. |

<PromptCard title="Défilement plus stable, sans clics accidentels (settings.json)">{`{
  "wheelScrollAccelerationEnabled": false,
  "env": {
    "CLAUDE_CODE_DISABLE_MOUSE_CLICKS": "1"
  }
}`}</PromptCard>

## Contraste et thèmes

Pour les utilisateurs malvoyants, le contraste du terminal est le premier levier. Claude Code fournit des thèmes clair et sombre et prend en charge des **thèmes personnalisés** que vous construisez depuis `/theme` ou fournissez dans un plugin — vous pouvez ainsi régler une palette à fort contraste qui répond à vos besoins. Voir [Réglages](/docs/claude-code/settings) pour savoir où le choix de thème est stocké, et [Styles de sortie](/docs/claude-code/output-styles) si vous voulez aussi une *prose* plus simple et moins décorée (les styles Explicatif et Apprentissage sont plus verbeux, pas moins — pour une sortie concise, restez sur Default).

:::tip Au-delà du CLI
Le mode lecteur d'écran couvre le terminal. Pour une interaction mains libres avec Claude de façon plus large, le [Mode vocal](/docs/claude-app/voice-mode) dans l'application vous permet de parler et d'écouter, et le [playbook Accessibilité](/docs/playbooks/accessibility) couvre l'usage de Claude lui-même comme outil d'assistance — simplifier du texte, construire des listes de contrôle et adapter l'information à votre façon de lire.
:::
