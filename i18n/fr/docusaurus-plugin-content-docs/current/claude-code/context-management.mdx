## La grande idée

Chaque session a une [fenêtre de contexte](/docs/foundations/tokens-and-context) finie. À mesure qu'elle se remplit, le contenu plus ancien se fait comprimer et Claude peut commencer à « dériver ». Gérer le contexte délibérément est ce qui garde les longues sessions affûtées.

## Les deux commandes essentielles

Deux commandes couvrent presque toutes les situations. L'astuce est de faire correspondre la commande à votre intention.

<Flashcards cards={[{front: "/clear", back: "Efface la conversation et repart de zéro. Utilisez-la quand vous passez à une tâche SANS RAPPORT. Le moyen le moins cher de retrouver un contexte propre et concentré."}, {front: "/compact", back: "Résume la conversation jusqu'ici sous une forme compacte — garde l'essentiel tout en libérant de l'espace. Utilisez-la quand vous voulez CONTINUER la même tâche mais que la fenêtre se remplit."}]} />

Décidez avec une seule question : est-ce une nouvelle tâche ou la même ?

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]
  A -->|Same task, context full| C[/compact]
```

## Auto-compaction

Quand le contexte approche de la limite, Claude Code peut **auto-compacter** pour que le travail continue sans arrêt brutal. C'est un filet de sécurité — mais faire proactivement `/clear` entre les tâches donne de meilleurs résultats que de s'y fier.

<Callout type="tip" items={["Traitez l'auto-compaction comme un filet de sécurité, pas une stratégie. Nettoyer entre les tâches sans rapport garde le contexte plus propre que de laisser la fenêtre se remplir jusqu'à déclencher l'auto-compaction."]} />

## Techniques pour rester efficace

Au-delà des deux commandes, quatre habitudes gardent votre contexte léger dès le départ.

<Steps items={[{title: "Une tâche par session", body: "Ne laissez pas un après-midi de travaux sans rapport s'entasser dans un seul contexte."}, {title: "Déléguer la lecture lourde aux sous-agents", body: "Confiez les grosses lectures aux sous-agents — ils consomment LEUR contexte et ne renvoient que la conclusion. Voir /docs/claude-code/statusline."}]} />

Cela correspond à des docs vivants : [sous-agents](/docs/claude-code/subagents), [CLAUDE.md](/docs/claude-code/claude-md) et [statusline](/docs/claude-code/statusline).

:::tip Symptôme → remède
Si Claude commence à oublier une instruction antérieure ou à se répéter, la fenêtre est probablement pleine. `/compact` (même tâche) ou `/clear` (nouvelle tâche) y remédie généralement instantanément. Plus dans [Dépannage](/docs/contribute/troubleshooting).
:::
