## Indépendance et marques déposées

AILmanac est une **ressource communautaire indépendante**. Elle n'est **ni affiliée à Anthropic ni approuvée par elle.** « Claude » et « Anthropic » sont des marques déposées d'Anthropic, utilisées à titre nominatif. La [documentation officielle](https://platform.claude.com/docs) fait foi.

## Licences (double)

Pour que les textes comme le code soient réutilisables :

- **Prose / documentation → CC BY 4.0** (réutilisation avec attribution).
- **Code, exemples, configuration → MIT.**

Voir `LICENSE-CONTENT`, `LICENSE-CODE` et `NOTICE` dans le dépôt. En contribuant, vous acceptez de placer votre prose sous licence CC BY 4.0 et votre code sous licence MIT.

## Code de conduite

Nous suivons le **Contributor Covenant** (voir `CODE_OF_CONDUCT.md`). Soyez bienveillant, constructif, présumez la bonne foi. Tout comportement inacceptable peut être signalé en privé aux mainteneurs.

## Modèle de contribution

- **Signature DCO**, pas un CLA — `git commit -s` certifie que vous êtes en droit de soumettre le travail. Barrière basse, provenance claire.
- **Pull requests** pour tout ; le [contrôle de build](/docs/contribute/automation) et une revue par un mainteneur maintiennent la qualité.
- **CODEOWNERS** dirige les différents domaines vers les relecteurs à mesure que le projet grandit.

## Étiquettes et triage

Les étiquettes définies en tant que code (`.github/labels.yml`) gardent le backlog lisible :

- **Type :** `content`, `code`, `i18n`, `automation`
- **Domaine :** `core-claude`, `ring-general`, `infra`
- **Workflow :** `good first issue`, `help wanted`, `upstream-change`, `needs-verification`, `link-rot`, `stale`

Nouveaux contributeurs : cherchez **`good first issue`** — ajouter un terme au glossaire, compléter une page prévue ou corriger un fait périmé sont des points de départ parfaits.

## Décisions et discussion

- **GitHub Discussions** pour les questions et les propositions ; **Issues** pour le travail concret.
- Les contributeurs sont reconnus dans le dépôt.
