AILmanac n'est « complet et toujours à jour » que parce que des personnes comme vous y contribuent. Vous n'avez **pas** besoin d'être développeur — corriger une faute de frappe, ajouter un terme au glossaire ou partager un modèle sont toutes de véritables contributions.

## La voie rapide

1. **Trouvez la page** sur le site et cliquez sur **« Edit this page »** en bas (cela ouvre le fichier sur GitHub).
2. Cliquez sur le crayon ✏️ pour éditer dans votre navigateur. GitHub crée automatiquement un fork du dépôt pour vous.
3. Apportez votre modification en Markdown.
4. En bas, rédigez une courte description et **proposez les changements** → **ouvrez une pull request**.
5. Un mainteneur la relit. C'est tout. 🎉

Vous préférez en local ? Clonez le dépôt, puis :

```bash
npm install
npm run start      # live preview at http://localhost:3000
npm run build      # must pass: it fails on broken links
```

## Les 4 règles que suit chaque page

Les respecter accélère la relecture et maintient une qualité élevée :

1. **Indiquez le niveau.** Commencez le corps par `<LevelBadge level="beginner" />` (ou `intermediate` / `advanced` / `all`).
2. **Liez la source pour les faits volatils.** Tout ce qui concerne les noms de modèles, les prix, les limites ou les fonctionnalités qui changent doit citer la page officielle — et utiliser `<VerifyNote lastVerified="YYYY-MM-DD" source="..." />`. Ne codez jamais en dur un fait relatif à un modèle ; renvoyez vers le [tableau des modèles](/docs/whats-new/models-and-pricing).
3. **Soyez tranché, puis honnête.** Donnez d'abord la manière recommandée ; mentionnez les alternatives. Ne dupliquez pas la documentation officielle — liez-la.
4. **Anglais simple, des exemples plutôt que de la théorie.** Des phrases courtes. Montrez, ne vous contentez pas de dire.

## Modèle de front-matter de page

```markdown
---
sidebar_position: 2
title: Your Page Title
description: One sentence shown in search and previews.
keywords: [relevant, search, terms]
---

<LevelBadge level="beginner" />

Your first paragraph...
```

## Ce qui fait une excellente première contribution

- Ajouter un terme manquant au **glossaire**.
- Ajouter un **modèle** ou un prompt que vous utilisez déjà.
- Corriger quelque chose de **récemment publié** qu'une page présente désormais de façon erronée (et mettre à jour sa date `lastVerified`).
- Traduire une page — voir le [Guide pratique de traduction](/docs/contribute).

## Licences et signature

En contribuant, vous acceptez que votre prose soit partagée sous **CC BY 4.0** et votre code sous **MIT**, et vous ajoutez une ligne `Signed-off-by` (DCO) à vos commits (`git commit -s`). Détails sur la [page de gouvernance](/docs/contribute).

Soyez bienveillant — nous suivons le [code de conduite](/docs/contribute) Contributor Covenant.
