Vous avez déjà choisi un modèle. Mais le programme qui *enveloppe* le modèle — la boucle qui lit les fichiers, planifie, exécute des commandes, vérifie son propre travail et réessaie — est un choix distinct, et il compte au moins autant. Le nom que la communauté donne à cet enveloppement est le **harnais** (ou **agent**), et le même modèle peut sembler brillant dans l'un et médiocre dans l'autre. Cette page cartographie les agents de codage en terminal de 2026, ce qui les distingue vraiment, et comment garder votre configuration portable pour maintenir des coûts de changement bas.

## Le modèle n'est pas l'agent

Un modèle de pointe est un prédicteur de texte. Un **agent de codage** est le harnais qui l'entoure : l'invite système, l'ensemble d'outils (lire, éditer, exécuter le shell, chercher), la boucle de planification, la stratégie de gestion du contexte, le modèle de permissions et de bac à sable, et la logique de réessai. Changez le harnais et vous changez les résultats — des retours de praticiens tout au long de 2026 décrivent le même modèle variant de dizaines de points sur de vraies tâches de dépôt selon l'agent qui le pilote.

Cela a deux conséquences pratiques :

- **Un chiffre de benchmark est un *appariement*, pas un score de modèle.** « 83 % sur Terminal-Bench » est un résultat de *modèle + harnais + échafaudage*. Changez n'importe quelle partie et il bouge.
- **Votre boucle est la vraie unité de comparaison.** Le meilleur agent est celui qui fonctionne le mieux sur *votre* stack, dans *votre* terminal, à un prix que vous pouvez assumer — exactement la même logique que pour choisir un modèle de codage. Voir [Claude vs GPT vs Gemini pour le codage](/docs/models/claude-gpt-gemini-for-coding) pour la moitié « modèle » de la décision ; cette page couvre la moitié « harnais ».

## Le paysage en 2026

L'espace des agents de terminal s'est scindé en deux camps : les **agents propriétaires** réglés pour les modèles d'un seul laboratoire, et les agents **indépendants du modèle** que vous pointez vers le modèle (ou les poids locaux) de votre choix.

| Agent | Qui | Ouverture | Modèles | Forme |
|---|---|---|---|---|
| **Claude Code** | Anthropic | Code fermé | Claude (propriétaire) | Travail multi-fichiers autonome et agentique ; bac à sable imposé par l'OS ; écosystème profond skills/MCP/sous-agents |
| **Codex CLI** | OpenAI | Ouvert (Apache-2.0, Rust) | GPT (propriétaire) | Agent de terminal autonome ; bac à sable Windows natif ; large couverture de plateformes |
| **Gemini CLI** | Google | Open source | Gemini (propriétaire) | Agent à grand contexte, intégrations à l'écosystème Google (voir la note de fraîcheur ci-dessous) |
| **opencode** | Communauté (anomalyco) | Ouvert (MIT, TypeScript) | N'importe lequel (indépendant du modèle) | L'agent ouvert le plus étoilé ; CLI + bureau ; apportez votre propre modèle |
| **Aider** | Communauté | Ouvert (Apache-2.0) | N'importe lequel, y compris local | Binôme de programmation natif Git, pas un orchestrateur pleinement autonome |
| **Cline / Goose / autres** | Communauté | Ouvert | N'importe lequel | Intégré à l'éditeur (Cline) ou agents ouverts généralistes (Goose) ; indépendants du modèle |

<VerifyNote lastVerified="2026-07-08" source="https://github.com/google-gemini/gemini-cli">
**Gemini CLI est en pleine évolution.** À la mi-2026, Google a modifié l'accès au palier gratuit et a orienté certains utilisateurs vers un CLI « Antigravity » distinct. Savoir si Gemini CLI, Antigravity ou quelque chose de plus récent est le bon point d'entrée Google est exactement le genre de fait qui se périme — confirmez l'état actuel dans la documentation officielle de Google avant de vous engager.
</VerifyNote>

## Portabilité : écrivez votre contexte une seule fois

Le plus grand coût de changement caché est le **contexte de votre projet** — commandes de build, conventions, règles de test, choses que l'agent ne peut pas deviner. En 2026, cela a un standard de fait : **`AGENTS.md`**, un fichier en Markdown simple à la racine de votre dépôt, géré par l'Agentic AI Foundation sous l'égide de la Linux Foundation et lu nativement par Codex, Gemini CLI, Cursor, Aider, l'agent de codage de Copilot, opencode, Goose et plus de 20 autres outils (plus de 60 000 dépôts l'ont adopté).

- **`AGENTS.md`** = contexte de projet inter-outils. Écrivez-le une fois ; la plupart des agents le lisent.
- **`CLAUDE.md`** = le fichier de mémoire de projet propre à Claude Code (voir [CLAUDE.md](/docs/claude-code/claude-md)). Vous pouvez conserver un petit `CLAUDE.md` qui pointe vers `AGENTS.md`, ou maintenir les deux.
- **`SKILL.md`** = la spécification distincte d'Anthropic pour empaqueter des [skills](/docs/claude-code/skills) réutilisables avec du frontmatter et des scripts groupés. Elle est *complémentaire* à `AGENTS.md`, pas concurrente — l'une décrit le projet, l'autre empaquette une procédure réutilisable.

Garder le contexte dans `AGENTS.md` signifie que le jour où vous essayez un autre agent, vos règles de projet suivent gratuitement.

<PromptCard title="AGENTS.md minimal — n'écrivez que ce qu'un agent ne peut pas déduire">{`# AGENTS.md

## Setup
- Install: \`pnpm install\`
- Dev server: \`pnpm dev\` (port 3000)

## Build & test (run before every commit)
- Typecheck: \`pnpm typecheck\`
- Lint: \`pnpm lint\`
- Tests: \`pnpm test\` — all must pass

## Conventions
- TypeScript strict; no \`any\`.
- Components in \`src/components\`, one per file.
- Never edit files under \`generated/\` by hand.

## Constraints
- Do NOT run \`git push\` or open PRs without being asked.
- Secrets live in \`.env.local\`; never commit them.
`}</PromptCard>

<Callout type="warning" items={["Ne mettez dans AGENTS.md que ce que l'agent ne peut pas découvrir par lui-même. ## Une décision, en une respiration

- **Travail Claude profond et autonome avec l'écosystème d'outils le plus riche** → **Claude Code**.
- **Agent propriétaire open source lié à GPT, avec une forte couverture plateforme/Windows** → **Codex CLI**.
- **Vous voulez changer de modèle librement ou faire tourner des poids locaux** → **opencode** ou **Cline**.
- **Éditions prudentes, natives Git, avec l'humain dans la boucle** → **Aider**.
- **Grand pari sur le contexte de l'écosystème Google** → **Gemini CLI** (confirmez d'abord l'accès actuel).

Quel que soit votre choix, gardez le contexte de votre projet dans `AGENTS.md` et prouvez ce choix avec une vraie tâche dans votre propre dépôt avant de vous engager.
