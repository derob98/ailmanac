Bonne nouvelle : **les fondamentaux du prompting se transfèrent à tous les modèles.** Si vous savez bien prompter Claude, vous savez bien prompter ChatGPT, Gemini ou Llama. Quelques détails de surface diffèrent — voici la pierre de Rosette.

## Ce qui se transfère partout

Cela fonctionne sur tous — il s'agit de *communication*, pas d'un fournisseur :

- **Donner du contexte** : objectif, audience, format, ton ([Les bases du prompting](/docs/prompting/basics)).
- **Des exemples** ([few-shot](/docs/prompting/few-shot)).
- **Demander le raisonnement** sur les tâches difficiles.
- **Contraindre la sortie** (longueur, format, « JSON uniquement »).
- **Itérer** en langage naturel.
- **Rôles / system prompts** — chaque grand modèle possède ce concept ([Rôles](/docs/foundations/roles)).

Apprenez-les une fois et vous êtes portable.

## Ce qui diffère (et comment l'ajuster)

| Aspect | À noter lors d'un changement |
|---|---|
| **Indices de structure** | Claude adore les [balises XML](/docs/prompting/xml-tags) ; d'autres modèles les acceptent mais peuvent préférer les titres Markdown ou le JSON. |
| **Intensité du ton** | Les anciens modèles toléraient les prompts énergiques/EN MAJUSCULES ; les modèles récents, plus littéraux, préfèrent des instructions calmes et spécifiques. Réajustez selon le modèle. |
| **Raisonnement** | « Réfléchis étape par étape » aide certains ; les modèles de raisonnement dédiés le font en interne — n'en rajoutez pas. |
| **Prefilling** | Mettre des mots dans la bouche de l'assistant est un levier propre à Claude ; le support varie ailleurs. |
| **Poids du system prompt** | Tous en ont un, mais le degré auquel il est suivi varie — testez. |
| **Appel d'outils/de fonctions** | Même concept, schémas de requête différents selon le fournisseur. |

## Un flux de travail portable

1. Rédigez le prompt avec une structure **universelle** (contexte + format + exemples).
2. **Testez sur votre modèle cible** avec quelques entrées réelles ([eval](/docs/foundations/evals)).
3. Ajustez uniquement les **détails de surface** ci-dessus si nécessaire.
4. Gardez les prompts sous gestion de version pour que changer de modèle coûte peu.

:::tip Ne réécrivez pas tout de zéro
Passer à un nouveau modèle nécessite rarement un nouveau prompt — d'ordinaire juste un ajustement de ton/format. Votre structure et vos exemples se conservent.
:::
