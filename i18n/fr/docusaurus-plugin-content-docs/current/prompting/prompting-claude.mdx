L'essentiel du [prompting](/docs/prompting/basics) est universel. Mais quelques techniques sont particulièrement efficaces avec Claude — et certaines habitudes issues d'anciens modèles sont désormais contre-productives. Voici ce qui est réellement propre à Claude.

## Misez là-dessus avec Claude

- **[Balises XML](/docs/prompting/xml-tags).** Claude est bien réglé pour les prompts structurés avec des balises nommées pour les entrées, les instructions et les exemples. C'est l'habitude Claude à plus forte valeur.
- **Soyez clair et direct.** Énoncez l'objectif, les contraintes et ce à quoi ressemble le « terminé ». Claude suit de près les instructions explicites.
- **Attribuez-lui un rôle** via le [system prompt](/docs/foundations/roles).
- **Laissez-le réfléchir sur les problèmes difficiles** — demandez un raisonnement étape par étape, ou utilisez la [réflexion étendue/l'effort](/docs/api/thinking-and-effort) sur les modèles de raisonnement (où vous n'avez souvent *pas* besoin de demander explicitement la chaîne de pensée).
- **Utilisez des exemples** ([few-shot](/docs/prompting/few-shot)) pour le format et les cas limites.
- **Préremplissez** le début de la réponse de Claude (via l'API) pour orienter le format — par ex. commencez le tour de l'assistant par `{` pour pousser du JSON, ou par un titre pour pousser une structure.

## Abandonnez ces vieilles habitudes

:::warning Les modèles récents, plus littéraux, changent les règles
- **Levez le pied sur les MAJUSCULES et les prompts en mode menace** (« TU DOIS… SINON »). Les modèles Claude récents suivent les instructions littéralement, et des instructions formulées poliment et de manière spécifique fonctionnent mieux que les cris.
- **Ne surconcevez pas** avec des échafaudages élaborés « façon jailbreak » — des instructions claires et honnêtes surpassent les astuces.
- **Souvent, moins c'est plus** — un prompt resserré bat un prompt tentaculaire qui enterre la demande.
:::

## Claude vs autres modèles

Les *principes* se transfèrent (voir [Traduction de prompts entre IA](/docs/prompting/cross-ai-translation)), mais l'**affinité pour les balises XML** et le **prefilling** sont les éléments à retenir comme propres à Claude. Lorsque vous changez de modèle, revérifiez l'intensité du ton et les indices de formatage.
