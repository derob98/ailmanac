أنت تفكّر بالفعل بلغة Claude. ثم يُقسِم زميلٌ بأن **Grok** هو الأفضل للأسئلة عمّا يحدث *الآن*، أو يحتاج مشروعٌ إلى مساعد موصول مباشرةً بـ X/تويتر، أو يسلّمك أحدهم قاعدة كود على شكل OpenAI ويطلب منك توجيهها إلى xAI بدلًا من ذلك. الخبر السار: كل ما تعرفه تقريبًا ينتقل معك — وفي جانب واحد (الـ API) يكاد التبديل يكون مجّانيًا. هذه الصفحة تُسقِط نموذجك الذهني الخاص بـ Claude على **Grok من xAI**، وتُشير إلى الحفنة القليلة من الأشياء التي تعمل فعلًا بشكل مختلف، وتخبرك متى يميل Grok إلى أن يكون الخيار الأفضل.

## خريطة المفاهيم في 60 ثانية

إن قرأت قسمًا واحدًا فقط، فاقرأ هذا. إليك كيف تصطفّ الأشياء التي تعرفها في Claude مع عالم Grok:

| في Claude تسمّيه… | في عالم Grok هو… | الفكرة نفسها؟ |
| --- | --- | --- |
| Claude.ai (تطبيق المحادثة) | **تطبيق Grok** (grok.com، وتطبيقات X، والموبايل) | نعم — الواجهة الاستهلاكية/المساعِدة |
| مُنتقي النموذج (Opus / Sonnet / Haiku) | متغيّرات Grok (نموذج رائد مثل **Grok 4.3**، ومتغيّر **سريع** لاستدعاء الأدوات، ومتغيّر **برمجي**) | نعم — الفكرة نفسها، بأسماء مختلفة |
| Claude Code (البرمجة الوكيلة في الطرفية) | **Grok Build** (واجهة سطر أوامر لوكيل البرمجة من xAI) | قريب — انظر أدناه |
| الكونسول / API + مساحة العمل | **واجهة xAI API** (`https://api.x.ai/v1`) + كونسول xAI | نعم — واجهة المطوّر للنمذجة والحصول على مفتاح |
| استخدام الأدوات / استدعاء الدوال (API) | **Function calling** (بشكل متوافق مع OpenAI) | نعم — الحلقة نفسها، وهنا حتى الـ JSON مألوف |
| البحث في الويب / الاستشهادات | **بحث فوري أصيل في X + الويب** مدمَج في النموذج | قريب — لكن أكثر "حيويةً" بوضوح |

الخلاصة: لا يكاد يوجد *مفهوم جديد* لتتعلّمه. الأمر في معظمه **إعادة تسمية وإعادة تموضع** لأشياء تفهمها بالفعل — زائد مجالين يتجاوز فيهما Grok حدود Claude فعلًا: **الوصول الفوري إلى بيانات X/الويب**، و**واجهة API متوافقة مع OpenAI جاهزة للاستبدال المباشر**.

## الـ API جاهز للاستبدال المباشر — وهذه هي النقطة الكبرى للبنّائين

هذا هو الفرق الذي يغيّر أكثر ما يغيّر للمطوّرين: **واجهة xAI API تتّبع مواصفة واجهة OpenAI.** عنوان الأساس هو `https://api.x.ai/v1`، فإن كان لديك كود يتحدّث بالفعل مع عميل متوافق مع OpenAI، فالهجرة غالبًا مجرّد *تغيير عنوان الأساس واستبدال المفتاح.* <VerifyNote lastVerified="2026-07-01" source="https://docs.x.ai/developers/quickstart">توثّق xAI واجهتها بأنها متوافقة مع OpenAI بعنوان أساس `https://api.x.ai/v1`؛ لكن السطح المتوافق الدقيق (أيّ المعاملات ونقاط النهاية تتطابق بنظافة) يتطوّر — تحقّق من دليل البدء السريع الحالي لدى xAI.</VerifyNote>

بالنسبة لمستخدم Claude يستحق هذا أن يُستوعب، لأن واجهة Messages في Claude لها شكلها *الخاص* (صيغة طلب/ردّ خاصة، وكتل `tool_use`/`tool_result` خاصة). قادمًا من Claude، عدّل ردّة فعلك هكذا:

## البحث الفوري في X + الويب هو العنوان الحقيقي

هنا يفترق Grok أكثر ما يفترق عن عاداتك في Claude. بُني Grok بواسطة xAI بـ **وصول فوري أصيل إلى المعلومات من X (تويتر) والويب**، وتضع xAI البحث الحيّ كجزء أساسي من طريقة إجابة النموذج. <VerifyNote lastVerified="2026-07-01" source="https://x.ai/news/grok-4">تصف xAI نموذج Grok باستخدام أدوات أصيل وتكامل بحث فوري (بما في ذلك بيانات من X)؛ لكن سلوك الاسترجاع الدقيق، ومدى الحداثة، وأيّ حدود بحسب الخطة تتغيّر — تحقّق من صفحات منتَج xAI الحالية.</VerifyNote>

بالنسبة لمستخدم Claude، التحوّل الذهني هو هذا: مع Claude تلجأ إلى أداة بحث، أو خطّ استرجاع، أو موصِّل حين تحتاج معلومات *حالية*، وتفكّر بعناية في حدود معرفتك الزمنية. مع Grok، "ما الذي يحدث الآن على X / على الويب" أقرب إلى إعداد افتراضي مدمَج. وهذا يجعل Grok خيارًا طبيعيًا للأسئلة **الحيّة، وعن الأحداث الجارية، ونبض الشبكات الاجتماعية** — ويغيّر ما ينبغي أن *تشكّك* فيه: النتائج الحيّة ما زالت تحتاج انضباط التحقّق من المصادر نفسه الذي تطبّقه على أي محتوى ويب.

## Grok Build هو جواب xAI على Claude Code

إن كنت تعيش في **Claude Code**، فإن **Grok Build** من xAI هو أقرب مكافئ: وكيل برمجة تقوده من الطرفية، ينافس في المضمار نفسه الذي فيه Claude Code وCodex CLI من OpenAI. بُني حول متغيّر Grok مضبوط للبرمجة ونهج وكلاء فرعيين متوازيين للعمل عبر قواعد كود أكبر، مع وضع مستقلّ طويل الأمد لتسليم المهام الأكبر. <VerifyNote lastVerified="2026-07-01" source="https://docs.x.ai/developers/models">أطلقت xAI واجهة سطر أوامر لوكيل البرمجة (Grok Build) ومتغيّر نموذج مضبوط للبرمجة؛ لكن التوفّر (مثلًا أيّ مستوى اشتراك)، ومجموعة الميزات الدقيقة، ومعرّف النموذج تتحرّك بسرعة — عامِل التفاصيل على أنها حسّاسة زمنيًّا وتحقّق منها.</VerifyNote>

تنتقل المفاهيم واحدًا لواحد تقريبًا من Claude Code:

| فكرة Claude Code | مكافئ Grok Build | ملاحظات |
| --- | --- | --- |
| تعديلات وكيلة من الطرفية | نفسها — وكيل برمجة بواجهة سطر أوامر | الحلقة الجوهرية (قراءة ← تخطيط ← تعديل ← تشغيل) هي الشكل المشترك لهذه الفئة بأكملها |
| وكلاء فرعيون / عمل متوازٍ | بنية وكلاء فرعيين متوازيين | كلاهما يقسّم العمل الكبير عبر مساعِدين؛ تختلف تفاصيل التنسيق |
| مهامّ مستقلّة طويلة الأمد | وضع "أنجِز الهدف" طويل الأمد | سلّم تنفيذًا أكبر ودَعه يخطّط-ينفّذ-يتحقّق |

النقطة الدائمة: **مهارة قيادة واجهة سطر أوامر برمجية وكيلة قابلة للنقل.** كتابة مهمة واضحة، وإبقاء السياق محكمًا، ومراجعة الفروقات، والتوجيه بنقاط تفتيش — كلّها الانضباطات نفسها سواء كان الوكيل هو Claude Code أو Codex أو Grok Build. انظر [ما هو Claude Code؟](/docs/claude-code/what-is-claude-code) للنموذج الذهني، و[وكلاء البرمجة المحلّيون](/docs/models/local-coding-agents) للمجال الأوسع.

## ما الذي يحتاج فعلًا إلى تعديل

معظم درايتك بـ Claude تنتقل. إليك القائمة القصيرة للأشياء المختلفة فعلًا:

- **لهجة الـ API (في صالحك).** يتحدّث Grok بشكل متوافق مع OpenAI، لا بـ Anthropic Messages. إن كنت قادمًا من واجهة Claude، فأنت تنتقل إلى شكل طلب/ردّ OpenAI — وتكسب إعادة استخدام سهلة لأي شيء مبنيّ سلفًا لنقاط النهاية المتوافقة مع OpenAI.
- **الفوري بشكل افتراضي.** التأريض على الأحداث الجارية أقرب إلى أن يكون مدمَجًا في Grok. عدّل متى تلجأ إليه — واستمرّ في التحقّق من المصادر.
- **الشخصية والحدود.** نبرة Grok وحدوده على المحتوى مضبوطة بشكل مختلف عن Claude. لا تفترض أن موجّه نظام معايَرًا لأسلوب Claude يُنتج النبرة نفسها أو الرفض نفسه.
- **التفاصيل المتقلّبة.** أسماء النماذج، والمتغيّرات، وحدود السياق، والأسعار تتحرّك بسرعة (أطلق Grok إصدارات متعدّدة عبر 2026). اكتسب عادة التأكيد من المصدر — تمامًا كما ينبغي أن تفعل مع Claude أيضًا.

## متى يميل Grok إلى التألّق

كلاهما مساعد عام قادر، والفارق في أي مهمة بعينها كثيرًا ما يكون صغيرًا ومتبدّلًا — فعامِل هذا على أنه *ميول*، لا قوانين، ودَع [تقييمك على بياناتك](/docs/models/choosing-a-model) يقرّر:

- **العمل الحيّ وعلى الأحداث الجارية.** "ما الذي يحدث الآن"، والمواضيع الرائجة، وأسئلة نبض الشبكات الاجتماعية تلعب لصالح بحث Grok الفوري الأصيل في X/الويب.
- **الهجرات السلسة من OpenAI.** إن كان لديك كود متوافق مع OpenAI، فتوجيهه إلى xAI من أقلّ عمليات التبديل احتكاكًا في المجال.
- **تسليمات البرمجة أوّلًا-في-الطرفية.** إن كنت تحبّ سير عمل Claude Code / Codex، فإن Grok Build خيار ثالث طبيعي لتجرّبه اختبارًا مقارنًا على مهامّك.

الجواب الصادق في معظم الأحيان: **أيّهما أنت (أو فريقك) مُعَدّ فيه بالفعل، وأيّهما يفوز في تقييمك على المهمة التي تهم.** المهارة قابلة للنقل؛ تكلفة الإعداد هي تكلفة التبديل الحقيقية. للطريقة المحايدة تجاه المزوّد في الاختيار، انظر [اختيار نموذج](/docs/models/choosing-a-model) و[ما تكلفة الذكاء الاصطناعي عبر المزوّدين](/docs/models/what-ai-costs-across-providers).
