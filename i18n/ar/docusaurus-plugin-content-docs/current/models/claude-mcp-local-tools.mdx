تريد أن يكون Claude هو **العقل** وأن يوفّر جهازك **الأيدي** — قراءة ملفاتك، الاستعلام من قاعدة بياناتك، تشغيل سكربتك، بل وحتى استدعاء نموذج محلي أو وكيل محلي، كل ذلك دون أن تغادر بياناتك حاسوبك المحمول. هذا التآزر يمتلك بالفعل معيارًا قياسيًا: **بروتوكول Model Context Protocol (MCP)**. في جملة واحدة: MCP هو معيار مفتوح يتيح لعميل ذكاء اصطناعي استدعاء أدوات وبيانات خارجية عبر برامج صغيرة تُسمّى **خوادم MCP** — وهذه الخوادم يمكن أن تعمل بخصوصية ومحليًا على جهازك الخاص. توضّح هذه الصفحة ما هو MCP، وكيف تتلاءم البنية المحلية معًا، وكيف **تُضيف** خادمًا محليًا إلى Claude، وكيف **تبني** خادمًا صغيرًا خاصًا بك.

## ما هو MCP (في جملة واحدة)

**MCP هو معيار مفتوح لربط تطبيقات الذكاء الاصطناعي بالأنظمة الخارجية.** يتحدث عميل ذكاء اصطناعي (مثل Claude) ببروتوكول واحد؛ وأي **خادم MCP** يتحدث به يمكنه الاتصال وتقديم قدرات جديدة للعميل. القياس الرسمي هو **منفذ USB-C للذكاء الاصطناعي**: بدلًا من تكامل مخصّص لكل أداة، تبني نحو موصّل واحد فيعمل في كل مكان.

قدّمت Anthropic بروتوكول MCP وجعلته مفتوح المصدر، ومنذ اليوم الأول جاء مصحوبًا بـ **دعم خادم MCP المحلي** كي يتمكن Claude من الاتصال بالأنظمة الداخلية ومجموعات البيانات التي تعمل على جهازك الخاص.

<VerifyNote lastVerified="2026-06-28" source="https://modelcontextprotocol.io/">
أرقام التبنّي، وتاريخ مراجعة المواصفات، والحوكمة (الجهة المشرفة على MCP)، والقائمة الدقيقة للعملاء الذين يدعمون MCP تتغير باستمرار. عامل أي رقم محدد على أنه قابل للتقادم وتأكّد منه في <a href="https://modelcontextprotocol.io/">modelcontextprotocol.io</a> وإعلان Anthropic. أما الحقائق الثابتة — أن MCP معيار مفتوح، عميل ↔ خادم، وأن الخوادم يمكن أن تعمل محليًا، وأنه مُتبنّى على نطاق واسع عبر العملاء — فهي مستقرة.
</VerifyNote>

يمكن لخادم MCP أن يعرض ثلاثة أنواع من الأشياء للعميل:

- **الأدوات (Tools)** — إجراءات يمكن للنموذج استدعاؤها (قراءة ملف، تشغيل استعلام، استدعاء API). لها الشكل نفسه لـ [استخدام الأدوات](/docs/api/tool-use) الأصلي في النموذج: قدرة مُسمّاة بمدخلات محدَّدة الأنواع.
- **الموارد (Resources)** — بيانات يمكن للعميل قراءتها (ملفات، سجلات، مستندات).
- **المطالبات (Prompts)** — قوالب مطالبات قابلة لإعادة الاستخدام والتخصيص يقدّمها الخادم.

بالنسبة لهذه الصفحة، العنوان الأبرز هو **الأدوات**: خادم MCP محلي هو الطريقة التي تمنح بها Claude قدرة جديدة وخاصة.

## لماذا هذا هو التآزر الذي تريده

ما يقصده الناس بعبارة "Claude كالعقل، وحاسوبي كالجسد" هو تحديدًا التقسيم بين عميل/خادم في MCP:

- **Claude هو المنسّق.** يقرأ هدفك، ويقرر أي أداة يستدعي، ويستدعيها، ويقرأ النتيجة، ويقرر الخطوة التالية.
- **خوادم MCP المحلية هي القدرات.** كل واحد منها برنامج صغير *تُشغّله أنت* على *جهازك أنت*: خادم نظام ملفات، خادم قاعدة بيانات، غلاف حول سكربت محلي، بل وحتى خادم يستدعي [نموذجًا محليًا](/docs/models/local-ai-agents) أو يُسلّم العمل إلى وكيل محلي.

ولأن الخادم يعمل محليًا ويتحدث إليه Claude عبر مدخلات/مخرجات جهازك القياسية، **تبقى بياناتك وأفعالك على عتادك** — يقوم النموذج بالتنسيق، لكن قراءات الملفات، وأوامر SQL، وتنفيذ السكربت تحدث محليًا. هذه هي النسخة الحافظة للخصوصية من العمل الوكيلي: العقل في السحابة (أو محليًا أيضًا)، والأيدي حصرًا على جهازك.

ولأن MCP هو **معيار مفتوح واحد مُتبنّى عبر عملاء كثيرين**، فإن الخادم الذي تبنيه ليس مقيّدًا بتطبيق واحد. الخادم المحلي نفسه الذي تربطه بـ [Claude Code](/docs/claude-code/what-is-claude-code) يمكن أن يستخدمه عملاء آخرون يتحدثون MCP — ابنِ مرة واحدة، وأعد الاستخدام في كل مكان.

## البنية المحلية

أصغر صورة مفيدة تتكون من جزأين وقناة واحدة:

- **عميل MCP (المضيف).** تطبيق الذكاء الاصطناعي — مثل [Claude Code](/docs/claude-code/what-is-claude-code) أو Claude Desktop. يحتوي على النموذج ويقرر أي الأدوات يستدعي.
- **خادم MCP.** عملية منفصلة تُعلن عن الأدوات/الموارد/المطالبات وتنفّذها عند الطلب.
- **النقل (Transport).** بالنسبة للخوادم المحلية، يُشغّل العميل الخادم كعملية فرعية ويتحادثان عبر **stdio** (المدخلات/المخرجات القياسية) باستخدام رسائل JSON-RPC. لا شبكة، لا منفذ — مجرد أنبوب محلي بين عمليتين على جهازك. (توجد أيضًا خوادم بعيدة، عبر HTTP؛ لكن هذه الصفحة تتناول الحالة المحلية عبر stdio.)

```
┌─────────────────────────┐         stdio (JSON-RPC)        ┌──────────────────────────┐
│  MCP CLIENT / HOST       │ ──── launches as subprocess ──▶ │  LOCAL MCP SERVER         │
│  Claude Code / Desktop   │ ◀── tools/resources/prompts ─── │  filesystem · db · script │
│  (the model = the brain) │                                 │  · local model / agent    │
└─────────────────────────┘                                  └──────────────────────────┘
                                                              runs on YOUR machine, YOUR data
```

لا يلمس النموذج قرصك مباشرة أبدًا — بل يطلب من الخادم المحلي، والخادم (الذي يعمل بصلاحياتك) يقوم بالعمل ويعيد النتيجة.

أبسط أمر `claude mcp add` لخادم نظام ملفات محلي (كل ما يلي `--` هو أمر التشغيل):

<PromptCard title="Add a local filesystem MCP server to Claude Code">{`claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /path/to/allowed/dir`}</PromptCard>

إن كنت تفضّل تحرير الإعداد مباشرة، فيُعرَّف خادم stdio بالأمر الذي يشغّله. الشكل يبدو هكذا:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    }
  }
}
```

<VerifyNote lastVerified="2026-06-28" source="https://modelcontextprotocol.io/">
الأوامر الفرعية الدقيقة لـ `claude mcp`، ونطاقات الإعداد، وموقع ملف الإعداد، والأسماء الدقيقة لحزم الخوادم المرجعية كلها تتطوّر. تأكّد من الصيغة الحالية في <a href="/docs/claude-code/mcp">دليل Claude Code MCP</a> ومستودع الخوادم المرجعية قبل الاعتماد على راية أو اسم حزمة محدد.
</VerifyNote>

## ابنِ خادم MCP صغيرًا

الخادم هو مجرد برنامج يستخدم إحدى حِزم SDK الخاصة بـ MCP، ويعلن عن أداة، ويبدأ نقل stdio. إليك الفكرة كاملة بلغة TypeScript — أداة واحدة تجمع رقمين (بدّل الجسم بـ "شغّل سكربتي"، "استعلم من قاعدة بياناتي"، "استدعِ نموذجي المحلي"):

```bash
npm install @modelcontextprotocol/sdk zod
```

```ts
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({ name: "my-local-tools", version: "0.1.0" });

// Declare ONE tool. Inputs are typed; the body runs locally with your privileges.
server.registerTool(
  "add",
  {
    description: "Add two numbers",
    inputSchema: { a: z.number(), b: z.number() },
  },
  async ({ a, b }) => ({
    content: [{ type: "text", text: String(a + b) }],
  }),
);

// Talk to the client over stdio — this is what makes it a LOCAL server.
await server.connect(new StdioServerTransport());
```

سجّله لدى Claude تمامًا مثل أي خادم stdio آخر — وجّه أمر التشغيل نحو ملفك:

<PromptCard title="Add your own local server to Claude Code">{`claude mcp add my-local-tools -- node /abs/path/to/server.js`}</PromptCard>

هذه هي الحلقة الكاملة: يرى Claude (العقل) أداة جديدة، ويستدعيها عند الحاجة، وتعمل شفرتك (الأيدي) محليًا وتعيد النتيجة. ولأنه يتحدث بالبروتوكول المفتوح نفسه، يعمل هذا الخادم أيضًا في عملاء MCP آخرين دون أي تغيير.

<Callout type="warning" items={["تعمل خوادم MCP بصلاحياتك ويمكنها اتخاذ إجراءات حقيقية — لا تُثبّت إلا خوادم تثق بها، وانتبه لحقن المطالبات (prompt-injection) عبر نتائج الأدوات."]} />
