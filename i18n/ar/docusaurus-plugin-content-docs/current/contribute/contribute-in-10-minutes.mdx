AILmanac "مكتمل ودائم التحديث" فقط لأن أشخاصًا مثلك يضيفون إليه. أنت **لست** بحاجة لأن تكون مطورًا — فإصلاح خطأ مطبعي، أو إضافة مصطلح في المسرد، أو مشاركة قالب، كلها مساهمات حقيقية.

## المسار السريع

1. **اعثر على الصفحة** على الموقع وانقر على **"حرّر هذه الصفحة"** في الأسفل (تفتح الملف على GitHub).
2. انقر على قلم التحرير ✏️ لتحرّر في متصفحك. ينشئ GitHub نسخة (fork) من المستودع لك تلقائيًا.
3. أجرِ تغييرك بصيغة Markdown.
4. في الأسفل، اكتب وصفًا قصيرًا و**اقترح التغييرات** ← **افتح طلب سحب**.
5. يراجعه أحد القائمين على الصيانة. هذا كل شيء. 🎉

تفضّل العمل محليًا؟ استنسخ المستودع، ثم:

```bash
npm install
npm run start      # live preview at http://localhost:3000
npm run build      # must pass: it fails on broken links
```

## القواعد الأربع التي تتبعها كل صفحة

الالتزام بهذه يجعل المراجعة سريعة ويُبقي الجودة عالية:

1. **حدِّد المستوى.** ابدأ المحتوى بـ `<LevelBadge level="beginner" />` (أو `intermediate` / `advanced` / `all`).
2. **اربط المصدر للحقائق المتقلّبة.** أي شيء يتعلق بأسماء النماذج، أو الأسعار، أو الحدود، أو الميزات التي تتغير يجب أن يستشهد بالصفحة الرسمية — واستخدم `<VerifyNote lastVerified="YYYY-MM-DD" source="..." />`. لا تُضمِّن حقيقة عن نموذج كقيمة ثابتة أبدًا؛ بل اربط بـ[جدول النماذج](/docs/whats-new/models-and-pricing).
3. **كن صاحب رأي، ثم كن أمينًا.** قدّم الطريقة المُوصى بها أولًا؛ وأشِر إلى البدائل. لا تكرّر الوثائق الرسمية — بل اربط بها.
4. **لغة بسيطة، والأمثلة قبل النظرية.** جُمل قصيرة. أرِ، لا تكتفِ بالإخبار.

## قالب البيانات الأمامية للصفحة

```markdown
---
sidebar_position: 2
title: Your Page Title
description: One sentence shown in search and previews.
keywords: [relevant, search, terms]
---

<LevelBadge level="beginner" />

Your first paragraph...
```

## ما الذي يصنع مساهمة أولى رائعة

- أضف مصطلحًا ناقصًا في **المسرد**.
- أضف **قالبًا** أو مُوجِّهًا (prompt) تستخدمه بالفعل.
- أصلح شيئًا **شُحن حديثًا** باتت صفحة ما تخطئ فيه (وحدّث تاريخ `lastVerified` الخاص بها).
- ترجم صفحة — راجع [دليل الترجمة](/docs/contribute).

## الترخيص والإقرار

بالمساهمة فأنت توافق على مشاركة نصوصك بموجب **CC BY 4.0** والكود بموجب **MIT**، وتضيف سطر `Signed-off-by` (DCO) إلى عمليات الإيداع (commits) الخاصة بك (`git commit -s`). التفاصيل في [صفحة الحوكمة](/docs/contribute).

كن لطيفًا — نحن نتبع [مدونة السلوك](/docs/contribute) المنبثقة عن Contributor Covenant.
