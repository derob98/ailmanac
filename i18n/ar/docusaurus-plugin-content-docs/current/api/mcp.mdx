تتطوّر طريقة إعلان MCP على طلبات/وكلاء الواجهة البرمجية وخيارات المصادقة — تأكّد منها في الوثائق الرسمية وعلى modelcontextprotocol.io.
</VerifyNote>

**بروتوكول سياق النموذج (MCP)** هو المعيار المفتوح لربط الذكاء الاصطناعي بالأدوات والبيانات الخارجية. على الواجهة البرمجية، يتيح لك MCP منح Claude وصولًا إلى **مجموعة أدوات خادم كاملة** عبر الإحالة — بدلًا من تعريف كل أداة يدويًا.

## MCP مقابل الأدوات المُعرَّفة يدويًا

| | [استخدام الأدوات](/docs/api/tool-use) (مخصّص) | MCP |
|---|---|---|
| أنت تُعرّف | مخطّط كل أداة، وتشغّلها بنفسك | اتصالًا بخادم *يَنشُر* الأدوات |
| الأفضل لـ | عدد قليل من الدوالّ المخصّصة في تطبيقك | إعادة استخدام تكاملات موجودة (GitHub، قواعد البيانات، المتصفّحات، خدمات SaaS) |
| المصادقة | شيفرتك | OAuth أو مفاتيح أصلية للخادم |

يتعايشان معًا: عرّف أدواتك الخاصة بتطبيقك مباشرةً، واستجلب قدرات جاهزة عبر MCP.

## إعلان خادم

تُرفِق خادم/خوادم MCP بطلب/وكيل (عبر عنوان URL أو أمر، إضافةً إلى المصادقة). عندئذٍ يستطيع Claude استدعاء أدوات الخادم ضمن [حلقة الوكيل](/docs/api/building-agents)، شأنها شأن أي أداة. حقل الطلب الدقيق وتدفّق المصادقة موجودان في [الوثائق الرسمية](https://platform.claude.com/docs/en/build-with-claude/mcp).

## المعيار نفسه، ثلاث واجهات

- **الواجهة البرمجية** (هذه الصفحة) — خوادم MCP على طلباتك/وكلائك.
- **[Claude Code](/docs/claude-code/mcp)** — خوادم MCP في جلسات التطوير لديك.
- **[التطبيقات](/docs/claude-app/connectors)** — MCP يشغّل الموصِّلات (Connectors).

تعلّمه مرة واحدة؛ فهو ينتقل معك.

## الثقة

:::warning خادم MCP هو شيفرة + وصول
لا تتّصل إلا بالخوادم التي تثق بها، وضيّق نطاقها إلى أقل صلاحية ممكنة، وتذكّر أن المحتوى الخارجي الذي تعيده قد يحمل [حقن المطالبات](/docs/security/prompt-injection). راجع خوادم الأطراف الثالثة أولًا — [مراجعة شيفرة الأطراف الثالثة](/docs/security/reviewing-third-party-code).
:::
