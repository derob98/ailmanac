تتغيّر رموز الحالة الدقيقة والترويسات وطبقات حدود المعدّل — تأكّد منها في وثائق الأخطاء/حدود المعدّل الرسمية.
</VerifyNote>

تتحدّث شيفرة الإنتاج إلى خدمة شبكية، لذا يجب أن تتوقّع الإخفاق. قليل من البنية هنا هو الفارق بين تكامل هشّ وآخر يُعتمد عليه.

## خريطة الأخطاء

حالات HTTP النموذجية التي ستتعامل معها:

| الحالة | المعنى | ما العمل |
|---|---|---|
| 400 | طلب غير صالح | أصلح الحمولة؛ لا تُعد المحاولة كما هي |
| 401 | مفتاح API خاطئ/مفقود | تحقّق من بيانات الاعتماد |
| 403 | غير مسموح به | تحقّق من الوصول/الصلاحيات |
| 429 | تجاوز حدّ المعدّل | **تراجع وأعِد المحاولة** (احترم `retry-after`) |
| 500/529 | خطأ في الخادم / تحميل زائد | **أعِد المحاولة مع التراجع التدريجي** |

<Callout type="tip" items={["تُظهر حِزَم SDK هذه الأخطاء كـاستثناءات مُصنَّفة بأنواع، لتتفرّع بوضوح بدلًا من تحليل السلاسل النصية."]} />

## إعادة المحاولة مع التراجع التدريجي

للأخطاء العابرة (429، 5xx)، أعِد المحاولة مع **تراجع أُسّي + اضطراب عشوائي (jitter)**، مع وضع حدّ أقصى:

```python
import time, random
for attempt in range(5):
    try:
        return client.messages.create(...)
    except (RateLimitError, APIStatusError) as e:
        if attempt == 4 or not should_retry(e):
            raise
        time.sleep(min(2 ** attempt + random.random(), 30))
```

<Callout type="warning" items={["كثير من حِزَم SDK تعيد المحاولة على الأخطاء العابرة تلقائيًا — اعرف الإعداد الافتراضي لعميلك قبل إضافة منطقك الخاص، وإلا فقد تتضاعف إعادات المحاولة."]} />

## حدود المعدّل

تُطبَّق الحدود لكل حساب/طبقة (الطلبات والرموز في الدقيقة). عند بلوغ حدٍّ تحصل على **429** مع تلميحات توقيت. استراتيجيات للبقاء تحت السقف:

<Steps items={[{title: "احترم retry-after", body: "عند الحصول على 429، اقرأ تلميح التوقيت في الاستجابة وانتظر تلك المدّة قبل إعادة المحاولة."}, {title: "نعّم الدفعات المفاجئة", body: "وزّع الطلبات على مدى الوقت بدلًا من إطلاقها كلها دفعة واحدة."}, {title: "جمّع العمل غير المتزامن في حزم", body: "انقل المهام عالية الحجم وغير التفاعلية إلى المعالجة بالحِزَم."}, {title: "استخدم نموذجًا أرخص", body: "وجّه الخطوات عالية الحجم إلى نموذج أرخص — راجع اختيار نموذج."}]} />

راجع [اختيار نموذج](/docs/api/choosing-a-model) لانتقاء النموذج المناسب للخطوات عالية الحجم.

## ترحيل النماذج

معرّفات النماذج مؤرّخة/مُصدَّرة وتُهجَر مع الوقت. اعزل نفسك:

- **اقرأ معرّف النموذج من الإعدادات**، لا من قيم حرفية متناثرة.
- **راقب عمليات الإهمال** — راجع [مراقبة الإهمال والترحيل](/docs/whats-new/deprecations) و[جدول النماذج](/docs/whats-new/models-and-pricing).
- **أعِد تشغيل [تقييماتك](/docs/foundations/evals)** عند تبديل النماذج.
