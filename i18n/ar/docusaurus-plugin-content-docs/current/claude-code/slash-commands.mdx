يمكن أن تتغير مواقع ملفات الأوامر وخيارات الواجهة الأمامية (frontmatter) — تأكد من ذلك في وثائق الأوامر المائلة الرسمية.
</VerifyNote>

يحزّم الأمر المائل المخصص مطالبةً تعيد كتابتها باستمرار في `/word` واحد. عرّفه مرة واحدة؛ وأعد استخدامه إلى الأبد (وشاركه مع فريقك).

## كيف تعمل

الأمر مجرد ملف Markdown. ضعه في دليل الأوامر لديك ويصبح اسم ملفه هو الأمر:

- `.claude/commands/changelog.md` ← `/changelog` (أمر مشروع، قابل للمشاركة)
- `~/.claude/commands/changelog.md` ← `/changelog` (شخصي، لكل المشاريع)

متن الملف هو المطالبة التي يشغّلها Claude عند استدعائك له.

## مثال بسيط

```markdown
---
description: Draft a Conventional Commit message from the staged diff
---

Look at the staged changes with `git diff --cached` and write a single
Conventional Commit message (feat/fix/docs/refactor/chore). Output only the
message, no preamble.
```

الآن `/changelog`… أي `/commit` (سمِّه باسم الملف أيًّا كان) يفعل ذلك في كل مرة.

## اللبنات المفيدة

- **`$ARGUMENTS`** — احقن ما كتبه المستخدم بعد الأمر. `/review src/auth.ts` ← يصل المسار إلى مطالبتك.
- **شغّل bash وضمّن المخرَج** — يمكن للأوامر تضمين مخرَج الصدفة (مثل الفرق الحالي) ليتصرف Claude بناءً على الحالة الحية.
- **أشِر إلى الملفات** — وجّه إلى ملفات لتضمين محتوياتها.
- **الواجهة الأمامية (Frontmatter)** — `description` (تُعرَض في القائمة) وخيارات أخرى.

راجع [الوثائق الرسمية](https://docs.anthropic.com/en/docs/claude-code/slash-commands) للصياغة المحددة للوسائط، وbash، ومراجع الملفات.

## أوامر تستحق السرقة

توجد مجموعة بدء في [مكتبة الأوامر المائلة](/docs/templates/slash-commands): رسالة commit، ووصف طلب سحب، ومراجعة شيفرة، وسجل تغييرات، وتوليد مكوّن (scaffold-a-component).

## الأمر مقابل المهارة مقابل الوكيل الفرعي

- **الأمر المائل** — *أنت* تحفّز سير عمل معروف عند الطلب.
- **[المهارة](/docs/claude-code/skills)** — يحمّل Claude الخبرة *تلقائيًا* عند الصلة.
- **[الوكيل الفرعي](/docs/claude-code/subagents)** — فوّض جزءًا من العمل إلى وكيل معزول.
