تمنح **المهارة** Claude خبرة مركّزة يحمّلها *عند الطلب* عندما يتطابق طلبك معها — حتى لا تعيد شرح احتياجاتك في كل مرة. يأتي AILmanac بمجموعة من **حزم المهارات الجاهزة للاستخدام**، واحدة لكل نوع من المستخدمين. تعلّم المفهوم بعمق في صفحة [المهارات](/docs/claude-code/skills).

## حزم جاهزة

احصل على أي منها من مجلد [`/skills`](https://github.com/derob98/ailmanac/tree/main/skills) في المستودع:

| الحزمة | لمن | ما تفعله |
|---|---|---|
| `writing-content-pro` | الكتّاب والمسوّقون والمبدعون | موجز ← مسودة على مراحل ← مطابقة أسلوبك ← إعادة الاستخدام |
| `developer-workflow` | المطورون | برمجة تبدأ بالسياق، مراجعات، اختبارات، رسائل التزام واضحة |
| `research-analyst` | الباحثون والمحللون | توليف مستند إلى المصادر مع انضباط في الاستشهاد |
| `data-analyst` | مستخدمو البيانات وجداول البيانات | SQL/صيغ + قراءات بلغة بسيطة + فحوصات إحصائية منطقية |
| `business-marketing` | المؤسسون وأصحاب الأعمال الصغيرة | التموضع، النصوص الإعلانية، تواصل العملاء، تخطيط خفيف |
| `student-learning` | الطلاب والمتعلمون | تدريس سقراطي + شرح عكسي (تعلّم لا مجرد نسخ) |
| `everyday-productivity` | الجميع | البريد الإلكتروني، الملخصات، التخطيط، القرارات |

كل واحدة تدمج قواعد البيت: **قدّم السياق، وكرّر، وتحقّق مما يهم.**

## ثبّت واحدة (Claude Code)

```bash
# personal — available in all your projects
mkdir -p ~/.claude/skills && cp -r skills/writing-content-pro ~/.claude/skills/

# or per-project — shareable with your team via the repo
mkdir -p .claude/skills && cp -r skills/writing-content-pro .claude/skills/
```

يُفعّل Claude المهارة تلقائياً عندما يتطابق طلبك مع `description` الخاص بها. في Claude.ai / Cowork، أنشئ مهارة والصق `SKILL.md` الخاص بالحزمة (يختلف التوافر حسب الخطة).

## اكتب مهارتك الخاصة — القالب الأدنى

المهارة عبارة عن مجلد يحتوي على `SKILL.md`: مقدمة YAML + تعليمات. الـ `description` هو **المُحفّز** — اكتبه على هيئة "استخدم عند…".

```markdown
---
name: my-skill
description: Use when the user is doing X — be specific about the trigger so Claude knows when to load this and when not to.
---

# My Skill

Short statement of the role/goal.

## Workflow
1. Step one…
2. Step two…

## Rules
- The non-negotiables (accuracy, privacy, output format).

## Default output
What a good response looks like.
```

### قائمة فحص الصلاحية

- [ ] `name` مكتوب بصيغة kebab-case ويطابق المجلد.
- [ ] `description` يوضّح **متى تُستخدم** (المُحفّز)، لا مجرد ماهيتها.
- [ ] التعليمات محددة ومحدودة — مهمة واحدة واضحة.
- [ ] لا أسرار أو بيانات اعتماد أو بيانات شخصية/خاصة بالعميل.
- [ ] تنص على كيفية التعامل مع عدم اليقين (لا تختلق؛ تحقّق).
