Buone notizie: **i fondamenti del prompting si trasferiscono su ogni modello.** Se sai fare buoni prompt con Claude, sai farli bene anche con ChatGPT, Gemini o Llama. Qualche dettaglio di superficie cambia — ecco la stele di Rosetta.

## Cosa si trasferisce ovunque

Questi funzionano su tutti — riguardano la *comunicazione*, non un fornitore:

- **Dai contesto**: obiettivo, pubblico, formato, tono ([Fondamenti del Prompting](/docs/prompting/basics)).
- **Esempi** ([few-shot](/docs/prompting/few-shot)).
- **Chiedi il ragionamento** sui compiti difficili.
- **Vincola l'output** (lunghezza, formato, "solo JSON").
- **Itera** in linguaggio naturale.
- **Ruoli / system prompt** — ogni modello importante ha questo concetto ([Ruoli](/docs/foundations/roles)).

Imparali una volta e sei portabile.

## Cosa differisce (e cosa regolare)

| Aspetto | Nota quando cambi modello |
|---|---|
| **Segnali di struttura** | Claude adora i [tag XML](/docs/prompting/xml-tags); altri modelli li accettano ma possono preferire titoli Markdown o JSON. |
| **Intensità del tono** | I modelli più vecchi tolleravano prompt energici/in TUTTO-MAIUSCOLO; i nuovi modelli letterali preferiscono istruzioni pacate e specifiche. Ricalibra per ogni modello. |
| **Ragionamento** | "Pensa passo dopo passo" aiuta alcuni; i modelli di reasoning dedicati lo fanno internamente — non raddoppiare. |
| **Precompilamento** | Mettere parole in bocca all'assistant è una leva tipica di Claude; il supporto altrove varia. |
| **Peso del system prompt** | Tutti ne hanno uno, ma quanto fedelmente venga seguito varia — testa. |
| **Tool/function calling** | Stesso concetto, schemi di richiesta diversi per ogni fornitore. |

## Un flusso di lavoro portabile

1. Scrivi il prompt con una struttura **universale** (contesto + formato + esempi).
2. **Testalo sul modello di destinazione** con alcuni input reali ([eval](/docs/foundations/evals)).
3. Regola solo i **dettagli di superficie** elencati sopra, se necessario.
4. Tieni i prompt sotto controllo di versione, così cambiare modello costa poco.

:::tip Non riscrivere da zero
Passare a un nuovo modello raramente richiede un nuovo prompt — di solito basta una piccola modifica di tono/formato. La tua struttura e i tuoi esempi si portano dietro.
:::
