AILmanac è "completo e sempre aggiornato" solo perché persone come te lo arricchiscono. **Non** devi essere uno sviluppatore — correggere un refuso, aggiungere un termine al glossario o condividere un template sono tutte contribuzioni concrete.

## Il percorso veloce

1. **Trova la pagina** sul sito e clicca su **"Modifica questa pagina"** in fondo (apre il file su GitHub).
2. Clicca sulla matita ✏️ per modificare nel browser. GitHub fa automaticamente un fork del repo per te.
3. Apporta la tua modifica in Markdown.
4. In fondo, scrivi una breve descrizione e **proponi le modifiche** → **apri una pull request**.
5. Un manutentore la esamina. Tutto qui. 🎉

Preferisci lavorare in locale? Clona il repo, poi:

```bash
npm install
npm run start      # live preview at http://localhost:3000
npm run build      # must pass: it fails on broken links
```

## Le 4 regole che ogni pagina segue

Rispettarle rende veloce la revisione e mantiene alta la qualità:

1. **Indica il livello.** Inizia il corpo con `<LevelBadge level="beginner" />` (oppure `intermediate` / `advanced` / `all`).
2. **Collega la fonte per i fatti volatili.** Tutto ciò che riguarda nomi dei modelli, prezzi, limiti o funzionalità che cambiano deve citare la pagina ufficiale — e usare `<VerifyNote lastVerified="YYYY-MM-DD" source="..." />`. Non scrivere mai un fatto sul modello a codice fisso; collega la [tabella dei modelli](/docs/whats-new/models-and-pricing).
3. **Sii deciso, poi onesto.** Indica per primo il modo raccomandato; segnala le alternative. Non duplicare la documentazione ufficiale — collegala.
4. **Inglese semplice, esempi più che teoria.** Frasi brevi. Mostra, non limitarti a raccontare.

## Template del front-matter della pagina

```markdown
---
sidebar_position: 2
title: Your Page Title
description: One sentence shown in search and previews.
keywords: [relevant, search, terms]
---

<LevelBadge level="beginner" />

Your first paragraph...
```

## Cosa rende ottima una prima contribuzione

- Aggiungi un termine mancante al **glossario**.
- Aggiungi un **template** o un prompt che usi già.
- Correggi qualcosa **appena rilasciato** che una pagina ora riporta in modo errato (e aggiorna la sua data `lastVerified`).
- Traduci una pagina — vedi il [Manuale di traduzione](/docs/contribute).

## Licenza e firma

Contribuendo accetti che la tua prosa venga condivisa con licenza **CC BY 4.0** e il codice con licenza **MIT**, e aggiungi una riga `Signed-off-by` (DCO) ai tuoi commit (`git commit -s`). I dettagli sono nella [pagina sulla governance](/docs/contribute).

Sii gentile — seguiamo il [Codice di condotta](/docs/contribute) Contributor Covenant.
