Il **Model Context Protocol (MCP)** è lo standard aperto per collegare l'AI a strumenti e dati esterni. Sull'API, MCP ti consente di dare a Claude accesso a un intero **insieme di strumenti di un server** per riferimento — invece di definire ogni strumento a mano.

## MCP vs strumenti definiti a mano

| | [Uso degli strumenti](/docs/api/tool-use) (personalizzati) | MCP |
|---|---|---|
| Cosa definisci | Lo schema di ogni strumento, e lo esegui tu stesso | Una connessione a un server che *pubblica* strumenti |
| Ideale per | Poche funzioni su misura nella tua app | Riutilizzare integrazioni esistenti (GitHub, database, browser, SaaS) |
| Auth | Il tuo codice | OAuth o chiavi native verso il server |

Coesistono: definisci direttamente gli strumenti specifici della tua app e attingi a capacità già pronte tramite MCP.

## Dichiarare un server

Colleghi uno o più server MCP a una richiesta/un agent (URL o comando, più auth). Claude può quindi chiamare gli strumenti del server all'interno del [loop dell'agent](/docs/api/building-agents), come qualsiasi altro strumento. Il campo esatto della richiesta e il flusso di auth sono nella [documentazione ufficiale](https://platform.claude.com/docs/en/docs/build-with-claude/mcp).

## Stesso standard, tre superfici

- **API** (questa pagina) — server MCP nelle tue richieste/nei tuoi agent.
- **[Claude Code](/docs/claude-code/mcp)** — server MCP nelle tue sessioni di sviluppo.
- **[Le app](/docs/claude-app/connectors)** — MCP alimenta i Connettori.

Imparalo una volta; si trasferisce.

## Fiducia

:::warning Un server MCP è codice + accesso
Collega solo server di cui ti fidi, limitali al privilegio minimo e ricorda che i contenuti esterni che restituiscono possono trasportare [prompt injection](/docs/security/prompt-injection). Esamina prima i server di terze parti — [Esaminare codice di terze parti](/docs/security/reviewing-third-party-code).
:::
