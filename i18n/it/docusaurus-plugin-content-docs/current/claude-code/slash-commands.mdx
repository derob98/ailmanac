Un comando slash personalizzato impacchetta un prompt che continui a riscrivere in un singolo `/word`. Definiscilo una volta; riutilizzalo per sempre (e condividilo con il tuo team).

## Come funzionano

Un comando è semplicemente un file Markdown. Inseriscilo nella tua directory dei comandi e il suo nome di file diventa il comando:

- `.claude/commands/changelog.md` → `/changelog` (comando di progetto, condivisibile)
- `~/.claude/commands/changelog.md` → `/changelog` (personale, tutti i progetti)

Il corpo del file è il prompt che Claude esegue quando lo invochi.

## Un esempio minimo

```markdown
---
description: Draft a Conventional Commit message from the staged diff
---

Look at the staged changes with `git diff --cached` and write a single
Conventional Commit message (feat/fix/docs/refactor/chore). Output only the
message, no preamble.
```

Ora `/changelog`… ehm, `/commit` (chiamalo come si chiama il file) fa questo ogni volta.

## I blocchi costruttivi utili

- **`$ARGUMENTS`** — inietta ciò che l'utente ha digitato dopo il comando. `/review src/auth.ts` → il percorso arriva nel tuo prompt.
- **Esegui bash e incorpora l'output** — i comandi possono includere l'output della shell (ad esempio il diff corrente) così Claude agisce sullo stato in tempo reale.
- **Riferisci file** — punta ai file per includerne i contenuti.
- **Frontmatter** — `description` (mostrata nel menu) e altre opzioni.

Vedi la [documentazione ufficiale](https://code.claude.com/docs/en/slash-commands) per la sintassi esatta di argomenti, bash e riferimenti ai file.

## Comando contro skill contro subagent

- **Comando slash** — sei *tu* ad attivare un workflow noto su richiesta.
- **[Skill](/docs/claude-code/skills)** — Claude carica competenza *automaticamente* quando è pertinente.
- **[Subagent](/docs/claude-code/subagents)** — delega una porzione di lavoro a un agente isolato.
