Non sei obbligato a vivere nel terminale. Claude Code si integra con il tuo editor così rivedi le modifiche **come diff, proprio dove scrivi il codice**.

## Cosa ottieni

- **Diff inline.** Le modifiche proposte appaiono come un diff revisionabile nell'editor, non solo come testo in un terminale.
- **Contesto dell'editor.** I tuoi file aperti, la selezione e il progetto sono disponibili a Claude.
- **Lo stesso cervello.** È lo stesso Claude Code — il tuo [CLAUDE.md](/docs/claude-code/claude-md), i [permessi](/docs/claude-code/permissions), gli [hook](/docs/claude-code/hooks) e le [impostazioni](/docs/claude-code/settings) si applicano in modo identico tra la CLI e l'IDE. Configura una volta, usa ovunque.

## Editor supportati

- **VS Code** (e gli editor basati su VS Code) tramite l'estensione.
- **IDE JetBrains** (IntelliJ, PyCharm, WebStorm, ecc.) tramite il plugin.

Installa dal marketplace dell'editor e autenticati; vedi la [documentazione ufficiale](https://code.claude.com/docs/en/vs-code) per i passaggi attuali.

## CLI o IDE — quale?

Sono lo stesso strumento; scegli in base a preferenza e attività:

| Preferisci l'IDE quando… | Preferisci la CLI quando… |
|---|---|
| Vuoi rivedere i diff visivamente | Stai scriptando o sei in modalità [headless](/docs/claude-code/headless-and-agent-sdk) |
| Stai già scrivendo codice nell'editor | Sei su un server / SSH |
| Ti piace avere il contesto dell'editor integrato | Vuoi il massimo controllo e le pipe |

:::tip Le impostazioni sono condivise
Poiché la CLI e l'IDE leggono la stessa configurazione, imposta il tuo [CLAUDE.md](/docs/claude-code/claude-md) e i [permessi](/docs/claude-code/permissions) una volta sola e ti seguiranno tra l'uno e l'altro.
:::
