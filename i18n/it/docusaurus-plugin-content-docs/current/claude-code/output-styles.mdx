Gli **stili di output** cambiano *come* Claude Code comunica — il suo tono, la prolissità e se spiega il proprio ragionamento — senza cambiare *cosa* può fare. Pensalo come un preset di personalità/formato per le risposte.

## Perché usarli

- **Adattarsi al tuo workflow.** Conciso e orientato all'azione quando vai veloce; esplicativo quando stai imparando.
- **Modalità didattica.** Alcuni stili aggiungono commenti educativi — ottimi mentre stai prendendo confidenza con un codebase o una tecnica.
- **Coerenza.** Imposta uno stile preferito così ogni sessione si legge come piace a te.

## Stili comuni

Gli stili integrati tipici vanno da un valore predefinito conciso a modalità più **esplicative** o **orientate all'apprendimento** che illustrano il "perché". Imposta lo stile attivo tramite il comando/impostazione pertinente (vedi la [documentazione ufficiale](https://code.claude.com/docs/en/output-styles) per l'elenco attuale e il meccanismo esatto).

## Stili di output contro CLAUDE.md contro stili di risposta

- **Stile di output** = preset di presentazione per le risposte di Claude Code (tono/prolissità).
- **[CLAUDE.md](/docs/claude-code/claude-md)** = fatti e regole *di progetto* persistenti (cosa fare, convenzioni, protezioni).
- **[Istruzioni personalizzate / stili di risposta](/docs/claude-app/custom-instructions)** = il cugino dell'app consumer Claude.ai.

:::tip Usa la leva giusta
Vuoi che Claude "esegua sempre i test prima di concludere"? Quella è una **protezione di CLAUDE.md**, non uno stile. Vuoi che "spieghi ogni passaggio man mano che procede"? Quello è uno **stile di output**. Mantieni le regole di comportamento in CLAUDE.md e la presentazione negli stili.
:::
