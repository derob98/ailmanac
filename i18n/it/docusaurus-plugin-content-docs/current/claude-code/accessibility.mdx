Claude Code gira nel terminale — una UI testuale, che è un buon punto di partenza per l'accessibilità, ma il rendering di default si affida a colori, glifi di disegno a riquadri e spinner che screen reader e alcune configurazioni motorie gestiscono male. A luglio 2026 esiste una **modalità screen reader** dedicata, oltre a impostazioni per l'ergonomia vim e il comportamento di mouse/scroll. Questa pagina raccoglie le leve che rendono la CLI utilizzabile con le tecnologie assistive.

## Modalità screen reader

La modalità screen reader sostituisce la TUI decorativa con un rendering in testo semplice che le tecnologie assistive possono leggere in modo pulito. Attivala in uno di tre modi — un flag una tantum, una variabile d'ambiente per ogni sessione, o un'impostazione persistente.

<Steps items={[
  { title: "Provala per una sessione", body: "Avvia con il flag: claude --ax-screen-reader. Non cambia nient'altro — stessa sessione, output più semplice." },
  { title: "Attivala per ogni sessione", body: "Esporta la variabile d'ambiente CLAUDE_AX_SCREEN_READER=1 nel profilo della tua shell in modo che si applichi ovunque." },
  { title: "Rendila permanente nelle impostazioni", body: "Imposta \"axScreenReader\": true in un file settings.json (a livello utente ~/.claude/settings.json si applica a tutti i progetti)." }
]} />

<PromptCard title="Modalità screen reader persistente (~/.claude/settings.json)">{`{
  "axScreenReader": true
}`}</PromptCard>

Quest'ultimo punto conta: ciclare tra le modalità dei permessi con **Shift+Tab** è un'interazione centrale di Claude Code (vedi [Permessi](/docs/claude-code/permissions)), e la modalità screen reader pronuncia a voce la nuova modalità così sai sempre se sei in modalità plan, accept-edits o auto senza leggere la barra di stato.

## Ergonomia vim: rimappare i tasti della modalità insert

Se usi la modalità vim e allungare la mano verso `Escape` la affatica, mappa al suo posto una sequenza di due tasti della modalità insert. L'impostazione `vimInsertModeRemaps` trasforma sequenze come `jj` in `Escape`, così non lasci mai la home row.

<PromptCard title="Mappa jj su Escape in modalità vim (settings.json)">{`{
  "editorMode": "vim",
  "vimInsertModeRemaps": {
    "jj": "Escape"
  }
}`}</PromptCard>

<Callout type="warning" items={[
  "Il remap scatta solo in modalità insert, quindi digitare jj come testo letterale richiede una breve pausa tra i tasti — scegli una sequenza che raramente digiti in fretta (jj e kj sono scelte comuni).",
  "editorMode: vim abilita i keybinding di vim; vimInsertModeRemaps ha effetto solo mentre la modalità vim è attiva."
]} />

## Controlli mouse e scroll

I menu a schermo intero ora accettano **i click del mouse** — inclusi i menu a selezione multipla e le righe di input \"Other\" — il che aiuta se la navigazione solo da tastiera è faticosa. Due impostazioni regolano il comportamento del puntatore per sensibilità al movimento o precisione:

| Impostazione | Cosa fa |
|---|---|
| `CLAUDE_CODE_DISABLE_MOUSE_CLICKS` | Disabilita click, trascinamento e hover del mouse mantenendo lo scroll con la rotellina — utile se i click accidentali ti disturbano. |
| `wheelScrollAccelerationEnabled` | Impostala a `false` per disattivare l'accelerazione dello scroll con la rotellina a schermo intero, per uno scorrimento più stabile e prevedibile. |

<PromptCard title="Scroll più stabile, nessun click accidentale (settings.json)">{`{
  "wheelScrollAccelerationEnabled": false,
  "env": {
    "CLAUDE_CODE_DISABLE_MOUSE_CLICKS": "1"
  }
}`}</PromptCard>

## Contrasto e temi

Per gli utenti ipovedenti, il contrasto del terminale è la prima leva. Claude Code include temi chiari e scuri e supporta **temi personalizzati** che costruisci da `/theme` o distribuisci in un plugin — così puoi calibrare una palette ad alto contrasto adatta alle tue esigenze. Vedi [Impostazioni](/docs/claude-code/settings) per dove viene memorizzata la scelta del tema, e [Output Styles](/docs/claude-code/output-styles) se vuoi anche una *prosa* più semplice e meno decorata (gli stili Explanatory e Learning sono più verbosi, non meno — per un output conciso resta su Default).

:::tip Oltre la CLI
La modalità screen reader copre il terminale. Per l'interazione a mani libere con Claude più in generale, [Voice Mode](/docs/claude-app/voice-mode) nell'app ti permette di parlare e ascoltare, e il [playbook Accessibilità](/docs/playbooks/accessibility) tratta l'uso di Claude stesso come strumento assistivo — semplificare testi, costruire checklist e adattare le informazioni al modo in cui leggi.
:::
