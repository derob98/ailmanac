Le policy di gestione dei dati e di addestramento variano per provider, prodotto e piano (consumer vs API vs enterprise) e cambiano nel tempo. Conferma sempre le condizioni attuali con il tuo provider.
</VerifyNote>

Una domanda semplice ma dalle conseguenze reali: **cosa succede al testo che invii a un'AI?** La risposta dipende dal provider, dal prodotto e dal tuo piano — quindi l'abitudine sicura è *sapere prima di incollare*.

## Le domande a cui rispondere per il tuo strumento

- **Il mio input viene usato per addestrare i modelli?** Varia molto — le fasce chat consumer, API ed enterprise spesso differiscono. Molte offerte **API/enterprise** **non** si addestrano sui tuoi dati per impostazione predefinita; alcune impostazioni consumer potrebbero farlo, a seconda delle tue scelte. Verifica.
- **Per quanto tempo vengono conservati i dati**, e chi può accedervi?
- **Dove vengono elaborati** (regione/residenza), e sono cifrati?
- **Esiste un'opzione a retention zero o business/enterprise** per il lavoro sensibile?

## Regole pratiche

- **Non incollare segreti** — chiavi API, password, token — in nessuno strumento AI. Mai.
- **Riduci al minimo i dati personali.** Invia solo ciò che serve al task; oscura nomi, ID, numeri di conto quando puoi.
- **I dati altrui sono dati altrui.** Fai attenzione con PII di clienti, dati sanitari o finanziari — potrebbero esserci obblighi legali (GDPR, HIPAA, ecc.).
- **Usa la fascia giusta.** Per il lavoro regolamentato o riservato, usa un'opzione business/enterprise/retention zero, non un account personale.
- **Considera modelli locali/self-hosted** quando i dati davvero non possono lasciare il tuo ambiente — sacrifichi un po' di capacità per il pieno controllo.

## On-device vs cloud

| | Cloud (hosted) | Locale / self-hosted |
|---|---|---|
| Capacità | Massima | Inferiore (ma in miglioramento) |
| I dati lasciano la tua macchina | Sì (secondo le condizioni del provider) | No |
| Setup/costo | Facile, paghi a consumo | Più sforzo, il tuo hardware |
