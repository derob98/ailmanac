每次会话都有一个有限的[上下文窗口](/docs/foundations/tokens-and-context)。随着它被填满，较旧的内容会被挤压，Claude 可能开始"漂移"。有意识地管理上下文，正是让长会话保持敏锐的关键。

## 两个必备命令

- **`/clear`**——清空对话并从头开始。当你切换到一个**不相关的任务**时使用。这是重获干净、聚焦上下文的最省钱方式。
- **`/compact`**——把目前为止的对话总结为一个紧凑形式，在腾出空间的同时保留要点。当你想**继续同一个任务**但窗口快满了时使用。

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]
  A -->|Same task, context full| C[/compact]
```

## 自动压缩

当上下文接近上限时，Claude Code 可以**自动压缩**，让工作得以继续而不硬性中断。它是一张安全网——但在任务之间主动 `/clear`，比依赖它能得到更好的结果。

## 保持高效的技巧

- **一次会话一个任务。** 别让一下午不相关的工作全堆进一个上下文。
- **把繁重的阅读委派**给[子智能体](/docs/claude-code/subagents)——它们消耗的是*它们自己*的上下文，只返回结论。
- **引用，别粘贴。** 用 `@path` 把 Claude 指向文件，而不是粘贴大块内容；保持 [CLAUDE.md](/docs/claude-code/claude-md) 精简。
- **盯着预算。** 一个显示上下文用量的[状态栏](/docs/claude-code/statusline)能告诉你何时该压缩。

:::tip 症状 → 处方
如果 Claude 开始忘记早先的某条指令或自我重复，那窗口很可能满了。`/compact`（同一任务）或 `/clear`（新任务）通常能立刻解决。更多内容见[故障排查](/docs/contribute/troubleshooting)。
:::
