你不必一直待在终端里。Claude Code 与你的编辑器集成，让你**在写代码的地方，把改动当作 diff 来审阅**。

## 你会得到什么

- **内联 diff。** 提议的编辑会以可审阅的 diff 形式出现在编辑器里，而不只是终端里的文本。
- **编辑器上下文。** 你打开的文件、选区和项目都可供 Claude 使用。
- **同一个大脑。** 这就是同一个 Claude Code——你的 [CLAUDE.md](/docs/claude-code/claude-md)、[权限](/docs/claude-code/permissions)、[钩子](/docs/claude-code/hooks)和[设置](/docs/claude-code/settings)在 CLI 和 IDE 中完全一致地生效。配置一次，处处可用。

## 受支持的编辑器

- **VS Code**（及基于 VS Code 的编辑器），通过扩展。
- **JetBrains IDE**（IntelliJ、PyCharm、WebStorm 等），通过插件。

从编辑器市场安装并完成认证；当前步骤见[官方文档](https://docs.anthropic.com/en/docs/claude-code/ide-integrations)。

## CLI 还是 IDE——选哪个？

它们是同一个工具；按偏好和任务来选：

| 当出现以下情况时选 IDE…… | 当出现以下情况时选 CLI…… |
|---|---|
| 你想直观地审阅 diff | 你在写脚本或处于[无头](/docs/claude-code/headless-and-agent-sdk)模式 |
| 你已经在编辑器里写代码 | 你在服务器上 / 通过 SSH |
| 你喜欢内建的编辑器上下文 | 你想要最大程度的控制和管道操作 |

:::tip 设置是共享的
因为 CLI 和 IDE 读取同一份配置，所以把你的 [CLAUDE.md](/docs/claude-code/claude-md) 和[权限](/docs/claude-code/permissions)设置好一次，它们就会在两者之间跟随你。
:::
