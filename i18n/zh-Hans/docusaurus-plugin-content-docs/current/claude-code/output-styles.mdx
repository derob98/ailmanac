**输出风格**改变 Claude Code *如何*沟通——它的语气、详略程度，以及是否解释其推理——而不改变它*能做什么*。可以把它看作回应的人格/格式预设。

## 为什么要用它们

- **匹配你的工作流。** 你赶进度时要简洁、行动优先；你在学习时要带解释。
- **教学模式。** 有些风格会添加教育性注解——在你熟悉一个代码库或一种技巧时很棒。
- **一致性。** 设定一个首选风格，让每次会话都按你喜欢的方式呈现。

## 常见风格

典型的内建风格从一个简洁的默认值，到更**讲解式**或**面向学习**、会带你走一遍"为什么"的模式不等。通过相关的命令/设置来设定当前风格（当前列表和确切机制见[官方文档](https://code.claude.com/docs/en/output-styles)）。

## 输出风格 vs CLAUDE.md vs 回应风格

- **输出风格** = Claude Code 回复的呈现预设（语气/详略）。
- **[CLAUDE.md](/docs/claude-code/claude-md)** = 持久化的*项目*事实与规则（做什么、约定、护栏）。
- **[自定义指令 / 回应风格](/docs/claude-app/custom-instructions)** = Claude.ai 消费级应用里的对应物。

:::tip 用对杠杆
想让 Claude "完成前总是先跑测试"？那是一条 **CLAUDE.md 护栏**，不是风格。想让它"边做边解释每一步"？那是一种**输出风格**。把行为规则放在 CLAUDE.md，把呈现放在风格里。
:::
