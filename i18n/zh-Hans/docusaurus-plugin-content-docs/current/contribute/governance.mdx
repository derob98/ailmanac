## 独立性与商标

AILmanac 是一个**独立的社区资源**。它**与 Anthropic 没有关联，也未获其背书。** “Claude” 和 “Anthropic” 是 Anthropic 的商标，此处为指称性使用。[官方文档](https://docs.anthropic.com)是事实的唯一来源。

## 许可（双重）

为了让文字与代码都可被复用：

- **文字／文档 → CC BY 4.0**（署名后可复用）。
- **代码、示例、配置 → MIT。**

参见仓库中的 `LICENSE-CONTENT`、`LICENSE-CODE` 和 `NOTICE`。通过贡献，你即同意将你的文字以 CC BY 4.0 许可、代码以 MIT 许可。

## 行为准则

我们遵循 **Contributor Covenant**（参见 `CODE_OF_CONDUCT.md`）。请保持友善、有建设性，并假定他人出于善意。不可接受的行为可私下向维护者举报。

## 贡献模式

- **DCO 签署**，而非 CLA——`git commit -s` 证明你有权提交这份工作。门槛低，来源清晰。
- 所有改动都走**拉取请求**；[构建门禁](/docs/contribute/automation)与维护者审阅共同保证质量。
- **CODEOWNERS** 会随着项目壮大，把各个领域分派给相应的审阅者。

## 标签与分诊

标签即代码（`.github/labels.yml`）让待办列表清晰可读：

- **类型：** `content`、`code`、`i18n`、`automation`
- **领域：** `core-claude`、`ring-general`、`infra`
- **工作流：** `good first issue`、`help wanted`、`upstream-change`、`needs-verification`、`link-rot`、`stale`

新贡献者：请寻找 **`good first issue`** ——添加术语条目、补全规划中的页面，或修正过时事实，都是完美的起点。

## 决策与讨论

- 提问与提案用 **GitHub Discussions**；可执行的工作用 **Issues**。
- 贡献者会在仓库中得到致谢。
