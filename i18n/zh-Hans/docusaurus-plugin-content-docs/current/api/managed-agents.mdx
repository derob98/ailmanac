如果 [自己构建智能体循环](/docs/api/building-agents) 所需的基础设施超出了你愿意维护的范围，那么 **托管的**（由 Anthropic 托管的）智能体选项会替你运行循环——让你专注于智能体的 *任务*，而非会话管线、重试和状态。

## "托管"为你带来什么

与其自己手写并托管循环，你得到的是诸如以下的托管构件：

- **会话** — 可恢复的持久智能体运行。
- **环境** — 一个供智能体连同其工具进行操作的场所。
- **记忆** — 跨轮次/跨运行延续的状态，无需你接线数据库。
- **调度** — 按计划无人值守运行的智能体。

（确切的名称和功能范围在演进——请将 [官方文档](https://docs.anthropic.com/en/docs/agents-and-tools) 视为权威。）

## 何时选择托管 vs 自定义

| 在以下情况选择 **托管**…… | 在以下情况选择 **自定义循环 / SDK**…… |
|---|---|
| 你希望托管、状态和调度都被处理好 | 你需要对循环和工具的完全控制 |
| 你在快速做原型 | 你有严格的自定义基础设施/合规需求 |
| 运维的简洁性比控制更重要 | 你要深度嵌入到自己的技术栈中 |

这是一道光谱——单次调用 → 工作流 → 自定义智能体（SDK）→ 托管。从任务允许的最简单方案开始；只在需要时才升级。

## 同样的护栏依然适用

无论是否托管，自主智能体仍会采取行动。请坚持 **最小权限**、**有上限的成本/迭代次数**，以及 **高风险步骤需人工批准**——参见 [保护智能体安全](/docs/security/securing-agents) 和 [加固自主运行](/docs/security/hardening-autonomous-runs)。
