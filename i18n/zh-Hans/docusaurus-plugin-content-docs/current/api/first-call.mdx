## 1. 获取 API 密钥

在 [Anthropic Console](https://console.anthropic.com) 中创建一个。然后将它设置为环境变量，这样它就永远不会出现在你的代码里：

<PromptCard title="导出你的 API 密钥">{`export ANTHROPIC_API_KEY="sk-ant-..."`}</PromptCard>

<Callout type="warning" items={["切勿提交你的密钥。把密钥保存在环境变量或密钥管理器中，永远不要放进源代码管理。参见安全页面 (/docs/security)。"]} />

## 2. 安装 SDK

选择你的语言。SDK 会自动从你刚刚设置的环境变量中读取密钥。

<Tabs groupId="lang">
<TabItem value="python" label="Python">

<PromptCard title="安装（Python）">{`pip install anthropic`}</PromptCard>

</TabItem>
<TabItem value="ts" label="TypeScript">

<PromptCard title="安装（TypeScript）">{`npm install @anthropic-ai/sdk`}</PromptCard>

</TabItem>
<TabItem value="curl" label="cURL">

无需安装任何东西——你只需要 `curl`。

</TabItem>
</Tabs>

## 3. 发起调用

每个请求都是一个 `messages` 列表。模型以 `content` 回复。运行你所用语言的代码片段，你就会看到 Claude 回答这个问题。

<Tabs groupId="lang">
<TabItem value="python" label="Python">

```python
import anthropic

client = anthropic.Anthropic()  # reads ANTHROPIC_API_KEY from the environment

message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "In one sentence, what is an API?"}
    ],
)

print(message.content[0].text)
```

</TabItem>
<TabItem value="ts" label="TypeScript">

```ts
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic(); // reads ANTHROPIC_API_KEY from the environment

const message = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [
    { role: "user", content: "In one sentence, what is an API?" },
  ],
});

console.log(message.content[0].text);
```

</TabItem>
<TabItem value="curl" label="cURL">

```bash
curl https://api.anthropic.com/v1/messages \
  --header "x-api-key: $ANTHROPIC_API_KEY" \
  --header "anthropic-version: 2023-06-01" \
  --header "content-type: application/json" \
  --data '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "In one sentence, what is an API?"}
    ]
  }'
```

</TabItem>
</Tabs>

## 刚才发生了什么

你发送了四条信息。下面是每一条所控制的内容：

- **`model`** — 使用哪个 Claude。不要盲目硬编码；参见[选择模型](/docs/api/choosing-a-model)。
- **`max_tokens`** — 回复长度的上限（以 [token](/docs/foundations) 计）。它 *不* 设置上下文窗口。
- **`messages`** — 目前为止的对话。该 API 是 **无状态的**：要继续一次对话，每次都要把全部历史发回去。
- **`system`**（可选）— 一条顶层指令，为本次调用设定 Claude 的角色。
