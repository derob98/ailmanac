默认情况下，每个对话都是一块[白板](/docs/foundations/tokens-and-context)。**记忆**功能让 Claude 能跨对话保留关于你的有用事实——这样你就不必每次都重新介绍自己。

## 这里的"记忆"指什么

人们口中的"记忆"其实有几种不同的东西——不要混淆：

| 类型 | 它是什么 |
|---|---|
| **上下文窗口** | 单次对话*内部*的工作记忆（对话结束即遗忘）——参见 [Token 与上下文](/docs/foundations/tokens-and-context) |
| **跨对话记忆** | Claude *跨*对话记住关于你的事实（本页主题） |
| **[自定义指令](/docs/claude-app/custom-instructions)** | *你*明确设定的固定事实 |
| **[项目](/docs/claude-app/projects)** | 限定在某个工作区内的知识 |

## 跨对话记忆如何帮助你

启用后，Claude 能回想起诸如你的偏好、进行中的项目，或你分享过的细节等内容——让它更像一个连续的助手，而不是每次都形同陌路。其可用性和控制选项取决于你的方案。

## 保持掌控

- **查看并编辑**被记住的内容（在应用提供控制选项的地方）。
- 如果你想要一块白板或更多隐私，可以**清除或禁用**它。
- **不要依赖它来保证重要事实的准确性**——核实关键细节，而不是信任它的回忆。
- **留意隐私**——记忆会持久保存信息；不要存储你不希望被保留的敏感数据。参见[隐私](/docs/foundations/privacy)。

:::tip 显式胜过隐式
对于*必须*被应用的事实（你的角色、单位、硬性规则），请设置[自定义指令](/docs/claude-app/custom-instructions)或使用[项目](/docs/claude-app/projects)——这比指望记忆回想起它们更可靠。
:::
