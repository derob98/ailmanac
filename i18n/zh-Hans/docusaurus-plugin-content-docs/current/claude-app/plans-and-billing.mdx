你需要哪个 Claude 方案？下面教你如何思考这个问题——而不去引用那些下个月就会过时的价格（这些请查看[官方定价](https://www.anthropic.com/pricing)）。

## 按用途划分的各个层级

| 层级 | 大致适合 |
|---|---|
| **Free** | 试用 Claude、轻度个人使用 |
| **Pro** | 个人常规使用——更高用量、更多能力 |
| **Max** | 经常触及 Pro 限额的重度个人用户 |
| **Team** | 小型团队——共享工作区与管理功能 |
| **Enterprise** | 需要管理、安全、规模和管控的组织 |

（确切的名称/结构可能变化——请在定价页面确认。）

## "限额"是什么感受

消费级方案会限制**你在一个时间窗口内能用多少**（消息/用量会随时间重置），更高层级会提高这些上限。当你触及限额时，通常会被要求等待其重置或进行升级——你的工作不会丢失，只是暂停一下。

更重的模型（以及更长的对话）会更快消耗你的额度。如果你频繁触及限额，一个快速的解决办法是为新话题**开启全新的对话**，并让提示词保持聚焦（与[上下文管理](/docs/foundations/tokens-and-context)是同一个思路）。

## 如何选择

1. **从 Free 开始。**搞清楚你实际用它来做什么。
2. 当限额打断真实工作时，**升级到 Pro**。
3. 如果你是仍然触及上限的重度用户，就**升级到 Max**。
4. 当同事需要共享访问、管理或合规时，选择 **Team/Enterprise**。

## 应用计费 vs API 计费——两回事

- **应用方案** = 带使用限额的固定订阅（本页）。
- **API 计费** = 按你所构建内容的 [token](/docs/api/tokens-and-pricing) 付费。不同的模式，不同的页面。
