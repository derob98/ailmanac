[提示](/docs/prompting/basics)的大部分内容都是通用的。但有几项技巧对 Claude 尤其有效——而且有一些来自旧模型的习惯如今已经起反作用了。下面是真正 Claude 独有的内容。

## 在 Claude 上多用这些

- **[XML 标签](/docs/prompting/xml-tags)。** Claude 针对带命名标签的结构化提示词（用于标注输入、指令和示例）做了良好调优。这是最有价值的 Claude 习惯。
- **清晰且直接。** 说明目标、约束以及"完成"是什么样子。Claude 会严格遵循明确的指令。
- 通过[系统提示](/docs/foundations/roles)**赋予它一个角色**。
- **在难题上让它思考**——要求它逐步推理，或者在推理模型上使用[扩展思考／努力程度](/docs/api/thinking-and-effort)（这种情况下你往往*不需要*显式要求思维链）。
- 为格式和边缘情况**使用示例**（[少样本](/docs/prompting/few-shot)）。
- **预填充** Claude 回复的开头（在 API 上）以引导格式——例如以 `{` 开始助手回合来推动输出 JSON，或以一个标题来推动结构化输出。

## 抛弃这些旧习惯

:::warning 更新、更字面遵循指令的模型改变了规则
- **少用全大写和威胁式提示词**（"你必须……否则……"）。较新的 Claude 模型会从字面遵循指令，措辞礼貌、具体的指令比大喊大叫更有效。
- **不要过度设计**那种繁复的"越狱式"脚手架——清晰、诚实的指令胜过各种花招。
- **少即是多**——一条紧凑的提示词胜过一条把诉求埋没其中的冗长提示词。
:::

## Claude 与其他模型的对比

这些*原则*是可以迁移的（参见[跨 AI 提示词转换](/docs/prompting/cross-ai-translation)），但最值得记住的 Claude 独有之处是**对 XML 标签的亲和力**和**预填充**。当你切换模型时，要重新检查语气强度和格式提示。
