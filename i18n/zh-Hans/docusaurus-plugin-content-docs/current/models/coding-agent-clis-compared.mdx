你已经选好了模型。但*包裹*模型的那个程序——读取文件、规划、运行命令、检查自身工作并重试的循环——是一个独立的选择，而且它的重要性至少不亚于模型本身。社区把这个包裹层称为 **外壳（harness）**（或 **智能体**），同一个模型在一种外壳里可能表现出色，在另一种里却平平无奇。本页梳理 2026 年的终端编程智能体、真正把它们区分开的因素，以及如何保持配置可移植，从而把切换成本压到最低。

## 模型不等于智能体

前沿模型是一个文本预测器。而**编程智能体**是围绕它的外壳：系统提示词、工具集（读取、编辑、运行 shell、搜索）、规划循环、上下文管理策略、权限与沙箱模型，以及重试逻辑。改变外壳就会改变结果——2026 年整年的从业者报告都描述了同一个模型在真实仓库任务上，因为驱动它的智能体不同而上下浮动几十分。

这带来两个实际后果：

- **一个基准数字是一个*配对*，而不是模型分数。** "Terminal-Bench 上 83%"是一个*模型 + 外壳 + 脚手架*的结果。换掉任何一部分，它都会变。
- **你的循环才是真正的比较单位。** 最好的智能体是在*你的*技术栈、*你的*终端里、以你能接受的价格表现最好的那个——这和选择编程模型的逻辑完全一样。关于这个决策中的模型这一半，见 [编程用 Claude vs GPT vs Gemini](/docs/models/claude-gpt-gemini-for-coding)；本页讲的是外壳这一半。

## 2026 年的格局

终端智能体领域分裂成两大阵营：为某一家实验室的模型调校的**第一方智能体**，以及你可以指向任意模型（或本地权重）的**模型无关**智能体。

| 智能体 | 出品方 | 开放性 | 模型 | 形态 |
|---|---|---|---|---|
| **Claude Code** | Anthropic | 闭源 | Claude（第一方） | 自主式、能动的多文件工作；操作系统强制沙箱；深度的 skills/MCP/子智能体生态 |
| **Codex CLI** | OpenAI | 开放（Apache-2.0，Rust） | GPT（第一方） | 自主式终端智能体；原生 Windows 沙箱；广泛的平台覆盖 |
| **Gemini CLI** | Google | 开源 | Gemini（第一方） | 大上下文智能体，Google 生态联动（见下方时效性说明） |
| **opencode** | 社区（anomalyco） | 开放（MIT，TypeScript） | 任意（模型无关） | 星标最多的开源智能体；CLI + 桌面端；自带模型 |
| **Aider** | 社区 | 开放（Apache-2.0） | 任意，含本地 | Git 原生的结对编程伙伴，而非完整的自主编排器 |
| **Cline / Goose / 其他** | 社区 | 开放 | 任意 | 嵌入编辑器（Cline）或通用开源智能体（Goose）；模型无关 |

<VerifyNote lastVerified="2026-07-08" source="https://github.com/google-gemini/gemini-cli">
**Gemini CLI 正处于变动之中。** 2026 年年中，Google 更改了免费额度的访问方式，并把部分用户引向一个单独的"Antigravity"CLI。到底是 Gemini CLI、Antigravity 还是更新的东西才是正确的 Google 入口，正是那种会迅速过时的事实——在投入之前，请在 Google 自己的文档上确认当前状态。
</VerifyNote>

## 可移植性：上下文只写一次

最大的隐性切换成本是你的**项目上下文**——构建命令、约定、测试规则，这些是智能体无法推断的东西。2026 年这方面有了事实标准：**`AGENTS.md`**，一个位于仓库根目录的纯 Markdown 文件，由 Linux Foundation 下的 Agentic AI Foundation 托管，被 Codex、Gemini CLI、Cursor、Aider、Copilot 的编程智能体、opencode、Goose 以及 20 多个其他工具原生读取（已有 6 万多个仓库采用）。

- **`AGENTS.md`** = 跨工具的项目上下文。写一次；大多数智能体都会读取。
- **`CLAUDE.md`** = Claude Code 自己的项目记忆文件（见 [CLAUDE.md](/docs/claude-code/claude-md)）。你可以保留一个指向 `AGENTS.md` 的小 `CLAUDE.md`，或者两者都维护。
- **`SKILL.md`** = Anthropic 用于打包可复用 [skills](/docs/claude-code/skills) 的单独规范，含 frontmatter 和捆绑脚本。它与 `AGENTS.md` *互补*，不是竞争者——一个描述项目，另一个打包可复用的流程。

把上下文放在 `AGENTS.md` 里意味着：你尝试另一个智能体的那天，你的项目规则会免费随之而来。

## 一句话做决定

- **需要深度、自主的 Claude 工作，配最丰富的工具生态** → **Claude Code**。
- **想要绑定 GPT 的开源第一方智能体，且有强大的平台/Windows 覆盖** → **Codex CLI**。
- **想自由切换模型或运行本地权重** → **opencode** 或 **Cline**。
- **谨慎的、git 原生的、人在环中的编辑** → **Aider**。
- **押注 Google 生态的大上下文玩法** → **Gemini CLI**（先确认当前访问方式）。

无论你选哪个，把项目上下文放在 `AGENTS.md` 里，并在投入之前用你自己仓库里的一个真实任务来验证这个选择。
