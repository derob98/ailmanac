Der schnellste Weg, um mit Claude Code ein Level aufzusteigen, ist, aufzuhören, alles von Hand zu schreiben, und stattdessen **Fähigkeiten zu installieren, die andere bereits perfektioniert haben**. Skills geben Claude bei Bedarf Expertenwissen; Plugins bündeln Skills, Agenten, Befehle, Hooks und MCP-Server zu einer versionierten, installierbaren Einheit. Diese Seite ist eine kuratierte, installationsbereite Auswahlliste — was jedes Element tut, wann man darauf zurückgreift und der genaue Befehl zum Hinzufügen.

Wenn dir die Mechanik unklar ist, lies zuerst [Skills](/docs/claude-code/skills) und [Plugins & Marketplaces](/docs/claude-code/plugins-marketplaces) — diese Seite setzt voraus, dass du weißt, was eine `SKILL.md` ist.

<VerifyNote lastVerified="2026-06-24" source="https://code.claude.com/docs/en/discover-plugins">
Marketplace-Namen, der Katalog offizieller Plugins und Star-Zahlen von Community-Repos ändern sich häufig. Plugins führen Code mit deinen Berechtigungen aus — installiere nur aus Quellen, denen du vertraust, und bestätige aktuelle Befehle in der offiziellen Dokumentation, bevor du dich darauf verlässt.
</VerifyNote>

## Skill vs. Plugin: das 20-Sekunden-Modell

Ein **Skill** ist ein Ordner mit einer `SKILL.md` (Frontmatter-`description` + Anweisungen). Claude lädt ihn nur, wenn er relevant ist, oder du rufst ihn mit `/skill-name` auf. Er liegt in `~/.claude/skills/` (persönlich), im `.claude/skills/` eines Projekts oder innerhalb eines Plugins. Laut der offiziellen Dokumentation wurden benutzerdefinierte Slash-Befehle mit Skills zusammengeführt — sowohl eine `.claude/commands/deploy.md` als auch eine `.claude/skills/deploy/SKILL.md` erzeugen `/deploy`.

Ein **Plugin** ist ein verteilbares Bündel, das Skills, Agenten, Hooks, MCP-Server und LSP-Server zusammen ausliefern kann, mit einem `.claude-plugin/plugin.json`-Manifest. Plugins beziehst du aus einem **Marketplace** — einem Katalog, den du einmal hinzufügst und aus dem du dann installierst. Plugin-Skills sind mit Namespaces versehen, z. B. `/commit-commands:commit`.

## Einen Marketplace hinzufügen, ein Plugin installieren

Der offizielle Anthropic-Marketplace (`claude-plugins-official`) ist automatisch verfügbar. Zwei weitere lohnt es sich, manuell hinzuzufügen: der kuratierte **Community**-Marketplace und der **Demo**-Marketplace mit Beispiel-Plugins.

## Offizielle Plugins, die sich zuerst zu installieren lohnen

Diese werden in Anthropics offiziellem Marketplace (`claude-plugins-official`) ausgeliefert und sind der Startpunkt mit dem höchsten Signal. Installiere mit `/plugin install <name>@claude-plugins-official`.

### Code-Intelligenz (LSP) — Claude Typfehler sofort sehen lassen

LSP-Plugins binden einen Language Server in Claude Code ein, sodass Claude nach jeder Bearbeitung automatische Diagnosen (Typfehler, fehlende Imports) und präzise Navigation (Gehe-zu-Definition, Referenzen finden) erhält statt grep-Vermutungen. Sie erfordern das passende Language-Server-Binary auf deinem Rechner.

- **Wann verwenden:** jede nicht-triviale Coding-Sitzung in einer typisierten Sprache. Das ist wahrscheinlich das größte Einzel-Upgrade an Genauigkeit für Entwicklungsarbeit.
- **Kompromiss:** Server wie `rust-analyzer` und `pyright` können bei großen Repos viel Speicher verbrauchen; deaktivieren, wenn es stört.

<PromptCard title="Install LSP for your stack">
{`/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls`}
</PromptCard>

### Git- & PR-Workflow

- **`commit-commands`** — Git-Workflows: Stagen, eine Commit-Nachricht generieren, pushen, einen PR öffnen. Verwenden, wenn du saubere, konsistente Commits ohne das Tippen der Zeremonie willst. Skills sind mit Namespaces versehen, z. B. `/commit-commands:commit`.
- **`pr-review-toolkit`** — spezialisierte Agenten zum Überprüfen von Pull Requests. Vor dem Mergen verwenden oder um ein zweites Paar Augen auf ein Diff zu bekommen.

<PromptCard title="Git/PR essentials">
{`/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins`}
</PromptCard>

### Automatische Sicherheitsüberprüfung

- **`security-guidance`** — überprüft jede Änderung, die Claude vornimmt, auf gängige Schwachstellen und weist Claude an, sie in derselben Sitzung zu beheben. Verwende es als ständiges Sicherheitsnetz in jedem Projekt, in dem Code geschrieben wird; es ergänzt eine echte Überprüfung (ersetzt sie nicht).

<PromptCard title="Always-on security pass">
{`/plugin install security-guidance@claude-plugins-official`}
</PromptCard>

### Externe Integrationen (MCP, vorkonfiguriert)

Diese bündeln einen einsatzbereiten [MCP-Server](/docs/claude-code/mcp), sodass du die manuelle Einrichtung überspringst. Greife zu einem davon, wenn Claude direkt in diesem Dienst handeln soll.

- **Versionskontrolle:** `github`, `gitlab`
- **Projektmanagement:** `linear`, `notion`, `asana`, `atlassian` (Jira/Confluence)
- **Infrastruktur:** `vercel`, `firebase`, `supabase`
- **Design / Kommunikation / Monitoring:** `figma`, `slack`, `sentry`

<PromptCard title="Connect Claude to your tools">
{`/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official`}
</PromptCard>

### Eigene Toolkits & Output-Stile bauen

- **`plugin-dev`** — ein Toolkit zum Erstellen eigener Plugins. Verwende es in dem Moment, in dem dein `.claude/`-Setup es wert ist, geteilt zu werden.
- **`agent-sdk-dev`** — Tools zum Bauen auf dem Claude Agent SDK.
- **`explanatory-output-style` / `learning-output-style`** — ändern, *wie* Claude antwortet: Erklärungen, die im Verlauf vermitteln, oder ein interaktiver Lernmodus. Großartig für die Einarbeitung oder wenn du die Änderungen wirklich verstehen willst.

## Hochwertige Skills nach zu erledigender Aufgabe

Skills glänzen bei wiederholbaren, kontextgeladenen Aufgaben. Anthropic veröffentlicht eine öffentliche Referenzsammlung im Repo [`anthropics/skills`](https://github.com/anthropics/skills) — dieselben Muster, die Claudes Dokument- und Kreativfähigkeiten antreiben. Du kannst einen Skill-Ordner nach `~/.claude/skills/` (persönlich) oder `.claude/skills/` (Projekt) kopieren, und er wird in der nächsten Sitzung geladen.

| Aufgabe | Skill (aus anthropics/skills) | Wann darauf zurückgreifen |
| :-- | :-- | :-- |
| **Dokumente** | `docx`, `pdf`, `pptx`, `xlsx` | Echte Office-/PDF-Dateien generieren oder bearbeiten — Berichte, Präsentationen, Tabellen, ausfüllbare Formulare. (Quell-verfügbare Referenzimplementierungen.) |
| **UIs bauen** | `web-artifacts-builder` / `artifacts-builder` | Eigenständige HTML/JS-Artefakte und Mini-Apps erzeugen. |
| **Claude erweitern** | `mcp-builder` | Einen neuen MCP-Server gerüsten, wenn noch keine Integration existiert. |
| **Marke & Kommunikation** | brand-guidelines / internal-comms Stil-Skills | Tonfall, Benennung und Formatierung über alle Ausgaben hinweg markenkonform halten. |

<Callout type="tip" items={["Wirf nicht 30 Skills auf einmal in ~/.claude/skills. Skills nutzen progressive Offenlegung — nur Name + Beschreibung werden geladen, bis ausgelöst — aber ein aufgeblähter, vage beschriebener Haufen macht das Auslösen unzuverlässig. Installiere wenige, schreibe scharfe \"Verwenden, wenn …\"-Beschreibungen und wachse gezielt.", "AILmanac liefert fertige Skill-Pakete, die du zum Ausprobieren einfügen kannst — siehe /docs/templates/skills."]} />

### Ein selbstauslösender Skill, den du gleich einfügen kannst

Für rechercheintensive Arbeit schlägt ein Skill, der Claude in frischen Quellen verankert, das wiederholte Erklären deiner Methode jedes Mal. Lege dies in `~/.claude/skills/research-brief/SKILL.md` ab:

<PromptCard title="~/.claude/skills/research-brief/SKILL.md">
{`---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.`}
</PromptCard>

## Wo die Community lebt (nutze die kuratierten Listen, überspringe das Rauschen)

Es gibt Dutzende von "awesome"-Repos; die meisten sind automatisch generierte Spiegel. Einige sind wirklich kuratiert und gepflegt — fang dort an.

- **[hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code)** — der bekannteste handkuratierte Index von Skills, Hooks, Slash-Befehlen, Agent-Orchestratoren, Anwendungen und Plugins. Zehntausende Sterne; aktiv neu organisiert.
- **[ComposioHQ/awesome-claude-skills](https://github.com/ComposioHQ/awesome-claude-skills)** — ein großer kuratierter Katalog von Claude Skills, gruppiert nach Funktion (Dokumente, Entwicklung, Daten, Marketing, Sicherheit, SaaS-Automatisierung).
- **[VoltAgent/awesome-agent-skills](https://github.com/VoltAgent/awesome-agent-skills)** — über 1.400 Community- + offizielle Skills (Anthropic, Vercel, Stripe, Cloudflare, Sentry, Figma und mehr), portabel über Claude Code, Cursor, Gemini CLI, Codex und andere.
- **[anthropics/claude-plugins-community](https://github.com/anthropics/claude-plugins-community)** — der offizielle Community-**Plugin**-Marketplace; Drittanbieter-Plugins, die Anthropics automatisierte Validierung und Sicherheitsprüfung bestanden haben.
