Du musst nicht im Terminal leben. Claude Code integriert sich in deinen Editor, sodass du Änderungen **als Diffs prüfst, direkt dort, wo du programmierst**.

## Was du bekommst

- **Inline-Diffs.** Vorgeschlagene Änderungen erscheinen als prüfbares Diff im Editor, nicht nur als Text im Terminal.
- **Editor-Kontext.** Deine geöffneten Dateien, deine Auswahl und dein Projekt stehen Claude zur Verfügung.
- **Dasselbe Gehirn.** Es ist dasselbe Claude Code — deine [CLAUDE.md](/docs/claude-code/claude-md), [Berechtigungen](/docs/claude-code/permissions), [Hooks](/docs/claude-code/hooks) und [Einstellungen](/docs/claude-code/settings) gelten identisch über CLI und IDE hinweg. Einmal konfigurieren, überall nutzen.

## Unterstützte Editoren

- **VS Code** (und VS-Code-basierte Editoren) über die Erweiterung.
- **JetBrains-IDEs** (IntelliJ, PyCharm, WebStorm usw.) über das Plugin.

Installiere aus dem Marketplace des Editors und authentifiziere dich; aktuelle Schritte siehe [offizielle Dokumentation](https://code.claude.com/docs/en/vs-code).

## CLI oder IDE — was?

Es ist dasselbe Werkzeug; wähle nach Vorliebe und Aufgabe:

| Bevorzuge die IDE, wenn… | Bevorzuge die CLI, wenn… |
|---|---|
| Du Diffs visuell prüfen willst | Du skriptest oder im [Headless](/docs/claude-code/headless-and-agent-sdk)-Modus bist |
| Du bereits im Editor programmierst | Du auf einem Server / per SSH bist |
| Du eingebauten Editor-Kontext magst | Du maximale Kontrolle und Pipes willst |

:::tip Einstellungen werden geteilt
Weil CLI und IDE dieselbe Konfiguration lesen, richtest du deine [CLAUDE.md](/docs/claude-code/claude-md) und [Berechtigungen](/docs/claude-code/permissions) einmal ein, und sie folgen dir zwischen beiden.
:::
