Claude Code läuft im Terminal – einer Text-Oberfläche, was ein guter Ausgangspunkt für Barrierefreiheit ist, aber die Standard-Darstellung setzt auf Farbe, Rahmen-Zeichen (Box-Drawing-Glyphen) und Spinner, mit denen Screenreader und manche motorischen Setups schlecht umgehen. Seit Juli 2026 gibt es einen dedizierten **Screenreader-Modus** sowie Einstellungen für Vim-Ergonomie und Maus-/Scroll-Verhalten. Diese Seite sammelt die Hebel, die das CLI mit assistiver Technik nutzbar machen.

## Screenreader-Modus

Der Screenreader-Modus tauscht die dekorative TUI gegen eine reine Text-Darstellung, die assistive Technik sauber lesen kann. Aktiviere ihn auf eine von drei Arten – ein einmaliges Flag, eine Umgebungsvariable für jede Sitzung oder eine dauerhafte Einstellung.

<Steps items={[
  { title: "Für eine Sitzung ausprobieren", body: "Starte mit dem Flag: claude --ax-screen-reader. Sonst ändert sich nichts – gleiche Sitzung, schlichtere Ausgabe." },
  { title: "Für jede Sitzung aktivieren", body: "Exportiere die Umgebungsvariable CLAUDE_AX_SCREEN_READER=1 in deinem Shell-Profil, damit sie überall gilt." },
  { title: "Dauerhaft in den Einstellungen setzen", body: "Setze \"axScreenReader\": true in einer settings.json-Datei (die Datei auf Nutzerebene ~/.claude/settings.json gilt für alle Projekte)." }
]} />

<PromptCard title="Dauerhafter Screenreader-Modus (~/.claude/settings.json)">{`{
  "axScreenReader": true
}`}</PromptCard>

### Was er verändert

<Callout type="tip" items={[
  "Reine Text-Darstellung ersetzt die gestylte TUI, damit Screenreader sauberen, linearen Text erhalten.",
  "Dekorative Glyphen werden vor Screenreadern verborgen, statt als Rauschen vorgelesen zu werden.",
  "Transkript-Symbole werden als kurze Beschriftungen angesagt statt als rohe Zeichen.",
  "Verschachtelte Tabellen werden als Zeilen Header: Wert. gelesen – flach zu sprechbaren Zeilen aufgelöst.",
  "Änderungen des Berechtigungsmodus werden laut angesagt, wenn du mit Shift+Tab durch die Modi wechselst."
]} />

Der letzte Punkt ist wichtig: Das Wechseln der Berechtigungsmodi mit **Shift+Tab** ist eine zentrale Claude-Code-Interaktion (siehe [Berechtigungen](/docs/claude-code/permissions)), und der Screenreader-Modus spricht den neuen Modus laut aus, damit du immer weißt, ob du im Plan-, Accept-Edits- oder Auto-Modus bist, ohne die Statuszeile lesen zu müssen.

## Vim-Ergonomie: Insert-Mode-Tasten neu belegen

Wenn du den Vim-Modus nutzt und das Greifen nach `Escape` deine Hand belastet, ordne ihm stattdessen eine Zwei-Tasten-Sequenz im Insert-Modus zu. Die Einstellung `vimInsertModeRemaps` verwandelt Sequenzen wie `jj` in `Escape`, sodass du die Grundreihe nie verlässt.

<PromptCard title="jj im Vim-Modus auf Escape abbilden (settings.json)">{`{
  "editorMode": "vim",
  "vimInsertModeRemaps": {
    "jj": "Escape"
  }
}`}</PromptCard>

<Callout type="warning" items={[
  "Das Remap greift nur im Insert-Modus, also braucht das Tippen von jj als wörtlichem Text eine kurze Pause zwischen den Tasten – wähle eine Sequenz, die du selten schnell tippst (jj und kj sind gängige Wahl).",
  "editorMode: vim aktiviert Vim-Tastenkürzel; vimInsertModeRemaps wirkt nur, solange der Vim-Modus an ist."
]} />

## Maus- und Scroll-Steuerung

Vollbild-Menüs akzeptieren jetzt **Mausklicks** – einschließlich Mehrfachauswahl-Menüs und der „Other“-Eingabezeilen – was hilft, wenn reine Tastaturnavigation mühsam ist. Zwei Einstellungen justieren das Zeiger-Verhalten bei Bewegungsempfindlichkeit oder Präzision:

| Einstellung | Was sie bewirkt |
|---|---|
| `CLAUDE_CODE_DISABLE_MOUSE_CLICKS` | Deaktiviert Mausklick, Ziehen und Hover, behält aber das Scrollrad – nützlich, wenn versehentliche Klicks dich stören. |
| `wheelScrollAccelerationEnabled` | Auf `false` setzen, um die Scrollrad-Beschleunigung im Vollbild abzuschalten – für ruhigeres, vorhersehbares Scrollen. |

<PromptCard title="Ruhigeres Scrollen, keine versehentlichen Klicks (settings.json)">{`{
  "wheelScrollAccelerationEnabled": false,
  "env": {
    "CLAUDE_CODE_DISABLE_MOUSE_CLICKS": "1"
  }
}`}</PromptCard>

## Kontrast und Themes

Für Nutzer mit Sehbehinderung ist der Terminal-Kontrast der erste Hebel. Claude Code liefert helle und dunkle Themes und unterstützt **benutzerdefinierte Themes**, die du über `/theme` erstellst oder in einem Plugin ausliefern kannst – so kannst du eine kontrastreiche Palette einstellen, die deinen Bedürfnissen entspricht. Siehe [Einstellungen](/docs/claude-code/settings) dafür, wo die Theme-Auswahl gespeichert wird, und [Output Styles](/docs/claude-code/output-styles), wenn du zusätzlich schlichtere, weniger dekorierte *Prosa* möchtest (die Stile Explanatory und Learning sind ausführlicher, nicht knapper – für knappe Ausgabe bleib bei Default).

:::tip Über das CLI hinaus
Der Screenreader-Modus deckt das Terminal ab. Für freihändige Interaktion mit Claude im weiteren Sinne lässt dich der [Voice Mode](/docs/claude-app/voice-mode) in der App sprechen und zuhören, und das [Accessibility-Playbook](/docs/playbooks/accessibility) behandelt die Nutzung von Claude selbst als assistives Werkzeug – Text vereinfachen, Checklisten bauen und Informationen an deine Art zu lesen anpassen.
:::
