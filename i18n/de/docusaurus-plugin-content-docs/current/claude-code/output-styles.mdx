Verfügbare Output-Styles und wie man sie setzt, können sich ändern — überprüfe das in der offiziellen Dokumentation.
</VerifyNote>

**Output-Styles** ändern, *wie* Claude Code kommuniziert — seinen Tonfall, seine Ausführlichkeit und ob es seine Überlegungen erklärt — ohne zu ändern, *was* es tun kann. Stell es dir als Persönlichkeits-/Format-Voreinstellung für Antworten vor.

## Warum sie nutzen

- **Passe sie an deinen Workflow an.** Knapp und handlungsorientiert, wenn du schnell vorankommst; erklärend, wenn du lernst.
- **Lehrmodus.** Manche Styles fügen lehrreiche Kommentare hinzu — großartig, während du eine Codebasis oder eine Technik kennenlernst.
- **Konsistenz.** Setze einen bevorzugten Style, damit sich jede Session so liest, wie du es magst.

## Gängige Styles

Typische eingebaute Styles reichen von einer prägnanten Voreinstellung bis zu eher **erklärenden** oder **lernorientierten** Modi, die das "Warum" durchgehen. Setze den aktiven Style über den entsprechenden Befehl/die entsprechende Einstellung (die aktuelle Liste und den genauen Mechanismus siehe in der [offiziellen Dokumentation](https://docs.anthropic.com/en/docs/claude-code/output-styles)).

## Output-Styles vs. CLAUDE.md vs. Response-Styles

- **Output-Style** = Präsentations-Voreinstellung für Claude Codes Antworten (Tonfall/Ausführlichkeit).
- **[CLAUDE.md](/docs/claude-code/claude-md)** = dauerhafte *Projekt*-Fakten und -Regeln (was zu tun ist, Konventionen, Leitplanken).
- **[Eigene Anweisungen / Response-Styles](/docs/claude-app/custom-instructions)** = das Pendant in der Claude.ai-Consumer-App.

:::tip Nutze den richtigen Hebel
Willst du, dass Claude "vor dem Abschluss immer Tests ausführt"? Das ist eine **CLAUDE.md-Leitplanke**, kein Style. Willst du, dass es "jeden Schritt unterwegs erklärt"? Das ist ein **Output-Style**. Halte Verhaltensregeln in der CLAUDE.md und die Präsentation in Styles.
:::
