Das **Model Context Protocol (MCP)** ist der offene Standard zur Verbindung von KI mit externen Tools und Daten. Auf der API ermöglicht dir MCP, Claude per Verweis Zugriff auf einen ganzen **Server voller Tools** zu geben — statt jedes Tool von Hand zu definieren.

## MCP vs. handdefinierte Tools

| | [Tool-Nutzung](/docs/api/tool-use) (eigen) | MCP |
|---|---|---|
| Du definierst | Das Schema jedes Tools und führst es selbst aus | Eine Verbindung zu einem Server, der Tools *veröffentlicht* |
| Am besten für | Einige wenige maßgeschneiderte Funktionen in deiner App | Wiederverwendung bestehender Integrationen (GitHub, Datenbanken, Browser, SaaS) |
| Auth | Dein Code | OAuth oder native Schlüssel für den Server |

Sie existieren nebeneinander: Definiere deine app-spezifischen Tools direkt und ziehe fertige Fähigkeiten über MCP hinzu.

## Einen Server deklarieren

Du hängst MCP-Server an eine Anfrage/einen Agenten an (URL oder Befehl, plus Auth). Claude kann dann die Tools des Servers innerhalb der [Agentenschleife](/docs/api/building-agents) aufrufen, genau wie jedes andere Tool. Das exakte Anfragefeld und der Auth-Flow stehen in der [offiziellen Dokumentation](https://docs.anthropic.com/en/docs/build-with-claude/mcp).

## Derselbe Standard, drei Oberflächen

- **API** (diese Seite) — MCP-Server bei deinen Anfragen/Agenten.
- **[Claude Code](/docs/claude-code/mcp)** — MCP-Server in deinen Entwicklungssitzungen.
- **[Die Apps](/docs/claude-app/connectors)** — MCP treibt Connectors an.

Lerne es einmal; es lässt sich übertragen.

## Vertrauen

:::warning Ein MCP-Server ist Code + Zugriff
Verbinde nur Server, denen du vertraust, beschränke sie auf Least Privilege und denke daran, dass externe Inhalte, die sie zurückgeben, [Prompt Injection](/docs/security/prompt-injection) enthalten können. Überprüfe Server von Drittanbietern zuerst — [Code von Drittanbietern überprüfen](/docs/security/reviewing-third-party-code).
:::
