Ein Werkzeugkasten wiederverwendbarer Prompt-Muster. Jede Karte sagt dir, was es ist, wann du danach greifst, und gibt dir eine Copy-paste-Vorlage. Sie sind stapelbar — kombiniere sie frei. Neu hier? Beginne zuerst mit den [Grundlagen des Promptings](/docs/prompting/basics).

<Callout type="objectives" items={["Die 11 Kern-Prompt-Muster erkennen und wann jedes anwendbar ist","Eine Copy-paste-Vorlage für jede Aufgabe greifen — Zero-Shot, Few-Shot, Chain-of-Thought und mehr","Muster kombinieren (Rolle + Beispiel + Ausgabe-Beschränkung + Erlaubnis zum Verzicht) zu einem starken Prompt","Grounding nutzen, um Halluzinationen zu reduzieren, und Meta-Prompting, um schwache Prompts zu reparieren"]} />

## Zero-Shot (einfach fragen, klar)

**Wann:** einfache, häufige Aufgaben. **Warum:** geringster Aufwand, oft genug.

<PromptCard title="Zero-shot summary">{`Summarize the text below in 5 bullets for a busy manager.
TEXT: """{paste}"""`}</PromptCard>

## Few-Shot (Beispiele zeigen)

**Wann:** du brauchst ein bestimmtes Format oder einen Stil. **Warum:** Beispiele lehren schneller als Beschreibungen. Siehe [Few-Shot richtig gemacht](/docs/prompting/few-shot).

<PromptCard title="Few-shot sentiment classifier">{`Classify the sentiment as positive/negative/neutral.
Review: "Loved it, shipped fast" → positive
Review: "Broke on day one" → negative
Review: "{new review}" →`}</PromptCard>

## Chain-of-Thought (Schritt für Schritt denken)

**Wann:** Mathematik, Logik, mehrstufiges Reasoning. **Warum:** Reasoning vor der Antwort verbessert die Genauigkeit.

<PromptCard title="Chain-of-thought">{`Work through this step by step, then give the final answer on its own line.
PROBLEM: {problem}`}</PromptCard>

## Zerlegung (aufbrechen)

**Wann:** große, unscharfe Aufgaben. **Warum:** kleinere Teilaufgaben sind einfacher und zuverlässiger.

<PromptCard title="Decomposition">{`First, list the sub-tasks needed to {goal}. Then do them one at a time,
showing each result before moving on.`}</PromptCard>

## Rollen-Prompting (eine Persona zuweisen)

**Wann:** du willst eine bestimmte Perspektive oder einen Standard. **Warum:** bahnt relevantes Verhalten an.

<PromptCard title="Role prompting">{`You are a meticulous senior copy editor. Edit the text for clarity and concision
without changing meaning. Show a bullet list of the changes you made.`}</PromptCard>

## Ausgabeformat-Beschränkung

**Wann:** das Ergebnis speist ein anderes Tool oder du willst Konsistenz. **Warum:** vorhersehbare Formen sind verwendbar.

<PromptCard title="Output-format constraint">{`Reply with ONLY valid JSON matching: {"title": string, "tags": string[]}.
No prose, no markdown fences.`}</PromptCard>

## Selbstkonsistenz / Selbstkritik

**Wann:** Korrektheit zählt. **Warum:** ein zweiter Durchgang fängt Fehler des ersten Durchgangs ab.

<PromptCard title="Self-critique">{`Answer the question. Then, in a section called "Check", critique your own answer
for errors or missing cases and revise if needed.`}</PromptCard>

## Grounding (nur aus dem bereitgestellten Kontext antworten)

**Wann:** Fakten zählen — Q&A über deine Docs, RAG, alles, wo eine falsche Antwort schlimmer ist als keine Antwort. **Warum:** Der größte einzelne Hebel gegen [Halluzinationen](/docs/foundations/hallucinations) ist, dem Modell das Erfinden zu verbieten und ihm ausdrücklich zu erlauben, zu verzichten.

<PromptCard title="Grounded Q&A">{`Answer the QUESTION using ONLY the CONTEXT below. Quote the exact sentence(s)
you relied on. If the context does not contain the answer, reply exactly:
"Not found in the provided context." Do not use outside knowledge.

CONTEXT: """{paste your sources}"""
QUESTION: {question}`}</PromptCard>

## Meta-Prompting (die KI deinen Prompt verbessern lassen)

**Wann:** dein Prompt liefert zu wenig. **Warum:** das Modell ist gut darin, zu erkennen, was ihm fehlt.

<PromptCard title="Meta-prompting">{`Here's my prompt: "{prompt}". Ask me the 3 questions that would most improve it,
then rewrite it incorporating sensible defaults.`}</PromptCard>

## Iterative Verfeinerung

**Wann:** fast immer. **Warum:** die erste Ausgabe ist ein Entwurf, nicht das Ziel.

<PromptCard title="Iterative refinement">{`{after a result} Shorter and more concrete. Keep the second paragraph. Add one
example. Drop the marketing tone.`}</PromptCard>

:::tip Kombiniere sie
Ein starker realer Prompt ist oft: eine **Rolle** + ein **Beispiel** + eine **Ausgabe-Beschränkung** + die Erlaubnis, **„Ich weiß es nicht.“** zu sagen. Schichte die Karten.
:::
