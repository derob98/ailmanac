Gute Nachricht: **Die Grundlagen des Promptings lassen sich auf jedes Modell übertragen.** Wer Claude gut prompten kann, kann auch ChatGPT, Gemini oder Llama gut prompten. Ein paar oberflächliche Details unterscheiden sich — hier ist der Stein von Rosetta.

## Was sich überall übertragen lässt

Das funktioniert bei allen — es geht um *Kommunikation*, nicht um einen Anbieter:

- **Kontext geben**: Ziel, Zielgruppe, Format, Tonfall ([Grundlagen des Promptings](/docs/prompting/basics)).
- **Beispiele** ([Few-Shot](/docs/prompting/few-shot)).
- **Um Schlussfolgerungen bitten** bei schwierigen Aufgaben.
- **Das Ergebnis beschränken** (Länge, Format, „nur JSON").
- **Iterieren** in einfacher Sprache.
- **Rollen / System-Prompts** — jedes große Modell kennt das Konzept ([Rollen](/docs/foundations/roles)).

Lerne das einmal, und du bist portabel.

## Was sich unterscheidet (und was anzupassen ist)

| Aspekt | Beachte beim Wechsel |
|---|---|
| **Strukturhinweise** | Claude liebt [XML-Tags](/docs/prompting/xml-tags); andere Modelle akzeptieren sie, bevorzugen aber möglicherweise Markdown-Überschriften oder JSON. |
| **Tonfall-Intensität** | Ältere Modelle tolerierten forsche/GROSSBUCHSTABEN-Prompts; neuere, wörtlich folgende Modelle bevorzugen ruhige, konkrete Anweisungen. Justiere pro Modell neu. |
| **Schlussfolgern** | „Denk Schritt für Schritt" hilft manchen; dedizierte Reasoning-Modelle tun das intern — verdopple es nicht. |
| **Prefilling** | Dem Assistenten Worte in den Mund zu legen, ist ein Claude-typischer Hebel; die Unterstützung anderswo variiert. |
| **Gewicht des System-Prompts** | Alle haben einen, aber wie strikt er befolgt wird, variiert — teste es. |
| **Tool-/Function-Calling** | Gleiches Konzept, unterschiedliche Request-Schemata je Anbieter. |

## Ein portabler Workflow

1. Schreibe den Prompt mit **universeller** Struktur (Kontext + Format + Beispiele).
2. **Teste auf deinem Zielmodell** mit ein paar echten Eingaben ([Eval](/docs/foundations/evals)).
3. Passe bei Bedarf nur die **oberflächlichen Details** oben an.
4. Halte Prompts unter Versionskontrolle, damit ein Modellwechsel günstig bleibt.

:::tip Schreib nicht von Grund auf neu
Der Umstieg auf ein neues Modell braucht selten einen neuen Prompt — meist nur eine Anpassung von Tonfall/Format. Deine Struktur und deine Beispiele werden übernommen.
:::
