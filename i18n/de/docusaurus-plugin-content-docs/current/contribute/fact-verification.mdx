„Immer aktuell" ist ein Versprechen, das wir mit einem einfachen, ehrlichen System einhalten — nicht durch Magie. So hältst du es ein, wenn du beiträgst.

## Beständig vs. volatil

- **Beständig** — Konzepte, die sich nicht ändern (was ein LLM ist, was RAG ist). **Kein Datum** nötig.
- **Volatil** — alles, was sich von Release zu Release ändert: Modellnamen/-IDs, Preise, Limits, UI-Bezeichnungen, Beta-/Feature-Verfügbarkeit, exakte CLI-Flags. **Muss ein Datum + eine Quelle tragen.**

## Volatile Fakten kennzeichnen

Verwende die Komponente `<VerifyNote>` direkt unter dem Level-Badge:

```mdx
<VerifyNote lastVerified="2026-06-20" source="https://docs.anthropic.com/...">
One line on what's volatile here and to confirm at the source.
</VerifyNote>
```

- **`lastVerified`** = das Datum, an dem *du* es tatsächlich gegen die Quelle geprüft hast.
- **`source`** = die offizielle Seite, die maßgeblich ist.

## Modellfakten niemals fest verdrahten

Modell-IDs/Preise/Kontext stehen **ausschließlich** in `data/models.json` und werden von `<ModelTable />` auf der [Modellseite](/docs/whats-new/models-and-pricing) gerendert. Im Fließtext **verlinke dorthin**, statt eine Modell-ID oder einen Preis hinzuschreiben. Modelle aktualisieren = eine einzeilige Änderung an diesem JSON (ein perfekter erster PR).

## Auf `needs-verification` reagieren

Die Automatisierung (und Leser) eröffnen Issues mit dem Label `needs-verification`, wenn ein Fakt veraltet sein könnte. So löst du eines:

1. Prüfe die Aussage gegen die offizielle Quelle.
2. Wenn korrekt → setze `lastVerified` auf heute.
3. Wenn geändert → korrigiere den Inhalt **und** das Datum und verlinke die Quelle.
4. Wenn es ein Modellfakt ist → aktualisiere `data/models.json`, nicht den Fließtext.

## Das Ehrlichkeitsprinzip

:::tip Ein altes Datum ist ein Feature
Ein `lastVerified` von vor Monaten bedeutet nicht, dass wir versagt haben — es sagt dem Leser, *diesen* Fakt nochmals zu prüfen. Lieber ein ehrliches altes Datum als ein selbstbewusst falsches. Setze kein Datum hoch, das du nicht tatsächlich erneut verifiziert hast.
:::
