AILmanac ist nur „vollständig und immer aktuell", weil Menschen wie du etwas beitragen. Du musst **kein** Entwickler sein — einen Tippfehler korrigieren, einen Glossarbegriff hinzufügen oder eine Vorlage teilen sind alles echte Beiträge.

## Der schnelle Weg

1. **Finde die Seite** auf der Website und klicke unten auf **„Diese Seite bearbeiten"** (öffnet die Datei auf GitHub).
2. Klicke auf den ✏️-Stift, um im Browser zu bearbeiten. GitHub forkt das Repo automatisch für dich.
3. Nimm deine Änderung in Markdown vor.
4. Schreibe unten eine kurze Beschreibung und **schlage Änderungen vor** → **eröffne einen Pull Request**.
5. Ein Maintainer prüft ihn. Das war's. 🎉

Lieber lokal? Klone das Repo, dann:

```bash
npm install
npm run start      # live preview at http://localhost:3000
npm run build      # must pass: it fails on broken links
```

## Die 4 Regeln, denen jede Seite folgt

Diese einzuhalten macht das Review schnell und hält die Qualität hoch:

1. **Kennzeichne das Level.** Beginne den Textkörper mit `<LevelBadge level="beginner" />` (oder `intermediate` / `advanced` / `all`).
2. **Verlinke die Quelle für volatile Fakten.** Alles über Modellnamen, Preise, Limits oder Features, das sich ändert, muss die offizielle Seite zitieren — und `<VerifyNote lastVerified="YYYY-MM-DD" source="..." />` verwenden. Verdrahte niemals einen Modellfakt fest; verlinke auf die [Modelltabelle](/docs/whats-new/models-and-pricing).
3. **Sei meinungsstark, dann ehrlich.** Gib zuerst den empfohlenen Weg an; nenne Alternativen. Dupliziere nicht die offizielle Dokumentation — verlinke darauf.
4. **Klares Deutsch, Beispiele statt Theorie.** Kurze Sätze. Zeige, statt nur zu erzählen.

## Vorlage für das Seiten-Frontmatter

```markdown
---
sidebar_position: 2
title: Your Page Title
description: One sentence shown in search and previews.
keywords: [relevant, search, terms]
---

<LevelBadge level="beginner" />

Your first paragraph...
```

## Was einen großartigen ersten Beitrag ausmacht

- Füge einen fehlenden **Glossarbegriff** hinzu.
- Füge eine **Vorlage** oder einen Prompt hinzu, den du bereits verwendest.
- Korrigiere etwas **frisch Ausgeliefertes**, das eine Seite nun falsch wiedergibt (und aktualisiere ihr `lastVerified`-Datum).
- Übersetze eine Seite — siehe das [Übersetzungs-Playbook](/docs/contribute).

## Lizenzierung & Sign-off

Mit deinem Beitrag erklärst du dich einverstanden, dass deine Prosa unter **CC BY 4.0** und dein Code unter **MIT** geteilt wird, und du fügst deinen Commits eine `Signed-off-by`-Zeile (DCO) hinzu (`git commit -s`). Details auf der [Governance-Seite](/docs/contribute).

Sei freundlich — wir folgen dem [Verhaltenskodex](/docs/contribute) des Contributor Covenant.
