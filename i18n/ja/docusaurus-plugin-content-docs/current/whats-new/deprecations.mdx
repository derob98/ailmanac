非推奨のスケジュールや移行の詳細は Anthropic に由来し、変更されます — 移行する前に、必ず公式ドキュメントで日付と代替手段を確認してください。
</VerifyNote>

モデルは廃止され、パラメータは名称変更され、挙動は変化します。このページは、アップグレードで驚かされることがないよう **破壊的変更** を追跡します。API 上で構築するなら、定期的にざっと目を通しましょう。

## なぜこれが重要なのか

固定したモデル ID は、いずれ **非推奨** となり、その後 **削除** されます。それをハードコーディングしたコードは壊れます。少しの衛生管理が、深夜 2 時の不意打ちを防ぎます。

## 生き残るためのルール

- **モデル ID は設定ファイルから読み込み**、リテラルをコードベース全体に散らばらせないこと — そうすれば移行は 1 行の変更で済みます。
- 非推奨の日付や推奨される代替手段について **発表をウォッチ** しましょう。
- モデルを切り替えたら **[評価（evals）](/docs/foundations/evals) を再実行** しましょう — 「より新しく優れた」モデルでも挙動は変化することがあります。
- 削除後ではなく **締め切り前にテスト** しましょう。

## よくある変更の種類

| 変更 | 例 | 対処法 |
|---|---|---|
| モデルの廃止 | 古いモデル ID が動作しなくなる | 推奨される後継へ移行し、再評価する |
| パラメータの置き換え | 推論/予算のつまみが `effort` に置き換えられる | 呼び出しを更新する。[思考とエフォート](/docs/api/thinking-and-effort) を参照 |
| デフォルトの変更 | 新しいデフォルトのモデル/挙動 | 意図的に固定し、プロンプトが引き続き機能するか検証する |
| エンドポイント/SDK の更新 | SDK の新しいメジャーバージョン | 移行ガイドを読み、慎重にバージョンを上げる |

## 現在のウォッチ

信頼できるスケジュールは [公式モデル非推奨ページ](https://platform.claude.com/docs/en/about-claude/model-deprecations) にあります — 移行する前に、必ずそこで日付を確認してください。Anthropic は、一般公開されたモデルを廃止する前に、少なくとも 60 日前に通知します。

### 非推奨のモデル

| モデル ID | 非推奨 | 削除 | 移行先 |
|---|---|---|---|
| `claude-opus-4-1-20250805` (Opus 4.1) | 2026年6月5日 | **2026年8月5日** | `claude-opus-4-8` |

削除日を過ぎたモデルへのリクエストは **失敗** します — Opus 4.1 を固定している場合は、締め切り前に移行してください。

### 最近削除されたモデル

`claude-opus-4-20250514` (Opus 4) と `claude-sonnet-4-20250514` (Sonnet 4) は **2026年6月15日に削除** されました。これらの ID への呼び出しは現在エラーになります — それぞれ `claude-opus-4-8` と `claude-sonnet-4-6` に移行してください。

### パラメータの非推奨

**Claude Opus 4.7 以降**（Opus 4.8 を含む）では、サンプリングのつまみ `temperature`、`top_p`、`top_k` は非推奨です。これらのいずれかをデフォルト以外の値に設定すると **400 エラー** が返されます。これらは省略し、代わりに [プロンプティング](/docs/prompting/prompting-claude) と [思考とエフォート](/docs/api/thinking-and-effort) で挙動を調整してください。

[現行モデルと価格](/docs/whats-new/models-and-pricing) の表に対して、あなたのモデル ID を必ず確認してください。古い ID を使っている場合は、移行を計画しましょう。
