データの取り扱いと訓練のポリシーは、プロバイダー、製品、プラン（コンシューマー vs API vs エンタープライズ）によって異なり、時とともに変化します。現在の条件は必ずプロバイダーで確認してください。
</VerifyNote>

シンプルだが現実的な重みのある問い。**AIに送ったテキストはどうなるのか？** 答えはプロバイダー、製品、そしてあなたのプランによって変わります。だから安全な習慣は、*貼り付ける前に知っておくこと*です。

## あなたのツールについて答えるべき問い

- **私の入力はモデルの訓練に使われるか？** これは大きく異なります。コンシューマー向けチャット、API、エンタープライズの各ティアはしばしば違います。多くの**API/エンタープライズ**の提供物はデフォルトであなたのデータで**訓練しません**。一部のコンシューマー設定は、あなたの選択次第で訓練することがあります。確認しましょう。
- **データはどれくらい保持されるか**、そして誰がアクセスできるか？
- **どこで処理されるか**（リージョン/データ所在地）、そして暗号化されているか？
- 機微な作業のための**ゼロ保持（zero-retention）またはビジネス/エンタープライズの選択肢**はあるか？

## 実践的な経験則

- **秘密情報を貼り付けない。** APIキー、パスワード、トークンを、いかなるAIツールにも、決して。
- **個人データを最小化する。** タスクに必要なものだけを送り、可能なら名前、ID、口座番号を伏せましょう。
- **他人のデータは他人のもの。** 顧客のPII、健康情報、金融情報には注意してください。法的義務（GDPR、HIPAAなど）があるかもしれません。
- **正しいティアを使う。** 規制対象または機密の作業には、個人アカウントではなく、ビジネス/エンタープライズ/ゼロ保持の選択肢を使いましょう。
- **ローカル/セルフホストのモデルを検討する。** データが本当に自分の環境を離れられない場合に。能力をいくらか犠牲にして完全な制御を得ます。

## オンデバイス vs クラウド

| | クラウド（ホスト型） | ローカル/セルフホスト |
|---|---|---|
| 能力 | 最高 | より低い（ただし向上中） |
| データがマシンを離れるか | はい（プロバイダーの条件による） | いいえ |
| セットアップ/コスト | 簡単、従量課金 | 手間がかかる、自前のハードウェア |
