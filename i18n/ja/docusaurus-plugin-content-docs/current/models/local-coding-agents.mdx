**ローカルコーディングエージェント**とは、あなた自身のマシン上で動作し、リポジトリを読み取り、ファイルを編集してくれるプログラムです。あなたは変更内容を自然な言葉で説明し、エージェントはそれを計画し、複数のファイルにわたってコードを書き、テストを実行し、失敗を読み取り、再び試みます — これは単一の答えではなく、ループです。ブラウザタブのチャットアシスタントは貼り付け用のスニペットを渡してくれますが、コーディングエージェントは貼り付けも実行も修正も、あなたのリポジトリの中で自分自身で行います。

このページを読む価値を生む重要な洞察は次の点です。**これらのツールのほとんどはモデル非依存（model-agnostic）である**ということ。エージェント（ファイルに触れるループ）はモデル（何を書くかを決める頭脳）とは別物です。だから*同じ*ツールを、最高のコード品質のために**Claude**へ向けることも、コードがノートPCから一切外に出ないように**[Ollama](/docs/models/run-models-locally-ollama)経由のローカルなオープンウェイトモデル**へ向けることもできます。エージェントはワークフローへの適合度で選び、モデルは品質とプライバシーのトレードオフで選ぶ — それぞれ独立にです。

## 原型：チャットボックスではなく、エージェントループ

このページのすべてのツールは、同じループのバリエーションです。一度このループが見えれば、各ツールはその周りに付いた形の違いにすぎません。

1. **読む（Read）** — エージェントはコンテキストを集めます：あなたが指定したファイル、リポジトリの構造、ときにはテスト出力やエラーログ。
2. **計画する（Plan）** — どのファイルで何を変更するかを決め、しばしば何かに触れる前に計画を提案します。
3. **編集する（Edit）** — ディスク上の実際のファイルに変更を書き込みます（通常はあなたが承認する差分として表示されます）。
4. **実行する（Run）** — コマンドを実行します：テストスイート、ビルド、リンター、スクリプト。
5. **観察して反復する（Observe and iterate）** — 何が起きたか（テストは通った？スタックトレースが出た？）を読み取り、タスクが完了するか、あるいは制御をあなたに返すまでステップ2に戻ります。

これはエージェント的な[ツール利用](/docs/api/tool-use)と同じ形です：モデルにはツール（ファイルを読む、ファイルを編集する、コマンドを実行する）が与えられ、いつそれを呼び出すかを決めます。ツール間の違いは主に、ループが*どれだけの自律性*を持つか、そして*どこに存在するか* — あなたのターミナル、エディタ、あるいは独立して動くサンドボックス環境 — にあります。

そのループの中心にあるモデルは交換可能です。それこそが、1つのエージェントがまったく異なる2種類のユーザーに対応できる理由です：可能なかぎり最強のコードを求める人（Claudeへ向ける）と、法的にコードをネットワークの外に出せない人（ローカルモデルへ向ける）です。

## なぜモデル非依存が目玉機能なのか

エージェントとモデルは別々のパーツです。エージェントはループを担います — ファイルを読み、差分を適用し、テストを実行する。モデルは判断を担います — 何を書くか、なぜ書くか、いつ止めるか。両者を切り離すことで、独立した2つのダイヤルが手に入ります。

- **品質ダイヤル → Claude。** 難しい推論、複数ファイルにまたがるリファクタリング、長いコンテキストの作業では、Claudeのようなフロンティアモデルが顕著に優れた編集を生み出し、失敗からもより確実に立ち直ります。これらのツールのほとんどが、Claudeを推奨またはデフォルトの選択肢として記載しています。（どのClaudeのティアを選ぶかは別の判断です — [モデルの選び方](/docs/models/choosing-a-model)を参照。）
- **プライバシーダイヤル → ローカルモデル。** 同じエージェントを[Ollama](/docs/models/run-models-locally-ollama)上で動くオープンウェイトモデルへ向ければ、**あなたのコード、プロンプト、差分はマシンから一切外に出ません** — API呼び出しもなし、第三者もなし、通信上に何も流れません。これは規制対象、エアギャップ環境、機密のコードベースにとって決定的な要因です。正直なトレードオフ：ノートPCで動かせるほど小さいモデルは、最も難しいタスクではフロンティアモデルと同じ土俵には立てません。あなたは、データを完全に支配する代わりに、いくらかの能力を手放しているのです。

<Callout type="info" items={[
  "モデル非依存とは、どのモデルも等しく優れているという意味ではありません。ループの動きは同じでも、弱いモデルはより悪い編集をし、テストの失敗を読み違え、非生産的にループします。プライバシーと能力は本当にトレードオフの関係にある — 一度きりではなく、タスクごとに選びましょう。",
]} />

## 実在するツール（検証済み）

知っておく価値のある5つのツール。最初の4つはオープンソースでモデル非依存。5つ目はAnthropic自身のエージェントで、Claudeネイティブな体験がどこに位置づけられるかを見られるように含めています。

### Aider — CLIのペアプログラマー

[Aider](https://aider.chat/)はあなたの**ターミナル**で動作し、gitを中心に作られています：リポジトリ内のファイルを編集し、**各変更を妥当なメッセージとともに自動でコミット**するので、すべてのAI編集はあなたがレビュー・取り消し・差し戻しできる差分になります。モデル非依存です — Claudeのようなフロンティアモデルで最もよく動きますが、**Ollama経由のローカルモデル**にも接続できます（`aider --model ollama_chat/<model>`）。ターミナルで暮らし、厳格なgit衛生を望むなら、AiderはCLIの定番の選択肢です。

### Cline — あなたのIDE内のエージェント

[Cline](https://github.com/cline/cline)は、あなたの**エディタ**（VS CodeとJetBrains）内で動作し、ターミナル/CLIの面も備えたオープンソースのコーディングエージェントです。ファイルを作成し、コマンドを実行し、**人間が介在する承認（human-in-the-loop）**を伴ってツールを使います — 各ステップをあなたが見て承認します。モデル非依存で、**Claude**のほか**OllamaやLM Studio経由のローカルモデル**を明示的にサポートします。エディタを離れずに自律的に近いエージェントが欲しいときは、Clineを選びましょう。

> **Roo Code**はClineのよく知られたフォークで、カスタムな「モード」（例：計画用のArchitectモード、編集用のCoderモード）を追加しました。これもモデル非依存でした。ただし、以下のそのステータスに注意してください — もはやかつてのような安全なデフォルトではありません。

<VerifyNote lastVerified="2026-06-28" source="https://github.com/cline/cline">
**Roo Codeのステータスは流動的です。** 検証日時点で、Roo CodeのチームはVS Code拡張機能の縮小と、クラウドエージェントへの方針転換を発表していました — そのため、初期の解説が述べているような、メンテナンスされたローカルファーストのデフォルトではもはやありません。Cline系のフォークを特に求めるなら、採用する前に*今日*アクティブにメンテナンスされているものを確認してください。Cline自体（上流のプロジェクト）は、より安全なローカルファーストの選択肢であり続けています。現在の状態は<a href="https://github.com/cline/cline">github.com/cline/cline</a>で確認してください。
</VerifyNote>

### Continue — オープンなIDEアシスタント

[Continue](https://continue.dev/)は、**VS CodeとJetBrains**（およびCLI）向けのオープンソースアシスタントで、チャット、編集、オートコンプリート、エージェントモードをカバーします。徹底してモデル非依存です：そのドキュメントには**Anthropic（Claude）**、ローカルモデル向けの**Ollama**、その他多くのプロバイダーが列挙されており、役割（チャット、編集、オートコンプリート）ごとに異なるモデルを割り当てられます。どのモデルが何をするかを正確に自分で配線したい、1つの設定可能なアシスタントが欲しいなら、強力な選択肢です。

<VerifyNote lastVerified="2026-06-28" source="https://continue.dev/">
**Continueのメンテナンス状況は確認する価値があります。** 検証日の前後で、プロジェクトは2.0リリースを出し、アクティブなメンテナンスに関するシグナルは変化しつつありました。ツールとそのClaude/Ollamaサポートは実在しますが、チームでの標準化を決める前に、<a href="https://github.com/continuedev/continue">github.com/continuedev/continue</a>で現在のリリースとメンテナンス状況を確認してください。
</VerifyNote>

### OpenHands — 自律型エージェント

[OpenHands](https://github.com/OpenHands/OpenHands)（**旧称OpenDevin**）は4つの中で最も自律的です：サンドボックス環境で動くAIソフトウェア開発エージェントのためのプラットフォームで、タスク（例：GitHubのissue）を分解し、ステップごとの承認をあまり挟まずにそれを最後までやり遂げます。モデル非依存です — 「どんなLLMでも使える」、**Claude**を含む — し、セルフホストで動かせます。各編集を承認するのではなく、タスク全体を引き渡して後で確認したいときは、OpenHandsを選びましょう。

<VerifyNote lastVerified="2026-06-28" source="https://github.com/OpenHands/OpenHands">
OpenHandsは**OpenDevin**から改名され、そのGitHub組織は`All-Hands-AI`から`OpenHands`へ改名されました（古い`All-Hands-AI/OpenHands`のパスは今もリダイレクトされます）。ドキュメントは歴史的に<a href="https://docs.all-hands.dev/">docs.all-hands.dev</a>に置かれてきました。現在の正規のリポジトリとドキュメントのURLは<a href="https://github.com/OpenHands/OpenHands">github.com/OpenHands/OpenHands</a>で確認してください。
</VerifyNote>

### Claude Code — Anthropic自身のエージェント

[Claude Code](https://code.claude.com/)はAnthropicの公式な**エージェント型コーディングツール**です：コードベースを読み、ファイルを編集し、コマンドやテストを実行し、ターミナル、IDE、デスクトップアプリ、Web をまたいで動作します。上記の4つとは異なり、「どんなモデルでも向けられる」わけではありません — Claudeを中心に作られ、Claude向けにチューニングされており、まさにそれが、難しいタスクでループがよりタイトに感じられ、編集がより高い確率で決まる理由です。最大限のモデル交換可能性ではなく、最高のClaude駆動コーディング体験を優先するなら、これがネイティブな道です。詳細は[Claude Codeとは](/docs/claude-code/what-is-claude-code)で。

## 選び方

2つの軸でほぼすべてが決まります。

**軸1 — エージェントがどこに存在するか：**

- **CLI / ターミナル**（Aider、Claude Code） — スクリプト化でき、git親和性が高く、他のツールへパイプできる。シェルで暮らすなら最適。
- **IDE統合型**（Cline、Continue、Claude Codeのエディタ面） — 差分と承認がエディタ内に、コードがすぐそこにある。日常の編集に最適。
- **自律型 / サンドボックス**（OpenHands） — タスク全体を引き渡し、結果をレビューする。バッチ作業や手離れの作業に最適。

**軸2 — Claude駆動 vs 完全ローカル：**

- **Claude駆動** — 最高品質、とりわけ難しい複数ファイルや長コンテキストのタスクで。あなたのコードはモデルのAPIへ送られます。品質が最も重要で、かつコードをプロバイダーへ送ってよい場合に選びましょう。
- **完全ローカル（Ollama）** — あなたのコードはマシンから一切出ない。無料で動かせる。オフラインでも動く。プライバシー、規制、エアギャップが譲れない場合に選びましょう — ノートPCサイズのモデルは最も難しいタスクでは弱いと受け入れたうえで。

長く通用する一手：**ワークフローに合う面でエージェントを選び、そのうえでタスクごとにモデルを選ぶ。** 機密のリポジトリにはローカルモデルでIDEエージェントを動かし、それ以外すべてには同じエージェントをClaudeで動かす — それを妨げるものは何もありません。

Aiderを**ローカル**モデルへ向けた具体的な最初の実行（プライベート、無料、オフライン） — まずOllamaでモデルをpullしてから：

<PromptCard title="ローカルのOllamaモデルでAiderを動かす（コードはマシン上に留まる）">{`aider --model ollama_chat/qwen3`}</PromptCard>

同じツールを代わりに**Claude**へ向けるには（最高品質、コードはAPIへ送られる）、Anthropicのキーを設定してClaudeモデルを選びます — エージェントループは同一で、変わるのは頭脳だけです：

```bash
export ANTHROPIC_API_KEY=sk-ant-...
aider --model sonnet
```

その切り替え — 1つのフラグ、まったく異なる2つのプライバシー/品質プロファイル、同じエージェント — こそが、このページの全要点です。
