あなたはすでにモデルを選びました。しかし、そのモデルを*ラップする*プログラム — ファイルを読み、計画を立て、コマンドを実行し、自らの作業を確認して再試行するループ — は別の選択であり、少なくとも同じくらい重要です。このラッパーのコミュニティでの呼び名が**ハーネス**（または**エージェント**）であり、同じモデルでも一方では見事に、もう一方では平凡に見えることがあります。このページでは、2026年のターミナルコーディングエージェントを俯瞰し、実際に何が違いを生むのか、そして切り替えコストを低く保つために設定をどうポータブルにするかを解説します。

## モデルはエージェントではない

フロンティアモデルはテキスト予測器です。**コーディングエージェント**とは、その周りにあるハーネスです：システムプロンプト、ツールセット（読み取り、編集、シェル実行、検索）、計画ループ、コンテキスト管理戦略、パーミッションとサンドボックスのモデル、そして再試行のロジック。ハーネスを変えれば結果も変わります — 2026年を通じた実務者の報告では、どのエージェントが駆動するかによって、同じモデルが実際のリポジトリタスクで数十ポイントも上下すると述べられています。

これには2つの実践的な帰結があります：

- **ベンチマークの数字はモデルのスコアではなく、*組み合わせ*である。** 「Terminal-Benchで83%」というのは*モデル + ハーネス + スキャフォールド*の結果です。どれか一つを入れ替えれば数字は動きます。
- **あなたのループこそが真の比較単位である。** 最良のエージェントとは、*あなたの*スタックで、*あなたの*ターミナルで、あなたが許容できる価格で、最もうまく機能するものです — コーディングモデルを選ぶのとまったく同じ論理です。決定のモデル側については[コーディングのためのClaude vs GPT vs Gemini](/docs/models/claude-gpt-gemini-for-coding)を参照してください。このページはハーネス側を扱います。

## 2026年の勢力図

ターミナルエージェントの領域は2つの陣営に分かれました：一つのラボのモデルに調整された**ファーストパーティエージェント**と、好きなモデル（またはローカルの重み）に向けられる**モデル非依存**エージェントです。

| エージェント | 提供元 | オープン性 | モデル | 特徴 |
|---|---|---|---|---|
| **Claude Code** | Anthropic | クローズドソース | Claude（ファーストパーティ） | 自律的でエージェント的な複数ファイル作業；OS強制サンドボックス；深いスキル/MCP/サブエージェントのエコシステム |
| **Codex CLI** | OpenAI | オープン（Apache-2.0、Rust） | GPT（ファーストパーティ） | 自律的なターミナルエージェント；ネイティブWindowsサンドボックス；幅広いプラットフォーム対応 |
| **Gemini CLI** | Google | オープンソース | Gemini（ファーストパーティ） | 大規模コンテキストのエージェント、Googleエコシステムとの連携（下記の鮮度に関する注記を参照） |
| **opencode** | コミュニティ（anomalyco） | オープン（MIT、TypeScript） | 任意（モデル非依存） | 最もスターが多いオープンエージェント；CLI + デスクトップ；好きなモデルを持ち込める |
| **Aider** | コミュニティ | オープン（Apache-2.0） | 任意、ローカルを含む | Git ネイティブのペアプログラマー、完全な自律オーケストレーターではない |
| **Cline / Goose / その他** | コミュニティ | オープン | 任意 | エディタ組み込み型（Cline）または汎用オープンエージェント（Goose）；モデル非依存 |

<VerifyNote lastVerified="2026-07-08" source="https://github.com/google-gemini/gemini-cli">
**Gemini CLIは流動的です。** 2026年半ばにGoogleは無料ティアのアクセスを変更し、一部のユーザーを別の「Antigravity」CLIへ誘導しました。Gemini CLI、Antigravity、あるいはさらに新しい何かのうち、どれが正しいGoogleの入り口なのかは、まさに陳腐化しやすい類の事実です — 決める前にGoogle自身のドキュメントで現在の状態を確認してください。
</VerifyNote>

## ポータビリティ：コンテキストは一度だけ書く

最大の隠れた切り替えコストは、あなたの**プロジェクトコンテキスト** — ビルドコマンド、規約、テストルール、エージェントが推測できない事柄です。2026年には、これに事実上の標準があります：**`AGENTS.md`** です。リポジトリのルートに置くプレーンなMarkdownファイルで、Linux Foundation傘下のAgentic AI Foundationが管理し、Codex、Gemini CLI、Cursor、Aider、Copilotのコーディングエージェント、opencode、Goose、そして20以上の他ツールがネイティブに読み込みます（6万以上のリポジトリが採用しています）。

- **`AGENTS.md`** = クロスツールのプロジェクトコンテキスト。一度書けば、ほとんどのエージェントが読みます。
- **`CLAUDE.md`** = Claude Code独自のプロジェクトメモリファイル（[CLAUDE.md](/docs/claude-code/claude-md)を参照）。`AGENTS.md`を指し示す小さな`CLAUDE.md`を維持することも、両方を保守することもできます。
- **`SKILL.md`** = 再利用可能な[スキル](/docs/claude-code/skills)を、フロントマターとバンドルされたスクリプトとともにパッケージ化するためのAnthropic独自の仕様です。これは`AGENTS.md`の*補完*であり、競合ではありません — 一方はプロジェクトを記述し、もう一方は再利用可能な手順をパッケージ化します。

コンテキストを`AGENTS.md`に保管しておけば、別のエージェントを試す日が来ても、プロジェクトのルールが無償でついてきます。

## 一息で決める

- **最も豊富なツールエコシステムを備えた、深く自律的なClaude作業** → **Claude Code**。
- **GPTに紐づいた、強力なプラットフォーム/Windows対応を持つオープンソースのファーストパーティエージェント** → **Codex CLI**。
- **モデルを自由に切り替えたい、またはローカルの重みを実行したい** → **opencode**または**Cline**。
- **慎重で、Gitネイティブで、人間が介在する編集** → **Aider**。
- **大規模なGoogleエコシステムのコンテキスト活用** → **Gemini CLI**（まず現在のアクセス状況を確認してください）。

何を選ぶにせよ、プロジェクトのコンテキストは`AGENTS.md`に保管し、コミットする前に自分のリポジトリの実際のタスクでその選択を証明してください。
