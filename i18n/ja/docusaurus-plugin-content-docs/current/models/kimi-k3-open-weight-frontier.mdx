## 一言でいうと

Kimi K3はMoonshot AIによる**オープンウェイト**(全ウェイトは2026年7月27日公開)、**2.8兆パラメータのスパースMoE**で、長期的なエージェンティックコーディングと知識労働に向けてチューニングされ、ネイティブなビジョンと100万トークンのコンテキストウィンドウを持つ。価格はフロンティアAPI並みだが、コーディングワークロードでは90%以上の頻度でプロンプトキャッシュにヒットするため、実際には安く済むことが多い。

## 見出しからは見えないK3の3つのポイント

### 1. 「2.8兆パラメータ」はトークンあたり2.8兆の活性化を意味しない

見出しの数字は劇的だが、K3は**Mixture-of-Experts**モデルだ。896個のエキスパートサブネットワークを持ち、**トークンあたり16エキスパートのみが発火する**——これが「Stable LatentMoE」コンポーネントである。モデルはストレージ上は巨大だが、フォワードパスあたりの計算量はコンパクトだ。推論時のアクティブパラメータフットプリントは2.8Tよりもはるかに小さい。これはDeepSeek V4 Pro(同じくMoEを使用)と同じアーキテクチャ的トレードオフであり、この規模のモデルがウェイトをセルフホストすれば管理可能なハードウェアで効率的に動く理由でもある。

実務上の含意:密なモデルとMoEモデルでパラメータ数を直接比較してはいけない。16/896エキスパートアクティブの2.8T MoEは、70Bの密モデルとは異なる計算プロファイルを持ち——セルフホストするハードウェア要件も大きく異なる。

<VerifyNote lastVerified="2026-07-23" source="https://www.kimi.com/blog/kimi-k3">
MoE構成(896エキスパート、16アクティブ)はMoonshotの公式技術ブログより。完全な2.8Tモデルをセルフホストするハードウェア要件は量子化に依存する。MXFP4ウェイトはメモリフットプリントを大きく削減するが、完全なオープンウェイトリリースの正確な数値はまだ確認されていない。
</VerifyNote>

### 2. キャッシュヒット率がコスト計算を完全に反転させる

K3のAPI価格は入力$3.00/M、出力$15.00/M——一見すると高い。しかしMoonshotは**キャッシュヒット価格$0.30/M入力**(10倍の割引)を提供しており、コーディングワークロードでは**キャッシュヒット率が90%を超える**と同社は報告している。つまり進行中のコーディングセッションにおける実質的な100万トークンあたりの入力価格は**$0.30〜$0.45**であり、$3.00ではない。

タスク単位のコストで比較すると(第三者評価より):

| モデル | エージェンティックタスクあたりコスト |
|---|---|
| **Kimi K3** | **$0.94** |
| GPT-5.6 Sol | $1.04 |
| Claude Opus 4.8 | $1.80 |

さらにK3は**K2.6より21%少ない出力トークン**を使う——より簡潔になるように学習した——ので、請求額はさらに下がる。見出し価格ではK3はプレミアムに見えるが、現実的なタスク価格ではフロンティア勢を下回る。

### 3. アーキテクチャの革新は事後ではなく学習時のもの

K3は学習の段階で組み込まれた3つの新しいコンポーネントを出荷する:

- **Kimi Delta Attention (KDA)** — 1Mトークンのコンテキストウィンドウを長距離での標準アテンションの二乗コストなしで効率的に扱うハイブリッド線形アテンション機構。
- **Attention Residuals (AttnRes)** — 深さ全体にわたる一様な蓄積の代わりに、AttnResは以前の層からの表現を選択的に取得し、長いエージェンティックランの間、モデルが自身の初期の思考を「振り返る」ことを可能にする。
- **Per-Head Muon と Sigmoid Tanh Unit (SiTU)** — オプティマイザと活性化の改良で、Moonshotが主張する**全体的なスケーリング効率のK2比2.5倍の改善**に寄与した。

MXFP4ウェイト量子化(MXFP8活性化と共に)も事後適用ではなく学習を意識したものだ。これは品質にとって重要である:事後の量子化は通常ベンチマークポイントを削るが、量子化を意識した学習はその代償を大幅に取り除く。

## ベンチマーク地図におけるK3の位置

K3はArtificial Analysis Intelligence Indexで**総合4位**にランクした(2026年7月時点):

| 順位 | モデル | スコア |
|---|---|---|
| 1 | Claude Fable 5 | ~60 |
| 2 | GPT-5.6 Sol | ~59 |
| 3 | (クローズドの拮抗領域) | — |
| **4** | **Kimi K3** | **~57** |
| 5 | Claude Opus 4.8 | ~56 |

K3が**全モデル**(オープンとクローズド)をリードする領域:**Frontend Code Arena**(1,679ポイントで1位)——UIコンポーネント生成と反復的なビジュアルコーディングの実世界評価である。モデルの1Mトークンコンテキストとネイティブなビジョン能力がここで寄与する。

注意すべき点:K3の**ハルシネーション率は独立評価で約51%**——フロンティアのクローズドモデルよりも高い。ベンチマーク精度はK2.6の33%からK3の46%へと跳ね上がったが、事実的グラウンディングは同じ速度では改善しなかった。K3の出力は事実的主張について検証すること。コードと構造化されたタスク完了で最も強く、検索なしのオープンエンドな知識質問では信頼性が下がる。

<VerifyNote lastVerified="2026-07-23" source="https://codersera.com/blog/kimi-k3-complete-guide-2026/">
ハルシネーション率、ベンチマークスコア、タスクあたりコストの数値は、Moonshotの自己報告ではなく第三者ベンチマーク集計元による。Artificial Analysis Intelligence Indexのスコアはローンチ週時点のおおよその値で、インデックスがより多くの評価で更新されるにつれて変動する可能性がある。
</VerifyNote>

**DeepSWE**(長期的なソフトウェアエンジニアリング):K3のスコアは**67.3**で、本稿執筆時点でオープンウェイトモデル上の最高公開スコアであり、フロンティア領域に匹敵する。

**BrowseComp**(コンテキスト管理なしの長文コンテキストナビゲーション):1Mトークンウィンドウを有効化して**90.4**——特筆すべきは、大部分のモデルがこの規模で必要とするチャンク化や検索戦略なしに達成した点である。

## K3が設計されている用途

Moonshot K3の学習は**長期・最小監督タスク**——モデルが計画し、ツールを呼び出し、デバッグし、反復し、大きなリポジトリやマルチ時間ワークフローを通じて一貫性を保たなければならない類の作業——に傾倒した。公式ブログが示す具体的な能力:

- **GPUカーネル最適化**(Pythonラッパーだけでなく、CUDA/Triton レベルの作業)
- **GPUコンパイラ開発**(ベンチマークタスクとしてMiniTritonコンパイラをゼロから構築)
- **チップ設計とRTL検証**(ハードウェアエンジニアリング)
- **マルチエージェントのサブエージェントオーケストレーション** — K3は独自のKimi Work環境で20以上の同時サブエージェントを協調させられる
- **対話的な研究可視化**とダッシュボード生成(ビジョン中心の設計とFrontend Code Arena 1位のランキングが理にかなうところ)

「thinking mode」はローンチ時に常時有効(最大努力)だ。APIはまだチューナブルな努力レベルをサポートしていない——ローンチ時点では、すべての呼び出しが最大の思考を使い、トークンを追加する。追加の推論努力レベルはロードマップにあるが、まだ利用できない。

## K2 vs K3:どちらに手を伸ばすか

[Kimi K2 for Claude users](/docs/models/kimi-k2-for-claude-users) ページから既にK2のツーリングを持っているかもしれない。判断の仕方はこうだ:

| | Kimi K2 (1Tパラメータ) | Kimi K3 (2.8Tパラメータ) |
|---|---|---|
| 総パラメータ数 | 1兆 | 2.8兆 |
| トークンあたりアクティブパラメータ | ~32B | ~16/896エキスパート |
| ネイティブビジョン | なし | あり |
| コンテキストウィンドウ | 128K | 1M |
| オープンウェイト利用可能 | 現在 | 2026年7月27日 |
| ベンチマーク順位 | ~7位 (Artificial Analysis) | 4位 |
| 出力コスト | より低い | $15/M (キャッシュで相殺) |
| 得意分野 | コスト重視のエージェンティックツール使用、今すぐセルフホスト | 長期のビジュアルコーディング、大規模リポジトリ、マルチモーダルタスク |

**K3を選ぶ場合**:フロンティア級の品質(トップ5モデル)が必要、タスクが長くプロンプトキャッシュに繰り返しヒットする可能性が高い、あるいはビジョンが必須のとき。コンプライアンス/プライバシーのためにオープンウェイトが欲しいが7月27日まで待てる場合に特に有用。

**K2にとどまる場合**:今すぐセルフホストが必要、コストが主要な制約でタスクがK3の追加の深さから恩恵を受けない、あるいは既にK2を動くツーリングと共に本番で運用している場合。
