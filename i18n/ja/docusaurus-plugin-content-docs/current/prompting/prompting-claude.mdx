Anthropicのプロンプトエンジニアリングのガイダンスはモデルごとに進化します — 現時点でのベストプラクティスは公式のプロンプトエンジニアリングドキュメントで確認してください。

[プロンプティング](/docs/prompting/basics)のほとんどは普遍的です。しかし、いくつかのテクニックはClaudeで特に効果的であり — 古いモデル由来の習慣のいくつかは、いまや逆効果になっています。ここでは、本当にClaude固有のものを紹介します。

## Claudeで積極的に活用すべきこと

- **[XMLタグ](/docs/prompting/xml-tags)。** Claudeは、入力・指示・例に名前付きタグを使った構造化プロンプトに、よく調整されています。これは最も価値の高いClaudeの習慣です。
- **明確かつ率直に伝える。** 目的、制約、そして「完了」がどう見えるかを述べましょう。Claudeは明示的な指示に忠実に従います。
- [システムプロンプト](/docs/foundations/roles)を通じて**役割を与える**。
- 難しい問題では**考えさせる** — ステップごとの推論を求めるか、推論モデルで[拡張思考/努力度](/docs/api/thinking-and-effort)を使いましょう（その場合、CoTを明示的に求める必要が*ない*ことがよくあります）。
- 形式やエッジケースのために**例を使う**（[few-shot](/docs/prompting/few-shot)）。
- Claudeの応答の冒頭を**プレフィル**して形式を誘導する（API上で） — 例えばアシスタントのターンを `{` で始めてJSONを促したり、見出しで構造を促したりします。

## やめるべき古い習慣

:::warning 指示に忠実な新しいモデルはルールを変える
- **ALL-CAPSや脅し口調のプロンプト**（「YOU MUST… OR ELSE」）は控えましょう。新しいClaudeモデルは指示を文字どおりに従うため、丁寧な言い回しの具体的な指示のほうが、怒鳴るよりうまくいきます。
- 凝った「ジェイルブレイク風」の足場で**過剰に作り込まない**でください — 明確で誠実な指示は小手先のテクニックに勝ります。
- **少ないほうが良いことが多い** — 引き締まったプロンプトは、依頼を埋もれさせる冗長なものに勝ります。
:::

## Claude対他のモデル

*原則*は引き継がれます（[AI横断のプロンプト翻訳](/docs/prompting/cross-ai-translation)を参照）が、Claude固有として最も覚えておく価値があるのは**XMLタグへの親和性**と**プレフィル**です。モデルを切り替えるときは、トーンの強さと形式の手がかりを再確認しましょう。
