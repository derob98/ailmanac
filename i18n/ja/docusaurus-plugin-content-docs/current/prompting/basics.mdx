「AIの出力がいまいち」と感じる原因の多くは、実は**仕様が曖昧なプロンプト**にあります。プロンプトを直せば出力は劇的に変わります — 小手先のテクニックは不要です。これが土台であり、[プロンプティング](/docs/prompting)で扱う内容はすべてこの上に積み重なります。そして、どんなAIにも応用できます。

## たった一つの習慣: 明確かつ率直に伝える

モデルを、頭は切れるが言葉どおりにしか受け取れない、あなたの心を読めない新しい同僚だと考えてください。次の5つを伝えましょう。

| 要素 | 自問すること | 例 |
|---|---|---|
| **目的** | 本当に欲しいものは何か? | 「コールドメールを書いて…」 |
| **読み手** | 誰に向けたものか? | 「…多忙な病院の調達担当者に」 |
| **背景** | 何を知っておく必要があるか? | 「当社はスケジューリングソフトを販売しており、週に約6時間を節約できます」 |
| **形式** | 出力はどんな形にすべきか? | 「120語以内、短い3段落、CTAは1つ」 |
| **トーン** | どんな印象であるべきか? | 「プロフェッショナルで温かく、押し付けがましくない」 |

まとめると、こうなります。

> 「多忙な病院の調達担当者に向けたコールドメールを書いてください。当社は週に約6時間を節約できるスケジューリングソフトを販売しています。120語以内、短い3段落、プロフェッショナルで温かく、しかし押し付けがましくないトーンで、最後は明確な1つの行動喚起で締めくくってください。」

このプロンプトは、「コールドメールを書いて」よりも毎回優れた結果を出します。

## 効果が大きい5つのアップグレード

1. **例を示す。** 欲しいスタイルや形式のサンプルを1つ示すこと（「この声のトーンに合わせて: …」）は、言葉で説明するよりも早く伝わります。これが*few-shot*プロンプティングです — [Few-Shotを正しく使う](/docs/prompting)を参照してください。
2. **役割を与える。** 「あなたは几帳面なコピーエディターです」と伝えると、適切な振る舞いが引き出されます。
3. **難問では思考を求める。** 「最終的な答えを出す前に、ステップごとに考えて」と伝えると推論が改善されます（*chain-of-thought*）。
4. **出力を制約する。** 「前置きなしでJSONだけを返して。」「ちょうど5つの箇条書き。」具体的な形は使いやすくなります。
5. **反復する。** 最初のプロンプトは下書きです。平易な言葉で改善しましょう: 「もっと短く」「もっと専門的に」「反論を1つ追加して」。

## やめるべきアンチパターン

- ❌ **キーワード羅列**（「メール 顧客 割引」）— 検索エンジンを使っているわけではありません。完全な文章で書きましょう。
- ❌ 大量の背景情報の中に**依頼を埋もれさせる**こと。指示は冒頭に置き、入力が長い場合は末尾でも繰り返しましょう。
- ❌ それが自分にとって何を意味するかを言わずに使う**曖昧な品質表現**（「良い感じに/プロっぽく」）。
- ❌ 過去のチャットを**覚えていると思い込む**こと。デフォルトでは覚えていません — [トークン、コンテキスト、メモリ](/docs/foundations)を参照してください。
