**出力スタイル** は、Claude Code が *どのように* コミュニケーションするか — トーン、冗長さ、推論を説明するかどうか — を変えます。*何が* できるかは変えません。応答のためのパーソナリティ/フォーマットのプリセットだと考えてください。

## なぜ使うのか

- **ワークフローに合わせる。** 速く進めているときは簡潔で行動優先、学んでいるときは説明的に。
- **教育モード。** いくつかのスタイルは教育的なコメントを加えます — コードベースやテクニックに慣れていく間に最適です。
- **一貫性。** 好みのスタイルを設定すれば、どのセッションも自分の好きな読み味になります。

## 一般的なスタイル

典型的な組み込みスタイルは、簡潔なデフォルトから、「なぜ」を丁寧にたどる、より **説明的** または **学習指向** のモードまで幅があります。アクティブなスタイルは、該当するコマンド/設定で指定します（現在の一覧と正確な仕組みは [公式ドキュメント](https://code.claude.com/docs/en/output-styles) を参照）。

## 出力スタイル vs CLAUDE.md vs レスポンススタイル

- **出力スタイル** = Claude Code の返答の表示プリセット（トーン/冗長さ）。
- **[CLAUDE.md](/docs/claude-code/claude-md)** = 永続的な *プロジェクト* の事実とルール（何をするか、規約、ガードレール）。
- **[カスタム指示 / レスポンススタイル](/docs/claude-app/custom-instructions)** = Claude.ai コンシューマーアプリ版のいとこ。

:::tip 正しいレバーを使おう
Claude に「終わる前に常にテストを実行」させたい？ それは **CLAUDE.md のガードレール** であって、スタイルではありません。「進めながら各ステップを説明」させたい？ それは **出力スタイル** です。挙動のルールは CLAUDE.md に、表示はスタイルに保ちましょう。
:::
