**出力スタイル**は、システムプロンプトを直接変更することで、Claude Code が*どのように*コミュニケーションするか — その役割、トーン、冗長さ、そして自身の推論を説明するかどうか — を変えます。これらは Claude が*どのように*応答するかを変えるものであり、*何を*知っているかを変えるものではありません。毎ターン同じ口調や形式を求めて繰り返しプロンプトを書き直しているなら、これを使う場面です。

## なぜ使うのか

- **ワークフローに合わせる。** 急いで進めているときは簡潔でアクション優先に、学んでいるときは説明的に。
- **教育モード。** 一部のスタイルは教育的な解説を加えます。コードベースや手法を学んでいるときに最適です。
- **一貫性。** 好みのスタイルを一度設定しておけば、すべてのセッションが好きな読み心地になります。

## 組み込みスタイル

Claude Code には4つが付属しています。

| スタイル | 何をするか |
|---|---|
| **Default** | 標準的なソフトウェアエンジニアリングのシステムプロンプト — 効率的で、アクション優先。 |
| **Proactive** | すぐに実行し、定型的な判断のために立ち止まる代わりに、妥当な前提を置きます。自動モードよりも強い自律実行のガイダンスがありますが、ツールが実行される前には依然として許可プロンプトが表示されます。 |
| **Explanatory** | ステップの合間に教育的な「Insights」を加え、実装上の選択やコードベースのパターンを説明します。 |
| **Learning** | 協働型で、手を動かして学ぶスタイル。Insights を共有する*とともに*、あなた自身に小さな部分を書くよう求め、コード内に `[TODO(human)]` マーカーを残します。 |

Explanatory と Learning は設計上、より長い応答を生成するため、Default よりも多くの出力トークンを消費します。

## 切り替え方法

`/config` を実行し、**Output style** の下からスタイルを選びます。選択内容はプロジェクトレベルの `.claude/settings.local.json` に保存されます。メニューを使わずに設定するには、任意の設定ファイルの `outputStyle` フィールドを編集します。

```json
{
  "outputStyle": "Explanatory"
}
```

:::warning 変更には新しいセッションが必要
出力スタイルはシステムプロンプトの一部であり、Claude Code は**セッション開始時に一度だけ**それを読み込みます。変更は `/clear` の後、または新しいセッションでのみ反映されます。（単独の `/output-style` コマンドは v2.1.91 で削除されました — `/config` または上記の設定を使ってください。）
:::

## カスタムスタイルを作成する

カスタムスタイルは Markdown ファイルです。フロントマター、その後にシステムプロンプトへ追加する指示が続きます。ユーザーレベル（`~/.claude/output-styles/`）またはプロジェクトレベル（`.claude/output-styles/`）に保存します。`name` を設定しない限り、ファイル名がスタイル名になります。

```markdown
---
name: Diagrams first
description: Lead every explanation with a diagram
keep-coding-instructions: true
---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.
```

Claude が*どのように*コミュニケーションするかだけを変えつつ、通常どおりコーディングも続けてほしい場合は `keep-coding-instructions: true` を設定します。Claude がソフトウェアエンジニアリングをまったく行わない場合 — たとえば純粋な文章作成やデータ分析のアシスタント — は、組み込みのコーディング指示が外れるように、これを省略します（デフォルト）。

## 出力スタイル vs CLAUDE.md vs その他のレバー

- **出力スタイル** = 役割、トーン、形式のためのシステムプロンプトのプリセット — *すべての*応答に適用されます。
- **[CLAUDE.md](/docs/claude-code/claude-md)** = 永続的な*プロジェクト*の事実とルール（規約、ガードレール）。システムプロンプトの後にメッセージとして追加されます。
- **[カスタム指示 / 応答スタイル](/docs/claude-app/custom-instructions)** = Claude.ai 消費者向けアプリにおける同類のもの。

:::tip 適切なレバーを使う
Claude に「終わる前に必ずテストを実行する」ようにしたいですか？それは**CLAUDE.md のガードレール**であって、スタイルではありません。「進めながら各ステップを説明する」ようにしたいですか？それは**出力スタイル**です。振る舞いのルールは CLAUDE.md に、提示の仕方はスタイルに留めましょう。
:::
