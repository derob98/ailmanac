カスタムスラッシュコマンドは、何度も打ち直しているプロンプトを 1 つの `/word` にまとめます。一度定義すれば、ずっと再利用できます（そしてチームと共有できます）。

## 仕組み

コマンドはただの Markdown ファイルです。コマンドディレクトリに置けば、そのファイル名がコマンドになります。

- `.claude/commands/changelog.md` → `/changelog`（プロジェクトコマンド、共有可能）
- `~/.claude/commands/changelog.md` → `/changelog`（個人、すべてのプロジェクト）

ファイルの本文が、呼び出したときに Claude が実行するプロンプトです。

## 最小限の例

```markdown
---
description: Draft a Conventional Commit message from the staged diff
---

Look at the staged changes with `git diff --cached` and write a single
Conventional Commit message (feat/fix/docs/refactor/chore). Output only the
message, no preamble.
```

これで `/changelog`… いや `/commit`（ファイル名のとおりに名付けます）が、毎回それを実行します。

## 役立つ構成要素

- **`$ARGUMENTS`** — コマンドの後にユーザーが入力したものを注入します。`/review src/auth.ts` → そのパスがプロンプトに入ります。
- **bash を実行して出力を埋め込む** — コマンドはシェルの出力（例: 現在の差分）を含められるので、Claude はライブな状態に基づいて動けます。
- **ファイルを参照する** — ファイルを指して、その内容を取り込みます。
- **フロントマター** — `description`（メニューに表示される）やその他のオプション。

引数、bash、ファイル参照の正確な構文は [公式ドキュメント](https://docs.anthropic.com/en/docs/claude-code/slash-commands) を参照してください。

## 盗む価値のあるコマンド

出発点となるセットが [スラッシュコマンドライブラリ](/docs/templates/slash-commands) にあります: コミットメッセージ、PR の説明、コードレビュー、changelog、コンポーネントのscaffold。

## コマンド vs スキル vs サブエージェント

- **スラッシュコマンド** — *あなた* が、既知のワークフローをオンデマンドでトリガーする。
- **[スキル](/docs/claude-code/skills)** — Claude が関連するときに *自動的に* 専門知識を読み込む。
- **[サブエージェント](/docs/claude-code/subagents)** — 作業のひと塊を、隔離されたエージェントに委譲する。
