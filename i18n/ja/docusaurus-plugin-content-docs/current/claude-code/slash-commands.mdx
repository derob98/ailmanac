カスタムスラッシュコマンドは、何度も打ち直しているプロンプトを単一の `/word` にパッケージ化します。一度定義すれば、永遠に再利用できます（そしてチームと共有できます）。

## どう動くか

コマンドはただの Markdown ファイルです。コマンドディレクトリに置くと、そのファイル名がコマンドになります。ファイルの本文は、呼び出したときに Claude が実行するプロンプトです。

ファイルの*場所*がそのスコープを決めます：

## 最小限の例

完全なコマンドファイルがこれです。フロントマターは `description` を保持し、その下のすべてがプロンプトです。

<PromptCard title="commit.md — Conventional Commit を起草する">{`---
description: Draft a Conventional Commit message from the staged diff
---

Look at the staged changes with \`git diff --cached\` and write a single
Conventional Commit message (feat/fix/docs/refactor/chore). Output only the
message, no preamble.`}</PromptCard>

これを `commit.md` として保存すれば、`/commit` が毎回それを実行します。（ファイル名のとおりに名付けます — 同じ本文を `changelog.md` として保存すれば `/changelog` になります。）

## 便利な構成要素

4 つの材料が、静的なプロンプトを柔軟なコマンドに変えます：

| 構成要素 | 何をするか |
| --- | --- |
| **`$ARGUMENTS`** | コマンドの後にユーザーが入力したものを注入します。`/review src/auth.ts` → そのパスがプロンプトに入ります。 |
| **bash を実行して出力を埋め込む** | コマンドはシェル出力（例：現在の diff）を含められるので、Claude はライブの状態に基づいて動けます。 |
| **ファイルを参照する** | ファイルを指定してその内容を含めます。 |
| **フロントマター** | `description`（メニューに表示）やその他のオプション。 |

引数、bash、ファイル参照の正確な構文については[公式ドキュメント](https://code.claude.com/docs/en/slash-commands)を参照してください。

## コマンド vs スキル vs サブエージェント

作業をパッケージ化する 3 つの方法 — 違いは*誰が起動するか*と*どれだけ隔離して動くか*です：

- **スラッシュコマンド** — *あなた*が既知のワークフローをオンデマンドで起動する。
- **[スキル](/docs/claude-code/skills)** — Claude が関連するときに*自動で*専門知識を読み込む。
- **[サブエージェント](/docs/claude-code/subagents)** — 作業の一塊を隔離されたエージェントに委任する。
