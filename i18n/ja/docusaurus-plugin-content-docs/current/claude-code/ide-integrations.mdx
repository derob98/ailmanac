ターミナルの中で暮らす必要はありません。Claude Code はエディタと統合され、**コードを書いているまさにその場で、変更を差分として** レビューできます。

## 得られるもの

- **インライン差分。** 提案された編集は、ターミナルのテキストではなく、エディタ内でレビュー可能な差分として表示されます。
- **エディタのコンテキスト。** 開いているファイル、選択範囲、プロジェクトが Claude に利用可能になります。
- **同じ頭脳。** 中身は同じ Claude Code です — あなたの [CLAUDE.md](/docs/claude-code/claude-md)、[権限](/docs/claude-code/permissions)、[フック](/docs/claude-code/hooks)、[設定](/docs/claude-code/settings) が、CLI と IDE で同一に適用されます。一度設定すれば、どこでも使えます。

## サポートされるエディタ

- **VS Code**（および VS Code ベースのエディタ）— 拡張機能経由。
- **JetBrains IDE**（IntelliJ、PyCharm、WebStorm など）— プラグイン経由。

エディタのマーケットプレイスからインストールして認証します。現在の手順は [公式ドキュメント](https://docs.anthropic.com/en/docs/claude-code/ide-integrations) を参照してください。

## CLI か IDE か — どちら？

同じツールです。好みとタスクで選びましょう。

| IDE を好む場面 | CLI を好む場面 |
|---|---|
| 差分を視覚的にレビューしたい | スクリプトを書いている、または [ヘッドレス](/docs/claude-code/headless-and-agent-sdk) モード |
| すでにエディタでコーディングしている | サーバー上 / SSH 経由 |
| エディタのコンテキストが組み込まれているのが好き | 最大限の制御とパイプが欲しい |

:::tip 設定は共有される
CLI と IDE は同じ設定を読むので、[CLAUDE.md](/docs/claude-code/claude-md) と [権限](/docs/claude-code/permissions) を一度セットアップすれば、両者の間で付いてきます。
:::
