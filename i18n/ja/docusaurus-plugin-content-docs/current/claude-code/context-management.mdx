## 大きな考え方

すべてのセッションには有限の[コンテキストウィンドウ](/docs/foundations/tokens-and-context)があります。それが埋まるにつれ、古い内容が押し出され、Claude は「ドリフト」し始めます。コンテキストを意図的に管理することが、長いセッションを鋭く保つ秘訣です。

## 二つの必須コマンド

ほとんどすべての状況は二つのコマンドでカバーできます。コツは、意図にコマンドを合わせることです。

<Flashcards cards={[{front: "/clear", back: "会話を消去して最初からやり直す。**無関係な**タスクに切り替えるときに使う。クリーンで集中したコンテキストを取り戻す最も安価な方法。"}, {front: "/compact", back: "ここまでの会話をコンパクトな形に要約する——要旨を保ちつつ空きを作る。同じタスクを**続け**たいがウィンドウが満杯になりつつあるときに使う。"}]} />

一つの問いで決める：これは新しいタスクか、同じタスクか？

```mermaid
flowchart TD
  A{新しいタスク？それとも同じタスク？} -->|新規／無関係| B[/clear]
  A -->|同じタスク、コンテキスト満杯| C[/compact]
```

## 自動コンパクション

コンテキストが上限に近づくと、Claude Code は**自動コンパクション**を行い、ハードな停止なしに作業を続けられます。これはセーフティネットです——しかしタスクの合間に先回りで `/clear` する方が、それに頼るより良い結果になります。

<Callout type="tip" items={["自動コンパクションは戦略ではなく最後の防波堤として扱いましょう。無関係なタスクの合間にクリアする方が、ウィンドウが満杯になり自動コンパクションが発動するまで放置するよりコンテキストをクリーンに保てます。"]} />

## 効果を保つテクニック

二つのコマンドの他に、最初からコンテキストを引き締めておく 4 つの習慣があります。

<Steps items={[{title: "1 セッション 1 タスク", body: "無関係な作業を一日分も一つのコンテキストに詰め込まないこと。"}, {title: "重い読み込みはサブエージェントに委譲", body: "大量の読み込みはサブエージェントに任せましょう——彼らは自分のコンテキストを消費し、結論だけを返します。/docs/claude-code/subagents を参照。"}, {title: "貼り付けず参照する", body: "大きな塊を貼り付ける代わりに @path で Claude にファイルを指し示し、CLAUDE.md を引き締めて保ちましょう（/docs/claude-code/claude-md を参照）。"}, {title: "予算を見張る", body: "コンテキスト使用量を表示するステータスラインが、いつコンパクトすべきかを教えてくれます。/docs/claude-code/statusline を参照。"}]} />

これらは実際のドキュメントに対応しています：[サブエージェント](/docs/claude-code/subagents)、[CLAUDE.md](/docs/claude-code/claude-md)、[ステータスライン](/docs/claude-code/statusline)。

:::tip 症状 → 修正
Claude が以前の指示を忘れ始めたり同じことを繰り返したりしたら、ウィンドウがおそらく満杯です。`/compact`（同じタスク）または `/clear`（新しいタスク）でたいてい即座に直ります。詳しくは[トラブルシューティング](/docs/contribute/troubleshooting)を参照。
:::
