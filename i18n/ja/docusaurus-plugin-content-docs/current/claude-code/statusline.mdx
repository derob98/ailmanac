**ステータスライン** は、Claude Code がセッションについて表示する常設の帯です。これをカスタマイズすると、あなたが気にする情報 — どのモデルか、どこにいるか、コンテキストがどれだけ残っているか、git ブランチ — を常に視界に入れておけます。

## 表示できるもの

- 使用中の **モデル**（些細なタスクに高価なものを誤って使わないように）。
- **現在のディレクトリ / プロジェクト**。
- **コンテキスト予算** — ウィンドウがどれだけ埋まっているか。[`/compact`](/docs/claude-code/context-management) の合図。
- **git の状態** — ブランチ、dirty/clean、そしてどの [ワークツリー](/docs/claude-code/worktrees) にいるか。

## 仕組み

ステータスラインを **スクリプト**（[設定](/docs/claude-code/settings) で構成）に向けます。そのスクリプトはセッションデータを受け取り、1 行を出力します。ただのスクリプトなので、計算できるものなら何でも表示できます。

```bash
#!/usr/bin/env bash
# reads session JSON on stdin; prints a status line
input=$(cat)
model=$(echo "$input" | jq -r '.model.display_name // "claude"')
branch=$(git branch --show-current 2>/dev/null)
printf "%s · %s · %s" "$model" "$(basename "$PWD")" "${branch:-no-git}"
```

（正確な入力フィールドはドキュメント化されています — 特定のキーに依存する前に確認してください。）

## スクリプトを書きたくない？

コミュニティのツール（例: `ccstatusline`）は、自前で書く代わりに導入できる、すぐ使える設定可能なステータスラインを提供しています。サードパーティのスクリプトはすべて、信頼する前に [レビュー](/docs/security/reviewing-third-party-code) すべきコードとして扱いましょう。

:::tip 最も役立つフィールド
たいていの人にとって、**コンテキスト予算 + モデル** はその位置に値します。最もよくある 2 つの不意打ち — タスク途中でのコンテキスト切れと、誤った（より高価な）モデルの消費 — を防いでくれます。
:::
