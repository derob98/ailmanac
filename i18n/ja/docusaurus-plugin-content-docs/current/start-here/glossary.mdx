### Agent（エージェント）
**ループの中でアクションを実行する**ように設定されたAIのこと。ゴールを読み取り、ツールを呼び出し、結果を確認し、完了するまでこれを繰り返します。Claude Code はあなたのコードベースのためのエージェントです。

### Anthropic
Claude を作っている会社。

### API（Application Programming Interface）
チャット画面に入力する代わりに、*あなたの*ソフトウェアからプログラム的に Claude と対話する手段。[初めてのAPI呼び出し](/docs/api/first-call)を参照。

### Artifact（アーティファクト）
claude.ai 上で Claude がチャットとは別のサイドパネルにレンダリングする、その場で実行できる成果物（アプリ、チャート、ドキュメントなど）。

### Chain-of-thought（CoT、思考連鎖）
回答する前にモデルに**段階的に推論させる**ことで、難しい問題での精度を高める手法。

### Claude
Anthropic が作る AI モデル（およびアプリ）の総称。「Claude」は文脈によって、チャットアプリ、モデル群、あるいはブランドを指します。

### Claude Code
Anthropic の**エージェント型コマンドライン/IDEツール**。あなたのコードベースを読み書きし、コマンドを実行し、ツールを使えます。[Claude Code とは](/docs/claude-code/what-is-claude-code)を参照。

### CLAUDE.md
プロジェクトについての永続的な指示を Claude Code に与えるプレーンテキストファイル。最もレバレッジの高いカスタマイズ手段です。

### Compaction（圧縮）
[コンテキストウィンドウ](/docs/foundations/tokens-and-context)がいっぱいになったときに、古い会話のやり取りを自動的に短い要約にまとめること。これにより、長いセッションでも文脈を見失わずに続けられます。[コンテキスト管理](/docs/claude-code/context-management)を参照。

### Context engineering（コンテキストエンジニアリング）
最良の結果を得るために、モデルの[コンテキストウィンドウ](/docs/foundations/tokens-and-context)に何を入れるか（指示、例、取得したデータ）を意図的に決めること。プロンプティング、RAG、メモリの背後にある一連の技術です。

### Context window（コンテキストウィンドウ）
モデルが一度に考慮できるテキストの最大量（**トークン**単位で測定）。あなたのプロンプト*と*その返答を合わせた量です。これを超えると古い内容から順に落ちていきます。

### Effort（エフォート）
新しめの Claude モデルにある、回答前に**モデルがどれくらい考えるか**をおおまかに制御する設定。最大に振り切るのではなく、タスクに応じてレベルを合わせましょう。考えることにはレイテンシとコストという対価が伴います。[拡張思考とエフォート](/docs/api/thinking-and-effort)を参照。

### Embedding（埋め込み）
テキストの*意味*を表す数値のリスト。これにより、ソフトウェアは単なるキーワード一致ではなく意味的に似たものを見つけられます。セマンティック検索や RAG を支える技術です。

### Eval（evaluation、評価）
AI出力の品質を繰り返し測定できるテスト。変更によって良くなったか悪くなったかを判断できます。

### Few-shot
求めるパターンをモデルに示すために、プロンプトに**いくつかの完成例**を含めること。

### Fine-tuning（ファインチューニング）
独自の例を使ってモデルをさらに訓練し、特化させること。通常は最終手段です。まずは[プロンプティングとRAGを試しましょう](/docs/foundations)。

### Grounding（グラウンディング）
**あなたが提供する文脈だけ**から回答するようモデルに指示し、答えがそこにないときは「わかりません」と言わせること。ハルシネーションに対する主要なプロンプトレベルの防御策です。[パターンライブラリ](/docs/prompting/pattern-library)を参照。

### Guardrails（ガードレール）
AIの振る舞いを安全かつ目的に沿った状態に保つルールと制限。例えば、権限ルール、「完了前にテストを実行する」、リスクの高いアクションへの人間による承認など。[エージェントのセキュリティ確保](/docs/security/securing-agents)を参照。

### Hallucination（ハルシネーション）
モデルが**誤っているのに自信ありげに**何かを述べること。核心となるスキルは検証です。[ハルシネーション](/docs/foundations)を参照。

### Hook（フック）
Claude Code において、ライフサイクルイベント時（例: ファイル編集のたび）に**自動的に**実行されるコマンド。フォーマット整形、ゲート、安全性の強制に使います。

### Inference（推論実行）
モデルを実行して出力を得る行為（訓練するのとは対照的）。

### Knowledge cutoff（ナレッジカットオフ）
それ以降についてモデルが**学習データを持たない**日付 — プロンプトで事実を補わない限り、それより後に起きたことを本当に知りません。Claude のカットオフは各モデルのリリースノートで公開されています。だからこそ、文脈なしでごく最近の出来事について尋ねると、古い答えや捏造された答えが出ることがあります。対処法は、関連する事実を貼り付けるか、ライブ情報を取得するツールを使うことです。

### Latency（レイテンシ）
応答を待つ時間。大きなモデルは通常遅くなります。[ストリーミング](/docs/api/streaming)は、生成と同時に出力を表示することで*体感*レイテンシを改善します。

### LLM（Large Language Model、大規模言語モデル）
Claude の背後にある種類のAI。**次のテキストのかたまりを予測する**ように訓練されており、それが結果的に文章作成、コーディング、要約、推論に十分であることがわかっています。[LLMとは？](/docs/foundations)を参照。

### MCP（Model Context Protocol）
AIを外部のツールやデータ（データベース、GitHub、ブラウザなど）に接続するための**オープン標準**。「MCPサーバー」がツールを公開し、Claude がそれを呼び出せます。[MCP](/docs/claude-code)を参照。

### Multimodal（マルチモーダル）
**テキスト以外も**扱えるモデル。例えば画像やPDFを入力として受け取れます。

### Output style（出力スタイル）
Claude Code において、できることは変えずに**Claude のコミュニケーションの仕方**（例: 簡潔、説明的、学習向け）を変えるプリセット。[出力スタイル](/docs/claude-code/output-styles)を参照。

### Permission mode（権限モード）
Claude Code において、**確認なしで**どこまで実行できるか。あらゆるアクションで確認を求めるものから、編集を自動承認するものまで。速度と安全性のバランスを決める主要なつまみです。[権限](/docs/claude-code/permissions)を参照。

### Plan mode（プランモード）
ファイルに触れる**前に**調査して計画を提案する、読み取り専用の Claude Code モード。実行前にあなたが承認します。[プランモード](/docs/claude-code/plan-mode)を参照。

### Prompt（プロンプト）
あなたがモデルに送るもの。指示、質問、そして文脈となる情報です。

### Prompt caching（プロンプトキャッシュ）
複数の呼び出しにまたがってプロンプトの変化しない前半部分を再利用し、コストとレイテンシを削減すること。[プロンプトキャッシュ](/docs/api)を参照。

### Prompt injection（プロンプトインジェクション）
AIが読み込むコンテンツ（Webページ、ドキュメントなど）に**悪意ある指示を隠し込み**、AIを乗っ取ろうとする攻撃。[プロンプトインジェクション](/docs/security)を参照。

### RAG（Retrieval-Augmented Generation、検索拡張生成）
*あなたの*データから関連する断片を取得してプロンプトに追加し、モデルにそれらについて回答させること。[RAG](/docs/foundations)を参照。

### Reasoning / extended thinking（推論 / 拡張思考）
より難しいタスクのために、モデルが回答前に追加の計算リソースを使って「考える」モード。新しめのモデルでは[エフォート](/docs/api/thinking-and-effort)で調整します。

### Skill（スキル）
パッケージ化された機能（指示文＋任意のスクリプト）。関連する場面で Claude が**必要に応じて**読み込みます。[スキル](/docs/claude-code)を参照。

### Slash command（スラッシュコマンド）
Claude Code における `/` で始まるショートカット。組み込みのもの（`/clear` など）や自作の再利用可能なプロンプトで、定義済みのアクションを起動します。[スラッシュコマンド](/docs/claude-code/slash-commands)を参照。

### Streaming（ストリーミング）
応答全体を待つ代わりに、生成されるそばからモデルの出力を**トークンごとに**送ること。ユーザーから見るとモデルがリアルタイムで入力しているように見え、開発者から見ると1つの応答オブジェクトではなくチャンクのストリームを処理することになります。長い出力では*体感*レイテンシを劇的に改善し、モデルが終わる前に描画を始められます。[ストリーミング](/docs/api/streaming)を参照。

### Structured output（構造化出力）
結果を自由なテキストではなく、厳密な機械可読の形式（例: スキーマに一致するJSON）で取得すること。これにより他のソフトウェアがその結果に依存できます。[構造化出力](/docs/api/structured-output)を参照。

### Subagent（サブエージェント）
独自のコンテキストとツールを持つ別の Claude インスタンス。タスクの一部（例: 調査）を**委譲**して、メインのセッションが集中を保てるようにします。

### System prompt（システムプロンプト）
会話全体にわたってモデルの役割とルールを定める高レベルの指示。最も強力な操舵レバーです。

### Temperature（温度）
ランダム性を制御する設定（一部のモデルで利用可能）。低い＝集中的で再現性が高い、高い＝多様で創造的。

### Token（トークン）
モデルが読み取り、カウントする単位。英語ではおおよそ1単語の¾に相当します。料金とコンテキスト上限はトークン単位で測定されます。[トークンとコンテキスト](/docs/foundations)を参照。

### Tool use / function calling（ツール利用 / 関数呼び出し）
あなたが定義した関数（検索、計算機、あなたのAPIなど）をモデルに呼び出させ、その結果を使わせること。[ツール利用](/docs/api)を参照。

### Zero-shot（ゼロショット）
**例を一切示さず**に — 指示だけで — タスクをモデルにやらせること。日常的なプロンプトのほとんどはゼロショットです。形式やスタイルを導くために完成例を含める[フューショット](#few-shot)と対照的です。素直なタスクならゼロショットで十分です。望む出力の形をモデルが外し続けるときは例を加えてください。
