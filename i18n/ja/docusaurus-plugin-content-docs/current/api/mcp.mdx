**Model Context Protocol（MCP）** は、AI を外部のツールやデータに接続するためのオープン標準です。API では、各ツールを手作業で定義する代わりに、参照によって**サーバー 1 台分のツール**へのアクセスを Claude に与えられます。

## MCP vs 手作業で定義するツール

| | [ツールの利用](/docs/api/tool-use)（カスタム） | MCP |
|---|---|---|
| あなたが定義するもの | 各ツールのスキーマと、それを自分で実行すること | ツールを*公開する*サーバーへの接続 |
| 向いている用途 | アプリ内の少数の専用関数 | 既存の連携（GitHub、DB、ブラウザ、SaaS）の再利用 |
| 認証 | あなたのコード | サーバーへの OAuth またはネイティブキー |

両者は共存します。アプリ固有のツールは直接定義し、出来合いの機能は MCP 経由で取り込みます。

## サーバーを宣言する

MCP サーバーをリクエスト/エージェントにアタッチします（URL またはコマンド、加えて認証）。その後、Claude は他のツールと同様に、[エージェントループ](/docs/api/building-agents)内でサーバーのツールを呼び出せます。正確なリクエストフィールドと認証フローは[公式ドキュメント](https://platform.claude.com/docs/en/docs/build-with-claude/mcp)にあります。

## 同じ標準、3 つの領域

- **API**（このページ） — リクエスト/エージェント上の MCP サーバー。
- **[Claude Code](/docs/claude-code/mcp)** — 開発セッション内の MCP サーバー。
- **[各アプリ](/docs/claude-app/connectors)** — MCP はコネクターを支えています。

一度学べば、どこでも応用できます。

## 信頼

:::warning MCP サーバーはコード + アクセス権である
信頼できるサーバーだけを接続し、最小権限にスコープし、サーバーが返す外部コンテンツが[プロンプトインジェクション](/docs/security/prompt-injection)を含み得ることを忘れないでください。サードパーティのサーバーはまず確認を — [サードパーティコードのレビュー](/docs/security/reviewing-third-party-code)。
:::
