[自前のエージェントループを構築する](/docs/api/building-agents)のが、自分で抱えたい以上のインフラになる場合、**マネージド**（Anthropic がホストする）エージェントオプションがループを代わりに実行します — セッションの配線、リトライ、状態管理ではなく、エージェントの*仕事*に集中できます。

## 「マネージド」が手に入れてくれるもの

ループを自前で組み上げてホストする代わりに、次のようなホスト型の構成要素が得られます:

- **セッション** — 再開できる、永続的なエージェントの実行。
- **環境** — エージェントが自分のツールとともに動作する場所。
- **メモリ** — データベースを自分で配線しなくても、ターンや実行をまたいで引き継がれる状態。
- **スケジューリング** — スケジュールに従って無人で実行されるエージェント。

（正確な名称や対象範囲は進化します — [公式ドキュメント](https://docs.anthropic.com/en/docs/agents-and-tools)を正典として扱ってください。）

## マネージド vs カスタムの選び方

| **マネージド**を選ぶのは… | **カスタムループ / SDK** を選ぶのは… |
|---|---|
| ホスティング、状態、スケジューリングを任せたいとき | ループとツールを完全に制御する必要があるとき |
| すばやくプロトタイピングしているとき | 厳格な独自インフラ/コンプライアンス要件があるとき |
| 制御よりも運用のシンプルさが重要なとき | 自社スタックに深く組み込んでいるとき |

これは連続的なスペクトラムです — 単一呼び出し → ワークフロー → カスタムエージェント（SDK）→ マネージド。タスクが許す限りシンプルに始め、必要になったときだけ上に移行しましょう。

## 同じガードレールが適用される

ホスト型かどうかにかかわらず、自律エージェントはアクションを起こします。**最小権限**、**コスト/反復回数の上限**、**リスクのあるステップへの人間の承認**を維持してください — [エージェントのセキュリティ](/docs/security/securing-agents)と[自律実行のハードニング](/docs/security/hardening-autonomous-runs)を参照。
