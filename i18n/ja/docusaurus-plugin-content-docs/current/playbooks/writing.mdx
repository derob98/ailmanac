AIは、*あなたが指示すれば*素晴らしいライティングのパートナーです。目標はボタンを押して公開することではありません — 白紙のページから強力なドラフトへ、**あなた**の声で素早く到達し、それを磨くことです。

## うまくいくワークフロー

1. **ブリーフを与える。** 目標、読者、形式、長さ、トーン。(「初めての創業者向けの600語のブログ序文、励ます調子で、専門用語なし。」)
2. 数段落より長いものは何でも、まず**アウトラインを作る** — 散文の前に構造を承認する。
3. **ドラフトを作り**、それから**段階的に編集する**(「引き締めて」「もっと具体的な例を」「もったいぶった前置きを削って」)。
4. **あなたの声に合わせる。** あなたの文章のサンプルを貼り付ける: *「この声に合わせて。」* これが品質を左右する最大のレバーです。
5. **転用する。** 1つの作品 → スレッド、ニュースレターの一節、LinkedIn投稿、メール。

## スタート用のプロンプト

```text
Brief: blog intro, 150 words, for {audience}, tone {tone}. Hook in the first line.
Here's my voice to match: """{paste a paragraph you wrote}"""
```
```text
Edit this for clarity and concision without changing meaning. Show a bullet list
of the changes. Text: """{paste}"""
```

## 厳守すべきルール

:::warning 事実をでっち上げさせない
AIは、間違っていてももっともらしい**統計、引用、名前、情報源**を喜んで生み出します。検証していない事実の主張を決して公表しないでください。実際の情報源が必要なものには `[verify]` で印を付けるよう求め、それらは自分で埋めてください。[ハルシネーション](/docs/foundations/hallucinations)を参照。
:::

- **あなたが編集者です。** 判断、センス、説明責任を保ち続けてください。
- **ありきたりな「AIの声」を避ける。** 具体性、例、そしてあなたのスタイルの実際のサンプルがそれを直します。
- 重要な場面ではAIの支援を**開示**してください。
