AILmanac の運営方法です。少人数のチームとボランティアで維持できるよう、軽量に保たれています。

## 独立性と商標

AILmanac は**独立したコミュニティリソース**です。**Anthropic と提携しておらず、その推奨も受けていません。**「Claude」および「Anthropic」は Anthropic の商標であり、名称的に使用されています。[公式ドキュメント](https://docs.anthropic.com)が信頼できる情報源です。

## ライセンス（デュアル）

文章もコードも再利用できるようにするためです。

- **文章／ドキュメント → CC BY 4.0**（帰属表示のうえで再利用可）。
- **コード、サンプル、設定 → MIT。**

リポジトリ内の `LICENSE-CONTENT`、`LICENSE-CODE`、`NOTICE` を参照してください。貢献することで、あなたの文章を CC BY 4.0 の下で、コードを MIT の下でライセンスすることに同意したことになります。

## 行動規範

私たちは **Contributor Covenant** に従います（`CODE_OF_CONDUCT.md` を参照）。親切に、建設的に、善意を前提に。容認できない行為はメンテナーに非公開で報告できます。

## 貢献モデル

- **DCO サインオフ**であり CLA ではありません — `git commit -s` は、あなたがその成果物を提出できることを証明します。障壁が低く、出所が明確です。
- すべてに**プルリクエスト**を用います。[ビルドゲート](/docs/contribute/automation)とメンテナーのレビューが品質を保ちます。
- **CODEOWNERS** は、プロジェクトの成長に合わせて各領域をレビュアーに振り分けます。

## ラベルとトリアージ

コードとしてのラベル（`.github/labels.yml`）がバックログを読みやすく保ちます。

- **タイプ:** `content`、`code`、`i18n`、`automation`
- **領域:** `core-claude`、`ring-general`、`infra`
- **ワークフロー:** `good first issue`、`help wanted`、`upstream-change`、`needs-verification`、`link-rot`、`stale`

新しい貢献者へ: **`good first issue`** を探してみてください。用語集への項目追加、計画段階のページの執筆、古くなった事実の修正は、いずれも絶好のスタートです。

## 意思決定と議論

- 質問や提案には **GitHub Discussions** を、実行可能な作業には **Issues** を用います。
- 貢献者はリポジトリ内で称えられます。
