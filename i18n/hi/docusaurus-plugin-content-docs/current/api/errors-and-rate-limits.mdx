<VerifyNote lastVerified="2026-06-20" source="https://platform.claude.com/docs/en/api/errors">
सटीक स्टेटस कोड, हेडर, और रेट-लिमिट टियर बदलते रहते हैं — आधिकारिक errors/rate-limit दस्तावेज़ों में पुष्टि करें।
</VerifyNote>

प्रोडक्शन कोड एक नेटवर्क सेवा से बात करता है, इसलिए इसे विफलता की अपेक्षा करनी चाहिए। यहाँ थोड़ी-सी संरचना एक अस्थिर इंटीग्रेशन और एक भरोसेमंद इंटीग्रेशन के बीच का अंतर है।

## त्रुटि मानचित्र

सामान्य HTTP स्टेटस जिन्हें आप संभालेंगे:

| स्टेटस | अर्थ | क्या करें |
|---|---|---|
| 400 | अमान्य अनुरोध | पेलोड ठीक करें; उसी रूप में रीट्राई न करें |
| 401 | ग़लत/अनुपस्थित API कुंजी | क्रेडेंशियल जाँचें |
| 403 | अनुमति नहीं | एक्सेस/अनुमतियाँ जाँचें |
| 429 | रेट लिमिटेड | **बैक ऑफ़ करें और रीट्राई करें** (`retry-after` का सम्मान करें) |
| 500/529 | सर्वर त्रुटि / ओवरलोड | **बैकऑफ़ के साथ रीट्राई करें** |

<Callout type="tip" items={["SDK इन्हें टाइप्ड एक्सेप्शन के रूप में सामने लाते हैं, ताकि आप स्ट्रिंग पार्स करने के बजाय स्पष्ट रूप से शाखाएँ बना सकें।"]} />

## बैकऑफ़ के साथ रीट्राई

अस्थायी त्रुटियों (429, 5xx) के लिए, **एक्सपोनेंशियल बैकऑफ़ + जिटर** के साथ रीट्राई करें, सीमित रूप में:

```python
import time, random
for attempt in range(5):
    try:
        return client.messages.create(...)
    except (RateLimitError, APIStatusError) as e:
        if attempt == 4 or not should_retry(e):
            raise
        time.sleep(min(2 ** attempt + random.random(), 30))
```

<Callout type="warning" items={["कई SDK अस्थायी त्रुटियों को स्वचालित रूप से रीट्राई करते हैं — अपनी ख़ुद की रीट्राई जोड़ने से पहले अपने क्लाइंट के डिफ़ॉल्ट को जानें, वरना आप रीट्राई दोगुनी कर सकते हैं।"]} />

## मॉडल माइग्रेशन

मॉडल ID दिनांकित/संस्करणित होते हैं और बंद कर दिए जाते हैं। ख़ुद को सुरक्षित रखें:

- **मॉडल ID को कॉन्फ़िग से पढ़ें**, बिखरे हुए लिटरल्स से नहीं।
- **डेप्रिकेशन पर नज़र रखें** — देखें [डेप्रिकेशन और माइग्रेशन वॉच](/docs/whats-new/deprecations) और [मॉडल तालिका](/docs/whats-new/models-and-pricing)।
- मॉडल बदलते समय अपने [evals](/docs/foundations/evals) **फिर से चलाएँ**।
