API अनुरोधों/एजेंटों पर MCP को कैसे घोषित किया जाता है और auth विकल्प विकसित होते रहते हैं — आधिकारिक दस्तावेज़ों और modelcontextprotocol.io पर पुष्टि करें।
</VerifyNote>

**Model Context Protocol (MCP)** AI को बाहरी टूल और डेटा से जोड़ने का खुला मानक है। API पर, MCP आपको Claude को संदर्भ द्वारा एक पूरे **सर्वर भर के टूल** तक पहुँच देने देता है — हर टूल को हाथ से परिभाषित करने के बजाय।

## MCP बनाम हाथ से परिभाषित टूल

| | [टूल उपयोग](/docs/api/tool-use) (कस्टम) | MCP |
|---|---|---|
| आप परिभाषित करते हैं | प्रत्येक टूल का स्कीमा, और उसे ख़ुद चलाते हैं | एक ऐसे सर्वर से कनेक्शन जो टूल *प्रकाशित* करता है |
| इसके लिए सर्वश्रेष्ठ | आपके ऐप में कुछ अनुकूलित फ़ंक्शन | मौजूदा इंटीग्रेशन का पुनः उपयोग (GitHub, DB, ब्राउज़र, SaaS) |
| Auth | आपका कोड | सर्वर के लिए OAuth या नेटिव कुंजियाँ |

ये साथ-साथ रहते हैं: अपने ऐप-विशिष्ट टूल सीधे परिभाषित करें, और MCP के माध्यम से तैयार क्षमताएँ खींचें।

## एक सर्वर घोषित करना

आप एक अनुरोध/एजेंट से MCP सर्वर(रों) को जोड़ते हैं (URL या कमांड, साथ ही auth)। फिर Claude [एजेंट लूप](/docs/api/building-agents) के भीतर सर्वर के टूल कॉल कर सकता है, किसी भी टूल की तरह। सटीक अनुरोध फ़ील्ड और auth फ़्लो [आधिकारिक दस्तावेज़ों](https://docs.anthropic.com/en/docs/build-with-claude/mcp) में हैं।

## वही मानक, तीन सतहें

- **API** (यह पेज) — आपके अनुरोधों/एजेंटों पर MCP सर्वर।
- **[Claude Code](/docs/claude-code/mcp)** — आपके डेव सेशन में MCP सर्वर।
- **[ऐप्स](/docs/claude-app/connectors)** — MCP कनेक्टर्स को सशक्त बनाता है।

इसे एक बार सीखें; यह स्थानांतरित हो जाता है।

## भरोसा

:::warning एक MCP सर्वर कोड + एक्सेस है
केवल उन सर्वरों से जुड़ें जिन पर आप भरोसा करते हैं, उन्हें न्यूनतम विशेषाधिकार तक सीमित करें, और याद रखें कि उनके द्वारा लौटाई गई बाहरी सामग्री [प्रॉम्प्ट इंजेक्शन](/docs/security/prompt-injection) ले जा सकती है। तीसरे-पक्ष के सर्वरों की पहले समीक्षा करें — [तीसरे-पक्ष के कोड की समीक्षा करना](/docs/security/reviewing-third-party-code)।
:::
