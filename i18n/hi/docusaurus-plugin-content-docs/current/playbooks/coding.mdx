चाहे आप कोड करना सीख रहे हों या प्रोडक्शन सॉफ्टवेयर शिप कर रहे हों, AI लूप को बदल देता है। विजेता इसे एक तेज़, जानकार जोड़ीदार के रूप में मानते हैं — और **यह जो कुछ भी बनाता है उसे सत्यापित करते हैं**।

## यह किसमें बेहतरीन है

- अपरिचित कोड या त्रुटियों को सरल भाषा में **समझाना**।
- बॉयलरप्लेट, टेस्ट, और फंक्शन के पहले मसौदे **जेनरेट** करना।
- स्पष्टता के लिए **रीफैक्टर** करना, और स्टैक ट्रेस पर तर्क करके **डिबग** करना।
- भाषाओं/फ्रेमवर्क के बीच **अनुवाद** करना।
- बग्स और खामियों के लिए एक diff की **समीक्षा** करना।

वास्तविक कोडबेस के लिए, यह काम अपने रिपॉज़िटरी *में* [Claude Code](/docs/claude-code/what-is-claude-code) के साथ करें, जो फ़ाइलें पढ़ सकता है, टेस्ट चला सकता है, और आपकी मंजूरी के साथ संपादन कर सकता है।

## स्वर्णिम लूप

1. **संदर्भ दें** — संबंधित कोड, त्रुटि, आपने क्या अपेक्षा की थी बनाम क्या मिला। अस्पष्ट इनपुट, अस्पष्ट आउटपुट।
2. संपादन से पहले गैर-तुच्छ बदलावों पर **योजना मांगें** ([Plan Mode](/docs/claude-code/plan-mode))।
3. बदलाव **जेनरेट करें**।
4. **इसे पढ़ें** — स्वीकार करने से पहले समझें। कोड आपका है।
5. **इसे चलाएं** — टेस्ट/लिंट/बिल्ड। *बिना चलाए "यह काम करता है" पर कभी भरोसा न करें।*

## ऐसे प्रॉम्प्ट जो अपना भार उठाते हैं

```text
Explain what this function does and any edge cases it mishandles: {code}
```
```text
Write tests for {function}. Cover the happy path and the edge cases. {code}
```
```text
This throws {error}. Here's the code and stack trace. Find the root cause and
propose a minimal fix. {context}
```

## कड़े नियम

:::warning सत्यापित करें, और अपने सीक्रेट की रक्षा करें
- जेनरेट किए गए कोड को **चलाएं और समीक्षा करें** — यह सूक्ष्म रूप से गलत हो सकता है या ऐसे API गढ़ सकता है जो मौजूद ही नहीं हैं।
- किसी प्रॉम्प्ट में **कभी सीक्रेट/कीज़ न चिपकाएं** ([गोपनीयता](/docs/foundations/privacy))।
- एजेंटिक/स्वचालित कोडिंग के लिए, [permissions](/docs/claude-code/permissions) को लॉक डाउन करें और [Securing Agents](/docs/security/securing-agents) पढ़ें।
:::
