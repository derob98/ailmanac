/compact और /clear जैसे कमांड नाम और ऑटो-कॉम्पैक्शन व्यवहार बदल सकते हैं — आधिकारिक डॉक्स में पुष्टि करें।
</VerifyNote>

हर सत्र में एक सीमित [संदर्भ विंडो](/docs/foundations/tokens-and-context) होती है। जैसे-जैसे यह भरती है, पुरानी सामग्री दब जाती है और Claude "भटकना" शुरू कर सकता है। संदर्भ को सोच-समझकर प्रबंधित करना ही वह चीज़ है जो लंबे सत्रों को तेज़ बनाए रखती है।

## दो आवश्यक कमांड

- **`/clear`** — बातचीत को मिटा दें और नए सिरे से शुरू करें। इसका उपयोग तब करें जब आप किसी **असंबंधित कार्य** पर स्विच करें। एक साफ़, केंद्रित संदर्भ पुनः पाने का सबसे सस्ता तरीका।
- **`/compact`** — अब तक की बातचीत को एक संक्षिप्त रूप में सारांशित करें, सार को बनाए रखते हुए जगह खाली करें। इसका उपयोग तब करें जब आप **उसी कार्य को जारी रखना** चाहते हैं लेकिन विंडो भर रही है।

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]
  A -->|Same task, context full| C[/compact]
```

## ऑटो-कॉम्पैक्शन

जब संदर्भ सीमा के निकट पहुँचता है, तो Claude Code **ऑटो-कॉम्पैक्ट** कर सकता है ताकि काम बिना किसी कठोर रुकावट के जारी रहे। यह एक सुरक्षा जाल है — लेकिन कार्यों के बीच सक्रिय रूप से `/clear` करना इस पर निर्भर रहने की तुलना में बेहतर परिणाम देता है।

## प्रभावी बने रहने की तकनीकें

- **प्रति सत्र एक कार्य।** असंबंधित कामों की एक पूरी दोपहर को एक ही संदर्भ में ढेर न होने दें।
- **भारी पठन को सौंपें** [सबएजेंट्स](/docs/claude-code/subagents) को — वे *अपना* संदर्भ खर्च करते हैं और केवल निष्कर्ष लौटाते हैं।
- **संदर्भित करें, पेस्ट न करें।** बड़े blobs पेस्ट करने के बजाय Claude को फ़ाइलों की ओर इंगित करें (`@path`); [CLAUDE.md](/docs/claude-code/claude-md) को दुबला रखें।
- **बजट पर नज़र रखें।** संदर्भ उपयोग दिखाने वाली एक [स्टेटसलाइन](/docs/claude-code/statusline) आपको बताती है कि कब कॉम्पैक्ट करना है।

:::tip लक्षण → समाधान
अगर Claude किसी पुराने निर्देश को भूलने लगे या खुद को दोहराने लगे, तो संभवतः विंडो भर गई है। `/compact` (वही कार्य) या `/clear` (नया कार्य) आमतौर पर इसे तुरंत ठीक कर देता है। अधिक जानकारी [समस्या निवारण](/docs/contribute/troubleshooting) में है।
:::
