समर्थित IDE और सेटअप चरण बदलते रहते हैं — आधिकारिक IDE एकीकरण डॉक्स में पुष्टि करें।
</VerifyNote>

आपको टर्मिनल में ही रहने की ज़रूरत नहीं है। Claude Code आपके संपादक के साथ एकीकृत होता है ताकि आप बदलावों की समीक्षा **डिफ़्स के रूप में करें, ठीक वहीं जहाँ आप कोड लिखते हैं**।

## आपको क्या मिलता है

- **इनलाइन डिफ़्स।** प्रस्तावित संपादन संपादक में एक समीक्षा करने योग्य डिफ़ के रूप में दिखाई देते हैं, न कि केवल टर्मिनल में टेक्स्ट के रूप में।
- **संपादक संदर्भ।** आपकी खुली फ़ाइलें, चयन, और प्रोजेक्ट Claude के लिए उपलब्ध हैं।
- **वही दिमाग।** यह वही Claude Code है — आपका [CLAUDE.md](/docs/claude-code/claude-md), [अनुमतियाँ](/docs/claude-code/permissions), [हुक्स](/docs/claude-code/hooks), और [सेटिंग्स](/docs/claude-code/settings) CLI और IDE में समान रूप से लागू होती हैं। एक बार कॉन्फ़िगर करें, हर जगह उपयोग करें।

## समर्थित संपादक

- **VS Code** (और VS Code-आधारित संपादक) एक्सटेंशन के माध्यम से।
- **JetBrains IDEs** (IntelliJ, PyCharm, WebStorm, आदि) प्लगइन के माध्यम से।

संपादक के मार्केटप्लेस से इंस्टॉल करें और प्रमाणित करें; वर्तमान चरणों के लिए [आधिकारिक डॉक्स](https://code.claude.com/docs/en/ide-integrations) देखें।

## CLI या IDE — कौन सा?

ये एक ही टूल हैं; अपनी पसंद और कार्य के अनुसार चुनें:

| IDE को तब प्राथमिकता दें जब… | CLI को तब प्राथमिकता दें जब… |
|---|---|
| आप डिफ़्स की दृश्य समीक्षा करना चाहते हैं | आप स्क्रिप्टिंग कर रहे हैं या [हेडलेस](/docs/claude-code/headless-and-agent-sdk) मोड में हैं |
| आप पहले से ही संपादक में कोड लिख रहे हैं | आप एक सर्वर / SSH पर हैं |
| आपको संपादक संदर्भ अंतर्निहित पसंद है | आप अधिकतम नियंत्रण और पाइप चाहते हैं |

:::tip सेटिंग्स साझा होती हैं
चूँकि CLI और IDE एक ही config पढ़ते हैं, अपना [CLAUDE.md](/docs/claude-code/claude-md) और [अनुमतियाँ](/docs/claude-code/permissions) एक बार सेट करें और वे आपके साथ इनके बीच चलती रहेंगी।
:::
