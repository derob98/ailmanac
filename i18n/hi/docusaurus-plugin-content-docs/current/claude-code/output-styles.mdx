उपलब्ध आउटपुट स्टाइल्स और उन्हें कैसे सेट करें यह बदल सकता है — आधिकारिक डॉक्स में पुष्टि करें।
</VerifyNote>

**आउटपुट स्टाइल्स** यह बदलती हैं कि Claude Code *कैसे* संवाद करता है — इसका स्वर, विस्तार, और क्या यह अपने तर्क को समझाता है — यह बदले बिना कि यह *क्या* कर सकता है। इसे प्रतिक्रियाओं के लिए एक व्यक्तित्व/प्रारूप प्रीसेट समझें।

## इनका उपयोग क्यों करें

- **अपने वर्कफ़्लो से मेल खाएँ।** जब आप तेज़ी से आगे बढ़ रहे हों तो संक्षिप्त और क्रिया-पहले; जब आप सीख रहे हों तो व्याख्यात्मक।
- **शिक्षण मोड।** कुछ स्टाइल्स शैक्षिक टिप्पणी जोड़ती हैं — किसी कोडबेस या तकनीक को जानने के दौरान बढ़िया।
- **एकरूपता।** एक पसंदीदा स्टाइल सेट करें ताकि हर सत्र वैसे पढ़े जैसा आपको पसंद हो।

## सामान्य स्टाइल्स

विशिष्ट अंतर्निहित स्टाइल्स एक संक्षिप्त डिफ़ॉल्ट से लेकर अधिक **व्याख्यात्मक** या **सीखने-उन्मुख** मोड तक होती हैं जो "क्यों" को विस्तार से समझाती हैं। सक्रिय स्टाइल को संबंधित कमांड/सेटिंग के माध्यम से सेट करें (वर्तमान सूची और सटीक तंत्र के लिए [आधिकारिक डॉक्स](https://docs.anthropic.com/en/docs/claude-code/output-styles) देखें)।

## आउटपुट स्टाइल्स बनाम CLAUDE.md बनाम रिस्पॉन्स स्टाइल्स

- **आउटपुट स्टाइल** = Claude Code की प्रतिक्रियाओं के लिए प्रस्तुति प्रीसेट (स्वर/विस्तार)।
- **[CLAUDE.md](/docs/claude-code/claude-md)** = स्थायी *प्रोजेक्ट* तथ्य और नियम (क्या करना है, परंपराएँ, सुरक्षा कवच)।
- **[कस्टम निर्देश / रिस्पॉन्स स्टाइल्स](/docs/claude-app/custom-instructions)** = Claude.ai उपभोक्ता-ऐप का समकक्ष।

:::tip सही लीवर उपयोग करें
चाहते हैं कि Claude "समाप्त करने से पहले हमेशा टेस्ट चलाए"? वह एक **CLAUDE.md सुरक्षा कवच** है, स्टाइल नहीं। चाहते हैं कि यह "हर चरण को आगे बढ़ते हुए समझाए"? वह एक **आउटपुट स्टाइल** है। व्यवहार नियमों को CLAUDE.md में और प्रस्तुति को स्टाइल्स में रखें।
:::
