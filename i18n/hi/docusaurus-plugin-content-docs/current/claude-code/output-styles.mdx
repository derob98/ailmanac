बिल्ट-इन स्टाइल्स, सटीक स्विचिंग प्रणाली, और कस्टम-स्टाइल फ़ाइल फ़ॉर्मेट रिलीज़ के बीच बदल सकते हैं। आधिकारिक डॉक्स में पुष्टि करें (Claude Code डॉक्स `docs.anthropic.com` से `code.claude.com` पर चले गए हैं)।
</VerifyNote>

**आउटपुट स्टाइल्स** यह बदलते हैं कि Claude Code *कैसे* संवाद करता है — इसकी भूमिका, स्वर, विस्तार, और क्या यह अपने तर्क को समझाता है या नहीं — सीधे सिस्टम प्रॉम्प्ट को संशोधित करके। ये बदलते हैं कि Claude *कैसे* जवाब देता है, न कि यह *क्या* जानता है। जब आप हर बार एक ही आवाज़ या फ़ॉर्मेट के लिए बार-बार प्रॉम्प्ट करते रहें, तो किसी एक का उपयोग करें।

## इन्हें क्यों उपयोग करें

- **अपने वर्कफ़्लो से मेल खाएं।** जब आप तेज़ी से आगे बढ़ रहे हों तो संक्षिप्त और एक्शन-फ़र्स्ट; जब आप सीख रहे हों तो व्याख्यात्मक।
- **शिक्षण मोड।** कुछ स्टाइल्स शैक्षिक टिप्पणी जोड़ते हैं — जब आप किसी कोडबेस या तकनीक को जान रहे हों तो बहुत उपयोगी।
- **स्थिरता।** एक बार पसंदीदा स्टाइल सेट करें ताकि हर सेशन वैसा ही पढ़े जैसा आपको पसंद है।

## बिल्ट-इन स्टाइल्स

Claude Code चार के साथ आता है:

| स्टाइल | यह क्या करता है |
|---|---|
| **Default** | मानक सॉफ़्टवेयर-इंजीनियरिंग सिस्टम प्रॉम्प्ट — कुशल, एक्शन-फ़र्स्ट। |
| **Proactive** | तुरंत निष्पादित करता है और नियमित निर्णयों के लिए रुकने के बजाय उचित अनुमान लगाता है। ऑटो मोड की तुलना में मज़बूत स्वायत्त-निष्पादन मार्गदर्शन, लेकिन टूल्स चलने से पहले भी आपको अनुमति प्रॉम्प्ट दिखते हैं। |
| **Explanatory** | चरणों के बीच शैक्षिक "Insights" जोड़ता है, कार्यान्वयन विकल्पों और कोडबेस पैटर्न को समझाता है। |
| **Learning** | सहयोगात्मक, करके-सीखें: Insights साझा करता है *और* आपसे छोटे हिस्से खुद लिखने को कहता है, आपके कोड में `[TODO(human)]` मार्कर छोड़ता है। |

Explanatory और Learning डिज़ाइन के अनुसार लंबे जवाब उत्पन्न करते हैं, इसलिए इनकी आउटपुट टोकन लागत Default से अधिक होती है।

## कैसे स्विच करें

`/config` चलाएं और **Output style** के तहत एक स्टाइल चुनें। आपका चयन प्रोजेक्ट स्तर पर `.claude/settings.local.json` में सहेजा जाता है। मेनू के बिना इसे सेट करने के लिए, किसी भी सेटिंग्स फ़ाइल में `outputStyle` फ़ील्ड संपादित करें:

```json
{
  "outputStyle": "Explanatory"
}
```

:::warning बदलावों के लिए नया सेशन चाहिए
आउटपुट स्टाइल सिस्टम प्रॉम्प्ट का हिस्सा है, जिसे Claude Code **सेशन शुरू होने पर एक बार** पढ़ता है। कोई बदलाव केवल `/clear` या नए सेशन के बाद ही प्रभावी होता है। (स्टैंडअलोन `/output-style` कमांड को v2.1.91 में हटा दिया गया था — `/config` या ऊपर दी गई सेटिंग का उपयोग करें।)
:::

## एक कस्टम स्टाइल बनाएं

एक कस्टम स्टाइल एक Markdown फ़ाइल है: फ़्रंटमैटर, फिर सिस्टम प्रॉम्प्ट में जोड़ने के लिए निर्देश। इसे यूज़र स्तर (`~/.claude/output-styles/`) या प्रोजेक्ट स्तर (`.claude/output-styles/`) पर सहेजें; फ़ाइल का नाम स्टाइल का नाम बन जाता है जब तक कि आप `name` सेट न करें।

```markdown
---
name: Diagrams first
description: Lead every explanation with a diagram
keep-coding-instructions: true
---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.
```

जब आप केवल यह बदल रहे हों कि Claude *कैसे* संवाद करता है लेकिन फिर भी चाहते हैं कि वह सामान्य रूप से कोडिंग करे, तो `keep-coding-instructions: true` सेट करें। जब Claude सॉफ़्टवेयर इंजीनियरिंग कर ही नहीं रहा हो — मान लीजिए, एक शुद्ध लेखन या डेटा-विश्लेषण सहायक — तो इसे छोड़ दें (डिफ़ॉल्ट) ताकि बिल्ट-इन कोडिंग निर्देश हटा दिए जाएं।

## आउटपुट स्टाइल्स बनाम CLAUDE.md बनाम अन्य लीवर

- **आउटपुट स्टाइल** = भूमिका, स्वर, और फ़ॉर्मेट के लिए एक सिस्टम-प्रॉम्प्ट प्रीसेट — *हर* जवाब पर लागू होता है।
- **[CLAUDE.md](/docs/claude-code/claude-md)** = स्थायी *प्रोजेक्ट* तथ्य और नियम (परंपराएं, गार्डरेल), सिस्टम प्रॉम्प्ट के बाद एक संदेश के रूप में जोड़े गए।
- **[कस्टम निर्देश / रिस्पॉन्स स्टाइल्स](/docs/claude-app/custom-instructions)** = Claude.ai उपभोक्ता-ऐप का समकक्ष।

:::tip सही लीवर का उपयोग करें
चाहते हैं कि Claude "हमेशा समाप्त करने से पहले टेस्ट चलाए"? वह एक **CLAUDE.md गार्डरेल** है, स्टाइल नहीं। चाहते हैं कि यह "जैसे-जैसे आगे बढ़े वैसे-वैसे हर चरण समझाए"? वह एक **आउटपुट स्टाइल** है। व्यवहार के नियम CLAUDE.md में और प्रस्तुति स्टाइल्स में रखें।
:::
