</VerifyNote>

एक **Skill** Claude को केंद्रित विशेषज्ञता देती है जिसे वह *माँग पर* तब लोड करता है जब आपका
अनुरोध मेल खाता है — ताकि आपको हर बार अपनी ज़रूरतें दोबारा समझाने की आवश्यकता न हो। AILmanac
**तैयार-उपयोग स्किल पैक** का एक सेट प्रदान करता है, हर तरह के उपयोगकर्ता के लिए एक। इस अवधारणा को
[Skills](/docs/claude-code/skills) पेज पर गहराई से सीखें।

## तैयार-निर्मित पैक

इनमें से कोई भी रेपो के
[`/skills`](https://github.com/derob98/ailmanac/tree/main/skills) फ़ोल्डर से लें:

| पैक | किसके लिए | यह क्या करता है |
|---|---|---|
| `writing-content-pro` | लेखक, मार्केटर, क्रिएटर | ब्रीफ़ → चरणों में ड्राफ़्ट → आपकी आवाज़ से मेल → पुन: उपयोग |
| `developer-workflow` | डेवलपर | संदर्भ-पहले कोडिंग, समीक्षाएँ, टेस्ट, स्पष्ट कमिट |
| `research-analyst` | शोधकर्ता, विश्लेषक | उद्धरण अनुशासन के साथ स्रोत-आधारित संश्लेषण |
| `data-analyst` | डेटा और स्प्रेडशीट उपयोगकर्ता | SQL/फ़ॉर्मूले + सरल-भाषा रीडिंग + सांख्यिकीय जाँच |
| `business-marketing` | संस्थापक, SMB मालिक | पोज़िशनिंग, कॉपी, ग्राहक संचार, हल्की योजना |
| `student-learning` | विद्यार्थी और सीखने वाले | सॉक्रेटिक ट्यूटरिंग + टीच-बैक (सीखें, बस कॉपी न करें) |
| `everyday-productivity` | सभी के लिए | ईमेल, सारांश, योजना, निर्णय |

हर एक में घर के नियम शामिल हैं: **संदर्भ दें, दोहराएँ, और जो मायने रखता है उसे सत्यापित करें।**

## एक इंस्टॉल करें (Claude Code)

```bash
# personal — available in all your projects
mkdir -p ~/.claude/skills && cp -r skills/writing-content-pro ~/.claude/skills/

# or per-project — shareable with your team via the repo
mkdir -p .claude/skills && cp -r skills/writing-content-pro .claude/skills/
```

जब आपका अनुरोध इसके `description` से मेल खाता है तो Claude स्किल को स्वचालित रूप से सक्रिय करता है।
Claude.ai / Cowork पर, एक स्किल बनाएँ और पैक की `SKILL.md` पेस्ट करें (उपलब्धता प्लान के अनुसार
बदलती है)।

## अपनी खुद की लिखें — न्यूनतम टेम्पलेट

एक स्किल एक `SKILL.md` वाला फ़ोल्डर है: YAML फ़्रंटमैटर + निर्देश। `description` ही
**ट्रिगर** है — इसे "Use when…" के रूप में लिखें।

```markdown
---
name: my-skill
description: Use when the user is doing X — be specific about the trigger so Claude knows when to load this and when not to.
---

# My Skill

Short statement of the role/goal.

## Workflow
1. Step one…
2. Step two…

## Rules
- The non-negotiables (accuracy, privacy, output format).

## Default output
What a good response looks like.
```

### वैधता चेकलिस्ट

- [ ] `name` kebab-case है और फ़ोल्डर से मेल खाता है।
- [ ] `description` बताता है कि **इसका उपयोग कब करें** (ट्रिगर), न कि केवल यह क्या है।
- [ ] निर्देश विशिष्ट और सीमित हैं — एक स्पष्ट काम।
- [ ] कोई सीक्रेट, क्रेडेंशियल, या व्यक्तिगत/क्लाइंट डेटा नहीं।
- [ ] बताता है कि अनिश्चितता को कैसे संभालें (मनगढ़ंत न करें; सत्यापित करें)।
