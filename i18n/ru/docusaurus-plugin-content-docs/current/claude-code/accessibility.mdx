Claude Code работает в терминале — это текстовый UI, что уже неплохая отправная точка для доступности, но стандартное отображение опирается на цвет, глифы псевдографики и спиннеры, которые плохо воспринимаются программами экранного доступа и некоторыми моторными настройками. По состоянию на июль 2026 года есть отдельный **режим экранного диктора**, а также настройки для эргономики vim и поведения мыши/прокрутки. Эта страница собирает рычаги, которые делают CLI пригодным для использования со вспомогательными технологиями.

## Режим экранного диктора

Режим экранного диктора заменяет декоративный TUI на простой текстовый рендеринг, который вспомогательные технологии читают чисто. Включайте его любым из трёх способов — разовым флагом, переменной окружения для каждой сессии или постоянной настройкой.

<Steps items={[
  { title: "Попробуйте на одну сессию", body: "Запустите с флагом: claude --ax-screen-reader. Больше ничего не меняется — та же сессия, более простой вывод." },
  { title: "Включите для каждой сессии", body: "Экспортируйте переменную окружения CLAUDE_AX_SCREEN_READER=1 в профиле оболочки, чтобы она применялась везде." },
  { title: "Сделайте это постоянным в настройках", body: "Установите \"axScreenReader\": true в файле settings.json (пользовательский уровень ~/.claude/settings.json применяется ко всем проектам)." }
]} />

<PromptCard title="Постоянный режим экранного диктора (~/.claude/settings.json)">{`{
  "axScreenReader": true
}`}</PromptCard>

### Что он меняет

<Callout type="tip" items={[

  "Символы транскрипта озвучиваются короткими метками вместо сырых символов.",
  "Вложенные таблицы читаются как строки Заголовок: значение — сплющенные в произносимые строки.",
  "Изменения режима разрешений озвучиваются вслух, когда вы переключаете режимы с помощью Shift+Tab."

Последний пункт важен: переключение режимов разрешений через **Shift+Tab** — это ключевое взаимодействие Claude Code (см. [Разрешения](/docs/claude-code/permissions)), и режим экранного диктора произносит новый режим вслух, чтобы вы всегда знали, находитесь ли вы в plan, accept-edits или auto без чтения строки состояния.

## Эргономика vim: переназначение клавиш режима вставки

Если вы используете режим vim и тянуться к `Escape` напрягает руку, назначьте вместо этого двухклавишную последовательность в режиме вставки. Настройка `vimInsertModeRemaps` превращает последовательности вроде `jj` в `Escape`, чтобы вам не приходилось покидать основной ряд клавиш.

<PromptCard title="Назначение jj на Escape в режиме vim (settings.json)">{`{
  "editorMode": "vim",
  "vimInsertModeRemaps": {
    "jj": "Escape"
  }
}`}</PromptCard>

<Callout type="warning" items={[
  "Переназначение срабатывает только в режиме вставки, поэтому для ввода jj как обычного текста нужна короткая пауза между клавишами — выбирайте последовательность, которую вы редко печатаете быстро (jj и kj — распространённые варианты).",
  "editorMode: vim включает клавиатурные привязки vim; vimInsertModeRemaps работает только при включённом режиме vim."
]} />

## Управление мышью и прокруткой

Полноэкранные меню теперь принимают **клики мыши** — включая меню с множественным выбором и строки ввода «Other» — что помогает, если навигация только с клавиатуры доставляет неудобства. Две настройки регулируют поведение указателя для чувствительности к движению или точности:

| Настройка | Что она делает |
|---|---|
| `CLAUDE_CODE_DISABLE_MOUSE_CLICKS` | Отключает клик, перетаскивание и наведение мышью, сохраняя прокрутку колёсиком — полезно, если случайные клики мешают. |
| `wheelScrollAccelerationEnabled` | Установите `false`, чтобы отключить ускорение прокрутки колёсиком в полноэкранном режиме для более стабильной, предсказуемой прокрутки. |

<PromptCard title="Более стабильная прокрутка, без случайных кликов (settings.json)">{`{
  "wheelScrollAccelerationEnabled": false,
  "env": {
    "CLAUDE_CODE_DISABLE_MOUSE_CLICKS": "1"
  }
}`}</PromptCard>

## Контрастность и темы

Для пользователей со слабым зрением контрастность терминала — первый рычаг. Claude Code поставляется со светлой и тёмной темами и поддерживает **пользовательские темы**, которые вы создаёте через `/theme` или поставляете в плагине — так вы можете подобрать высококонтрастную палитру под свои потребности. См. [Настройки](/docs/claude-code/settings), где хранится выбор темы, и [Стили вывода](/docs/claude-code/output-styles), если хотите также более простой, менее декорированный *текст* (стили Explanatory и Learning более многословны, а не менее — для лаконичного вывода оставайтесь на Default).

:::tip За пределами CLI
Режим экранного диктора покрывает терминал. Для взаимодействия с Claude без рук в более широком смысле [Голосовой режим](/docs/claude-app/voice-mode) в приложении позволяет говорить и слушать, а [Плейбук по доступности](/docs/playbooks/accessibility) описывает использование самого Claude как вспомогательного инструмента — упрощение текста, создание чек-листов и адаптация информации под ваш способ чтения.
:::
