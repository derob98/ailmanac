**Стили вывода** меняют то, *как* Claude Code общается — его роль, тон, многословность и то, объясняет ли он свои рассуждения — изменяя системный промпт напрямую. Они меняют то, *как* Claude отвечает, а не то, *что* он знает. Прибегайте к ним, когда вы раз за разом просите один и тот же стиль или формат на каждом шаге.

## Зачем их использовать

- **Под ваш рабочий процесс.** Лаконично и с упором на действие, когда вы двигаетесь быстро; с пояснениями, когда вы учитесь.
- **Режим обучения.** Некоторые стили добавляют образовательные комментарии — отлично подходит, пока вы знакомитесь с кодовой базой или приёмом.
- **Постоянство.** Задайте предпочитаемый стиль один раз, чтобы каждая сессия читалась так, как вам нравится.

## Встроенные стили

Claude Code поставляется с четырьмя:

| Стиль | Что он делает |
|---|---|
| **Default** | Стандартный системный промпт для разработки ПО — эффективный, с упором на действие. |
| **Proactive** | Выполняет немедленно и делает разумные предположения вместо того, чтобы останавливаться на рутинных решениях. Более сильная установка на автономное выполнение, чем в автоматическом режиме, но вы всё равно видите запросы разрешений перед запуском инструментов. |
| **Explanatory** | Добавляет образовательные «Insights» между шагами, объясняя выбор реализации и паттерны кодовой базы. |
| **Learning** | Совместный режим обучения через практику: делится Insights *и* просит вас написать небольшие фрагменты самостоятельно, оставляя в вашем коде маркеры `[TODO(human)]`. |

Explanatory и Learning по своей задумке выдают более длинные ответы, поэтому они стоят больше выходных токенов, чем Default.

## Как переключить

Запустите `/config` и выберите стиль в разделе **Output style**. Ваш выбор сохраняется в `.claude/settings.local.json` на уровне проекта. Чтобы задать его без меню, отредактируйте поле `outputStyle` в любом файле настроек:

```json
{
  "outputStyle": "Explanatory"
}
```

:::warning Изменения требуют новой сессии
Стиль вывода является частью системного промпта, который Claude Code читает **один раз при старте сессии**. Изменение вступает в силу только после `/clear` или в новой сессии. (Отдельная команда `/output-style` была удалена в v2.1.91 — используйте `/config` или настройку выше.)
:::

## Создание пользовательского стиля

Пользовательский стиль — это файл Markdown: frontmatter, затем инструкции для добавления в системный промпт. Сохраните его на уровне пользователя (`~/.claude/output-styles/`) или на уровне проекта (`.claude/output-styles/`); имя файла становится именем стиля, если вы не зададите `name`.

```markdown
---
name: Diagrams first
description: Lead every explanation with a diagram
keep-coding-instructions: true
---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.
```

Установите `keep-coding-instructions: true`, когда вы меняете только то, *как* Claude общается, но при этом хотите, чтобы он работал с кодом как обычно. Опустите это (значение по умолчанию), когда Claude вообще не занимается разработкой ПО — скажем, это чисто писательский или аналитический ассистент по данным, — чтобы встроенные инструкции по работе с кодом были убраны.

## Стили вывода против CLAUDE.md против других рычагов

- **Стиль вывода** = пресет системного промпта для роли, тона и формата — применяется к *каждому* ответу.
- **[CLAUDE.md](/docs/claude-code/claude-md)** = постоянные *проектные* факты и правила (соглашения, ограничители), добавляемые как сообщение после системного промпта.
- **[Пользовательские инструкции / стили ответов](/docs/claude-app/custom-instructions)** = аналог в потребительском приложении Claude.ai.

:::tip Используйте правильный рычаг
Хотите, чтобы Claude «всегда запускал тесты перед завершением»? Это **ограничитель CLAUDE.md**, а не стиль. Хотите, чтобы он «объяснял каждый шаг по ходу дела»? Это **стиль вывода**. Держите правила поведения в CLAUDE.md, а подачу — в стилях.
:::
