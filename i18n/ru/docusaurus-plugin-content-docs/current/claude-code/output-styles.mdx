**Стили вывода** меняют то, *как* Claude Code общается — его тон, многословность и то, объясняет ли он своё рассуждение — не меняя того, *что* он может делать. Считайте это пресетом личности/формата для ответов.

## Зачем их использовать

- **Под ваш рабочий процесс.** Лаконично и с упором на действие, когда вы движетесь быстро; с объяснениями, когда вы учитесь.
- **Режим обучения.** Некоторые стили добавляют образовательные комментарии — отлично, пока вы знакомитесь с кодовой базой или техникой.
- **Согласованность.** Задайте предпочтительный стиль, чтобы каждая сессия читалась так, как вам нравится.

## Распространённые стили

Типичные встроенные стили варьируются от лаконичного стиля по умолчанию до более **объясняющих** или **ориентированных на обучение** режимов, которые проходят через «почему». Задайте активный стиль через соответствующую команду/настройку (актуальный список и точный механизм смотрите в [официальной документации](https://code.claude.com/docs/en/output-styles)).

## Стили вывода против CLAUDE.md против стилей ответа

- **Стиль вывода** = пресет подачи для ответов Claude Code (тон/многословность).
- **[CLAUDE.md](/docs/claude-code/claude-md)** = постоянные факты и правила *проекта* (что делать, соглашения, ограничители).
- **[Пользовательские инструкции / стили ответа](/docs/claude-app/custom-instructions)** = аналог из потребительского приложения Claude.ai.

:::tip Используйте правильный рычаг
Хотите, чтобы Claude «всегда прогонял тесты перед завершением»? Это **ограничитель CLAUDE.md**, а не стиль. Хотите, чтобы он «объяснял каждый шаг по ходу дела»? Это **стиль вывода**. Держите правила поведения в CLAUDE.md, а подачу — в стилях.
:::
