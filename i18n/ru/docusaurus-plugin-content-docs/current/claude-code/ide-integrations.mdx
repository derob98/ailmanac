Вам не обязательно жить в терминале. Claude Code интегрируется с вашим редактором, чтобы вы просматривали изменения **как diff'ы, прямо там, где вы пишете код**.

## Что вы получаете

- **Встроенные diff'ы.** Предлагаемые правки появляются как доступный для просмотра diff в редакторе, а не просто как текст в терминале.
- **Контекст редактора.** Ваши открытые файлы, выделение и проект доступны Claude.
- **Тот же мозг.** Это тот же Claude Code — ваши [CLAUDE.md](/docs/claude-code/claude-md), [разрешения](/docs/claude-code/permissions), [хуки](/docs/claude-code/hooks) и [настройки](/docs/claude-code/settings) применяются одинаково в CLI и IDE. Настройте один раз — используйте везде.

## Поддерживаемые редакторы

- **VS Code** (и редакторы на базе VS Code) через расширение.
- **IDE JetBrains** (IntelliJ, PyCharm, WebStorm и т. д.) через плагин.

Установите из маркетплейса редактора и пройдите аутентификацию; актуальные шаги смотрите в [официальной документации](https://docs.anthropic.com/en/docs/claude-code/ide-integrations).

## CLI или IDE — что выбрать?

Это один и тот же инструмент; выбирайте по предпочтениям и задаче:

| Предпочитайте IDE, когда… | Предпочитайте CLI, когда… |
|---|---|
| Вы хотите просматривать diff'ы визуально | Вы пишете скрипты или работаете в [headless](/docs/claude-code/headless-and-agent-sdk)-режиме |
| Вы уже пишете код в редакторе | Вы на сервере / по SSH |
| Вам нравится встроенный контекст редактора | Вам нужен максимальный контроль и конвейеры |

:::tip Настройки общие
Поскольку CLI и IDE читают одну и ту же конфигурацию, настройте [CLAUDE.md](/docs/claude-code/claude-md) и [разрешения](/docs/claude-code/permissions) один раз, и они будут следовать за вами между ними.
:::
