Вы хотите, чтобы Claude был **мозгом**, а ваша машина предоставляла **руки** — читала ваши файлы, обращалась к базе данных, запускала ваш скрипт, даже вызывала локальную модель или локального агента, и всё это без выхода ваших данных за пределы ноутбука. У этой синергии уже есть стандарт: **Model Context Protocol (MCP)**. В двух словах: MCP — это открытый стандарт, который позволяет ИИ-клиенту вызывать внешние инструменты и данные через небольшие программы, называемые **MCP-серверами** — и эти серверы могут работать приватно, локально, на вашей собственной машине. ## Что такое MCP (в двух словах)

**MCP — это открытый стандарт для подключения ИИ-приложений к внешним системам.** ИИ-клиент (например, Claude) говорит на одном протоколе; любой **MCP-сервер**, который на нём говорит, может подключиться и предложить клиенту новые возможности. Официальная аналогия — **порт USB-C для ИИ**: вместо отдельной интеграции под каждый инструмент вы строите под один разъём, и он работает везде.

Anthropic представила и открыла исходный код MCP, и с самого первого дня он поставлялся с **поддержкой локальных MCP-серверов**, чтобы Claude мог подключаться к внутренним системам и наборам данных, работающим на вашей собственной машине.

<VerifyNote lastVerified="2026-06-28" source="https://modelcontextprotocol.io/">
Показатели внедрения, дата ревизии спецификации, управление (руководство MCP) и точный список клиентов, поддерживающих MCP, часто меняются. Относитесь к любой конкретной цифре как к скоропортящейся и подтверждайте её на <a href="https://modelcontextprotocol.io/">modelcontextprotocol.io</a> и в анонсе Anthropic. Устойчивые факты — MCP является открытым стандартом, клиент ↔ сервер, серверы могут работать локально, широко принят множеством клиентов — стабильны.
</VerifyNote>

MCP-сервер может предоставлять клиенту три вида вещей:

- **Инструменты (Tools)** — действия, которые модель может вызвать (прочитать файл, выполнить запрос, обратиться к API). Это та же форма, что и у нативного [использования инструментов](/docs/api/tool-use) модели: именованная возможность с типизированными входными данными.
- **Ресурсы (Resources)** — данные, которые клиент может читать (файлы, записи, документы).
- **Промпты (Prompts)** — переиспользуемые параметризованные шаблоны промптов, которые предлагает сервер.

Для этой страницы главное — это **инструменты**: локальный MCP-сервер — это то, как вы вручаете Claude новую, приватную возможность.

## Почему это та синергия, которую вы хотите

То, что люди подразумевают под «Claude как мозг, мой компьютер как тело», — это в точности разделение MCP на клиент/сервер:

- **Claude — оркестратор.** Он читает вашу цель, решает, какой инструмент вызвать, вызывает его, читает результат и решает, каким будет следующий шаг.
- **Локальные MCP-серверы — это возможности.** Каждый из них — небольшая программа, которую *вы* запускаете на *своей* машине: файловый сервер, сервер базы данных, обёртка вокруг локального скрипта, даже сервер, который вызывает [локальную модель](/docs/models/local-ai-agents) или передаёт работу локальному агенту.

Поскольку сервер работает локально, а Claude общается с ним через стандартный ввод/вывод вашей собственной машины, **ваши данные и действия остаются на вашем оборудовании** — модель оркестрирует, но чтение файлов, SQL, выполнение скриптов происходят локально. Это версия агентной работы, сохраняющая приватность: мозг в облаке (или тоже локально), руки строго на вашей машине.

А поскольку MCP — это **один открытый стандарт, принятый множеством клиентов**, сервер, который вы создаёте, не привязан к одному приложению. Тот же локальный сервер, который вы подключаете к [Claude Code](/docs/claude-code/what-is-claude-code), может использоваться другими клиентами, говорящими на MCP — создайте один раз, переиспользуйте везде.

## Локальная архитектура

Наименьшая полезная картина состоит из двух частей и одного канала:

- **MCP-клиент (хост).** ИИ-приложение — например, [Claude Code](/docs/claude-code/what-is-claude-code) или Claude Desktop. Оно содержит модель и решает, какие инструменты вызывать.
- **MCP-сервер.** Отдельный процесс, который объявляет инструменты/ресурсы/промпты и выполняет их по запросу.
- **Транспорт.** Для локальных серверов клиент запускает сервер как подпроцесс, и они общаются через **stdio** (стандартный ввод/вывод) с помощью сообщений JSON-RPC. Никакой сети, никакого порта — просто локальный канал между двумя процессами на вашей машине. (Удалённые серверы тоже существуют, по HTTP; эта страница о локальном случае со stdio.)

```
┌─────────────────────────┐         stdio (JSON-RPC)        ┌──────────────────────────┐
│  MCP CLIENT / HOST       │ ──── launches as subprocess ──▶ │  LOCAL MCP SERVER         │
│  Claude Code / Desktop   │ ◀── tools/resources/prompts ─── │  filesystem · db · script │
│  (the model = the brain) │                                 │  · local model / agent    │
└─────────────────────────┘                                  └──────────────────────────┘
                                                              runs on YOUR machine, YOUR data
```

Модель никогда не касается вашего диска напрямую — она обращается к локальному серверу, а сервер (работающий с вашими правами) выполняет работу и возвращает результат.

## Добавьте локальный MCP-сервер в Claude

Самый быстрый путь — команда `claude mcp add` в Claude Code, которая регистрирует сервер, запускаемый клиентом через stdio. Общая форма — имя, затем команда, которая запускает сервер.

Минимальный `claude mcp add` для локального файлового сервера (всё после `--` — это команда запуска):

<PromptCard title="Добавить локальный файловый MCP-сервер в Claude Code">{`claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /path/to/allowed/dir`}</PromptCard>

Если вы предпочитаете править конфигурацию напрямую, stdio-сервер объявляется командой, которая его запускает. Форма выглядит так:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    }
  }
}
```

<VerifyNote lastVerified="2026-06-28" source="https://modelcontextprotocol.io/">
Точные подкоманды `claude mcp`, области видимости конфигурации, расположение конфигурационного файла и точные имена пакетов эталонных серверов развиваются. Подтвердите актуальный синтаксис в <a href="/docs/claude-code/mcp">руководстве по MCP для Claude Code</a> и репозитории эталонных серверов, прежде чем полагаться на конкретный флаг или имя пакета.
</VerifyNote>

## Создайте крошечный MCP-сервер

Сервер — это просто программа, которая использует MCP SDK, объявляет инструмент и запускает транспорт stdio. Вот вся идея на TypeScript — один инструмент, который складывает два числа (замените тело на «запустить мой скрипт», «обратиться к моей БД», «вызвать мою локальную модель»):

```bash
npm install @modelcontextprotocol/sdk zod
```

```ts
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({ name: "my-local-tools", version: "0.1.0" });

// Declare ONE tool. Inputs are typed; the body runs locally with your privileges.
server.registerTool(
  "add",
  {
    description: "Add two numbers",
    inputSchema: { a: z.number(), b: z.number() },
  },
  async ({ a, b }) => ({
    content: [{ type: "text", text: String(a + b) }],
  }),
);

// Talk to the client over stdio — this is what makes it a LOCAL server.
await server.connect(new StdioServerTransport());
```

Зарегистрируйте его в Claude точно так же, как любой другой stdio-сервер — укажите команде запуска на ваш файл:

<PromptCard title="Добавить свой собственный локальный сервер в Claude Code">{`claude mcp add my-local-tools -- node /abs/path/to/server.js`}</PromptCard>

Это полный цикл: Claude (мозг) видит новый инструмент, вызывает его, когда это уместно, а ваш код (руки) выполняется локально и возвращает результат. Поскольку он говорит на том же открытом протоколе, этот сервер также работает в других MCP-клиентах без изменений.

<Callout type="warning" items={["MCP-серверы работают с вашими привилегиями и могут совершать реальные действия — устанавливайте только те серверы, которым доверяете, и остерегайтесь prompt-инъекций через результаты инструментов."]} />
