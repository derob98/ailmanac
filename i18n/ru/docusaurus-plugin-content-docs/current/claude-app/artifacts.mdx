Возможности артефактов (интерактивность, сохранение, к чему они могут обращаться) быстро развиваются — уточняйте актуальное поведение в приложении/центре помощи.
</VerifyNote>

**Артефакты** — это результаты, которые Claude отображает в **боковой панели** рядом с чатом: документ, диаграмма, рабочее приложение, схема — то, что вы можете увидеть, использовать и дорабатывать отдельно от текста разговора.

## Что можно создать

- **Мини-веб-приложения и инструменты** — калькулятор, квиз, форму, небольшое интерактивное демо.
- **Документы** — структурированные тексты, которые можно дорабатывать и экспортировать.
- **Визуализации** — диаграммы, схемы и простые дашборды данных.
- **Код**, который вы можете прочитать и запустить.

## Почему это мощно для тех, кто не программирует

Вы можете создать что-то *пригодное к использованию* — «сделай мне калькулятор чаевых для ужина компанией», «дашборд из этого CSV» — просто описав это, а затем дорабатывать в разговоре («добавь поле для сервисного сбора», «сделай кнопки крупнее»). Это самый наглядный пример **создания с помощью ИИ без написания кода самостоятельно**.

## Как работать с артефактами

1. **Попросите то, что нужно**, с конкретикой (цель, входные данные, внешний вид).
2. **Итерируйте на обычном языке** — Claude обновляет тот же артефакт.
3. **Используйте его** в панели; **экспортируйте/делитесь** там, где это поддерживается.

```mermaid
flowchart LR
  A[Describe what you want] --> B[Claude builds an artifact]
  B --> C[You try it in the side panel]
  C --> D[“Tweak this…”] --> B
```

## Советы

- **Будьте конкретны** насчёт входов/выходов и аудитории — так же, как при хорошем [промптинге](/docs/prompting/basics).
- **Итерируйте небольшими шагами.** Одно изменение за раз проще довести до нужного результата.
- **Проверяйте любую логику/числа**, которые вычисляет артефакт, для важных задач ([Галлюцинации](/docs/foundations/hallucinations)).
