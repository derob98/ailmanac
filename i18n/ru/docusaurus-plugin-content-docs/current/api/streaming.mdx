Создание чат-подобных интерфейсов на API сводится к двум практическим реалиям: **стримьте**, чтобы пользователи сразу видели вывод, и **управляйте историей сами**, потому что API не хранит состояние. Освойте оба приёма — и ваш чат будет ощущаться быстрым и помнить всё.

## Стриминг

Без стриминга пользователь ждёт весь ответ целиком. Со стримингом токены приходят по мере генерации — гораздо лучше воспринимаемая скорость.

<Callout type="tip" items={["Используйте хелпер стриминга из SDK вместо ручного разбора сырых событий — он управляет жизненным циклом событий за вас."]} />

<Tabs groupId="lang">
<TabItem value="python" label="Python">

```python
with client.messages.stream(
    model="claude-sonnet-5", max_tokens=1024,
    messages=[{"role": "user", "content": "Explain RAG in two sentences."}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
```

</TabItem>
<TabItem value="ts" label="TypeScript">

```ts
const stream = client.messages.stream({
  model: "claude-sonnet-5", max_tokens: 1024,
  messages: [{ role: "user", content: "Explain RAG in two sentences." }],
});
for await (const event of stream) {
  if (event.type === "content_block_delta") process.stdout.write(event.delta.text ?? "");
}
```

</TabItem>

## Многоходовый диалог: историю держите вы

API **не имеет памяти** между вызовами ([почему](/docs/foundations/tokens-and-context)). Чтобы продолжить диалог, каждый раз отправляйте обратно **весь предыдущий обмен**.

<Steps items={[{title: "Начните диалог", body: "Отправьте первое сообщение пользователя в списке messages."}, {title: "Зафиксируйте ответ", body: "Прочитайте текст ответа ассистента из результата."}, {title: "Добавьте оба хода", body: "Добавьте ответ ассистента и следующее сообщение пользователя в тот же список messages."}, {title: "Отправьте полный список снова", body: "Отправьте полный список messages заново — именно так Claude «помнит» обмен."}]} />

```python
messages = [{"role": "user", "content": "Hi, I'm planning a trip."}]
# ... get assistant reply, then append both turns:
messages.append({"role": "assistant", "content": assistant_text})
messages.append({"role": "user", "content": "Make it 3 days."})
# send the full `messages` list again
```

## Длинные диалоги заполняют окно

По мере роста истории она съедает [контекстное окно](/docs/foundations/tokens-and-context), и стоимость растёт. Стратегии, чтобы держать это под контролем:

- **Суммируйте/уплотняйте** более старые ходы в короткую сводку, которую вы несёте дальше.
- **Обрезайте** нерелевантные ранние ходы.
- Сочетайте с **[кэшированием промптов](/docs/api/prompt-caching)**, чтобы не платить повторно за стабильный префикс.

<Callout type="warning" items={["Каждый вызов повторно отправляет всю историю — длинные диалоги стоят дороже и в итоге могут превысить контекстное окно, если вы никогда не уплотняете и не обрезаете её."]} />
