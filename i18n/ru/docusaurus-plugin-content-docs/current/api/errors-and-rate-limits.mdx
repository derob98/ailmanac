Точные коды состояния, заголовки и уровни лимитов частоты меняются — уточняйте в официальной документации по ошибкам/лимитам частоты.
</VerifyNote>

Продакшен-код общается с сетевым сервисом, поэтому он должен ожидать сбоев. Немного структуры здесь — это разница между ненадёжной интеграцией и надёжной.

## Карта ошибок

Типичные HTTP-статусы, которые вам придётся обрабатывать:

| Статус | Значение | Что делать |
|---|---|---|
| 400 | Некорректный запрос | Исправьте полезную нагрузку; не повторяйте как есть |
| 401 | Неверный/отсутствующий API-ключ | Проверьте учётные данные |
| 403 | Не разрешено | Проверьте доступ/разрешения |
| 429 | Превышен лимит частоты | **Сделайте задержку и повторите** (соблюдайте `retry-after`) |
| 500/529 | Ошибка сервера / перегрузка | **Повторите с задержкой** |

<Callout type="tip" items={["SDK представляют их как типизированные исключения, так что вы можете чисто ветвиться, а не разбирать строки."]} />

## Повторы с задержкой

При временных ошибках (429, 5xx) повторяйте с **экспоненциальной задержкой + джиттером**, с ограничением:

```python
import time, random
for attempt in range(5):
    try:
        return client.messages.create(...)
    except (RateLimitError, APIStatusError) as e:
        if attempt == 4 or not should_retry(e):
            raise
        time.sleep(min(2 ** attempt + random.random(), 30))
```

<Callout type="warning" items={["Многие SDK повторяют временные ошибки автоматически — узнайте поведение вашего клиента по умолчанию, прежде чем добавлять собственное, иначе вы можете удвоить число повторов."]} />

## Лимиты частоты

Лимиты применяются на уровне аккаунта/тарифа (запросы и токены в минуту). При достижении лимита вы получаете **429** с подсказками о таймингах. Стратегии, чтобы оставаться под потолком:

## Миграция моделей

ID моделей датированы/версионированы и со временем устаревают. Защитите себя:

- **Читайте ID модели из конфигурации**, а не из разбросанных литералов.
- **Следите за устареваниями** — см. [Мониторинг устаревания и миграции](/docs/whats-new/deprecations) и [таблицу моделей](/docs/whats-new/models-and-pricing).
- **Перезапускайте свои [оценки](/docs/foundations/evals)** при смене моделей.
