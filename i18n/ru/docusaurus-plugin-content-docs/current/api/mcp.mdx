**Model Context Protocol (MCP)** — это открытый стандарт для подключения ИИ к внешним инструментам и данным. На API MCP позволяет дать Claude доступ к целому **набору инструментов сервера** по ссылке — вместо того чтобы определять каждый инструмент вручную.

## MCP против вручную определённых инструментов

| | [Использование инструментов](/docs/api/tool-use) (собственные) | MCP |
|---|---|---|
| Вы определяете | Схему каждого инструмента и запускаете его сами | Подключение к серверу, который *публикует* инструменты |
| Лучше всего для | Нескольких индивидуальных функций в вашем приложении | Повторного использования существующих интеграций (GitHub, БД, браузеры, SaaS) |
| Аутентификация | Ваш код | OAuth или собственные ключи к серверу |

Они сосуществуют: определяйте специфичные для приложения инструменты напрямую и подтягивайте готовые возможности через MCP.

## Объявление сервера

Вы прикрепляете MCP-сервер(ы) к запросу/агенту (URL или команда плюс аутентификация). После этого Claude может вызывать инструменты сервера внутри [цикла агента](/docs/api/building-agents), так же как любой инструмент. Точное поле запроса и поток аутентификации описаны в [официальной документации](https://docs.anthropic.com/en/docs/build-with-claude/mcp).

## Один стандарт, три поверхности

- **API** (эта страница) — MCP-серверы в ваших запросах/агентах.
- **[Claude Code](/docs/claude-code/mcp)** — MCP-серверы в ваших сессиях разработки.
- **[Приложения](/docs/claude-app/connectors)** — MCP лежит в основе коннекторов.

Изучите один раз — знание переносимо.

## Доверие

:::warning MCP-сервер — это код + доступ
Подключайте только те серверы, которым доверяете, ограничивайте их минимумом привилегий и помните, что возвращаемый ими внешний контент может нести [инъекцию промпта](/docs/security/prompt-injection). Сначала проверяйте сторонние серверы — [Проверка стороннего кода](/docs/security/reviewing-third-party-code).
:::
