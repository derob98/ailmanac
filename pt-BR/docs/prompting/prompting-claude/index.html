Prompting Específico para o Claude

Intermediário
Última verificação: 2026-06-20.

A orientação de engenharia de prompt da Anthropic evolui a cada modelo — consulte a documentação oficial de engenharia de prompt para as melhores práticas atuais.

Confirme na fonte oficial.

A maior parte do prompting é universal. Mas algumas técnicas são especialmente eficazes com o Claude — e alguns hábitos de modelos mais antigos são agora contraproducentes. Esta lição separa o sinal do ruído: o que é de fato específico do Claude.

What you'll learn
  • Reconhecer as técnicas específicas do Claude que vale a pena adotar — tags XML, clareza, papéis, raciocínio, exemplos e prefilling
  • Desaprender os hábitos de modelos antigos (CAIXA-ALTA, ameaças, andaimes superdimensionados) que agora saem pela culatra
  • Saber quais dois hábitos — afinidade com tags XML e prefilling — carregar como sua assinatura com o Claude
  • Entender o que reavaliar quando você troca de modelo

Aposte nestes com o Claude​

Estes são os movimentos que mais compensam com o Claude. O primeiro é o hábito de maior valor que você pode construir.

  • Tags XML. O Claude é bem ajustado para prompts estruturados com tags nomeadas para entradas, instruções e exemplos. Este é o hábito de maior valor com o Claude.
  • Seja claro e direto. Declare o objetivo, as restrições e como é "estar pronto". O Claude segue de perto as instruções explícitas.
  • Atribua um papel via system prompt.
  • Deixe-o pensar em problemas difíceis — peça raciocínio passo a passo, ou use extended thinking/effort em modelos de raciocínio (onde muitas vezes você não precisa pedir CoT explicitamente).
  • Use exemplos (few-shot) para formato e casos extremos.
  • Faça prefill do início da resposta do Claude (na API) para direcionar o formato.

Experimente o prefilling​

Fazer prefill significa começar o turno do assistente você mesmo, para que o Claude continue no formato que você quer. Na API, comece o turno do assistente com um caractere de abertura ou um título:

## Summary
Pro tip
  • Comece o turno do assistente com { para forçar JSON, ou com um título para forçar saída estruturada. O prefilling é uma alavanca exclusiva da API — direciona o formato sem gastar palavras em instruções.

Abandone estes hábitos antigos​

Modelos Claude mais novos, que seguem instruções de forma literal, mudam as regras. Truques que ajudavam em modelos mais antigos agora atrapalham.

Watch out
  • Vá com calma no CAIXA-ALTA e em prompts em tom de ameaça ("VOCÊ DEVE… SENÃO"). Os modelos Claude mais novos seguem instruções literalmente, e instruções específicas e formuladas com polidez funcionam melhor do que gritar.
  • Não exagere na engenharia com andaimes elaborados em estilo "jailbreak" — instruções claras e honestas superam truques.
  • Menos costuma ser mais — um prompt enxuto vence um prolixo que esconde o pedido.

Claude vs. outros modelos​

Os princípios se transferem (veja Tradução de Prompts entre IAs), mas duas coisas são as mais dignas de lembrar como específicas do Claude.

Guided walkthrough1 of 3
  1. Envolva entradas, instruções e exemplos em tags nomeadas. A afinidade do Claude com tags torna este o hábito de estruturação mais confiável.
  2. Semeie o turno do assistente para travar o formato — uma chave de abertura para JSON, um título para estrutura.
  3. Quando você sai do Claude, reavalie a intensidade do tom e as pistas de formatação; o que é literal aqui pode não se transferir de forma limpa.

Key takeaways
  • Tags XML são o hábito de maior valor com o Claude; o prefilling é a alavanca de formato específica do Claude mais útil.
  • Os modelos Claude mais novos seguem instruções literalmente — seja claro e direto, não barulhento ou ameaçador.
  • Menos é mais: um prompt enxuto e honesto vence andaimes elaborados.
  • Os princípios se transferem entre modelos, mas reavalie a intensidade do tom e as pistas de formatação quando você trocar.
