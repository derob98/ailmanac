Tradução de Prompts entre IAs

Intermediário
What you'll learn
  • Saber quais fundamentos de prompting se transferem para qualquer modelo sem alteração
  • Identificar os poucos detalhes de superfície que mudam quando você troca de modelo ou de fornecedor
  • Executar um fluxo de trabalho portátil que leva um prompt para um novo modelo sem reescrevê-lo
  • Evitar a armadilha de pedir duas vezes a um modelo de raciocínio para pensar passo a passo

Boa notícia: os fundamentos de prompting se transferem para todos os modelos. Se você sabe fazer bons prompts para o Claude, sabe fazer bons prompts para o ChatGPT, o Gemini ou o Llama. Alguns detalhes de superfície diferem — aqui está a pedra de Roseta.

O que se transfere em todo lugar​

Estes funcionam em todos eles — têm a ver com comunicação, não com um fornecedor:

  • Dê contexto: objetivo, público, formato, tom (Fundamentos de Prompting).
  • Exemplos (few-shot).
  • Peça raciocínio em tarefas difíceis.
  • Restrinja a saída (tamanho, formato, "apenas JSON").
  • Itere em linguagem natural.
  • Papéis / prompts de sistema — todo modelo importante tem o conceito (Papéis).

Aprenda isto uma vez e você se torna portátil.

O que difere (e o que ajustar)​

AspectoObservação ao trocar
Sinais de estruturaO Claude adora tags XML; outros modelos as aceitam, mas podem preferir títulos em Markdown ou JSON.
Intensidade do tomModelos mais antigos toleravam prompts enfáticos/em CAIXA-ALTA; os mais novos, que seguem literalmente, preferem instruções calmas e específicas. Reajuste por modelo.
Raciocínio"Pense passo a passo" ajuda alguns; modelos de raciocínio dedicados fazem isso internamente — não duplique.
PrefillingColocar palavras na boca do assistente é uma alavanca no estilo Claude; o suporte varia em outros lugares.
Peso do prompt de sistemaTodos têm um, mas o quanto ele é seguido varia — teste.
Chamada de ferramentas/funçõesMesmo conceito, esquemas de requisição diferentes por fornecedor.

Um fluxo de trabalho portátil​

Guided walkthrough1 of 4
  1. Rascunhe o prompt com contexto + formato + exemplos — as partes que se transferem para qualquer modelo. Ainda não embuta sintaxe específica de fornecedor.
  2. Rode algumas entradas reais pelo modelo em que você realmente vai publicar (/docs/foundations/evals). Julgue as saídas, não sua intuição sobre o modelo.
  3. Se algo estiver errado, mude uma alavanca de superfície por vez — estilo de delimitador, intensidade do tom, se você pede raciocínio — não o prompt inteiro.
  4. Guarde os prompts como arquivos para que trocar de modelo seja um diff pequeno e revisável em vez de uma reescrita.

Como só a superfície muda, um único prompt bem estruturado normalmente migra com um ajuste de uma linha. Aqui está um escrito da forma universal — contexto, formato explícito e uma tarefa clara, sem sintaxe específica de fornecedor para remover:

Um prompt portátil e neutro em relação a fornecedores

You are a technical editor. Rewrite the text below to be clearer and more concise.

Constraints:
- Keep every fact unchanged.
- Target a general professional audience.
- Return only the rewritten text, no preamble.

Text:
"""
[paste the draft here]
"""

Esse prompt roda como está no Claude, no GPT, no Gemini ou em um modelo aberto. Se um modelo ignorar "retorne apenas o texto reescrito", esse é o seu único ajuste de superfície — aperte a restrição ou, no Claude, faça o prefill da resposta.

:::tip Não reescreva do zero Mudar para um novo modelo raramente exige um novo prompt — geralmente só um ajuste de tom/formato. Sua estrutura e seus exemplos se mantêm. :::

Teste você mesmo

0/3
  1. 1Qual destes NÃO se transfere entre modelos e precisa de reajuste quando você troca?
  2. 2Você move um prompt para um modelo de raciocínio dedicado. O que deve fazer com sua instrução 'pense passo a passo'?
  3. 3Qual é a forma recomendada de trocar um prompt para um novo modelo?
Key takeaways
  • Os fundamentos — contexto, exemplos, raciocínio, restrições de saída, papéis — transferem-se para qualquer modelo sem alteração.
  • Apenas seis detalhes de superfície diferem: sinais de estrutura, intensidade do tom, prompts de raciocínio, prefilling, peso do prompt de sistema e esquemas de ferramentas.
  • Trocar de modelo é um ajuste, não uma reescrita: mantenha a estrutura universal, teste no alvo, ajuste uma alavanca de superfície por vez.
  • Coloque seus prompts em controle de versão para que mover-se entre modelos seja um diff pequeno e revisável.
