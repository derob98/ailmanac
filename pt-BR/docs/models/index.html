AI Models & Assistants

Claude is our core — but the same skills travel. This section widens the lens to the whole AI world: the major assistants and models (ChatGPT, Gemini, Llama, Mistral, Grok, DeepSeek, open/local models), how they differ, when to use which, and the techniques that transfer across all of them.

TThe AI Model Landscape: Choosing Among Claude, ChatGPT, Gemini & Open Models

A durable framework for picking the right AI model for a task — across Claude, ChatGPT, Gemini, Llama, Mistral, Grok and the open/local models — plus what skills transfer across all of them.

CChatGPT for Claude Users

Fluent in Claude, need to use ChatGPT? The differences that matter, what transfers, and when to reach for which.

RRun AI Models Locally with Ollama

Run open-weight models (Llama, Mistral, Qwen…) on your own machine — private, offline, free to run — with Ollama. Setup, CLI, and calling it from code.

CClaude vs GPT vs Gemini for Coding

A framework (not a leaderboard) for choosing a frontier model for coding — the factors that matter, and why an eval on your own repo beats every benchmark.

GGemini for Claude Users

Fluent in Claude, need Google's Gemini? The differences that matter — Gems, Workspace integration, huge context, multimodal — and what transfers.

GGrok for Claude Users

Fluent in Claude, need xAI's Grok? The differences that matter — real-time X/web search, an OpenAI-compatible API, and the Grok Build coding agent — plus everything that transfers.

OOpen-Source AI Agent Frameworks

A provider-neutral map of the main open frameworks for building LLM agents — LangGraph, LlamaIndex, AutoGen, CrewAI and the minimal-loop approach — and how to choose.

PPorting Prompts Across Models

Move a prompt between Claude, GPT, Gemini and open models — what transfers cleanly, what to adjust per model, and a migration workflow.

DDeepSeek, Qwen & the Open-Weight Wave

Strong open-weight models (DeepSeek, Qwen, Llama, Mistral) closed much of the gap. What 'open weight' means, the trade-offs vs closed frontier, and how to use them.

GGenerative Media: Image, Audio & Video AI

Beyond text chat — a map of AI for images, video, voice and music: the major tools, the durable skills, and the rights/ethics that matter.

WWhat AI Actually Costs (Across Providers)

A durable framework for comparing AI cost across providers — the pricing archetypes, how to estimate a workload, the levers to cut the bill, and when open/local wins.

BBuilding Local AI Agents

Autonomous agents that run entirely on your machine with a local open-weight model — private, offline, free to run. The architecture, the trade-offs, and how to start.

CClaude + Local Models: Hybrid Patterns

Make Claude and local open-weight models work in synergy — router, draft-then-refine, privacy redaction, bulk pre-processing — for the reasoning of frontier with the privacy and cost of local.

LLocal Coding Agents (and How They Pair with Claude)

Coding agents that run on your machine and edit your repo — Aider, Cline, Continue, OpenHands, Claude Code — powered by Claude for quality or a local model for privacy.

CConnect Claude to Local Tools & Agents with MCP

The Model Context Protocol lets Claude orchestrate tools, data and agents running privately on your own machine — the Claude-as-brain, local-capabilities synergy. Add or build a local MCP server.

BBuild a Private Local AI Stack (End-to-End)

Tie it together: a local model (Ollama) + a model-agnostic agent + tools via a local MCP server — a private assistant on your own machine, with Claude as an optional smart layer.

LLocal Agent or Claude? A Decision Guide

Fully-local agent, Claude-powered, or hybrid? The factors that decide it — privacy, task difficulty, cost, latency, reliability — and why hybrid often wins.

SSecuring Local & Hybrid Agents

An agent that can edit files and run commands is powerful and dangerous. Least privilege, sandboxing, human approval, prompt-injection defense, budget caps — how to run agents safely.

RReasoning Models Compared

Thinking budgets and reasoning effort across Claude, GPT, Gemini, DeepSeek, and Qwen — when to spend tokens on thinking, and when not to.
