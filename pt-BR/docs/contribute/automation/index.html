Como Funciona Nossa Automação

Avançado

Página de transparência: aqui está a maquinaria que ajuda a manter o AILmanac atualizado e de alta qualidade com o mínimo de esforço dos mantenedores. O princípio que permeia tudo: a automação revela trabalho para que humanos aprovem — ela nunca publica silenciosamente afirmações não verificadas.

O portão de build (ativo)​

Toda alteração é compilada com onBrokenLinks: 'throw', então um link interno quebrado ou um MDX inválido faz o build falhar antes do merge. Esta é a verificação mais valiosa de todas — ela mantém a navegação confiável.

Workflows de CI​

Os arquivos de workflow ficam no branch ci-workflows e são habilitados assim que o repositório tem um token com o escopo workflow (veja o README). Até lá, o site é publicado a partir do branch gh-pages.

  • build-check — compila cada PR (links quebrados falham). Uma verificação obrigatória.
  • deploy — compila e publica no GitHub Pages a cada push para main.
  • link-check — o lychee roda semanalmente e nos PRs; em links externos quebrados ele abre uma issue de acompanhamento (rótulo link-rot) em vez de apenas ficar vermelho.

Automação de atualização (planejada / em fases)​

  • Monitor de releases — uma tarefa agendada que consulta a documentação/changelog da Anthropic e as superfícies de consumo que mudam mais rápido (apps, voz, mobile, marketplace). Em uma mudança, ele abre uma issue/PR rotulada com upstream-change + needs-verification apontando para as páginas provavelmente afetadas. Um humano escreve a atualização.
  • Verificador da tabela de modelos — compara data/models.json com a lista oficial de modelos e abre um PR quando algo diverge. A tabela de modelos mostra uma data de "última verificação".
  • Varredura de obsolescência — sinaliza páginas voláteis cujo lastVerified é mais antigo que um limite.

Por que humano no circuito​

:::tip Automatizado e honesto > automatizado e errado Uma IA que faz commit automático de "notícias" em um site de referência corre o risco de publicar alucinações — o oposto de ser confiável. Por isso nossa automação propõe (issues/PRs); humanos verificam e fazem merge. Veja Verificação de Fatos. :::

Quer ajudar com a automação?​

Melhorar esses workflows é uma ótima contribuição avançada — veja os arquivos em .github/workflows (no branch ci-workflows) e Contribuir.
