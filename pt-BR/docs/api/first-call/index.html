Sua Primeira Chamada à API

🟢Iniciante
✅
Última verificação: 2026-06-20.

O formato da API abaixo é estável, mas os IDs de modelo e as versões do SDK mudam. Veja a tabela de modelos atuais.

Confirme na fonte oficial.

A API permite que o seu programa converse com o Claude. Ao final desta lição, você terá executado uma requisição real e entenderá exatamente o que cada parte dela faz.

🎯What you'll learn
  • ✅Criar uma chave de API e armazená-la com segurança como uma variável de ambiente
  • ✅Instalar o SDK oficial para Python ou TypeScript (ou usar cURL puro)
  • ✅Enviar sua primeira requisição messages e ler a resposta do Claude
  • ✅Entender os quatro campos principais: model, max_tokens, messages e system

Toda a jornada em três passos​

Tudo abaixo se encaixa em uma sequência simples. Mantenha este mapa em mente conforme avança.

Guided walkthrough1 of 3
  1. Crie uma chave no Anthropic Console e exporte-a como uma variável de ambiente para que ela nunca fique no seu código.
  2. Adicione o pacote anthropic para Python ou @anthropic-ai/sdk para TypeScript. Com cURL não há nada para instalar.
  3. Envie uma lista de mensagens ao modelo e imprima o conteúdo que ele retorna.

1. Obtenha uma chave de API​

Crie uma no Anthropic Console. Depois, defina-a como uma variável de ambiente para que ela nunca fique no seu código:

✦

Exporte sua chave de API

export ANTHROPIC_API_KEY="sk-ant-..."
⚠️Watch out
  • •Nunca faça commit da sua chave. Mantenha as chaves em variáveis de ambiente ou em um gerenciador de segredos, nunca no controle de versão. Veja a página de Segurança (/docs/security).

2. Instale o SDK​

Escolha sua linguagem. O SDK lê a chave automaticamente da variável de ambiente que você acabou de definir.

  • Python
  • TypeScript
  • cURL
✦

Instalação (Python)

pip install anthropic
✦

Instalação (TypeScript)

npm install @anthropic-ai/sdk

Nada para instalar — você só precisa do curl.

3. Faça a chamada​

Toda requisição é uma lista de messages. O modelo responde com content. Execute o trecho de código para a sua linguagem e você verá o Claude responder à pergunta.

  • Python
  • TypeScript
  • cURL
import anthropic

client = anthropic.Anthropic()  # reads ANTHROPIC_API_KEY from the environment

message = client.messages.create(
    model="claude-sonnet-5",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "In one sentence, what is an API?"}
    ],
)

print(message.content[0].text)
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic(); // reads ANTHROPIC_API_KEY from the environment

const message = await client.messages.create({
  model: "claude-sonnet-5",
  max_tokens: 1024,
  messages: [
    { role: "user", content: "In one sentence, what is an API?" },
  ],
});

console.log(message.content[0].text);
curl https://api.anthropic.com/v1/messages \
  --header "x-api-key: $ANTHROPIC_API_KEY" \
  --header "anthropic-version: 2023-06-01" \
  --header "content-type: application/json" \
  --data '{
    "model": "claude-sonnet-5",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "In one sentence, what is an API?"}
    ]
  }'

O que acabou de acontecer​

Você enviou quatro informações. Veja o que cada uma controla:

  • model — qual Claude usar. Não codifique de forma fixa às cegas; veja Escolhendo um Modelo.
  • max_tokens — um limite para o tamanho da resposta (em tokens). Ele não define a janela de contexto.
  • messages — a conversa até o momento. A API é stateless: para continuar um chat, reenvie todo o histórico a cada vez.
  • system (opcional) — uma instrução de nível superior que define o papel do Claude para a chamada.

💡Key takeaways
  • ⭐Três passos: obtenha uma chave, instale o SDK, envie uma requisição messages.
  • ⭐Armazene a chave em ANTHROPIC_API_KEY para que o SDK a leia automaticamente e ela nunca entre no seu código.
  • ⭐Uma requisição é uma lista de mensagens; a resposta volta em content.
  • ⭐A API é stateless: reenvie todo o histórico da conversa para continuar um chat.
  • ⭐model, max_tokens, messages e system são os campos que você usará primeiro.
