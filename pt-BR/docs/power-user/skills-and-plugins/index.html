As Skills e Plugins Mais Úteis

Intermediário
What you'll learn
  • Saber a diferença entre uma skill e um plugin, e como cada um é instalado
  • Adicionar os marketplaces oficial, da comunidade e de demonstração, e instalar plugins a partir deles
  • Escolher os plugins oficiais da Anthropic certos para controle de versão, segurança, LSP e fluxos de git
  • Curar skills por tarefa a ser feita: desenvolvimento, escrita, pesquisa, dados, operações
  • Encontrar mais nas listas famosas da comunidade sem se afogar em forks de baixo sinal

A forma mais rápida de evoluir no Claude Code é parar de escrever tudo à mão e instalar capacidades que outras pessoas já aperfeiçoaram. As skills dão ao Claude expertise sob demanda; os plugins agrupam skills, agents, comandos, hooks e servidores MCP em uma única unidade versionada e instalável. Esta página é uma lista curta, curada e pronta para instalar — o que cada coisa faz, quando recorrer a ela e o comando exato para adicioná-la.

Se você ainda tem dúvidas sobre a mecânica, leia Skills e Plugins e Marketplaces primeiro — esta página presume que você sabe o que é um SKILL.md.

Última verificação: 2026-06-24.

Os nomes dos marketplaces, o catálogo de plugins oficiais e as contagens de estrelas dos repositórios da comunidade mudam com frequência. Os plugins executam código com os seus privilégios — instale apenas a partir de fontes em que você confia e confirme os comandos atuais na documentação oficial antes de depender deles.

Confirme na fonte oficial.

Skill vs plugin: o modelo de 20 segundos​

Uma skill é uma pasta com um SKILL.md (frontmatter description + instruções). O Claude a carrega apenas quando relevante, ou você a invoca com /skill-name. Ela fica em ~/.claude/skills/ (pessoal), no .claude/skills/ de um projeto, ou dentro de um plugin. Segundo a documentação oficial, os comandos de barra personalizados foram fundidos com as skills — um .claude/commands/deploy.md e um .claude/skills/deploy/SKILL.md ambos criam /deploy.

Um plugin é um pacote distribuível que pode entregar skills, agents, hooks, servidores MCP e servidores LSP juntos, com um manifesto .claude-plugin/plugin.json. Você obtém plugins de um marketplace — um catálogo que você adiciona uma vez e depois usa para instalar. As skills de plugins têm namespace, por exemplo /commit-commands:commit.

Adicione um marketplace, instale um plugin​

O marketplace oficial da Anthropic (claude-plugins-official) já vem disponível automaticamente. Dois outros valem a pena adicionar manualmente: o marketplace curado da comunidade e o marketplace de demonstração com plugins de exemplo.

Guided walkthrough1 of 4
  1. Execute /plugin marketplace add anthropics/claude-plugins-community e /plugin marketplace add anthropics/claude-code. Adicionar um marketplace apenas registra o catálogo — nada é instalado ainda.
  2. Execute /plugin para abrir o gerenciador (abas: Discover, Installed, Marketplaces, Errors). O painel de detalhes mostra uma estimativa de custo de contexto e uma lista 'Will install' de cada comando, skill, agent, hook e servidor — revise-a antes de instalar.
  3. Instale pela CLI com /plugin install <nome>@<marketplace>, ou escolha um escopo (User / Project / Local) na interface. O escopo Project compartilha o plugin com todos no repositório via .claude/settings.json.
  4. Execute /reload-plugins para captar novos plugins no meio da sessão sem reiniciar. Depois chame a skill com namespace, por exemplo /commit-commands:commit.

Configure seus marketplaces (execute dentro do Claude Code)

/plugin marketplace add anthropics/claude-plugins-community
/plugin marketplace add anthropics/claude-code
/plugin
Watch out
  • Plugins e marketplaces são componentes altamente confiáveis — eles podem executar código arbitrário com os seus privilégios. Adicione marketplaces e instale plugins apenas a partir de fontes em que você confia, e revise a lista 'Will install' primeiro.
  • As skills de plugins só aparecem após /reload-plugins (ou um reinício). Se uma skill ainda não aparecer, limpe o cache: rm -rf ~/.claude/plugins/cache, depois reinstale.

Plugins oficiais que valem a pena instalar primeiro​

Estes vêm no marketplace oficial da Anthropic (claude-plugins-official) e são o ponto de partida de maior sinal. Instale com /plugin install <nome>@claude-plugins-official.

Inteligência de código (LSP) — faça o Claude enxergar erros de tipo instantaneamente​

Os plugins LSP conectam um servidor de linguagem ao Claude Code, de modo que após cada edição o Claude recebe diagnósticos automáticos (erros de tipo, imports faltando) e navegação precisa (ir para definição, encontrar referências) em vez de palpites via grep. Eles exigem o binário do servidor de linguagem correspondente instalado na sua máquina.

  • Quando usar: qualquer sessão de programação não trivial em uma linguagem tipada. Este é provavelmente o maior upgrade de precisão para trabalho de desenvolvimento.
  • Contrapartida: servidores como rust-analyzer e pyright podem consumir bastante memória em repositórios grandes; desative se isso atrapalhar.

Instale LSP para a sua stack

/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls

Fluxo de git e PR​

  • commit-commands — fluxos de git: preparar (stage), gerar uma mensagem de commit, fazer push, abrir um PR. Use quando quiser commits limpos e consistentes sem digitar a cerimônia. As skills têm namespace, por exemplo /commit-commands:commit.
  • pr-review-toolkit — agents especializados para revisar pull requests. Use antes de fazer merge, ou para ter um segundo par de olhos sobre um diff.

Essenciais de Git/PR

/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins

Revisão de segurança automática​

  • security-guidance — revisa cada alteração que o Claude faz em busca de vulnerabilidades comuns e instrui o Claude a corrigi-las na mesma sessão. Use-o como uma rede de segurança permanente em qualquer projeto que escreve código; ele complementa (não substitui) uma revisão de verdade.

Passagem de segurança sempre ativa

/plugin install security-guidance@claude-plugins-official

Integrações externas (MCP, pré-configurado)​

Estes agrupam um servidor MCP pronto para uso, para que você pule a configuração manual. Recorra a um quando quiser que o Claude aja diretamente naquele serviço.

  • Controle de versão: github, gitlab
  • Gestão de projetos: linear, notion, asana, atlassian (Jira/Confluence)
  • Infra: vercel, firebase, supabase
  • Design / comunicação / monitoramento: figma, slack, sentry

Conecte o Claude às suas ferramentas

/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official

Toolkits faça-você-mesmo e estilos de saída​

  • plugin-dev — um toolkit para criar seus próprios plugins. Use no momento em que a sua configuração .claude/ valer a pena ser compartilhada.
  • agent-sdk-dev — ferramentas para construir sobre o Claude Agent SDK.
  • explanatory-output-style / learning-output-style — mudam como o Claude responde: explicações que ensinam à medida que avançam, ou um modo de aprendizado interativo. Ótimos para onboarding ou quando você quer realmente entender as mudanças.

Skills de alto valor por tarefa a ser feita​

As skills brilham em tarefas repetíveis e carregadas de contexto. A Anthropic publica um conjunto de referência público no repositório anthropics/skills — os mesmos padrões que dão poder às capacidades de documentos e criativas do Claude. Você pode copiar uma pasta de skill para ~/.claude/skills/ (pessoal) ou .claude/skills/ (projeto) e ela carrega na próxima sessão.

TarefaSkill (de anthropics/skills)Quando recorrer a ela
Documentosdocx, pdf, pptx, xlsxGere ou edite arquivos reais de Office/PDF — relatórios, apresentações, planilhas, formulários preenchíveis. (Implementações de referência com código disponível.)
Construir UIsweb-artifacts-builder / artifacts-builderProduza artefatos HTML/JS autocontidos e mini-apps.
Estender o Claudemcp-builderCrie a estrutura de um novo servidor MCP quando ainda não existir integração.
Marca e comunicaçãoskills de estilo brand-guidelines / internal-commsMantenha tom, nomenclatura e formatação alinhados à marca em todas as saídas.
Pro tip
  • Não despeje 30 skills no ~/.claude/skills de uma vez. As skills usam divulgação progressiva — apenas o nome + a descrição carregam até serem acionadas — mas uma pilha inchada e descrita de forma vaga torna o acionamento não confiável. Instale algumas, escreva descrições afiadas de "Use quando…" e cresça de forma deliberada.
  • A AILmanac traz pacotes de skills prontos que você pode adicionar para experimentar — veja /docs/templates/skills.

Uma skill autoacionável que você pode colar agora mesmo​

Para trabalho intensivo em pesquisa, uma skill que ancora o Claude em fontes frescas é melhor do que reexplicar o seu método toda vez. Coloque isto em ~/.claude/skills/research-brief/SKILL.md:

~/.claude/skills/research-brief/SKILL.md

---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.

Onde a comunidade vive (use as listas curadas, pule o ruído)​

Existem dezenas de repositórios "awesome"; a maioria são espelhos gerados automaticamente. Alguns são genuinamente curados e mantidos — comece por eles.

  • hesreallyhim/awesome-claude-code — o índice curado à mão mais conhecido de skills, hooks, comandos de barra, orquestradores de agents, aplicações e plugins. Dezenas de milhares de estrelas; ativamente reorganizado.
  • ComposioHQ/awesome-claude-skills — um grande catálogo curado de Claude Skills agrupado por função (documentos, desenvolvimento, dados, marketing, segurança, automação de SaaS).
  • VoltAgent/awesome-agent-skills — mais de 1.400 skills da comunidade + oficiais (Anthropic, Vercel, Stripe, Cloudflare, Sentry, Figma e outras), portáveis entre Claude Code, Cursor, Gemini CLI, Codex e outros.
  • anthropics/claude-plugins-community — o marketplace oficial de plugins da comunidade; plugins de terceiros que passaram pela validação automatizada e triagem de segurança da Anthropic.
Watch out
  • Contagens de estrelas e alegações de 'mais abrangente' nesses repositórios são voláteis e às vezes infladas por forks gerados por IA. Trate os números como aproximados, julgue pela manutenção e curadoria, e sempre revise o conteúdo real de uma skill ou plugin antes de instalar.
Key takeaways
  • Skill = uma capacidade sob demanda (SKILL.md); plugin = um pacote versionado (skills + agents + hooks + MCP/LSP) instalado a partir de um marketplace.
  • Adicione marketplaces com /plugin marketplace add <owner/repo>, instale com /plugin install <nome>@<marketplace>, depois /reload-plugins.
  • Maiores ganhos no primeiro dia: um plugin LSP para a sua linguagem, commit-commands, security-guidance e a integração MCP do serviço em que você vive.
  • As skills são portáveis e de divulgação progressiva — cure umas poucas e afiadas em vez de acumular centenas.
  • Confiança não é opcional: plugins e marketplaces executam código com os seus privilégios. Revise antes de instalar.
