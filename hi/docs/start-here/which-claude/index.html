मुझे कौन-सा Claude उपयोग करना चाहिए?

🟢शुरुआती

"Claude" कुछ रूपों में आता है। यह इस आधार पर चुनें कि आप क्या करने की कोशिश कर रहे हैं, न कि इस आधार पर कि आपने किसके बारे में सुना है।

🎯What you'll learn
  • ✅अपने लक्ष्य को सही Claude सतह से मिलाएँ: चैट, Claude Code, या API
  • ✅जानें कि तस्वीर में मोबाइल और आवाज़ कब फिट होते हैं
  • ✅समझें कि जैसे-जैसे आप आगे बढ़ते हैं, तीनों सतहें कैसे साथ मिलकर काम करती हैं
  • ✅जब आप निर्माण शुरू करें तो कौन-सा मॉडल चुनना है, इसका एक त्वरित अंदाज़ा पाएँ

30-सेकंड का निर्णय​

एक नज़र में तीन सतहें​

सतहकिसके लिए सर्वोत्तमकौनयहाँ से शुरू करें
Claude.aiलेखन, शोध, विश्लेषण, सीखना, योजना बनाना, रोज़मर्रा के सवालहर कोई, कोई सेटअप नहींClaude.ai के साथ शुरुआत करना
Claude Codeएक कोडबेस के भीतर काम करना — पढ़ना, संपादित करना, कमांड चलाना, टेस्ट ठीक करनाडेवलपर (और तकनीकी रूप से जिज्ञासु लोग)Claude Code क्या है
API और Agent SDKऐप्स, स्वचालन, और एजेंट जो प्रोग्रामेटिक रूप से Claude को कॉल करते हैंऐसे डेवलपर जो कोई उत्पाद या पाइपलाइन शिप कर रहे हैंआपका पहला API कॉल

Claude.ai — चैट ऐप्स​

Claude.ai हर किसी के लिए बिना किसी सेटअप वाला शुरुआती बिंदु है। आपको यह मोबाइल (iOS/Android) पर और आवाज़ द्वारा भी मिलता है — चलते-फिरते विचार दर्ज करने के लिए बढ़िया। इसे प्रोजेक्ट्स, कस्टम निर्देशों, और Artifacts से और सशक्त बनाएँ।

Claude Code — एजेंटिक कोडिंग उपकरण​

Claude Code आपके प्रोजेक्ट के भीतर काम करता है। यह पढ़ता है, संपादित करता है, कमांड चलाता है, और टेस्ट ठीक करता है — आपकी अनुमति से आपकी फ़ाइलों पर कार्य करते हुए।

API और Agent SDK — Claude को अपने सॉफ़्टवेयर में बनाएँ​

API और Agent SDK आपके अपने सॉफ़्टवेयर को प्रोग्रामेटिक रूप से Claude को कॉल करने देते हैं, ताकि आप AI सुविधाएँ, स्वचालन, और एजेंट शिप कर सकें।

ये साथ मिलकर काम करते हैं​

ये प्रतिद्वंद्वी उत्पाद नहीं हैं — अधिकांश लोग इनके बीच आगे बढ़ते हैं:

आप चाहते हैं…उपयोग करें
एक ईमेल का मसौदा बनाना, एक PDF सारांशित करना, विचार-मंथन करनाClaude.ai (या आवाज़/मोबाइल)
एक मॉड्यूल रिफैक्टर करना, टेस्ट जोड़ना, एक बग ठीक करनाClaude Code
आपके ऐप में एक AI सुविधा जोड़नाAPI / Agent SDK

:::tip तय नहीं कर पा रहे? चैट से शुरू करें Claude.ai को किसी सेटअप की ज़रूरत नहीं है और यह आपको सिखाता है कि Claude कैसे "सोचता" है। ये कौशल बाक़ी हर जगह काम आते हैं। :::

जब आप निर्माण करने लगें, तो कौन-सा मॉडल?​

एक सतह चुनना पहला कदम है। जब आप Claude Code या API की ओर बढ़ते हैं, तो आप एक मॉडल भी चुनते हैं — Haiku, Sonnet, या Opus। तीन त्वरित सवालों के जवाब दें और यह पिकर एक शुरुआती बिंदु सुझाता है:

काम की कठिनाई
सबसे ज़्यादा क्या मायने रखता है
रिक्वेस्ट वॉल्यूम
Claude Sonnet

संतुलित डिफ़ॉल्ट — Opus की लागत के एक अंश में मज़बूत तर्क और कोडिंग। यहीं से शुरू करें, और तभी ऊपर बढ़ें जब आप किसी असली गुणवत्ता सीमा तक पहुंचें।

सटीक मॉडल ID यहां देखें मॉडल टेबल. केवल एक मोटा अनुमान — पक्का करने के लिए अपने इनपुट पर एक त्वरित eval चलाएं।

:::note नामों को हार्ड-कोड न करें मॉडल लाइनअप और कीमतें बदलती रहती हैं। शिप करने से पहले हमेशा एक Claude मॉडल चुनना पेज पर वर्तमान मॉडल ID की पुष्टि करें। :::

  2. 2आपको एक वास्तविक प्रोजेक्ट के भीतर एक मॉड्यूल रिफैक्टर करना और विफल हो रहे टेस्ट ठीक करने हैं। कौन-सी सतह?

💡Key takeaways
  • ⭐Claude.ai: लेखन, शोध, और रोज़मर्रा के काम के लिए बिना सेटअप वाली चैट — मोबाइल पर और आवाज़ द्वारा भी
  • ⭐Claude Code: एक एजेंट जो आपके कोडबेस के भीतर कार्य करता है
  • ⭐API और Agent SDK: Claude को अपने स्वयं के सॉफ़्टवेयर में बनाएँ
  • ⭐ये एक-दूसरे के साथ जुड़ते हैं — अधिकांश लोग चैट से शुरू करते हैं और Code तथा API की ओर बढ़ते हैं
  • ⭐एक मॉडल (Haiku / Sonnet / Opus) तभी चुनें जब आप निर्माण करने लगें, और शिप करने से पहले वर्तमान ID सत्यापित करें
