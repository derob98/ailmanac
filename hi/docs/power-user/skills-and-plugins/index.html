सबसे उपयोगी Skills और Plugins

🟡मध्यम
🎯What you'll learn
  • ✅जानें कि एक skill और एक plugin में क्या अंतर है, और हर एक कैसे इंस्टॉल होता है
  • ✅आधिकारिक, सामुदायिक और डेमो marketplaces जोड़ें और उनसे plugins इंस्टॉल करें
  • ✅source control, security, LSP और git वर्कफ़्लो के लिए सही आधिकारिक Anthropic plugins चुनें
  • ✅job-to-be-done के अनुसार skills चुनें: dev, लेखन, शोध, डेटा, ops
  • ✅कम-सिग्नल वाले forks में डूबे बिना प्रसिद्ध सामुदायिक सूचियों से और खोजें

Claude Code को तेज़ी से बेहतर बनाने का सबसे आसान तरीका है हर चीज़ को हाथ से लिखना बंद करना और उन क्षमताओं को इंस्टॉल करना जिन्हें दूसरे लोग पहले ही परिपूर्ण कर चुके हैं। Skills, Claude को माँग पर विशेषज्ञता देते हैं; plugins, skills, agents, commands, hooks और MCP servers को एक संस्करण-युक्त, इंस्टॉल-योग्य इकाई में बंडल करते हैं। यह पेज एक चयनित, इंस्टॉल-के-लिए-तैयार शॉर्टलिस्ट है — हर चीज़ क्या करती है, कब इसे काम में लें, और इसे जोड़ने की सटीक कमांड।

अगर आप तंत्र को लेकर अस्पष्ट हैं, तो पहले Skills और Plugins & Marketplaces पढ़ें — यह पेज मानता है कि आप जानते हैं कि SKILL.md क्या होती है।

✅
आखिरी बार सत्यापित: 2026-06-24।

Marketplace के नाम, आधिकारिक plugins का कैटलॉग, और सामुदायिक-रिपॉज़िटरी के स्टार-काउंट बार-बार बदलते रहते हैं। Plugins आपके विशेषाधिकारों के साथ कोड चलाते हैं — केवल उन्हीं स्रोतों से इंस्टॉल करें जिन पर आप भरोसा करते हैं, और उन पर निर्भर रहने से पहले आधिकारिक डॉक्स में मौजूदा कमांड की पुष्टि करें।

इसके विरुद्ध पुष्टि करें आधिकारिक स्रोत.

Skill बनाम plugin: 20-सेकंड का मॉडल​

एक skill एक फ़ोल्डर है जिसमें एक SKILL.md होती है (frontmatter में description + निर्देश)। Claude इसे केवल तभी लोड करता है जब यह प्रासंगिक हो, या आप इसे /skill-name से इनवोक करते हैं। यह ~/.claude/skills/ (व्यक्तिगत), किसी प्रोजेक्ट के .claude/skills/, या किसी plugin के अंदर रहती है। आधिकारिक डॉक्स के अनुसार, कस्टम slash commands को skills में मिला दिया गया है — एक .claude/commands/deploy.md और एक .claude/skills/deploy/SKILL.md दोनों /deploy बनाते हैं।

एक plugin एक वितरण-योग्य बंडल है जो skills, agents, hooks, MCP servers और LSP servers को एक साथ शिप कर सकता है, जिसमें एक .claude-plugin/plugin.json manifest होती है। आप plugins किसी marketplace से प्राप्त करते हैं — एक कैटलॉग जिसे आप एक बार जोड़ते हैं, फिर उससे इंस्टॉल करते हैं। Plugin skills namespaced होती हैं, जैसे /commit-commands:commit।

एक marketplace जोड़ें, एक plugin इंस्टॉल करें​

आधिकारिक Anthropic marketplace (claude-plugins-official) स्वतः उपलब्ध है। दो अन्य को मैन्युअल रूप से जोड़ना सार्थक है: चयनित community marketplace और उदाहरण plugins का demo marketplace।

Guided walkthrough1 of 4
  1. /plugin marketplace add anthropics/claude-plugins-community और /plugin marketplace add anthropics/claude-code चलाएँ। किसी marketplace को जोड़ना केवल कैटलॉग रजिस्टर करता है — अभी तक कुछ भी इंस्टॉल नहीं होता।
  2. मैनेजर खोलने के लिए /plugin चलाएँ (टैब: Discover, Installed, Marketplaces, Errors)। डिटेल पैनल एक context-cost अनुमान और हर command, skill, agent, hook और server की 'Will install' सूची दिखाता है — इंस्टॉल करने से पहले इसकी समीक्षा करें।
  3. CLI से /plugin install <name>@<marketplace> के साथ इंस्टॉल करें, या UI में एक scope (User / Project / Local) चुनें। Project scope, .claude/settings.json के माध्यम से plugin को रिपॉज़िटरी पर मौजूद हर किसी के साथ साझा करता है।
  4. रीस्टार्ट किए बिना सत्र के बीच में नए plugins लेने के लिए /reload-plugins चलाएँ। फिर namespaced skill को कॉल करें, जैसे /commit-commands:commit।
✦

अपने marketplaces को बूटस्ट्रैप करें (Claude Code के अंदर चलाएँ)

/plugin marketplace add anthropics/claude-plugins-community
/plugin marketplace add anthropics/claude-code
/plugin
⚠️Watch out
  • •Plugins और marketplaces अत्यधिक भरोसे वाले घटक हैं — वे आपके विशेषाधिकारों के साथ मनमाना कोड निष्पादित कर सकते हैं। केवल उन्हीं स्रोतों से marketplaces जोड़ें और plugins इंस्टॉल करें जिन पर आप भरोसा करते हैं, और पहले 'Will install' सूची की समीक्षा करें।
  • •Plugin skills केवल /reload-plugins (या रीस्टार्ट) के बाद ही दिखती हैं। अगर कोई skill फिर भी नहीं दिखे, तो कैश साफ़ करें: rm -rf ~/.claude/plugins/cache, फिर पुनः इंस्टॉल करें।

पहले इंस्टॉल करने योग्य आधिकारिक plugins​

ये Anthropic के आधिकारिक marketplace (claude-plugins-official) में शिप होते हैं और सबसे उच्च-सिग्नल शुरुआती बिंदु हैं। /plugin install <name>@claude-plugins-official के साथ इंस्टॉल करें।

Code intelligence (LSP) — Claude को तुरंत type errors दिखाएँ​

LSP plugins एक language server को Claude Code में जोड़ते हैं, ताकि हर एडिट के बाद Claude को स्वचालित निदान (type errors, गायब imports) और सटीक नेविगेशन (go-to-definition, find references) मिले, बजाय grep के अनुमानों के। इनके लिए आपकी मशीन पर मिलान करने वाली language-server बाइनरी की आवश्यकता होती है।

  • कब उपयोग करें: किसी टाइप-युक्त भाषा में कोई भी गैर-तुच्छ कोडिंग सत्र। dev कार्य के लिए यह संभवतः सबसे बड़ा अकेला सटीकता-सुधार है।
  • समझौता: rust-analyzer और pyright जैसे servers बड़े रिपॉज़िटरीज़ पर बहुत सारी मेमोरी उपयोग कर सकते हैं; अगर यह परेशान करे तो अक्षम कर दें।
✦

अपने stack के लिए LSP इंस्टॉल करें

/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls

Git और PR वर्कफ़्लो​

  • commit-commands — git वर्कफ़्लो: stage करें, एक commit संदेश बनाएँ, push करें, एक PR खोलें। तब उपयोग करें जब आप औपचारिकता टाइप किए बिना साफ़, सुसंगत commits चाहते हों। Skills namespaced हैं, जैसे /commit-commands:commit।
  • pr-review-toolkit — pull requests की समीक्षा के लिए विशेष agents। merge करने से पहले उपयोग करें, या किसी diff पर दूसरी नज़र पाने के लिए।
✦

Git/PR के अनिवार्य

/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins

स्वचालित security समीक्षा​

  • security-guidance — Claude द्वारा किए गए हर परिवर्तन की सामान्य कमज़ोरियों के लिए समीक्षा करता है और Claude को उन्हें उसी सत्र में ठीक करने का निर्देश देता है। किसी भी कोड-लेखन प्रोजेक्ट पर इसे एक स्थायी सुरक्षा-जाल के रूप में उपयोग करें; यह असली समीक्षा का पूरक है (उसका विकल्प नहीं)।
✦

हमेशा-चालू security पास

/plugin install security-guidance@claude-plugins-official

बाहरी इंटीग्रेशन (MCP, पहले से कॉन्फ़िगर किए गए)​

ये एक तैयार MCP server को बंडल करते हैं ताकि आप मैन्युअल सेटअप छोड़ सकें। तब इनमें से किसी एक का सहारा लें जब आप चाहते हों कि Claude सीधे उस सेवा में कार्य करे।

  • Source control: github, gitlab
  • Project management: linear, notion, asana, atlassian (Jira/Confluence)
  • Infra: vercel, firebase, supabase
  • Design / comms / monitoring: figma, slack, sentry
✦

Claude को अपने टूल्स से जोड़ें

/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official

खुद-के-टूलकिट बनाएँ और output styles​

  • plugin-dev — अपने स्वयं के plugins लिखने के लिए एक टूलकिट। जिस क्षण आपका .claude/ सेटअप साझा करने लायक हो, इसका उपयोग करें।
  • agent-sdk-dev — Claude Agent SDK पर निर्माण के लिए टूल्स।
  • explanatory-output-style / learning-output-style — बदलें कि Claude कैसे जवाब देता है: चलते-चलते सिखाने वाली व्याख्याएँ, या एक इंटरैक्टिव लर्निंग मोड। ऑनबोर्डिंग के लिए शानदार, या जब आप परिवर्तनों को वास्तव में समझना चाहते हों।

job-to-be-done के अनुसार उच्च-मूल्य वाली skills​

Skills दोहराने योग्य, context-लोडेड कार्यों के लिए चमकती हैं। Anthropic anthropics/skills रिपॉज़िटरी में एक सार्वजनिक संदर्भ सेट प्रकाशित करता है — वही पैटर्न जो Claude की दस्तावेज़ और रचनात्मक क्षमताओं को शक्ति देते हैं। आप किसी skill फ़ोल्डर को ~/.claude/skills/ (व्यक्तिगत) या .claude/skills/ (प्रोजेक्ट) में कॉपी कर सकते हैं और यह अगले सत्र पर लोड हो जाती है।

JobSkill (anthropics/skills से)इसका सहारा कब लें
दस्तावेज़docx, pdf, pptx, xlsxअसली Office/PDF फ़ाइलें बनाएँ या संपादित करें — रिपोर्ट्स, decks, स्प्रेडशीट, भरने योग्य फ़ॉर्म। (स्रोत-उपलब्ध संदर्भ कार्यान्वयन।)
UIs बनाएँweb-artifacts-builder / artifacts-builderस्व-निहित HTML/JS artifacts और मिनी-ऐप्स बनाएँ।
Claude का विस्तार करेंmcp-builderजब अभी तक कोई इंटीग्रेशन मौजूद न हो तो एक नए MCP server का ढाँचा बनाएँ।
Brand और commsbrand-guidelines / internal-comms style skillsसभी आउटपुट्स में टोन, नामकरण और फ़ॉर्मैटिंग को ब्रांड के अनुरूप रखें।
✨Pro tip
  • →~/.claude/skills में एक साथ 30 skills मत डालें। Skills प्रगतिशील-प्रकटीकरण वाली हैं — ट्रिगर होने तक केवल नाम + description लोड होते हैं — लेकिन एक फूली हुई, अस्पष्ट रूप से वर्णित ढेर ट्रिगरिंग को अविश्वसनीय बना देती है। कुछ ही इंस्टॉल करें, तीखे "Use when…" descriptions लिखें, और सोच-समझकर बढ़ाएँ।
  • →AILmanac आज़माने के लिए तैयार-निर्मित skill packs शिप करता है जिन्हें आप डाल सकते हैं — देखें /docs/templates/skills।

एक स्व-ट्रिगरिंग skill जिसे आप अभी पेस्ट कर सकते हैं​

शोध-गहन कार्य के लिए, एक ऐसी skill जो Claude को ताज़ा स्रोतों में आधार देती है, हर बार अपनी विधि को फिर से समझाने से बेहतर है। इसे ~/.claude/skills/research-brief/SKILL.md में डालें:

✦

~/.claude/skills/research-brief/SKILL.md

---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.

समुदाय कहाँ रहता है (चयनित सूचियों का उपयोग करें, शोर छोड़ें)​

दर्जनों "awesome" रिपॉज़िटरीज़ हैं; अधिकांश स्वतः-उत्पन्न मिरर हैं। कुछ वास्तव में चयनित और अनुरक्षित हैं — वहीं से शुरू करें।

  • hesreallyhim/awesome-claude-code — skills, hooks, slash-commands, agent orchestrators, अनुप्रयोगों और plugins का सबसे प्रसिद्ध हाथ-से-चयनित सूचकांक। हज़ारों-हज़ार स्टार; सक्रिय रूप से पुनर्व्यवस्थित।
  • ComposioHQ/awesome-claude-skills — कार्य के अनुसार समूहीकृत Claude Skills का एक बड़ा चयनित कैटलॉग (दस्तावेज़, dev, डेटा, मार्केटिंग, security, SaaS automation)।
  • VoltAgent/awesome-agent-skills — 1,400+ सामुदायिक + आधिकारिक skills (Anthropic, Vercel, Stripe, Cloudflare, Sentry, Figma, और अधिक), जो Claude Code, Cursor, Gemini CLI, Codex और अन्य में पोर्टेबल हैं।
  • anthropics/claude-plugins-community — आधिकारिक सामुदायिक plugin marketplace; तृतीय-पक्ष plugins जो Anthropic की स्वचालित सत्यापन और सुरक्षा जाँच को पार कर गए।
⚠️Watch out
  • •इन रिपॉज़िटरीज़ पर स्टार-काउंट और 'सबसे व्यापक' दावे अस्थिर हैं और कभी-कभी AI-उत्पन्न forks द्वारा बढ़ा-चढ़ाकर दिखाए जाते हैं। संख्याओं को मोटा-मोटी मानें, रखरखाव और चयन से आँकें, और इंस्टॉल करने से पहले हमेशा किसी skill या plugin की वास्तविक सामग्री की समीक्षा करें।
💡Key takeaways
  • ⭐Skill = एक माँग-पर क्षमता (SKILL.md); plugin = एक संस्करण-युक्त बंडल (skills + agents + hooks + MCP/LSP) जो किसी marketplace से इंस्टॉल होता है।
  • ⭐/plugin marketplace add <owner/repo> से marketplaces जोड़ें, /plugin install <name>@<marketplace> से इंस्टॉल करें, फिर /reload-plugins।
  • ⭐पहले-दिन की सबसे बड़ी जीतें: आपकी भाषा के लिए एक LSP plugin, commit-commands, security-guidance, और जिस सेवा में आप रहते हैं उसके लिए MCP इंटीग्रेशन।
  • ⭐Skills पोर्टेबल और प्रगतिशील-प्रकटीकरण वाली हैं — सैकड़ों जमा करने के बजाय कुछ तीखी skills चुनें।
  • ⭐भरोसा वैकल्पिक नहीं है: plugins और marketplaces आपके विशेषाधिकारों के साथ कोड चलाते हैं। इंस्टॉल करने से पहले समीक्षा करें।
