Claude Code में MCP सर्वर

🔴उन्नत
✅
आखिरी बार सत्यापित: 2026-06-23।

claude mcp कमांड, कॉन्फ़िगरेशन स्कोप, और ट्रांसपोर्ट विकसित होते रहते हैं — आधिकारिक Claude Code MCP डॉक्स और modelcontextprotocol.io पर पुष्टि करें।

इसके विरुद्ध पुष्टि करें आधिकारिक स्रोत.

Model Context Protocol (MCP) AI को बाहरी टूल्स और डेटा से जोड़ने के लिए एक खुला मानक है। एक MCP सर्वर क्षमताएँ उजागर करता है (किसी डेटाबेस को क्वेरी करना, एक GitHub PR खोलना, किसी ब्राउज़र को चलाना); Claude Code इससे जुड़ता है और एक सत्र के दौरान उन टूल्स को कॉल कर सकता है। यह वह तरीका है जिससे आप Claude को अपने फ़ाइलसिस्टम और शेल से परे विस्तारित करते हैं।

🎯What you'll learn
  • ✅समझाएँ कि MCP सर्वर क्या है और Claude Code इसके टूल्स को कैसे कॉल करता है
  • ✅दो ट्रांसपोर्ट्स में अंतर करें: स्थानीय stdio बनाम रिमोट HTTP/SSE
  • ✅claude mcp add के साथ एक सर्वर जोड़ें और उसके द्वारा लिखे गए JSON को पढ़ें
  • ✅यह तय करने के लिए सही स्कोप (local, project, user) चुनें कि सर्वर किसे दिखता है
  • ✅एक वास्तविक डेटाबेस को Claude से एंड-टू-एंड कनेक्ट करें
  • ✅उन सुरक्षा और कॉन्फ़िगरेशन जालों से बचें जो अधिकांश लोगों को परेशान करते हैं

इसका स्वरूप​

आप उन सर्वरों की घोषणा करते हैं जिन्हें Claude उपयोग कर सकता है; प्रत्येक सर्वर स्कीमा के साथ टूल्स का एक सेट प्रकाशित करता है; Claude उन्हें किसी भी अन्य टूल की तरह चुनता और कॉल करता है।

ट्रांसपोर्ट्स​

Claude किसी सर्वर तक पहुँचने के दो तरीके हैं। सर्वर कहाँ चलता है उसके आधार पर चुनें।

  • stdio — एक स्थानीय प्रोसेस जिसे Claude लॉन्च करता है (स्थानीय टूल्स/CLI के लिए बढ़िया)।
  • रिमोट (HTTP/SSE) — एक होस्ट किया गया सर्वर, अक्सर OAuth के साथ।

सर्वर कॉन्फ़िगर करना​

सबसे तेज़ रास्ता claude mcp add कमांड है — यह आपके लिए कॉन्फ़िग लिख देता है। शून्य से एक कनेक्टेड सर्वर तक पहुँचने के लिए इस क्रम का पालन करें।

Guided walkthrough1 of 4
  1. claude mcp add चलाएँ — -- के बाद की हर चीज़ वह लॉन्च कमांड है जिसे Claude आपके लिए चलाता है।
  2. --transport http और एक स्कोप पास करें, फिर सर्वर URL। रिमोट सर्वर अक्सर होस्ट किए जाते हैं और OAuth का उपयोग करते हैं।
  3. कॉन्फ़िगर किए गए सर्वरों और उनकी कनेक्शन स्थिति देखने के लिए claude mcp list चलाएँ।
  4. किसी सर्वर के टूल्स का निरीक्षण करने और रिमोट सर्वरों को प्रमाणित करने के लिए किसी सत्र के अंदर /mcp का उपयोग करें।
✦

एक स्थानीय stdio सर्वर जोड़ें

# A local stdio server (everything after -- is the launch command)
claude mcp add github -- npx -y @modelcontextprotocol/server-github
✦

एक रिमोट HTTP सर्वर जोड़ें (प्रोजेक्ट के साथ साझा)

# A remote HTTP server, shared with everyone on the project
claude mcp add --transport http --scope project linear https://mcp.linear.app/mcp

पर्दे के पीछे वह बस JSON है। एक project-स्कोप वाला सर्वर रेपो रूट पर एक .mcp.json में आता है — इसे चेक इन करें और आपकी पूरी टीम को समान टूल्स मिल जाते हैं:

{
  "mcpServers": {
    "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"] }
  }
}

स्कोप तय करता है कि सर्वर किसे दिखता है​

स्कोपकहाँ रहता हैकिसके लिए उपयोग करें
local (डिफ़ॉल्ट)आपकी उपयोगकर्ता सेटिंग्स, केवल यह प्रोजेक्टव्यक्तिगत प्रयोग, सीक्रेट्स
projectरेपो में .mcp.json (कमिट किया हुआ)वे टूल्स जिन्हें पूरी टीम को साझा करना चाहिए
userआपकी उपयोगकर्ता सेटिंग्स, सभी प्रोजेक्ट्सवे सर्वर जिन्हें आप हर जगह चाहते हैं

जो कनेक्ट है उसे देखने के लिए claude mcp list चलाएँ और टूल्स का निरीक्षण करने तथा रिमोट सर्वरों को प्रमाणित करने के लिए किसी सत्र के अंदर /mcp चलाएँ। कॉपी-पेस्ट स्टार्टर्स के लिए MCP Config और सर्वर स्कैफ़ोल्ड्स देखें।

व्यावहारिक उदाहरण: Claude को अपना डेटाबेस दें​

मान लीजिए आप चाहते हैं कि क्वेरी परिणाम पेस्ट करने के बजाय Claude एक स्थानीय Postgres के विरुद्ध प्रश्नों का उत्तर दे। सर्वर जोड़ें (project स्कोप, ताकि टीम के साथी इसे विरासत में पाएँ):

✦

project स्कोप पर एक Postgres सर्वर जोड़ें

claude mcp add --scope project db -- npx -y @modelcontextprotocol/server-postgres "postgresql://localhost/app"

अब किसी सत्र में आप प्रश्न को सरल भाषा में पूछ सकते हैं और Claude को आपके लिए क्वेरी लूप करने दे सकते हैं:

✦

डेटाबेस के विरुद्ध एक प्रश्न पूछें

How many users signed up last week? Check the DB.

Claude सर्वर का query टूल कॉल करता है, पंक्तियाँ वापस पाता है, और उत्तर देता है — कोई कॉपी-पेस्ट लूप नहीं। चूँकि यह project-स्कोप वाला है, रेपो खींचने वाले एक टीम साथी को Claude Code खोलते ही वही क्षमता मिल जाती है। यदि आप केवल पठन चाहते हैं तो कनेक्शन स्ट्रिंग को केवल-पठन रखें।

विश्वास और सुरक्षा​

⚠️Watch out
  • •एक MCP सर्वर कोड चलाता है और डेटा पढ़ सकता है तथा कार्रवाइयाँ कर सकता है — केवल उन्हीं सर्वरों से जुड़ें जिन पर आप भरोसा करते हैं।
  • •प्रत्येक सर्वर को उतना ही न्यूनतम विशेषाधिकार दें जितना उसे चाहिए।
  • •कोई भी बाहरी सामग्री जो सर्वर लौटाता है, वह प्रॉम्प्ट इंजेक्शन ले जा सकती है।
  • •तृतीय-पक्ष सर्वरों से जुड़ने से पहले उनकी समीक्षा करें।

:::warning MCP सर्वरों को सॉफ़्टवेयर इंस्टॉल करने जैसा मानें एक MCP सर्वर कोड चलाता है और डेटा पढ़ सकता है और कार्रवाई कर सकता है। केवल उन्हीं सर्वरों से जुड़ें जिन पर आप भरोसा करते हैं, उन्हें आवश्यक न्यूनतम विशेषाधिकार दें, और याद रखें कि वे जो भी बाहरी सामग्री लौटाते हैं वह प्रॉम्प्ट इंजेक्शन ले जा सकती है। तृतीय-पक्ष सर्वरों की पहले समीक्षा करें — देखें तृतीय-पक्ष कोड की समीक्षा करना। :::

ऐप्स में भी MCP​

MCP Claude ऐप्स में Connectors को भी शक्ति प्रदान करता है — वही मानक, अलग सतह। देखें ऐप्स में Connectors (MCP) और, API के लिए, MCP और टूल्स से कनेक्ट करना।

आम गलतियाँ​

  • गलत स्कोप। local स्कोप पर जोड़ा गया सर्वर टीम के साथियों के लिए नहीं दिखेगा; जो आप केवल अपने लिए चाहते थे उसे project स्कोप पर कमिट नहीं किया जाना चाहिए। सोच-समझकर चुनें।
  • बहुत सारे सर्वर, बहुत सारे टूल्स। प्रत्येक जुड़ा सर्वर अपने टूल स्कीमा को संदर्भ में जोड़ता है। कार्य को जो चाहिए उसे जोड़ें, अपना पूरा कैटलॉग नहीं।
  • अति-विशेषाधिकार प्राप्त कनेक्शन। किसी डेटाबेस सर्वर को एक केवल-पठन भूमिका दें जब तक कि Claude को सचमुच लिखने की ज़रूरत न हो। MCP क्षमताओं को वास्तविक बनाता है — उन्हें संकीर्ण करें।
  • इंजेक्शन जोखिम को भूलना। सर्वर जो भी लौटाता है (एक वेब पेज, एक issue बॉडी, एक पंक्ति) वह अविश्वसनीय टेक्स्ट है जो प्रॉम्प्ट इंजेक्शन ले जा सकता है। सोचे-समझे बिना एक शक्तिशाली write-सक्षम सर्वर को किसी अविश्वसनीय read-सक्षम सर्वर के बगल में न जोड़ें।

💡Key takeaways
  • ⭐MCP एक खुला मानक है; एक MCP सर्वर टूल्स उजागर करता है जिन्हें Claude Code किसी भी अन्य टूल की तरह कॉल करता है।
  • ⭐दो ट्रांसपोर्ट: स्थानीय stdio (एक प्रोसेस जिसे Claude लॉन्च करता है) और रिमोट HTTP/SSE (होस्ट किया हुआ, अक्सर OAuth)।
  • ⭐claude mcp add आपके लिए कॉन्फ़िग लिख देता है; पर्दे के पीछे यह JSON है, और project स्कोप एक कमिट की गई .mcp.json में रहता है।
  • ⭐स्कोप दृश्यता को नियंत्रित करता है: local (आप, यह प्रोजेक्ट), project (टीम के लिए कमिट किया हुआ), user (आप, हर जगह)।
  • ⭐सर्वरों को सॉफ़्टवेयर इंस्टॉल करने जैसा मानें: विश्वास, न्यूनतम विशेषाधिकार, और वे जो भी लौटाते हैं उसमें प्रॉम्प्ट इंजेक्शन पर नज़र रखें।
