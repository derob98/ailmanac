आउटपुट स्टाइल्स

🟡मध्यम
✅
आखिरी बार सत्यापित: 2026-06-21।

बिल्ट-इन स्टाइल्स, सटीक स्विचिंग प्रणाली, और कस्टम-स्टाइल फ़ाइल फ़ॉर्मेट रिलीज़ के बीच बदल सकते हैं। आधिकारिक डॉक्स में पुष्टि करें (Claude Code डॉक्स docs.anthropic.com से code.claude.com पर चले गए हैं)।

इसके विरुद्ध पुष्टि करें आधिकारिक स्रोत.
🎯What you'll learn
  • ✅आउटपुट स्टाइल क्या है — और यह Claude जो जानता है उसे बदलने से कैसे अलग है
  • ✅चार बिल्ट-इन स्टाइल्स: Default, Proactive, Explanatory, Learning
  • ✅/config के साथ स्टाइल्स कैसे स्विच करें — और बदलाव के लिए नया सेशन क्यों ज़रूरी है
  • ✅अपनी खुद की कस्टम स्टाइल एक Markdown फ़ाइल के रूप में कैसे लिखें
  • ✅आउटपुट स्टाइल बनाम CLAUDE.md बनाम Claude.ai कस्टम निर्देश — किसे कब चुनें

आउटपुट स्टाइल्स यह बदलते हैं कि Claude Code कैसे संवाद करता है — इसकी भूमिका, स्वर, विस्तार, और क्या यह अपने तर्क को समझाता है — सीधे सिस्टम प्रॉम्प्ट को संशोधित करके। ये बदलते हैं कि Claude कैसे जवाब देता है, न कि यह क्या जानता है। जब आप हर बार एक ही आवाज़ या फ़ॉर्मेट के लिए बार-बार प्रॉम्प्ट करते रहें, तो किसी एक का उपयोग करें।

इन्हें क्यों उपयोग करें​

  • अपने वर्कफ़्लो से मेल खाएं। जब आप तेज़ी से आगे बढ़ रहे हों तो संक्षिप्त और एक्शन-फ़र्स्ट; जब आप सीख रहे हों तो व्याख्यात्मक।
  • शिक्षण मोड। कुछ स्टाइल्स शैक्षिक टिप्पणी जोड़ते हैं — जब आप किसी कोडबेस या तकनीक को जान रहे हों तो बहुत उपयोगी।
  • स्थिरता। एक बार पसंदीदा स्टाइल सेट करें ताकि हर सेशन वैसा ही पढ़े जैसा आपको पसंद है।

बिल्ट-इन स्टाइल्स​

Claude Code चार के साथ आता है:

स्टाइलयह क्या करता है
Defaultमानक सॉफ़्टवेयर-इंजीनियरिंग सिस्टम प्रॉम्प्ट — कुशल, एक्शन-फ़र्स्ट।
Proactiveतुरंत निष्पादित करता है और नियमित निर्णयों के लिए रुकने के बजाय उचित अनुमान लगाता है। ऑटो मोड की तुलना में मज़बूत स्वायत्त-निष्पादन मार्गदर्शन, लेकिन टूल्स चलने से पहले भी आपको अनुमति प्रॉम्प्ट दिखते हैं।
Explanatoryचरणों के बीच शैक्षिक "Insights" जोड़ता है, कार्यान्वयन विकल्पों और कोडबेस पैटर्न को समझाता है।
Learningसहयोगात्मक, करके-सीखें: Insights साझा करता है और आपसे छोटे हिस्से खुद लिखने को कहता है, आपके कोड में [TODO(human)] मार्कर छोड़ता है।
⚠️Watch out
  • •Explanatory और Learning डिज़ाइन के अनुसार लंबे जवाब उत्पन्न करते हैं, इसलिए इनकी आउटपुट टोकन लागत Default से अधिक होती है।
Term shown.
1 / 4

कैसे स्विच करें​

/config चलाएं और Output style के तहत एक स्टाइल चुनें। आपका चयन प्रोजेक्ट स्तर पर .claude/settings.local.json में सहेजा जाता है। मेनू के बिना इसे सेट करने के लिए, किसी भी सेटिंग्स फ़ाइल में outputStyle फ़ील्ड संपादित करें।

Guided walkthrough1 of 4
  1. अपने सेशन में /config चलाएं और Output style विकल्प ढूंढें।
  2. Default, Proactive, Explanatory, या Learning में से एक चुनें। आपका चयन प्रोजेक्ट स्तर पर .claude/settings.local.json में सहेजा जाता है।
  3. किसी भी सेटिंग्स फ़ाइल में outputStyle फ़ील्ड संपादित करके मेनू को छोड़ दें (नीचे दिया गया JSON देखें)।
  4. स्टाइल सिस्टम प्रॉम्प्ट का हिस्सा है, जिसे सेशन शुरू होने पर एक बार पढ़ा जाता है — इसके प्रभावी होने के लिए /clear चलाएं या एक नया सेशन खोलें।
{
  "outputStyle": "Explanatory"
}

:::warning बदलावों के लिए नया सेशन चाहिए आउटपुट स्टाइल सिस्टम प्रॉम्प्ट का हिस्सा है, जिसे Claude Code सेशन शुरू होने पर एक बार पढ़ता है। कोई बदलाव केवल /clear या नए सेशन के बाद ही प्रभावी होता है। (स्टैंडअलोन /output-style कमांड को v2.1.91 में हटा दिया गया था — /config या ऊपर दी गई सेटिंग का उपयोग करें।) :::

एक कस्टम स्टाइल बनाएं​

एक कस्टम स्टाइल एक Markdown फ़ाइल है: फ़्रंटमैटर, फिर सिस्टम प्रॉम्प्ट में जोड़ने के लिए निर्देश। इसे यूज़र स्तर (~/.claude/output-styles/) या प्रोजेक्ट स्तर (.claude/output-styles/) पर सहेजें; फ़ाइल का नाम स्टाइल का नाम बन जाता है जब तक कि आप name सेट न करें।

✦

✨Pro tip
  • →जब आप केवल यह बदल रहे हों कि Claude कैसे संवाद करता है लेकिन फिर भी चाहते हों कि वह सामान्य रूप से कोडिंग करे, तो keep-coding-instructions: true सेट करें।
  • →जब Claude सॉफ़्टवेयर इंजीनियरिंग कर ही नहीं रहा हो — मान लीजिए, एक शुद्ध लेखन या डेटा-विश्लेषण सहायक — तो इसे छोड़ दें (डिफ़ॉल्ट) ताकि बिल्ट-इन कोडिंग निर्देश हटा दिए जाएं।

आउटपुट स्टाइल्स बनाम CLAUDE.md बनाम अन्य लीवर​

  • आउटपुट स्टाइल = भूमिका, स्वर, और फ़ॉर्मेट के लिए एक सिस्टम-प्रॉम्प्ट प्रीसेट — हर जवाब पर लागू होता है।
  • CLAUDE.md = स्थायी प्रोजेक्ट तथ्य और नियम (परंपराएं, गार्डरेल), सिस्टम प्रॉम्प्ट के बाद एक संदेश के रूप में जोड़े गए।
  • कस्टम निर्देश / रिस्पॉन्स स्टाइल्स = Claude.ai उपभोक्ता-ऐप का समकक्ष।

:::tip सही लीवर का उपयोग करें चाहते हैं कि Claude "हमेशा समाप्त करने से पहले टेस्ट चलाए"? वह एक CLAUDE.md गार्डरेल है, स्टाइल नहीं। चाहते हैं कि यह "जैसे-जैसे आगे बढ़े वैसे-वैसे हर चरण समझाए"? वह एक आउटपुट स्टाइल है। व्यवहार के नियम CLAUDE.md में और प्रस्तुति स्टाइल्स में रखें। :::

💡Key takeaways
  • ⭐आउटपुट स्टाइल्स यह बदलती हैं कि Claude कैसे संवाद करता है (भूमिका, स्वर, विस्तार), न कि यह क्या जानता है — ये सिस्टम प्रॉम्प्ट को संपादित करती हैं।
  • ⭐चार बिल्ट-इन: Default (एक्शन-फ़र्स्ट), Proactive (अनुमानों के साथ निष्पादित करता है), Explanatory (Insights जोड़ता है), Learning (Insights + [TODO(human)] मार्कर)।
  • ⭐Explanatory और Learning अधिक आउटपुट टोकन खर्च करती हैं क्योंकि जवाब डिज़ाइन के अनुसार लंबे होते हैं।
  • ⭐/config के माध्यम से स्विच करें (.claude/settings.local.json में सहेजा गया) या outputStyle सेटिंग के माध्यम से — बदलावों के लिए /clear या नया सेशन चाहिए।
  • ⭐कस्टम स्टाइल्स ~/.claude/output-styles/ या .claude/output-styles/ में Markdown फ़ाइलें हैं; कोडिंग व्यवहार बनाए रखने के लिए keep-coding-instructions: true का उपयोग करें।
  • ⭐व्यवहार के नियमों/गार्डरेल के लिए CLAUDE.md और प्रस्तुति के लिए आउटपुट स्टाइल्स का उपयोग करें।
