أنماط المخرَجات

متوسط
آخر تحقّق: 2026-06-21.

يمكن أن تتغير الأنماط المدمجة، وآلية التبديل الدقيقة، وتنسيق ملف النمط المخصص بين الإصدارات. تأكّد من ذلك في الوثائق الرسمية (انتقلت وثائق Claude Code من docs.anthropic.com إلى code.claude.com).

تأكّد من المصدر الرسمي.
What you'll learn
  • ما هو نمط المخرَجات — وكيف يختلف عن تغيير ما يعرفه Claude
  • الأنماط الأربعة المدمجة: Default وProactive وExplanatory وLearning
  • كيفية تبديل الأنماط باستخدام /config — ولماذا يحتاج التغيير إلى جلسة جديدة
  • كيفية تأليف نمطك المخصص كملف Markdown
  • متى تلجأ إلى نمط مخرَجات مقابل CLAUDE.md مقابل التعليمات المخصصة في Claude.ai

تغيّر أنماط المخرَجات كيفية تواصل Claude Code — دوره، ونبرته، وإسهابه، وما إذا كان يشرح استدلاله — عن طريق تعديل موجّه النظام مباشرةً. إنها تغيّر كيف يستجيب Claude، لا ما يعرفه. الجأ إلى أحدها عندما تجد نفسك تعيد المطالبة بالصوت أو التنسيق نفسه في كل دور.

لماذا تستخدمها​

  • طابِق سير عملك. مختصر وموجّه نحو الفعل عندما تتحرك بسرعة؛ شرحيّ عندما تتعلم.
  • وضع التعليم. تضيف بعض الأنماط تعليقات تعليمية — رائع بينما تتعرّف على قاعدة شيفرة أو تقنية.
  • الاتساق. اضبط النمط المفضّل مرة واحدة لتقرأ كل جلسة بالطريقة التي تحبها.

الأنماط المدمجة​

يأتي Claude Code مزوّداً بأربعة:

النمطماذا يفعل
Defaultموجّه النظام القياسي لهندسة البرمجيات — فعّال، وموجّه نحو الفعل.
Proactiveينفّذ فوراً ويضع افتراضات معقولة بدلاً من التوقّف عند القرارات الروتينية. توجيه أقوى للتنفيذ المستقل من الوضع التلقائي، لكنك لا تزال ترى مطالبات الإذن قبل تشغيل الأدوات.
Explanatoryيضيف "رؤى" تعليمية بين الخطوات، تشرح خيارات التنفيذ وأنماط قاعدة الشيفرة.
Learningتعاونيّ، تعلّم بالممارسة: يشارك الرؤى ويطلب منك كتابة أجزاء صغيرة بنفسك، تاركاً علامات [TODO(human)] في شيفرتك.
Watch out
  • يُنتج Explanatory وLearning استجابات أطول بحكم التصميم، فهما يكلّفان رموز مخرَجات أكثر من Default.
اضغط Enter أو مفتاح المسافة لقلب البطاقة. استخدم مفتاحي السهمين الأيسر والأيمن للتنقل بين البطاقات.تم إظهار المصطلح.

كيفية التبديل​

شغّل /config واختر نمطاً ضمن Output style. يُحفظ اختيارك في .claude/settings.local.json على مستوى المشروع. لضبطه دون القائمة، عدّل الحقل outputStyle في أي ملف إعدادات.

Guided walkthrough1 of 4
  1. شغّل /config في جلستك وابحث عن خيار Output style.
  2. اختر أحد Default أو Proactive أو Explanatory أو Learning. يُحفظ اختيارك في .claude/settings.local.json على مستوى المشروع.
  3. تجاوز القائمة بتعديل الحقل outputStyle في أي ملف إعدادات (انظر JSON أدناه).
  4. النمط جزء من موجّه النظام، يُقرأ مرة واحدة عند بدء الجلسة — شغّل /clear أو افتح جلسة جديدة لكي يسري.
{
  "outputStyle": "Explanatory"
}

:::warning التغييرات تحتاج إلى جلسة جديدة نمط المخرَجات جزء من موجّه النظام، الذي يقرؤه Claude Code مرة واحدة عند بدء الجلسة. لا يسري التغيير إلا بعد /clear أو جلسة جديدة. (أُزيل الأمر المستقل /output-style في v2.1.91 — استخدم /config أو الإعداد أعلاه.) :::

إنشاء نمط مخصص​

النمط المخصص هو ملف Markdown: مقدّمة (frontmatter)، ثم التعليمات المراد إلحاقها بموجّه النظام. احفظه على مستوى المستخدم (~/.claude/output-styles/) أو مستوى المشروع (.claude/output-styles/)؛ يصبح اسم الملف اسم النمط ما لم تضبط name.

مثال على نمط مخصص — المخططات أولاً

---
name: Diagrams first
description: Lead every explanation with a diagram
keep-coding-instructions: true
---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.
Pro tip
  • اضبط keep-coding-instructions: true عندما تكون فقط بصدد تغيير كيفية تواصل Claude لكنك لا تزال تريده يبرمج بشكل طبيعي.
  • اتركه دون تعيين (الوضع الافتراضي) عندما لا يقوم Claude بهندسة البرمجيات على الإطلاق — مثلاً، مساعد كتابة بحت أو مساعد لتحليل البيانات — حتى تُسقَط تعليمات البرمجة المدمجة.

أنماط المخرَجات مقابل CLAUDE.md مقابل الروافع الأخرى​

  • نمط المخرَجات = إعداد مُسبق لموجّه النظام يخص الدور والنبرة والتنسيق — ينطبق على كل استجابة.
  • CLAUDE.md = حقائق وقواعد المشروع الدائمة (الأعراف، الحواجز الوقائية)، مُضافة كرسالة بعد موجّه النظام.
  • التعليمات المخصصة / أنماط الاستجابة = النظير في تطبيق Claude.ai الاستهلاكي.

:::tip استخدم الرافعة الصحيحة تريد أن "يشغّل Claude الاختبارات دائماً قبل الانتهاء"؟ هذا حاجز وقائي في CLAUDE.md، وليس نمطاً. تريد أن "يشرح كل خطوة أثناء سيره"؟ هذا نمط مخرَجات. أبقِ قواعد السلوك في CLAUDE.md والعرض في الأنماط. :::

Key takeaways
  • تغيّر أنماط المخرَجات كيفية تواصل Claude (الدور والنبرة والإسهاب)، لا ما يعرفه — فهي تعدّل موجّه النظام.
  • أربعة أنماط مدمجة: Default (موجّه نحو الفعل)، وProactive (ينفّذ مع افتراضات)، وExplanatory (يضيف رؤى)، وLearning (رؤى + علامات [TODO(human)]).
  • يكلّف Explanatory وLearning رموز مخرَجات أكثر لأن استجاباتهما أطول بحكم التصميم.
  • بدّل عبر /config (يُحفظ في .claude/settings.local.json) أو الإعداد outputStyle — تحتاج التغييرات إلى /clear أو جلسة جديدة.
  • الأنماط المخصصة ملفات Markdown في ~/.claude/output-styles/ أو .claude/output-styles/؛ استخدم keep-coding-instructions: true للحفاظ على سلوك البرمجة.
  • استخدم CLAUDE.md لقواعد السلوك/الحواجز الوقائية وأنماط المخرَجات للعرض.
