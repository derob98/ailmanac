وكلاء البرمجة المحليون (وكيف يتكاملون مع Claude)

متوسط

وكيل البرمجة المحلي برنامج يعمل على جهازك أنت، يقرأ مستودعك، ويحرّر ملفاتك نيابةً عنك. تصف التغيير بلغة طبيعية؛ فيخطّط له الوكيل، ويكتب الكود عبر عدّة ملفات، ويشغّل اختباراتك، ويقرأ الإخفاقات، ثم يحاول مجدّدًا — حلقة، لا إجابة واحدة. مساعد الدردشة في تبويب المتصفّح يعطيك مقتطفًا لتلصقه؛ أما وكيل البرمجة فيقوم باللصق والتشغيل والإصلاح بنفسه، داخل مستودعك.

الفكرة المحورية التي تجعل هذه الصفحة جديرة بالقراءة: جميع هذه الأدوات تقريبًا محايدة تجاه النموذج. فالوكيل (الحلقة التي تلمس ملفاتك) منفصل عن النموذج (العقل الذي يقرّر ما يُكتب). فيمكن توجيه الأداة نفسها إلى Claude لأفضل جودة كود، أو إلى نموذج محلي مفتوح الأوزان عبر Ollama كي لا يغادر كودك حاسوبك المحمول أبدًا. تختار الوكيل بحسب مدى ملاءمته لسير عملك، وتختار النموذج بحسب مفاضلة الجودة مقابل الخصوصية — كلٌّ على حدة.

What you'll learn
  • فهم نموذج وكيل البرمجة الأصيل: حلقة وكيل تقرأ مستودعك، وتخطّط، وتحرّر الملفات، وتشغّل الاختبارات، وتكرّر
  • معرفة الأدوات المفتوحة الحقيقية — Aider وCline وContinue وOpenHands — وكيف يندرج وكيل Anthropic الخاص Claude Code
  • إدراك سبب أهمية 'الحياد تجاه النموذج': وجّه وكيلًا واحدًا إلى Claude من أجل الجودة أو إلى نموذج محلي من أجل الخصوصية
  • فهم زاوية الخصوصية — النموذج المحلي يعني أن كودك لا يغادر جهازك أبدًا
  • الاختيار على محورين: مدمج في المحرّر مقابل سطر الأوامر مقابل مستقلّ، ومدعوم بـ Claude مقابل محلي بالكامل

النموذج الأصيل: حلقة وكيل، لا صندوق دردشة​

كل أداة في هذه الصفحة تنويعٌ على الحلقة نفسها. وحين ترى الحلقة، تصبح كل أداة مجرّد شكل مختلف حولها:

  1. اقرأ — يجمع الوكيل السياق: الملفات التي توجّهه إليها، وبنية المستودع، وأحيانًا مخرجات الاختبار أو سجلات الأخطاء.
  2. خطّط — يقرّر ما يغيّره وفي أي ملفات، وغالبًا يقترح خطّة قبل أن يلمس أي شيء.
  3. حرّر — يكتب التغييرات إلى ملفات حقيقية على القرص (تُعرض عادةً كفرق تُوافق عليه).
  4. شغّل — ينفّذ أوامر: مجموعة اختباراتك، أو بناءً، أو مدقّق أسلوب، أو سكربتًا.
  5. راقِب وكرّر — يقرأ ما حدث (هل نجحت الاختبارات؟ أم ظهر أثر مكدّس؟) ويعود إلى الخطوة 2 إلى أن تكتمل المهمة أو يعيد التحكّم إليك.

هذا هو الشكل نفسه لـاستخدام الأدوات الوكيلي: يُمنح النموذج أدوات (اقرأ ملفًا، حرّر ملفًا، شغّل أمرًا) ويقرّر متى يستدعيها. والفارق بين الأدوات هو في الغالب مقدار الاستقلالية التي تملكها الحلقة ومكان وجودها — طرفيّتك، أو محرّرك، أو بيئة معزولة تعمل بمفردها.

النموذج في وسط تلك الحلقة قابل للتبديل. وهذا هو السبب الكامل في أن وكيلًا واحدًا يستطيع خدمة نوعين مختلفين تمامًا من المستخدمين: من يريد أقوى كود ممكن (وجّهه إلى Claude)، ومن لا يستطيع قانونيًّا السماح للكود بمغادرة شبكته (وجّهه إلى نموذج محلي).

لماذا الحياد تجاه النموذج هو الميزة الأبرز​

الوكيل والنموذج قطعتان منفصلتان. الوكيل يملك الحلقة — قراءة الملفات، وتطبيق الفروق، وتشغيل الاختبارات. والنموذج يملك الحُكم — ماذا يُكتب، ولماذا، ومتى يتوقّف. وفصلهما يمنحك قرصَي ضبط مستقلّين:

  • قرص الجودة → Claude. للاستدلال الصعب، وإعادة الهيكلة عبر ملفات متعدّدة، والعمل الطويل السياق، ينتج نموذج رائد مثل Claude تحريرات أفضل بوضوح ويتعافى من الإخفاقات بموثوقية أعلى. ومعظم هذه الأدوات توثّق Claude كخيار موصى به أو افتراضي. (أيّ مستوى من Claude تختار قرارٌ قائم بذاته — انظر اختيار نموذج.)
  • قرص الخصوصية → نموذج محلي. وجّه الوكيل نفسه إلى نموذج مفتوح الأوزان يعمل تحت Ollama، وكودك وأوامرك وفروقك لا تغادر جهازك أبدًا — لا استدعاء API، ولا طرف ثالث، ولا شيء على السلك. وهذا هو العامل الحاسم لقواعد الكود الخاضعة للتنظيم، أو المعزولة عن الشبكة، أو السرّية. المفاضلة الصادقة: نموذجٌ صغير بما يكفي ليعمل على حاسوب محمول ليس في مصافّ نموذج رائد في أصعب المهام. أنت تقايض بعض القدرة مقابل تحكّم كامل في البيانات.
What you'll learn
  • الحياد تجاه النموذج لا يعني أن كل نموذج جيّد بالقدر نفسه. تعمل الحلقة على الشكل ذاته، لكن نموذجًا ضعيفًا يجري تحريرات أسوأ، ويسيء قراءة إخفاقات الاختبار، ويدور في حلقات عقيمة. تتفاضل الخصوصية والقدرة فعلًا — فاختر بحسب المهمة، لا مرّة واحدة إلى الأبد.

الأدوات الحقيقية (مُحقَّقة)​

خمس أدوات تستحق المعرفة. الأربع الأولى مفتوحة المصدر ومحايدة تجاه النموذج؛ والخامسة هي وكيل Anthropic الخاص، أُدرِج كي ترى أين تقع تجربة أصيلة مبنية على Claude.

Aider — رفيق البرمجة في سطر الأوامر​

Aider يعمل في طرفيّتك وبُني حول git: يحرّر الملفات في مستودعك ويودِع تلقائيًّا كل تغيير برسالة معقولة، فيكون كل تحرير للذكاء الاصطناعي فرقًا يمكنك مراجعته أو التراجع عنه أو الرجوع منه. وهو محايد تجاه النموذج — يعمل على أفضل وجه مع النماذج الرائدة مثل Claude، ويتّصل أيضًا بـ النماذج المحلية عبر Ollama (aider --model ollama_chat/<model>). إن كنت تعيش في الطرفيّة وتريد نظافة git محكمة، فإن Aider هو الخيار القياسي لسطر الأوامر.

Cline — الوكيل داخل محرّرك​

Cline وكيل برمجة مفتوح المصدر يعيش في محرّرك (VS Code وJetBrains) ويقدّم أيضًا واجهة طرفيّة/سطر أوامر. ينشئ الملفات، ويشغّل الأوامر، ويستخدم الأدوات مع موافقة بشرية في الحلقة — ترى كل خطوة وتوافق عليها. وهو محايد تجاه النموذج ويدعم صراحةً Claude إضافةً إلى النماذج المحلية عبر Ollama أو LM Studio. الجأ إلى Cline حين تريد وكيلًا شبه مستقلّ دون مغادرة محرّرك.

Roo Code تفرّع معروف عن Cline أضاف "أوضاعًا" مخصّصة (مثل وضع المعماري للتخطيط، ووضع المبرمج للتحرير). وكان محايدًا تجاه النموذج أيضًا. لاحظ حالته أدناه — لم يعد الخيار الافتراضي الآمن كما كان.

آخر تحقّق: 2026-06-28.

حالة Roo Code متقلّبة. حتى تاريخ التحقّق، كان فريق Roo Code قد أعلن تقليصًا لإضافة VS Code وتحوّلًا نحو وكيل سحابي — فلم يعد الخيار الافتراضي المُصان ذا الأولوية المحلية الذي تصفه الكتابات الأقدم. إن أردت تفرّعًا بأسلوب Cline تحديدًا، فتأكّد ممّا يُصان بفاعلية اليوم قبل اعتماده. أما Cline نفسه (المشروع الأصل) فيبقى الخيار المحلّي الأكثر أمانًا. تحقّق من الحالة الراهنة عند github.com/cline/cline.

Continue — مساعد المحرّر المفتوح​

Continue مساعد مفتوح المصدر لـ VS Code وJetBrains (إضافةً إلى سطر أوامر) يغطّي الدردشة والتحرير والإكمال التلقائي ووضع الوكيل. وهو محايد تجاه النموذج تمامًا: تُدرِج وثائقه Anthropic (Claude) وOllama للنماذج المحلية والعديد من المزوّدين الآخرين، مع إمكانية إسناد نماذج مختلفة لأدوار مختلفة (دردشة، تحرير، إكمال تلقائي). وهو خيار قوي إن أردت مساعدًا واحدًا قابلًا للتهيئة توصِّل فيه بالضبط أيّ نموذج يقوم بأيّ دور.

آخر تحقّق: 2026-06-28.

حالة صيانة Continue تستحق التحقّق. حول تاريخ التحقّق أطلق المشروع إصدار 2.0 وكانت إشارات الصيانة النشطة تتبدّل. الأداة ودعمها لـ Claude/Ollama حقيقيان، لكن تأكّد من الإصدار الحالي وحالة الصيانة عند github.com/continuedev/continue قبل توحيد فريق عليه.

OpenHands — الوكيل المستقلّ​

OpenHands (المعروف سابقًا بـ OpenDevin) هو الأكثر استقلاليةً بين الأربعة: منصّة لوكلاء تطوير البرمجيات بالذكاء الاصطناعي يعملون في بيئة معزولة، ويفكّكون مهمّة (مثل مشكلة GitHub)، وينفّذونها من طرفها إلى طرفها بموافقة أقلّ خطوةً بخطوة. وهو محايد تجاه النموذج — "استخدمه مع أي نموذج لغوي"، بما في ذلك Claude — ويمكن استضافته ذاتيًّا. اختر OpenHands حين تريد تسليم مهمّة كاملة والعودة للاطّلاع لاحقًا، بدلًا من الموافقة على كل تحرير.

آخر تحقّق: 2026-06-28.

أُعيدت تسمية OpenHands من OpenDevin، وأُعيدت تسمية منظّمته على GitHub من All-Hands-AI إلى OpenHands (المسار القديم All-Hands-AI/OpenHands ما زال يعيد التوجيه). عاشت الوثائق تاريخيًّا عند docs.all-hands.dev. تأكّد من عنواني المستودع والوثائق القانونيّين الحاليّين عند github.com/OpenHands/OpenHands.

Claude Code — وكيل Anthropic الخاص​

Claude Code هو أداة البرمجة الوكيلية الرسمية من Anthropic: تقرأ قاعدة كودك، وتحرّر الملفات، وتشغّل الأوامر والاختبارات، وتعمل عبر طرفيّتك ومحرّرك وتطبيق سطح المكتب والويب. وخلافًا للأربع أعلاه، ليست "وجّهها إلى أي نموذج" — فهي مبنية حول Claude ومضبوطة له، وهذا بالضبط سبب شعور الحلقة بإحكام أكبر ونجاح التحريرات أكثر في المهام الصعبة. إن كانت أولويتك أفضل تجربة برمجة مدعومة بـ Claude لا أقصى قابلية لتبديل النموذج، فهذا هو المسار الأصيل. التفصيل الكامل في ما هو Claude Code.

كيف تختار​

محوران يقرّران كل شيء تقريبًا:

المحور 1 — أين يعيش الوكيل:

  • سطر الأوامر / الطرفيّة (Aider، Claude Code) — قابل للسكربتة، وودود مع git، وقابل للتمرير إلى أدوات أخرى. الأفضل إن كنت تعيش في الصدفة.
  • مدمج في المحرّر (Cline، Continue، واجهة محرّر Claude Code) — الفروق والموافقات داخل محرّرك، وكودك أمامك مباشرةً. الأفضل للتحرير اليومي.
  • مستقلّ / معزول (OpenHands) — سلّم مهمّة كاملة وراجع النتيجة. الأفضل للعمل بالدُّفعات أو دون تدخّل.

المحور 2 — مدعوم بـ Claude مقابل محلي بالكامل:

  • مدعوم بـ Claude — أفضل جودة، خصوصًا في المهام الصعبة متعدّدة الملفات أو الطويلة السياق. يُرسَل كودك إلى API النموذج. اختره حين تكون الجودة الأهمّ ويمكن إرسال كودك إلى مزوّد.
  • محلي بالكامل (Ollama) — كودك لا يغادر الجهاز أبدًا؛ تشغيله مجّاني؛ يعمل دون اتصال. اختره حين تكون الخصوصية أو التنظيم أو العزل عن الشبكة غير قابل للتفاوض — مع قبول أن نموذجًا بحجم حاسوب محمول أضعف في أصعب المهام.

الخطوة الراسخة: اختر الوكيل بحسب الواجهة التي تلائم سير عملك، ثم اختر النموذج بحسب المهمة. لا شيء يمنعك من تشغيل وكيل محرّر على نموذج محلي للمستودع السرّي، والوكيل نفسه على Claude لكل شيء آخر.

اختر وكيل برمجة وهيّئه​

Guided walkthrough1 of 5
  1. سطر الأوامر (Aider، Claude Code)، أو مدمج في المحرّر (Cline، Continue)، أو مستقلّ (OpenHands). طابِقها مع حيث تعمل بالفعل — الطرفيّة مقابل المحرّر مقابل التسليم والمراجعة.
  2. Claude لأفضل جودة (يُرسَل الكود إلى API) أو نموذج Ollama محلي لخصوصية كاملة (لا يغادر الكود الجهاز أبدًا). هذا مستقلّ عن الوكيل في كل أداة عدا Claude Code، المبني أصلًا حول Claude.
  3. اتّبع وثائق الأداة. يُثبَّت Aider عبر pip؛ ويُثبَّت Cline وContinue من متجر VS Code أو JetBrains؛ ويُستضاف OpenHands ذاتيًّا؛ ويُثبَّت Claude Code عبر مثبّته الرسمي.
  4. لـ Claude، اضبط مفتاح Anthropic API لديك (أو سجّل الدخول لـ Claude Code). للمحلّي، ثبّت Ollama واسحب نموذجًا أولًا — انظر تشغيل النماذج محليًّا مع Ollama — ثم وجّه الوكيل إليه.
  5. شغّل مهمّة واحدة محدودة (أصلِح اختبارًا فاشلًا، أضِف دالة صغيرة). راجع كل فرق قبل الموافقة. وثِق بالاستقلالية أكثر فقط بعد أن ترى الحلقة تتصرّف جيّدًا على مستودعك.

تشغيل أول ملموس مع Aider موجَّهًا إلى نموذج محلي (خاص، مجّاني، دون اتصال) — اسحب النموذج بـ Ollama أولًا، ثم:

شغّل Aider على نموذج Ollama محلي (يبقى الكود على جهازك)

aider --model ollama_chat/qwen3

ولتوجيه الأداة نفسها إلى Claude بدلًا من ذلك (أفضل جودة، يُرسَل الكود إلى API)، تضبط مفتاح Anthropic لديك وتختار نموذج Claude — حلقة الوكيل متطابقة؛ يتغيّر العقل فقط:

export ANTHROPIC_API_KEY=sk-ant-...
aider --model sonnet

ذلك التبديل — علامة واحدة، وملمحان مختلفان تمامًا للخصوصية/الجودة، والوكيل نفسه — هو مغزى هذه الصفحة بأكمله.

Key takeaways
  • وكيل البرمجة يشغّل حلقة على جهازك — اقرأ المستودع، خطّط، حرّر الملفات، شغّل الاختبارات، كرّر — بدلًا من تسليمك مقتطفات لتلصقها.
  • جميع هذه الأدوات تقريبًا محايدة تجاه النموذج: الوكيل والنموذج منفصلان، فتبدّل العقل دون تغيير الحلقة.
  • قرصان: Claude لأفضل جودة كود، أو نموذج محلي مفتوح الأوزان عبر Ollama لخصوصية كاملة — كودك لا يغادر الجهاز أبدًا.
  • Aider = سطر الأوامر/git؛ Cline = وكيل في محرّرك؛ Continue = مساعد محرّر قابل للتهيئة؛ OpenHands = مستقلّ/معزول؛ Claude Code = وكيل Anthropic الأصيل المبني على Claude.
  • اختر على محورين — أين يعيش الوكيل (سطر الأوامر مقابل المحرّر مقابل المستقلّ) ومدعوم بـ Claude مقابل محلي بالكامل — واختر النموذج بحسب المهمة، لا مرّة واحدة إلى الأبد.
