Grok لمستخدمي Claude

مبتدئ

أنت تفكّر بالفعل بلغة Claude. ثم يُقسِم زميلٌ بأن Grok هو الأفضل للأسئلة عمّا يحدث الآن، أو يحتاج مشروعٌ إلى مساعد موصول مباشرةً بـ X/تويتر، أو يسلّمك أحدهم قاعدة كود على شكل OpenAI ويطلب منك توجيهها إلى xAI بدلًا من ذلك. الخبر السار: كل ما تعرفه تقريبًا ينتقل معك — وفي جانب واحد (الـ API) يكاد التبديل يكون مجّانيًا. هذه الصفحة تُسقِط نموذجك الذهني الخاص بـ Claude على Grok من xAI، وتُشير إلى الحفنة القليلة من الأشياء التي تعمل فعلًا بشكل مختلف، وتخبرك متى يميل Grok إلى أن يكون الخيار الأفضل.

What you'll learn
  • ترجمة مفاهيم Claude إلى مكافئاتها في Grok (تطبيق Grok، وواجهة xAI API + الكونسول، ومتغيّرات النموذج، وGrok Build)
  • معرفة أيٌّ من عاداتك في صياغة الأوامر مع Claude ينتقل دون تغيير (جميعها تقريبًا)
  • فهم الميزتين المميّزتين الحقيقيتين لـ Grok: البحث الفوري الأصيل في X/الويب، وواجهة API متوافقة مع OpenAI
  • معرفة متى يميل Grok إلى التألّق — العمل الحيّ/على الأحداث الجارية، وعمليات الهجرة السلسة من OpenAI
آخر تحقّق: 2026-07-01.

أسماء النماذج الدقيقة، وأرقام الإصدارات، وأحجام نافذة السياق، ومستويات الخطط، والأسعار تتغيّر باستمرار في xAI (فقد أطلق Grok عدّة إصدارات فرعية في 2026 وحده). تُعلّمك هذه الصفحة النموذج الذهني الدائم وربط المفاهيم؛ ولأي اسم أو حدّ أو سعر محدّد، راجع وثائق xAI الحالية نفسها (المرتبطة في المصادر) بدلًا من الوثوق برقم هنا.

تأكّد من المصدر الرسمي.

خريطة المفاهيم في 60 ثانية​

إن قرأت قسمًا واحدًا فقط، فاقرأ هذا. إليك كيف تصطفّ الأشياء التي تعرفها في Claude مع عالم Grok:

في Claude تسمّيه…في عالم Grok هو…الفكرة نفسها؟
Claude.ai (تطبيق المحادثة)تطبيق Grok (grok.com، وتطبيقات X، والموبايل)نعم — الواجهة الاستهلاكية/المساعِدة
مُنتقي النموذج (Opus / Sonnet / Haiku)متغيّرات Grok (نموذج رائد مثل Grok 4.3، ومتغيّر سريع لاستدعاء الأدوات، ومتغيّر برمجي)نعم — الفكرة نفسها، بأسماء مختلفة

الخلاصة: لا يكاد يوجد مفهوم جديد لتتعلّمه. الأمر في معظمه إعادة تسمية وإعادة تموضع لأشياء تفهمها بالفعل — زائد مجالين يتجاوز فيهما Grok حدود Claude فعلًا: الوصول الفوري إلى بيانات X/الويب، وواجهة API متوافقة مع OpenAI جاهزة للاستبدال المباشر.

الـ API جاهز للاستبدال المباشر — وهذه هي النقطة الكبرى للبنّائين​

هذا هو الفرق الذي يغيّر أكثر ما يغيّر للمطوّرين: واجهة xAI API تتّبع مواصفة واجهة OpenAI. عنوان الأساس هو https://api.x.ai/v1، فإن كان لديك كود يتحدّث بالفعل مع عميل متوافق مع OpenAI، فالهجرة غالبًا مجرّد تغيير عنوان الأساس واستبدال المفتاح.

آخر تحقّق: 2026-07-01. توثّق xAI واجهتها بأنها متوافقة مع OpenAI بعنوان أساس https://api.x.ai/v1؛ لكن السطح المتوافق الدقيق (أيّ المعاملات ونقاط النهاية تتطابق بنظافة) يتطوّر — تحقّق من دليل البدء السريع الحالي لدى xAI. تأكّد من المصدر الرسمي.

بالنسبة لمستخدم Claude يستحق هذا أن يُستوعب، لأن واجهة Messages في Claude لها شكلها الخاص (صيغة طلب/ردّ خاصة، وكتل tool_use/tool_result خاصة). قادمًا من Claude، عدّل ردّة فعلك هكذا:

Guided walkthrough1 of 4
  1. يُوصَل إلى Grok عبر عملاء على طراز OpenAI. لا تترجم إلى لهجة ثالثة — بل تُعيد استخدام شكل طلب OpenAI الذي تعرفه أنت (أو مكتبة) بالفعل.
  2. اضبط عنوان الأساس على https://api.x.ai/v1 واستخدم مفتاح xAI. يظلّ الكثير من بنيتك التحتية المتوافقة مع OpenAI يعمل.
  3. لأن الصيغة متوافقة مع OpenAI، تنتقل تعريفات استدعاء الدوال بشكل مباشر أكثر بكثير ممّا تنتقل بين Claude وOpenAI — راحة حقيقية، لا فخّ.
  4. الشيء الوحيد الذي عليك ضبطه بدقّة هو اسم النموذج (مثلًا معرّف grok-4.x حالي). معرّفات النماذج هي أسرع التفاصيل تغيّرًا — راجع صفحة النماذج الحيّة.

هجرة بالاستبدال المباشر (Python، بـ OpenAI SDK)

from openai import OpenAI

client = OpenAI(
  api_key="YOUR_XAI_KEY",          # an xAI key, not an OpenAI key
  base_url="https://api.x.ai/v1",  # the only structural change
)

resp = client.chat.completions.create(
  model="grok-4.3",                # confirm the current id in xAI docs
  messages=[{"role": "user", "content": "Summarize today's top AI headline."}],
)
print(resp.choices[0].message.content)
Watch out
  • الاستبدال المباشر ≠ سلوك متطابق. صيغة السلك متوافقة مع OpenAI، لكن النموذج هو Grok — تختلف شخصيته، وحدود رفضه، ومعرفته الفورية. أعِد تشغيل تقييماتك؛ لا تفترض أن موجّهًا مضبوطًا لـ Claude (أو GPT) يؤدّي الأداء نفسه.
  • لا تلصق أبدًا مفتاح API حيًّا في موجّه، أو مستودع، أو محادثة. استخدم متغيّرات البيئة وإدارة الأسرار، تمامًا كما تفعل مع مفتاح Anthropic.

البحث الفوري في X + الويب هو العنوان الحقيقي​

هنا يفترق Grok أكثر ما يفترق عن عاداتك في Claude. بُني Grok بواسطة xAI بـ وصول فوري أصيل إلى المعلومات من X (تويتر) والويب، وتضع xAI البحث الحيّ كجزء أساسي من طريقة إجابة النموذج.

آخر تحقّق: 2026-07-01. تصف xAI نموذج Grok باستخدام أدوات أصيل وتكامل بحث فوري (بما في ذلك بيانات من X)؛ لكن سلوك الاسترجاع الدقيق، ومدى الحداثة، وأيّ حدود بحسب الخطة تتغيّر — تحقّق من صفحات منتَج xAI الحالية. تأكّد من المصدر الرسمي.

بالنسبة لمستخدم Claude، التحوّل الذهني هو هذا: مع Claude تلجأ إلى أداة بحث، أو خطّ استرجاع، أو موصِّل حين تحتاج معلومات حالية، وتفكّر بعناية في حدود معرفتك الزمنية. مع Grok، "ما الذي يحدث الآن على X / على الويب" أقرب إلى إعداد افتراضي مدمَج. وهذا يجعل Grok خيارًا طبيعيًا للأسئلة الحيّة، وعن الأحداث الجارية، ونبض الشبكات الاجتماعية — ويغيّر ما ينبغي أن تشكّك فيه: النتائج الحيّة ما زالت تحتاج انضباط التحقّق من المصادر نفسه الذي تطبّقه على أي محتوى ويب.

Pro tip
  • اعتمد على Grok على محور الحداثة: الأخبار العاجلة، والنقاشات الرائجة، و'ماذا يقول الناس اليوم'. هذا هو موطن قوّته بنيويًّا.
  • احتفظ بغرائز Claude في التحقّق: الفوري ≠ الموثوق. اطلب المصادر، وعامِل المنشور الفيروسي كادّعاء يُتحقّق منه، لا كحقيقة.
  • للحداثة عمر نصف. للمواد المرجعية الدائمة، قد تتقادم إجابة البحث الحيّ أو تُفرِط في الاتّكاء على أعلى الأصوات الأخيرة — انظر الملاحظة أدناه.

للفكرة الأعمق عن لماذا الحداثة والحقيقة ليستا الشيء نفسه، انظر الحداثة وعمر النصف والهلوسة — كلاهما ينطبق مباشرةً على أي مساعد يعتمد البحث الفوري.

Grok Build هو جواب xAI على Claude Code​

إن كنت تعيش في Claude Code، فإن Grok Build من xAI هو أقرب مكافئ: وكيل برمجة تقوده من الطرفية، ينافس في المضمار نفسه الذي فيه Claude Code وCodex CLI من OpenAI. بُني حول متغيّر Grok مضبوط للبرمجة ونهج وكلاء فرعيين متوازيين للعمل عبر قواعد كود أكبر، مع وضع مستقلّ طويل الأمد لتسليم المهام الأكبر.

آخر تحقّق: 2026-07-01. أطلقت xAI واجهة سطر أوامر لوكيل البرمجة (Grok Build) ومتغيّر نموذج مضبوط للبرمجة؛ لكن التوفّر (مثلًا أيّ مستوى اشتراك)، ومجموعة الميزات الدقيقة، ومعرّف النموذج تتحرّك بسرعة — عامِل التفاصيل على أنها حسّاسة زمنيًّا وتحقّق منها. تأكّد من المصدر الرسمي.

تنتقل المفاهيم واحدًا لواحد تقريبًا من Claude Code:

النقطة الدائمة: مهارة قيادة واجهة سطر أوامر برمجية وكيلة قابلة للنقل. كتابة مهمة واضحة، وإبقاء السياق محكمًا، ومراجعة الفروقات، والتوجيه بنقاط تفتيش — كلّها الانضباطات نفسها سواء كان الوكيل هو Claude Code أو Codex أو Grok Build. انظر ما هو Claude Code؟ للنموذج الذهني، ووكلاء البرمجة المحلّيون للمجال الأوسع.

ما الذي يحتاج فعلًا إلى تعديل​

معظم درايتك بـ Claude تنتقل. إليك القائمة القصيرة للأشياء المختلفة فعلًا:

  • لهجة الـ API (في صالحك). يتحدّث Grok بشكل متوافق مع OpenAI، لا بـ Anthropic Messages. إن كنت قادمًا من واجهة Claude، فأنت تنتقل إلى شكل طلب/ردّ OpenAI — وتكسب إعادة استخدام سهلة لأي شيء مبنيّ سلفًا لنقاط النهاية المتوافقة مع OpenAI.
  • الفوري بشكل افتراضي. التأريض على الأحداث الجارية أقرب إلى أن يكون مدمَجًا في Grok. عدّل متى تلجأ إليه — واستمرّ في التحقّق من المصادر.
  • الشخصية والحدود. نبرة Grok وحدوده على المحتوى مضبوطة بشكل مختلف عن Claude. لا تفترض أن موجّه نظام معايَرًا لأسلوب Claude يُنتج النبرة نفسها أو الرفض نفسه.
  • التفاصيل المتقلّبة. أسماء النماذج، والمتغيّرات، وحدود السياق، والأسعار تتحرّك بسرعة (أطلق Grok إصدارات متعدّدة عبر 2026). اكتسب عادة التأكيد من المصدر — تمامًا كما ينبغي أن تفعل مع Claude أيضًا.

متى يميل Grok إلى التألّق​

كلاهما مساعد عام قادر، والفارق في أي مهمة بعينها كثيرًا ما يكون صغيرًا ومتبدّلًا — فعامِل هذا على أنه ميول، لا قوانين، ودَع تقييمك على بياناتك يقرّر:

  • العمل الحيّ وعلى الأحداث الجارية. "ما الذي يحدث الآن"، والمواضيع الرائجة، وأسئلة نبض الشبكات الاجتماعية تلعب لصالح بحث Grok الفوري الأصيل في X/الويب.
  • الهجرات السلسة من OpenAI. إن كان لديك كود متوافق مع OpenAI، فتوجيهه إلى xAI من أقلّ عمليات التبديل احتكاكًا في المجال.
  • تسليمات البرمجة أوّلًا-في-الطرفية. إن كنت تحبّ سير عمل Claude Code / Codex، فإن Grok Build خيار ثالث طبيعي لتجرّبه اختبارًا مقارنًا على مهامّك.

الجواب الصادق في معظم الأحيان: أيّهما أنت (أو فريقك) مُعَدّ فيه بالفعل، وأيّهما يفوز في تقييمك على المهمة التي تهم. المهارة قابلة للنقل؛ تكلفة الإعداد هي تكلفة التبديل الحقيقية. للطريقة المحايدة تجاه المزوّد في الاختيار، انظر اختيار نموذج وما تكلفة الذكاء الاصطناعي عبر المزوّدين.

Key takeaways
  • لا يكاد يوجد مفهوم جديد: تطبيق Grok ≈ Claude.ai، ومتغيّرات Grok ≈ Opus/Sonnet/Haiku، وGrok Build ≈ Claude Code، وfunction calling ≈ استخدام الأدوات.
  • الـ API هو الفرق البارز — فهو متوافق مع OpenAI (عنوان الأساس api.x.ai/v1)، فالكود على شكل OpenAI يُهاجَر باستبدال عنوان الأساس + المفتاح، بخلاف واجهة Messages الخاصة بـ Claude.
  • البحث الفوري في X/الويب أقرب إلى أن يكون مدمَجًا في Grok — الجأ إليه للحداثة والأحداث الجارية، لكن احتفظ بانضباط Claude في التحقّق من المصادر.
  • Grok Build وكيل برمجة طرفي في مضمار Claude Code / Codex؛ ومهارة قيادة واجهة سطر أوامر وكيلة قابلة للنقل.
  • الأسماء والمتغيّرات وحدود السياق والأسعار تتقادم بسرعة في xAI — تحقّق من التفاصيل في وثائق xAI، تمامًا كما ينبغي مع Claude.

المصادر وقراءات إضافية​

  • النماذج | وثائق xAI — القائمة المرجعية الحالية لنماذج Grok، ونوافذ السياق، والأسعار.
  • البدء السريع | وثائق xAI — عنوان الأساس المتوافق مع OpenAI وإعداد أول استدعاء.
  • المخرجات المُنظَّمة | وثائق xAI — دعم المخرجات المفروضة بمخطّط.
  • نظرة عامة على الأدوات | وثائق xAI — استدعاء الدوال والأدوات الأصيلة (بحث الويب/X، والكود).
  • Grok 4 | xAI — تأطير xAI للنموذج الرائد، والاستخدام الأصيل للأدوات، والبحث الفوري.
  • API: النماذج الرائدة | xAI — نظرة عامة على منتَج المطوّرين وتموضعه.
