コンテキスト管理

中級
What you'll learn
  • /clear と /compact を見分け、正しい方を即座に選ぶ
  • 自動コンパクションを理解し、なぜそれでも先回りのクリアが勝るのかを知る
  • 長いセッションを鋭く保つ 4 つのテクニックを適用する
  • コンテキストウィンドウが満杯になった症状を見抜き、素早く修正する
最終検証: 2026-06-20。

/compact や /clear のようなコマンド名、自動コンパクションの挙動は変わることがあります。公式ドキュメントで確認してください。

次の情報源で確認してください: 公式情報源.

大きな考え方​

すべてのセッションには有限のコンテキストウィンドウがあります。それが埋まるにつれ、古い内容が押し出され、Claude は「ドリフト」し始めます。コンテキストを意図的に管理することが、長いセッションを鋭く保つ秘訣です。

二つの必須コマンド​

ほとんどすべての状況は二つのコマンドでカバーできます。コツは、意図にコマンドを合わせることです。

Enter キーまたはスペースキーでカードを裏返します。左右の矢印キーでカードを移動できます。用語を表示しました。
1 / 2

一つの問いで決める：これは新しいタスクか、同じタスクか？

自動コンパクション​

コンテキストが上限に近づくと、Claude Code は自動コンパクションを行い、ハードな停止なしに作業を続けられます。これはセーフティネットです——しかしタスクの合間に先回りで /clear する方が、それに頼るより良い結果になります。

Pro tip
  • 自動コンパクションは戦略ではなく最後の防波堤として扱いましょう。無関係なタスクの合間にクリアする方が、ウィンドウが満杯になり自動コンパクションが発動するまで放置するよりコンテキストをクリーンに保てます。

効果を保つテクニック​

二つのコマンドの他に、最初からコンテキストを引き締めておく 4 つの習慣があります。

Guided walkthrough1 of 4
  1. 無関係な作業を一日分も一つのコンテキストに詰め込まないこと。
  2. 大量の読み込みはサブエージェントに任せましょう——彼らは自分のコンテキストを消費し、結論だけを返します。/docs/claude-code/subagents を参照。
  3. 大きな塊を貼り付ける代わりに @path で Claude にファイルを指し示し、CLAUDE.md を引き締めて保ちましょう（/docs/claude-code/claude-md を参照）。
  4. コンテキスト使用量を表示するステータスラインが、いつコンパクトすべきかを教えてくれます。/docs/claude-code/statusline を参照。

これらは実際のドキュメントに対応しています：サブエージェント、CLAUDE.md、ステータスライン。

:::tip 症状 → 修正 Claude が以前の指示を忘れ始めたり同じことを繰り返したりしたら、ウィンドウがおそらく満杯です。/compact（同じタスク）または /clear（新しいタスク）でたいてい即座に直ります。詳しくはトラブルシューティングを参照。 :::

Key takeaways
  • /clear は会話を消去して新たに始める——無関係なタスクに使う
  • /compact はセッションを要約して空きを作る——同じタスクを続けたいときにウィンドウが満杯になったら使う
  • 自動コンパクションはセーフティネット。タスクの合間に先回りで /clear する方が良い結果になる
  • コンテキストを引き締める：1 セッション 1 タスク、重い読み込みはサブエージェントに委譲、貼り付けずファイルを参照、ステータスラインで予算を見張る
  • 忘却や繰り返しはウィンドウ満杯の合図——/compact（同じタスク）または /clear（新しいタスク）で直る
