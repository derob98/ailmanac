Claude ユーザーのための Gemini

🟢初級

あなたはすでに Claude で考える習慣が身についています。さて、あるプロジェクトが Google Workspace のなかにあったり、チームメイトがカスタム Gem を共有してきたり、あるいは2時間の動画と PDF の山を一つのプロンプトに放り込みたいとき — そこで自然に手が伸びるツールが Google の Gemini です。良い知らせがあります。あなたが知っていることのほぼすべてが、そのまま通用します。このページは、あなたの Claude のメンタルモデルを Gemini に対応づけ、本当に違う動きをする数少ないポイントを示し、Gemini に手を伸ばすほうが良い場面を教えます。

🎯What you'll learn
  • ✅Claude の概念を Gemini の対応物に翻訳する(Gems、Gemini アプリ、Workspace 連携、AI Studio + Gemini API)
  • ✅あなたの Claude のプロンプト習慣のうち、どれがそのまま使えるかを知る(ほとんどがそうです)
  • ✅本当に調整しなければならない数少ないものを見抜く
  • ✅Gemini が輝きやすい場面を知る — Google エコシステムのタスク、巨大なコンテキスト、ネイティブなマルチモーダル
✅
最終検証: 2026-06-28。

正確なモデル名やバージョン番号、プランのティア、ファイル/アップロードの上限、コンテキストウィンドウのサイズ、価格は、Google では絶えず変化します。このページが教えるのは、長持ちするメンタルモデルと概念のマッピングです。特定の名前・上限・価格については、ここの数値を信じるのではなく、Google 自身の最新ドキュメント(ソースにリンクあり)を確認してください。

次の情報源で確認してください: 公式情報源.

60秒でわかる概念マップ​

一つだけセクションを読むなら、これを読んでください。Claude で知っていることが Gemini とどう対応するかを示します。

Claude ではこう呼ぶもの…Gemini の世界では…同じ考え方?
Claude.ai(チャットアプリ)gemini.google.com にある Gemini アプリはい — コンシューマー/アシスタント向けの面
Claude Projects(ワークスペース + ファイル + 指示)+ 再利用可能な Custom StylesGems(指示 + 知識ファイルを備えた、保存・再利用可能なアシスタント)近い — 下記の違いを参照
アカウント設定 / 「どう応答してほしいか」Gem の指示 + アカウント設定はい — 同じ考え方
Artifacts / ファイルコンテキストGem に添付された知識ファイル(Google ドライブからも可)近い

要点はこうです。学ぶべき新しい概念はほとんどありません。多くはすでに理解しているものの名前の付け替えと配置換えです。そこに加えて、Gemini が Claude よりも本当に踏み込んでいる2つの領域があります。Google エコシステムとの連携と、ネイティブなマルチモーダル + 非常に大きなコンテキストです。

Gems は、あなたがすでに欲しがっている「再利用可能な Claude」​

Gem は、一度作って再利用できるカスタマイズ可能なアシスタントです。名前を付け、指示を書き、必要に応じて参照用の知識ファイルを添付します。Claude のシステムプロンプトを保存したり、参照ドキュメントを中心に Claude Project を組み立てたことがあるなら、Gem はまさに同じ発想を一つのオブジェクトにまとめたものです。

✅
最終検証: 2026-06-28。 Google は Gem を、指示(よくペルソナ・タスク・コンテキスト・フォーマットの枠組みで表現される)に、任意のアップロードした知識ファイル(PDF、Word/Google ドキュメント、ドライブのファイル)を加えて定義する、カスタマイズ可能なアシスタントと説明しています。正確な作成フロー、ファイルの上限、対応形式は変化します — Google の最新ヘルプページで確認してください。 次の情報源で確認してください: 公式情報源.

Google が推奨する指示の構造は、Claude ユーザーにとって馴染みやすいはずです。なぜなら、それはただの良いプロンプトだからです。

Guided walkthrough1 of 4
  1. その Gem が誰なのか — その役割と語り口。Claude のシステムプロンプトで使う「あなたは…です」と同じ枠組みです。
  2. 何をすべきか。ゴールを一つの明確な文で述べます。
  3. 前提とすべき背景 — そして参照できる知識ファイル。Google ドライブのライブドキュメントも含みます。
  4. 出力の形:表、リスト、JSON、構造化された文書。Claude に特定のフォーマットを求めるのとまったく同じです。

正しく押さえておきたい微妙な点があります。Gem は指示 + 参照資料を一つの再利用可能なアシスタントにまとめるもので、Claude Project の指示とファイルによく似ています。ただし、単なる自分専用のワークスペースというより、チームが動かせる共有可能なスペシャリストとして枠づけられています。Gem がドライブのファイルを参照できる場合、最新バージョンを読む傾向があります — チャットに静的なスナップショットを貼り付けるのとは、小さいけれど本物の違いです。

✅
最終検証: 2026-06-28。 Google は、共有可能な Gems と、Workspace の契約者向けにドライブのファイルを(その最新バージョンを使って)参照する機能を説明しています。提供状況、共有の範囲、ファイルの上限は変化します — Google の最新の発表を確認してください。 次の情報源で確認してください: 公式情報源.

Workspace 連携こそが本当の目玉​

ここが、あなたの Claude の習慣から Gemini が最も離れる場所であり、たいていは適切なタスクにとって良い方向にです。Gemini アプリは Google Workspace — Gmail、Docs、Drive、Calendar など — に組み込まれているので、あなたがコピペしなくても、Google のデータをまたいでコンテキストを引き出し、操作できます。

✅
最終検証: 2026-06-28。 Google は、Gemini が Workspace アプリ(Gmail、Docs、Drive など)と連携してコンテンツを表示・要約することを説明しています。どのアプリが接続されているか、そのアクセスに対する管理者/アカウントの制御は変化します — Google の最新の Workspace ドキュメントで確認してください。 次の情報源で確認してください: 公式情報源.

Claude ユーザーにとって、頭の切り替えはこうです。素材をモデルのところに持っていくのではなく、しばしばすでに Google アカウント内にある素材にモデルを向けることができます。これはプロンプトを変えるというより、ワークフローを変えます。プロンプトは依然として「これを要約して、あれを抽出して、これを下書きして」ですが — その「これ」が、貼り付けではなく、ライブのドライブ文書や受信トレイのスレッドになり得るのです。

✨Pro tip
  • →あなたの仕事がすでに Gmail/Docs/Drive にあるなら、Gemini の連携は、Claude なら必要だったコピペの手間を取り除きます。
  • →Gemini がアクセスできるものは、アカウントと管理者によって制御されると考えてください — 思い込まず、現在の権限を確認しましょう。
  • →それでもスコープを絞ること:意図する特定のドキュメントやスレッドを指し示し、ドライブ全体ではなく。Claude のプロンプトにすべてを詰め込まないのと同じ理由です。

ネイティブなマルチモーダル + 巨大なコンテキスト — そこに寄りかかろう​

2つの能力こそ、習慣化した Claude のワークフローよりも Gemini に手を伸ばす価値が本当に出る場所です。

  • ネイティブなマルチモーダル入力。 Gemini のモデルは、テキスト・画像・音声・動画を一つのリクエストでまとめて受け取るよう作られています — 画像が後付けで貼り付けられているだけではありません。タスクが動画を見る、音声を聴く、あるいは複数のメディアを一度に推論することを含むなら、それは Gemini の得意どころです。

    ✅
    最終検証: 2026-06-28。 Google は、Gemini が複数の入力モダリティ(テキスト、画像、そして — 多くのモデルでは — 音声と動画)を一つのリクエストで受け付けると文書化しています。各モデルがどのモダリティに対応するかはモデルごとに変化します — Google の最新のモデルおよびビジョンのドキュメントで確認してください。 次の情報源で確認してください: 公式情報源.

  • 非常に大きなコンテキストウィンドウ。 多くの Gemini モデルは極めて大きな入力を受け付けます — Google はこれを、コードベース全体、複数の長い文書、あるいは数時間のメディアを一度に処理し、チャンク分割や検索の回避策の必要性を減らすものとして枠づけています。正確なトークンの上限は変動しやすくモデル固有なので、ハードコードしないでください。長持ちする要点は、「丸ごと放り込む」という戦略が、あなたが慣れているよりもここでは現実的だということです。

    ✅
    最終検証: 2026-06-28。 Google は、多くの Gemini モデルが大きなコンテキストウィンドウ(100万トークン以上に対応できるモデルを挙げています)を持ち、長い文書、大きなコードベース、長い動画/音声を一つのリクエストで扱えると説明しています。モデルごとの正確な上限は変化します — Google の最新のロングコンテキストのドキュメントで確認してください。 次の情報源で確認してください: 公式情報源.

調整すべき習慣:Claude では、試す前から反射的に削ったり、チャンクに分けたり、検索を組んだりするかもしれません。大きなコンテキストの Gemini モデルなら、しばしばまず文書(または動画)を丸ごと一つのプロンプトで試す価値があり、実際の上限にぶつかったときだけ最適化すればよいのです。

AI Studio + Gemini API — 開発者向けの面​

チャットから構築へ移るとき、Claude ユーザーにはきれいな対応物があります。

  • Google AI Studio はプロトタイピングのワークベンチ — プロンプトを試し、マルチモーダル入力を試し、パラメータを調整し、API キーを取得する最速の手段です。何かを配線する前に実験しに行く場所だと考えてください。
    ✅
    最終検証: 2026-06-28。 Google は AI Studio を、Gemini でプロトタイプを作り、マルチモーダル入力を試し、パラメータを調整し、API キーを取得するための開発者環境として位置づけています。正確な機能や無料枠の条件は変化します — Google の最新の AI Studio ページで確認してください。 次の情報源で確認してください: 公式情報源.
  • Gemini API は、プロトタイプを作ったあとにコードから呼び出すものです — コンソールのプレイグラウンドから本番呼び出しへ移るのに相当します。

混同しないでおきたい枠組みの違いが一つあります。Gemini アプリ(gemini.google.com)は、Workspace 連携を備えた洗練されたアシスタントです。AI Studio は、モデルの挙動をテストしキーを取得するための開発者向けの面です。これらは別の入口であり、Claude ユーザーは時折、Claude.ai と API コンソールを混同するように、この2つを混同します。仕事に応じて入口を選びましょう。アシスタント的な作業 → アプリ。構築 → AI Studio、それから API。

Function calling — 同じ形、違う JSON の詳細​

Claude でツール使用をしたことがあるなら、Gemini の function calling はすぐに馴染むはずです。なぜならループが同じだからです。

Guided walkthrough1 of 5
  1. 各関数を、名前・説明・パラメータの JSON スキーマ(型、どれが必須か)で記述します。Claude のツール定義と同じ考え方です。
  2. プロンプトと宣言された関数を踏まえ、モデルはテキストの代わりに(またはテキストとともに)構造化された関数呼び出し — 名前と引数 — を返します。モデル自身は何も実行しません。
  3. あなたのアプリケーションが名前と引数を取り出し、実際の関数を実行します。モデルは提案し、あなたのコードが処理します。
  4. 関数の出力をモデルに戻して渡します。
  5. モデルはその結果を使って最終的な応答を生成します — そして終える前にさらに呼び出しを要求することもあります。

同一なのは、記述する → モデルが要求する → あなたが実行する → 結果を返す → 答える のループと、モデルは決してあなたのコードを実行しないという原則です。Gemini は並列呼び出し(一つのターンで複数の独立した関数)や合成的呼び出し(後の呼び出しが前の結果に依存するよう関数を連鎖させる)にも対応しています — どちらも Claude ですでに知っている概念です。

✅
最終検証: 2026-06-28。 Google は、名前/説明/JSON スキーマのパラメータで関数を宣言すること、定義する → モデルを呼ぶ → ローカルで実行する → 結果を返す のループ、加えて並列および合成的(逐次的)な呼び出しを文書化しています。正確なフィールド名、応答の形、オプションは変化します — Google の最新の function-calling ガイドで確認してください。 次の情報源で確認してください: 公式情報源.

異なり、コピペすると引っかかるのは、正確な JSON のフィールド名、関数呼び出しと結果がメッセージリストでどう表現されるか、パラメータ/スキーマの慣習、そして具体的なエンドポイントの形です。プロバイダーをまたぐ経験則として、プロンプトとアーキテクチャは移行できるが、ワイヤーフォーマットの詳細はできない — プロバイダーごとのスキーマを常に確認し直してください。(同じ注意は、Claude ユーザーのための ChatGPT から来る場合にも当てはまります:ループは普遍的、JSON はそうではありません。)

そのまま使えるもの(ネタバレ:ほぼすべて)​

苦労して身につけた Claude の習慣は無駄になりません。以下は本質的にそのまま通用します。

  • プロンプトの構造 — 明確な役割、明示的なタスク、例、要求する出力フォーマット → プロンプトの基本
  • 意図的にコンテキストを与えること — スコープを絞り、適切な素材を指す(今はライブのドライブ文書かもしれません)
  • 特定の出力フォーマットを求めること — 表、JSON、ステップ;同じように機能します
  • その場で反復すること — フォローアップで磨き込む
  • ツール使用のループ — ツールを記述し、モデルに呼ばせ、結果を返す → ツール使用
  • タスクの難しさでモデルを選ぶこと — 速くて安いなら Flash、深いなら Pro。Opus/Sonnet/Haiku と同じ判断です
✦

本当に調整すべきもの​

実際に違うものを、短く正直に列挙します。

  • 連携はワークフローを変えるが、プロンプトは変えない。 Gemini では、貼り付ける代わりにライブの Google データを指せます。何にアクセスできるかは確認しましょう — それはアカウントと管理者の制御です。
  • 大きな入力は「丸ごと」をデフォルトに。 大きなコンテキストとネイティブなマルチモーダルにより、チャンク化/検索の反射をしばしば飛ばし、まず文書や動画を丸ごと試せます。
  • 構築者には入口が2つ。 Gemini アプリ(アシスタント)と AI Studio(開発者向けプレイグラウンド + キー)は別物です。チャットアプリの中で API キーを探しに行かないこと。
  • ツールのワイヤーフォーマット。 function calling の概念は同一ですが、JSON の詳細はそうではありません — Claude の正確なツール JSON をそのまま再利用しないこと。
  • 変動しやすい具体値。 モデル名、バージョン番号、ティア、コンテキスト上限、価格は速く動きます。ソースで確認する習慣をつけましょう。Claude でもそうすべきなのと同じです。

Gemini が輝きやすい場面​

どちらも優れた汎用アシスタントで、特定のタスクでの差はしばしば小さく、移ろいやすいものです — なので、これらを法則ではなく傾向として扱い、最終的には自分のデータでの評価に判断を委ねてください。

  • Google エコシステムのタスク。 仕事がすでに Gmail、Docs、Drive、Calendar にあるなら、Gemini のネイティブな Workspace 連携は、Claude が手動の貼り付けなしには太刀打ちできない摩擦を取り除きます。
  • 巨大な一発のコンテキスト。 膨大な文書、コードベース全体、一度に推論したい長いメディア — 大きなコンテキストの Gemini モデルに寄せ、検索の足場を省きましょう。
  • ネイティブなマルチモーダル。 動画と音声の理解、あるいは混合メディアのプロンプトは、構造的に Gemini の強みです。
  • すでに Google で回っているチーム。 協力者が Gems を共有し Workspace で生活しているなら、そこで合流するほうが摩擦に勝ります。

たいていの場合の正直な答えは:あなた(またはチーム)がすでにセットアップされているほうです。スキルは持ち運べます。セットアップのコストこそが、本当の乗り換えコストです。プロバイダー中立な選び方については、モデルの選び方 と どの Claude を使うべき? を参照してください。

💡Key takeaways
  • ⭐新しい概念はほとんどありません — Gemini アプリ ≈ Claude.ai、Gem ≈ 再利用可能なシステムプロンプト + Project のファイル、モデルピッカー(Pro/Flash)≈ Opus/Sonnet/Haiku、function calling ≈ ツール使用。
  • ⭐本当の違いは、学び直しではなくレバレッジです:深い Google Workspace 連携、非常に大きなコンテキスト、ネイティブなマルチモーダル(画像/音声/動画)。
  • ⭐反射を調整しましょう:貼り付ける代わりにライブの Google データを指す;チャンク化の前に「丸ごと」を試す;アプリと AI Studio が別の入口であることを覚えておく。
  • ⭐あなたのプロンプト、コンテキストの規律、出力フォーマット、ツールループの習慣はそのまま移行できます — 異なるのはツール呼び出しの JSON の詳細だけです。
  • ⭐名前、バージョン、ティア、コンテキスト上限、価格はすぐに古くなります — Claude でもそうすべきなのと同じく、具体値は Google のドキュメントで確認しましょう。
