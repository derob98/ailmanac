Comparatif des CLI d'agents de codage

Intermédiaire

Vous avez déjà choisi un modèle. Mais le programme qui enveloppe le modèle — la boucle qui lit les fichiers, planifie, exécute des commandes, vérifie son propre travail et réessaie — est un choix distinct, et il compte au moins autant. Le nom que la communauté donne à cet enveloppement est le harnais (ou agent), et le même modèle peut sembler brillant dans l'un et médiocre dans l'autre. Cette page cartographie les agents de codage en terminal de 2026, ce qui les distingue vraiment, et comment garder votre configuration portable pour maintenir des coûts de changement bas.

What you'll learn
  • Séparer deux choix que l'on confond : le modèle et le harnais qui le pilote
  • Connaître la forme durable de chaque grand agent CLI — Claude Code, Codex CLI, Gemini CLI et le peloton open source
  • Choisir selon les axes qui décident réellement : autonomie, ouverture, indépendance vis-à-vis du modèle, bac à sable et écosystème
  • Rendre votre configuration portable avec AGENTS.md pour pouvoir changer d'agent sans réécrire le contexte de votre projet
Dernière vérification : 2026-07-08.

Le nombre d'étoiles, les numéros de version, les scores de benchmark, les prix et la disponibilité des plans/paliers sur cette page évoluent vite — les classements d'agents de codage se rebattent presque tous les mois et les fournisseurs modifient leurs paliers gratuits sans grand préavis. Traitez les archétypes et axes ci-dessous comme durables ; revérifiez tout chiffre précis dans le dépôt ou la documentation de chaque projet (liens dans les Sources) avant de décider.

Vérifiez auprès de la source officielle.

Le modèle n'est pas l'agent​

Un modèle de pointe est un prédicteur de texte. Un agent de codage est le harnais qui l'entoure : l'invite système, l'ensemble d'outils (lire, éditer, exécuter le shell, chercher), la boucle de planification, la stratégie de gestion du contexte, le modèle de permissions et de bac à sable, et la logique de réessai. Changez le harnais et vous changez les résultats — des retours de praticiens tout au long de 2026 décrivent le même modèle variant de dizaines de points sur de vraies tâches de dépôt selon l'agent qui le pilote.

Cela a deux conséquences pratiques :

  • Un chiffre de benchmark est un appariement, pas un score de modèle. « 83 % sur Terminal-Bench » est un résultat de modèle + harnais + échafaudage. Changez n'importe quelle partie et il bouge.
  • Votre boucle est la vraie unité de comparaison. Le meilleur agent est celui qui fonctionne le mieux sur votre stack, dans votre terminal, à un prix que vous pouvez assumer — exactement la même logique que pour choisir un modèle de codage. Voir Claude vs GPT vs Gemini pour le codage pour la moitié « modèle » de la décision ; cette page couvre la moitié « harnais ».

Le paysage en 2026​

L'espace des agents de terminal s'est scindé en deux camps : les agents propriétaires réglés pour les modèles d'un seul laboratoire, et les agents indépendants du modèle que vous pointez vers le modèle (ou les poids locaux) de votre choix.

Gemini CLI est en pleine évolution. À la mi-2026, Google a modifié l'accès au palier gratuit et a orienté certains utilisateurs vers un CLI « Antigravity » distinct. Savoir si Gemini CLI, Antigravity ou quelque chose de plus récent est le bon point d'entrée Google est exactement le genre de fait qui se périme — confirmez l'état actuel dans la documentation officielle de Google avant de vous engager.

Les axes qui décident réellement​

Oubliez le classement une minute. Voici les questions durables :

Guided walkthrough1 of 5
  1. Les agents propriétaires (Claude Code, Codex CLI, Gemini CLI) sont réglés pour leur propre modèle et offrent en général l'expérience de pointe la plus fluide. Les agents indépendants du modèle (opencode, Aider, Cline) vous laissent changer de modèle — de pointe aujourd'hui, bon marché ou local demain — sans changer d'outils. Choisissez l'indépendance si éviter le verrouillage ou faire tourner des poids locaux compte pour vous.
  2. Aider est un binôme de programmation : des éditions ciblées, au niveau du fichier, natives Git, avec vous dans la boucle à chaque commit. Claude Code et Codex CLI sont des orchestrateurs : donnez un objectif, ils planifient et exécutent sur de nombreux fichiers et commandes shell. Plus d'autonomie signifie plus de levier et plus de besoin de garde-fous.
  3. Un agent qui exécute des commandes shell peut supprimer des fichiers, pousser du code ou dépenser de l'argent. Préférez un bac à sable imposé par l'OS et un modèle de permissions explicite au « faites-moi confiance ». Claude Code et Codex CLI livrent tous deux du bac à sable en 2026 ; vérifiez ce que tout agent peut toucher avant de le laisser tourner sans surveillance.
  4. Au-delà de la boucle centrale : serveurs MCP pour les outils, skills réutilisables, sous-agents, hooks, intégration à l'IDE, actions CI. Un agent plus léger sur un excellent modèle peut perdre face à un agent plus épais sur un modèle juste suffisant, parce que l'écosystème fait davantage le travail.
  5. Abonnement vs facturation à l'API, fenêtre de contexte, mise en cache et volubilité du harnais font tous bouger la facture réelle. Un modèle moins cher dans un harnais gourmand en jetons peut coûter plus qu'un modèle plus onéreux dans un harnais sobre.
Pro tip
  • N'achetez pas un harnais sur un benchmark accrocheur. Achetez-le sur un essai de 30 minutes face à une vraie tâche dans votre propre dépôt — le même geste d'« évaluer sur son propre code » qui bat tout classement de modèles.
  • L'ouverture est une couverture, pas une fonctionnalité que vous utiliserez tous les jours. Sa valeur apparaît le jour où un prix change ou un palier gratuit disparaît et où vous voulez déménager.

Portabilité : écrivez votre contexte une seule fois​

Le plus grand coût de changement caché est le contexte de votre projet — commandes de build, conventions, règles de test, choses que l'agent ne peut pas deviner. En 2026, cela a un standard de fait : AGENTS.md, un fichier en Markdown simple à la racine de votre dépôt, géré par l'Agentic AI Foundation sous l'égide de la Linux Foundation et lu nativement par Codex, Gemini CLI, Cursor, Aider, l'agent de codage de Copilot, opencode, Goose et plus de 20 autres outils (plus de 60 000 dépôts l'ont adopté).

  • AGENTS.md = contexte de projet inter-outils. Écrivez-le une fois ; la plupart des agents le lisent.
  • CLAUDE.md = le fichier de mémoire de projet propre à Claude Code (voir CLAUDE.md). Vous pouvez conserver un petit CLAUDE.md qui pointe vers AGENTS.md, ou maintenir les deux.
  • SKILL.md = la spécification distincte d'Anthropic pour empaqueter des skills réutilisables avec du frontmatter et des scripts groupés. Elle est complémentaire à AGENTS.md, pas concurrente — l'une décrit le projet, l'autre empaquette une procédure réutilisable.

Garder le contexte dans AGENTS.md signifie que le jour où vous essayez un autre agent, vos règles de projet suivent gratuitement.

AGENTS.md minimal — n'écrivez que ce qu'un agent ne peut pas déduire

# AGENTS.md

## Setup
- Install: `pnpm install`
- Dev server: `pnpm dev` (port 3000)

## Build & test (run before every commit)
- Typecheck: `pnpm typecheck`
- Lint: `pnpm lint`
- Tests: `pnpm test` — all must pass

## Conventions
- TypeScript strict; no `any`.
- Components in `src/components`, one per file.
- Never edit files under `generated/` by hand.

## Constraints
- Do NOT run `git push` or open PRs without being asked.
- Secrets live in `.env.local`; never commit them.
Watch out
  • Ne mettez dans AGENTS.md que ce que l'agent ne peut pas découvrir par lui-même. Répéter l'évident coûte des jetons à chaque exécution et enterre les règles qui comptent. Curatez-le à la main — les fichiers de contexte auto-générés ont tendance à n'être que du bruit.

Une décision, en une respiration​

  • Travail Claude profond et autonome avec l'écosystème d'outils le plus riche → Claude Code.
  • Agent propriétaire open source lié à GPT, avec une forte couverture plateforme/Windows → Codex CLI.
  • Vous voulez changer de modèle librement ou faire tourner des poids locaux → opencode ou Cline.
  • Éditions prudentes, natives Git, avec l'humain dans la boucle → Aider.
  • Grand pari sur le contexte de l'écosystème Google → Gemini CLI (confirmez d'abord l'accès actuel).

Quel que soit votre choix, gardez le contexte de votre projet dans AGENTS.md et prouvez ce choix avec une vraie tâche dans votre propre dépôt avant de vous engager.
