Rédiger des prompts spécifiquement pour Claude

Intermédiaire
Dernière vérification : 2026-06-20.

Les recommandations d'Anthropic sur le prompt engineering évoluent avec chaque modèle — consultez la documentation officielle de prompt engineering pour les bonnes pratiques actuelles.

Vérifiez auprès de la source officielle.

L'essentiel du prompting est universel. Mais quelques techniques sont particulièrement efficaces avec Claude — et une paire d'habitudes héritées des anciens modèles sont désormais contre-productives. Cette leçon sépare le signal du bruit : ce qui est réellement propre à Claude.

What you'll learn
  • Reconnaître les techniques propres à Claude sur lesquelles miser — balises XML, clarté, rôles, réflexion, exemples et préremplissage
  • Désapprendre les vieilles habitudes (MAJUSCULES, menaces, échafaudages sur-conçus) qui se retournent maintenant contre vous
  • Savoir lesquelles des deux habitudes — l'affinité pour les balises XML et le préremplissage — adopter comme votre signature Claude
  • Comprendre ce qu'il faut revérifier quand vous changez de modèle

Misez sur celles-ci avec Claude​

Ce sont les leviers les plus payants avec Claude. Le premier est l'habitude à plus forte valeur que vous puissiez développer.

  • Balises XML. Claude est bien réglé pour les prompts structurés avec des balises nommées pour les entrées, les instructions et les exemples. C'est l'habitude Claude à plus forte valeur.
  • Soyez clair et direct. Énoncez l'objectif, les contraintes et ce à quoi ressemble « terminé ». Claude suit les instructions explicites de près.
  • Donnez-lui un rôle via le prompt système.
  • Laissez-le réfléchir sur les problèmes difficiles — demandez un raisonnement étape par étape, ou utilisez la réflexion étendue / l'effort sur les modèles de raisonnement (où souvent vous n'avez pas besoin de demander explicitement une chaîne de pensée).
  • Utilisez des exemples (few-shot) pour le format et les cas limites.
  • Préremplissez le début de la réponse de Claude (sur l'API) pour orienter le format.

Essayez le préremplissage​

Préremplir signifie commencer vous-même le tour de l'assistant pour que Claude continue dans le format voulu. Sur l'API, entamez le tour de l'assistant par un caractère d'ouverture ou un titre :

Préremplir pour forcer une sortie JSON

{

Préremplir un titre pour forcer la structure

## Summary
Pro tip
  • Commencez le tour de l'assistant par { pour forcer du JSON, ou par un titre pour forcer une sortie structurée. Le préremplissage est un levier réservé à l'API — il oriente le format sans dépenser de mots en instructions.

Abandonnez ces vieilles habitudes​

Les modèles Claude récents, qui suivent les instructions à la lettre, changent les règles. Les astuces qui aidaient sur les anciens modèles gênent désormais.

Watch out
  • Levez le pied sur les MAJUSCULES et les prompts menaçants (« TU DOIS… SINON »). Les modèles Claude récents suivent les instructions à la lettre, et des instructions polies et précises fonctionnent mieux que crier.
  • Ne sur-concevez pas avec des échafaudages élaborés de style « jailbreak » — des instructions claires et honnêtes surpassent les astuces.
  • Souvent, moins c'est plus — un prompt serré vaut mieux qu'un prompt tentaculaire qui noie la demande.

Claude vs autres modèles​

Les principes se transfèrent (voir Traduction de prompts entre IA), mais deux choses valent surtout la peine d'être retenues comme propres à Claude.

Guided walkthrough1 of 3
  1. Enveloppez les entrées, instructions et exemples dans des balises nommées. L'affinité de Claude pour les balises en fait l'habitude de structuration la plus fiable.
  2. Amorcez le tour de l'assistant pour verrouiller le format — une accolade d'ouverture pour le JSON, un titre pour la structure.
  3. Quand vous quittez Claude, revérifiez l'intensité du ton et les indices de formatage ; ce qui est littéral ici peut ne pas se transférer proprement.

Key takeaways
  • Les balises XML sont l'habitude Claude à plus forte valeur ; le préremplissage est le levier de format propre à Claude le plus utile.
  • Les modèles Claude récents suivent les instructions à la lettre — soyez clair et direct, pas bruyant ni menaçant.
  • Moins c'est plus : un prompt serré et honnête vaut mieux qu'un échafaudage élaboré.
  • Les principes se transfèrent d'un modèle à l'autre, mais revérifiez l'intensité du ton et les indices de formatage quand vous changez.
