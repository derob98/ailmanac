Styles de sortie

🟡Intermédiaire
✅
Dernière vérification : 2026-06-21.

Les styles intégrés, le mécanisme exact de changement et le format de fichier des styles personnalisés peuvent changer d'une version à l'autre. Vérifiez dans la documentation officielle (la doc de Claude Code est passée de docs.anthropic.com à code.claude.com).

Vérifiez auprès de la source officielle.
🎯What you'll learn

  • ✅Les quatre styles intégrés : Default, Proactive, Explanatory, Learning
  • ✅Comment changer de style avec /config — et pourquoi le changement nécessite une nouvelle session
  • ✅Comment créer votre propre style personnalisé sous forme de fichier Markdown
  • ✅Quand recourir à un style de sortie vs CLAUDE.md vs instructions personnalisées de Claude.ai

Les styles de sortie changent comment Claude Code communique — son rôle, son ton, sa verbosité, et s'il explique son raisonnement — en modifiant directement le prompt système. Ils changent comment Claude répond, pas ce qu'il sait. Recourez-y quand vous re-promptez sans cesse pour obtenir la même voix ou le même format à chaque tour.

Pourquoi les utiliser​

  • Adaptez-les à votre flux de travail. Concis et orienté action quand vous allez vite ; explicatif quand vous apprenez.
  • Mode pédagogique. Certains styles ajoutent un commentaire éducatif — parfait pendant que vous découvrez une base de code ou une technique.
  • Cohérence. Définissez un style préféré une fois pour que chaque session se lise comme vous l'aimez.

Les styles intégrés​

Claude Code est livré avec quatre :

StyleCe qu'il fait
DefaultLe prompt système standard d'ingénierie logicielle — efficace, orienté action.
ProactiveExécute immédiatement et fait des hypothèses raisonnables au lieu de s'arrêter pour des décisions de routine. Guidage d'exécution autonome plus fort que le mode auto, mais vous voyez toujours les invites de permission avant l'exécution des outils.
ExplanatoryAjoute des « Insights » pédagogiques entre les étapes, expliquant les choix d'implémentation et les patterns de la base de code.
LearningCollaboratif, apprentissage par la pratique : partage des Insights et vous demande d'écrire vous-même de petits morceaux, laissant des marqueurs [TODO(human)] dans votre code.
⚠️Watch out
  • •Explanatory et Learning produisent des réponses plus longues par conception, ils coûtent donc plus de tokens de sortie que Default.

Comment changer​

Exécutez /config et choisissez un style sous Output style. Votre choix est enregistré dans .claude/settings.local.json au niveau du projet. Pour le définir sans le menu, modifiez le champ outputStyle dans n'importe quel fichier de paramètres.

Guided walkthrough1 of 4
  1. Exécutez /config dans votre session et trouvez l'option Output style.
  2. Sélectionnez l'un de Default, Proactive, Explanatory ou Learning. Votre choix est enregistré dans .claude/settings.local.json au niveau du projet.
  3. Contournez le menu en modifiant le champ outputStyle dans n'importe quel fichier de paramètres (voir le JSON ci-dessous).
  4. Le style fait partie du prompt système, lu une seule fois au démarrage de la session — exécutez /clear ou ouvrez une nouvelle session pour qu'il prenne effet.
{
  "outputStyle": "Explanatory"
}

:::warning Les changements nécessitent une nouvelle session Le style de sortie fait partie du prompt système, que Claude Code lit une seule fois au démarrage de la session. Un changement ne prend effet qu'après /clear ou une nouvelle session. (La commande autonome /output-style a été supprimée en v2.1.91 — utilisez /config ou le paramètre ci-dessus.) :::

Créer un style personnalisé​

Un style personnalisé est un fichier Markdown : frontmatter, puis les instructions à ajouter au prompt système. Enregistrez-le au niveau utilisateur (~/.claude/output-styles/) ou projet (.claude/output-styles/) ; le nom du fichier devient le nom du style à moins que vous ne définissiez name.

✦

Exemple de style personnalisé — Diagrammes d'abord

---
name: Diagrams first
description: Lead every explanation with a diagram
keep-coding-instructions: true
---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.
✨Pro tip
  • →Mettez keep-coding-instructions: true quand vous changez seulement COMMENT Claude communique mais voulez qu'il code encore normalement.
  • →Omettez-le (par défaut) quand Claude ne fait pas du tout d'ingénierie logicielle — disons un assistant de pure rédaction ou d'analyse de données — pour que les instructions de codage intégrées soient supprimées.

Styles de sortie vs CLAUDE.md vs autres leviers​

  • Style de sortie = un préréglage du prompt système pour le rôle, le ton et le format — s'applique à chaque réponse.
  • CLAUDE.md = faits et règles projet persistants (conventions, garde-fous), ajoutés en tant que message après le prompt système.
  • Instructions personnalisées / styles de réponse = le cousin de l'application grand public Claude.ai.

:::tip Utilisez le bon levier Vous voulez que Claude « lance toujours les tests avant de terminer » ? C'est un garde-fou CLAUDE.md, pas un style. Vous voulez qu'il « explique chaque étape au fur et à mesure » ? C'est un style de sortie. Gardez les règles de comportement dans CLAUDE.md et la présentation dans les styles. :::

💡Key takeaways
  • ⭐Les styles de sortie changent COMMENT Claude communique (rôle, ton, verbosité), pas CE QU'il sait — ils modifient le prompt système.
  • ⭐Quatre intégrés : Default (orienté action), Proactive (exécute avec des hypothèses), Explanatory (ajoute des Insights), Learning (Insights + marqueurs [TODO(human)]).
  • ⭐Explanatory et Learning coûtent plus de tokens de sortie car les réponses sont plus longues par conception.
  • ⭐Changez via /config (enregistré dans .claude/settings.local.json) ou le paramètre outputStyle — les changements nécessitent /clear ou une nouvelle session.
  • ⭐Les styles personnalisés sont des fichiers Markdown dans ~/.claude/output-styles/ ou .claude/output-styles/ ; utilisez keep-coding-instructions: true pour conserver le comportement de codage.
  • ⭐Utilisez CLAUDE.md pour les règles de comportement/garde-fous et les styles de sortie pour la présentation.
