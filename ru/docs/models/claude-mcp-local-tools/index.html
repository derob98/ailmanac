Подключение Claude к локальным инструментам и агентам через MCP

Средний

Вы хотите, чтобы Claude был мозгом, а ваша машина предоставляла руки — читала ваши файлы, обращалась к базе данных, запускала ваш скрипт, даже вызывала локальную модель или локального агента, и всё это без выхода ваших данных за пределы ноутбука. У этой синергии уже есть стандарт: Model Context Protocol (MCP). В двух словах: MCP — это открытый стандарт, который позволяет ИИ-клиенту вызывать внешние инструменты и данные через небольшие программы, называемые MCP-серверами — и эти серверы могут работать приватно, локально, на вашей собственной машине. На этой странице показано, что такое MCP, как складывается локальная архитектура, как добавить локальный сервер в Claude и как создать свой собственный маленький сервер.

What you'll learn
  • Объяснить MCP одним предложением — открытый стандарт, чтобы ИИ-клиент мог вызывать внешние инструменты и данные через MCP-серверы
  • Увидеть локальную архитектуру: MCP-клиент (Claude Code / Claude Desktop), общающийся с локальным MCP-сервером через stdio
  • Понять, почему ЭТО и есть та самая синергия «Claude-как-мозг + локальные возможности», которую вы хотите
  • Добавить локальный MCP-сервер в Claude одной командой
  • Создать крошечный MCP-сервер, предоставляющий один инструмент
  • Знать границу доверия — MCP-серверы действуют с ВАШИМИ привилегиями, поэтому устанавливайте только то, чему доверяете

Что такое MCP (в двух словах)​

MCP — это открытый стандарт для подключения ИИ-приложений к внешним системам. ИИ-клиент (например, Claude) говорит на одном протоколе; любой MCP-сервер, который на нём говорит, может подключиться и предложить клиенту новые возможности. Официальная аналогия — порт USB-C для ИИ: вместо отдельной интеграции под каждый инструмент вы строите под один разъём, и он работает везде.

Anthropic представила и открыла исходный код MCP, и с самого первого дня он поставлялся с поддержкой локальных MCP-серверов, чтобы Claude мог подключаться к внутренним системам и наборам данных, работающим на вашей собственной машине.

Последняя проверка: 2026-06-28.

Показатели внедрения, дата ревизии спецификации, управление (руководство MCP) и точный список клиентов, поддерживающих MCP, часто меняются. Относитесь к любой конкретной цифре как к скоропортящейся и подтверждайте её на modelcontextprotocol.io и в анонсе Anthropic. Устойчивые факты — MCP является открытым стандартом, клиент ↔ сервер, серверы могут работать локально, широко принят множеством клиентов — стабильны.

Сверьтесь с официальным источником.

MCP-сервер может предоставлять клиенту три вида вещей:

  • Инструменты (Tools) — действия, которые модель может вызвать (прочитать файл, выполнить запрос, обратиться к API). Это та же форма, что и у нативного использования инструментов модели: именованная возможность с типизированными входными данными.
  • Ресурсы (Resources) — данные, которые клиент может читать (файлы, записи, документы).
  • Промпты (Prompts) — переиспользуемые параметризованные шаблоны промптов, которые предлагает сервер.

Для этой страницы главное — это инструменты: локальный MCP-сервер — это то, как вы вручаете Claude новую, приватную возможность.

Почему это та синергия, которую вы хотите​

То, что люди подразумевают под «Claude как мозг, мой компьютер как тело», — это в точности разделение MCP на клиент/сервер:

  • Claude — оркестратор. Он читает вашу цель, решает, какой инструмент вызвать, вызывает его, читает результат и решает, каким будет следующий шаг.
  • Локальные MCP-серверы — это возможности. Каждый из них — небольшая программа, которую вы запускаете на своей машине: файловый сервер, сервер базы данных, обёртка вокруг локального скрипта, даже сервер, который вызывает локальную модель или передаёт работу локальному агенту.

Поскольку сервер работает локально, а Claude общается с ним через стандартный ввод/вывод вашей собственной машины, ваши данные и действия остаются на вашем оборудовании — модель оркестрирует, но чтение файлов, SQL, выполнение скриптов происходят локально. Это версия агентной работы, сохраняющая приватность: мозг в облаке (или тоже локально), руки строго на вашей машине.

А поскольку MCP — это один открытый стандарт, принятый множеством клиентов, сервер, который вы создаёте, не привязан к одному приложению. Тот же локальный сервер, который вы подключаете к Claude Code, может использоваться другими клиентами, говорящими на MCP — создайте один раз, переиспользуйте везде.

Локальная архитектура​

Наименьшая полезная картина состоит из двух частей и одного канала:

  • MCP-клиент (хост). ИИ-приложение — например, Claude Code или Claude Desktop. Оно содержит модель и решает, какие инструменты вызывать.
  • MCP-сервер. Отдельный процесс, который объявляет инструменты/ресурсы/промпты и выполняет их по запросу.
  • Транспорт. Для локальных серверов клиент запускает сервер как подпроцесс, и они общаются через stdio (стандартный ввод/вывод) с помощью сообщений JSON-RPC. Никакой сети, никакого порта — просто локальный канал между двумя процессами на вашей машине. (Удалённые серверы тоже существуют, по HTTP; эта страница о локальном случае со stdio.)
┌─────────────────────────┐         stdio (JSON-RPC)        ┌──────────────────────────┐
│  MCP CLIENT / HOST       │ ──── launches as subprocess ──▶ │  LOCAL MCP SERVER         │
│  Claude Code / Desktop   │ ◀── tools/resources/prompts ─── │  filesystem · db · script │
│  (the model = the brain) │                                 │  · local model / agent    │
└─────────────────────────┘                                  └──────────────────────────┘
                                                              runs on YOUR machine, YOUR data

Модель никогда не касается вашего диска напрямую — она обращается к локальному серверу, а сервер (работающий с вашими правами) выполняет работу и возвращает результат.

Добавьте локальный MCP-сервер в Claude​

Самый быстрый путь — команда claude mcp add в Claude Code, которая регистрирует сервер, запускаемый клиентом через stdio. Общая форма — имя, затем команда, которая запускает сервер.

Guided walkthrough1 of 4
  1. Начните с официального эталонного сервера (например, файлового сервера), чтобы поведение было известным. Anthropic поддерживает коллекцию эталонных MCP-серверов, которые можно запускать локально — filesystem, git, memory, fetch и другие.
  2. Используйте `claude mcp add <name> -- <command...>`. Часть после `--` — это в точности команда, которую Claude Code выполнит, чтобы запустить сервер как локальный подпроцесс; он общается с ним через stdio.
  3. Запустите `claude mcp list`, чтобы увидеть зарегистрированные серверы, затем начните сессию и спросите Claude, какие инструменты у него теперь есть. Инструменты сервера должны появиться как вызываемые возможности.
  4. Дайте Claude цель, для которой нужна новая возможность («прочитай файлы в ./notes и составь их резюме»). Claude вызывает инструменты сервера; работа происходит локально.

Минимальный claude mcp add для локального файлового сервера (всё после -- — это команда запуска):

Добавить локальный файловый MCP-сервер в Claude Code

claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /path/to/allowed/dir

Если вы предпочитаете править конфигурацию напрямую, stdio-сервер объявляется командой, которая его запускает. Форма выглядит так:

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    }
  }
}
Последняя проверка: 2026-06-28.

Точные подкоманды claude mcp, области видимости конфигурации, расположение конфигурационного файла и точные имена пакетов эталонных серверов развиваются. Подтвердите актуальный синтаксис в руководстве по MCP для Claude Code и репозитории эталонных серверов, прежде чем полагаться на конкретный флаг или имя пакета.

Сверьтесь с официальным источником.

Создайте крошечный MCP-сервер​

Сервер — это просто программа, которая использует MCP SDK, объявляет инструмент и запускает транспорт stdio. Вот вся идея на TypeScript — один инструмент, который складывает два числа (замените тело на «запустить мой скрипт», «обратиться к моей БД», «вызвать мою локальную модель»):

npm install @modelcontextprotocol/sdk zod
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({ name: "my-local-tools", version: "0.1.0" });

// Declare ONE tool. Inputs are typed; the body runs locally with your privileges.
server.registerTool(
  "add",
  {
    description: "Add two numbers",
    inputSchema: { a: z.number(), b: z.number() },
  },
  async ({ a, b }) => ({
    content: [{ type: "text", text: String(a + b) }],
  }),
);

// Talk to the client over stdio — this is what makes it a LOCAL server.
await server.connect(new StdioServerTransport());

Зарегистрируйте его в Claude точно так же, как любой другой stdio-сервер — укажите команде запуска на ваш файл:

Добавить свой собственный локальный сервер в Claude Code

claude mcp add my-local-tools -- node /abs/path/to/server.js

Это полный цикл: Claude (мозг) видит новый инструмент, вызывает его, когда это уместно, а ваш код (руки) выполняется локально и возвращает результат. Поскольку он говорит на том же открытом протоколе, этот сервер также работает в других MCP-клиентах без изменений.

Watch out
  • MCP-серверы работают с вашими привилегиями и могут совершать реальные действия — устанавливайте только те серверы, которым доверяете, и остерегайтесь prompt-инъекций через результаты инструментов.

Key takeaways
  • MCP — это открытый стандарт, чтобы ИИ-клиент мог вызывать внешние инструменты и данные через MCP-серверы — Anthropic представила его с поддержкой локальных серверов с самого начала.
  • Локальная архитектура проста: MCP-клиент (Claude Code / Desktop) запускает локальный MCP-сервер как подпроцесс и общается с ним через stdio — ваши данные остаются на вашей машине.
  • Это синергия «Claude-как-мозг + локальные возможности»: Claude оркестрирует; локальные серверы (файловая система, БД, скрипт, даже локальная модель или агент) предоставляют руки.
  • Добавьте сервер одной командой (claude mcp add ... -- <команда запуска>); создайте его с помощью MCP SDK, зарегистрировав инструмент и подключив транспорт stdio.
  • MCP широко принят, поэтому созданный вами локальный сервер работает во многих клиентах — создайте один раз, переиспользуйте везде.
  • Серверы действуют с вашими привилегиями: устанавливайте только то, чему доверяете, и относитесь к результатам инструментов как к возможному вектору prompt-инъекции.
