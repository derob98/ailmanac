Сравнение CLI-агентов для программирования

Средний

Вы уже выбрали модель. Но программа, которая оборачивает модель — цикл, читающий файлы, планирующий, запускающий команды, проверяющий собственную работу и повторяющий попытки — это отдельный выбор, и он значит как минимум столько же. В сообществе эту обёртку называют обвязкой (harness) или агентом, и одна и та же модель может выглядеть блестяще в одной и посредственно в другой. На этой странице разобраны терминальные агенты для программирования 2026 года, что их действительно различает и как сохранить переносимость конфигурации, чтобы издержки перехода оставались низкими.

What you'll learn
  • Разделить два выбора, которые часто путают: модель и обвязку, которая ею управляет
  • Знать устойчивую форму каждого крупного CLI-агента — Claude Code, Codex CLI, Gemini CLI и пакет open-source
  • Выбирать по осям, которые реально всё решают: автономность, открытость, независимость от модели, песочница и экосистема
  • Сделать конфигурацию переносимой с помощью AGENTS.md, чтобы менять агентов без переписывания контекста проекта
Последняя проверка: 2026-07-08.

Количество звёзд, номера версий, результаты бенчмарков, цены и доступность планов/тарифов на этой странице меняются быстро — рейтинги агентов для программирования перетасовываются почти ежемесячно, а вендоры меняют бесплатные тарифы почти без предупреждения. Считайте архетипы и оси ниже устойчивыми; любую конкретную цифру перепроверяйте по собственному репозиторию или документации каждого проекта (ссылки в разделе «Источники») до того, как принять решение.

Сверьтесь с официальным источником.

Модель — это не агент​

Передовая модель — это предсказатель текста. Агент для программирования — это обвязка вокруг неё: системный промпт, набор инструментов (чтение, редактирование, запуск shell, поиск), цикл планирования, стратегия управления контекстом, модель прав и песочницы, а также логика повторных попыток. Смените обвязку — и вы смените результат: отчёты практиков на протяжении 2026 года описывают, как одна и та же модель колеблется на десятки пунктов на реальных задачах с репозиториями в зависимости от того, какой агент ею управляет.

Из этого следуют два практических вывода:

  • Число в бенчмарке — это пара, а не оценка модели. «83% на Terminal-Bench» — это результат модель + обвязка + каркас. Замените любую часть — и число сдвинется.
  • Ваш цикл — вот настоящая единица сравнения. Лучший агент — тот, что лучше всего работает на вашем стеке, в вашем терминале, по цене, с которой вы можете жить — ровно та же логика, что и при выборе модели для программирования. См. Claude против GPT против Gemini для программирования для модельной половины решения; эта страница — про половину с обвязкой.

Расстановка сил в 2026 году​

Пространство терминальных агентов разделилось на два лагеря: фирменные агенты, настроенные под модели одной лаборатории, и независимые от модели агенты, которые вы направляете на любую модель (или локальные веса) по своему выбору.

Gemini CLI в состоянии перемен. В середине 2026 года Google изменил доступ по бесплатному тарифу и направил часть пользователей к отдельному CLI «Antigravity». Что именно является правильной точкой входа со стороны Google — Gemini CLI, Antigravity или что-то более новое — это ровно тот тип факта, который устаревает: подтвердите текущее состояние в собственной документации Google, прежде чем на что-то полагаться.

Оси, которые действительно всё решают​

Забудьте на минуту про таблицу лидеров. Вот устойчивые вопросы:

Guided walkthrough1 of 5
  1. Фирменные агенты (Claude Code, Codex CLI, Gemini CLI) настроены под собственную модель и обычно дают самый гладкий пиковый опыт. Независимые от модели агенты (opencode, Aider, Cline) позволяют менять модели — сегодня передовую, завтра дешёвую или локальную — не меняя инструментов. Выбирайте независимый вариант, если для вас важно избежать привязки к вендору или запускать локальные веса.
  2. Aider — это напарник по программированию: сфокусированные, пофайловые, git-нативные правки, где вы участвуете в каждом коммите. Claude Code и Codex CLI — оркестраторы: дайте цель, и они спланируют и выполнят работу по множеству файлов и shell-команд. Больше автономности означает больше рычага и больше потребности в ограждениях.
  3. Агент, запускающий shell-команды, может удалить файлы, отправить код или потратить деньги. Предпочитайте песочницу на уровне ОС и явную модель прав вместо «доверься мне». Claude Code и Codex CLI в 2026 году оба поставляются с песочницей; проверьте, к чему может прикоснуться любой агент, прежде чем позволить ему работать без присмотра.
  4. Помимо основного цикла: MCP-серверы для инструментов, переиспользуемые skills, субагенты, хуки, интеграция с IDE, действия CI. Более тонкий агент на отличной модели может проиграть более толстому агенту на достаточно хорошей, потому что экосистема делает больше работы.
  5. Подписка против оплаты API по метрике, окно контекста, кеширование и то, насколько разговорчива обвязка — всё это двигает реальный счёт. Более дешёвая модель в прожорливой на токены обвязке может стоить дороже, чем более дорогая модель в экономной.
Pro tip
  • Не покупайте обвязку по громкому бенчмарку. Покупайте её по 30-минутному пробному прогону на реальной задаче в вашем собственном репозитории — тот же приём «оценивай на своём коде», который бьёт любую таблицу лидеров моделей.
  • Открытость — это страховка, а не функция, которую вы будете использовать ежедневно. Её ценность проявляется в тот день, когда меняется цена или исчезает бесплатный тариф, и вы захотите переехать.

Переносимость: опишите контекст один раз​

Самая большая скрытая издержка перехода — это контекст вашего проекта: команды сборки, соглашения, правила тестирования, вещи, которые агент не может вывести сам. В 2026 году для этого есть де-факто стандарт: AGENTS.md, файл в чистом Markdown в корне репозитория, который курирует Agentic AI Foundation под эгидой Linux Foundation и который нативно читают Codex, Gemini CLI, Cursor, Aider, агент для программирования Copilot, opencode, Goose и ещё 20+ инструментов (его приняли 60k+ репозиториев).

  • AGENTS.md = кросс-инструментальный контекст проекта. Напишите один раз; большинство агентов его читают.
  • CLAUDE.md = собственный файл памяти проекта у Claude Code (см. CLAUDE.md). Можно держать небольшой CLAUDE.md, который указывает на AGENTS.md, или поддерживать оба.
  • SKILL.md = отдельная спецификация Anthropic для упаковки переиспользуемых навыков с frontmatter и вложенными скриптами. Она дополняет AGENTS.md, а не конкурирует с ним — одна описывает проект, другая упаковывает переиспользуемую процедуру.

Хранение контекста в AGENTS.md означает, что в тот день, когда вы попробуете другого агента, правила вашего проекта поедут с вами бесплатно.

Минимальный AGENTS.md — пишите только то, что агент не может вывести сам

# AGENTS.md

## Setup
- Install: `pnpm install`
- Dev server: `pnpm dev` (port 3000)

## Build & test (run before every commit)
- Typecheck: `pnpm typecheck`
- Lint: `pnpm lint`
- Tests: `pnpm test` — all must pass

## Conventions
- TypeScript strict; no `any`.
- Components in `src/components`, one per file.
- Never edit files under `generated/` by hand.

## Constraints
- Do NOT run `git push` or open PRs without being asked.
- Secrets live in `.env.local`; never commit them.
Watch out
  • Помещайте в AGENTS.md только то, что агент не может обнаружить самостоятельно. Повторение очевидного тратит токены при каждом прогоне и хоронит правила, которые действительно важны. Составляйте файл вручную — автогенерируемые файлы контекста обычно превращаются в шум.

Решение, на одном дыхании​

  • Глубокая автономная работа с Claude и богатейшей экосистемой инструментов → Claude Code.
  • Открытый фирменный агент, привязанный к GPT, с сильным охватом платформ/Windows → Codex CLI.
  • Вы хотите свободно менять модели или запускать локальные веса → opencode или Cline.
  • Аккуратные, git-нативные правки с человеком в цикле → Aider.
  • Крупная ставка на контекст экосистемы Google → Gemini CLI (сначала подтвердите текущий доступ).

Что бы вы ни выбрали, храните контекст проекта в AGENTS.md и подтвердите выбор реальной задачей в собственном репозитории, прежде чем на нём остановиться.
