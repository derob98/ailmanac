Le Skill e i Plugin Più Utili

🟡Intermedio
🎯What you'll learn
  • ✅Capire la differenza tra una skill e un plugin, e come si installa ciascuno
  • ✅Aggiungere i marketplace ufficiale, community e demo e installare plugin da essi
  • ✅Scegliere i plugin ufficiali Anthropic giusti per controllo di versione, sicurezza, LSP e flussi git
  • ✅Selezionare le skill in base al lavoro da svolgere: sviluppo, scrittura, ricerca, dati, operations
  • ✅Trovarne altre dalle famose liste della community senza perderti tra fork a basso valore

Il modo più rapido per fare il salto di qualità con Claude Code è smettere di scrivere tutto a mano e installare capacità che altri hanno già perfezionato. Le skill danno a Claude competenze su richiesta; i plugin raggruppano skill, agenti, comandi, hook e server MCP in un'unica unità versionata e installabile. Questa pagina è una shortlist curata e pronta all'installazione: cosa fa ogni cosa, quando ricorrervi e il comando esatto per aggiungerla.

Se i meccanismi non ti sono chiari, leggi prima Skill e Plugin e Marketplace — questa pagina dà per scontato che tu sappia cos'è un SKILL.md.

✅
Ultima verifica: 2026-06-24.

I nomi dei marketplace, il catalogo dei plugin ufficiali e il conteggio delle stelle dei repository della community cambiano di frequente. I plugin eseguono codice con i tuoi privilegi: installa solo da fonti di cui ti fidi e verifica i comandi attuali nella documentazione ufficiale prima di farci affidamento.

Conferma confrontando con la fonte ufficiale.

Skill vs plugin: il modello in 20 secondi​

Una skill è una cartella con un SKILL.md (frontmatter description + istruzioni). Claude la carica solo quando è pertinente, oppure la invochi con /skill-name. Vive in ~/.claude/skills/ (personale), nel .claude/skills/ di un progetto, o all'interno di un plugin. Secondo la documentazione ufficiale, gli slash command personalizzati sono stati uniti alle skill: sia un .claude/commands/deploy.md sia un .claude/skills/deploy/SKILL.md creano entrambi /deploy.

Un plugin è un pacchetto distribuibile che può portare insieme skill, agenti, hook, server MCP e server LSP, con un manifest .claude-plugin/plugin.json. I plugin si ottengono da un marketplace — un catalogo che aggiungi una volta, e da cui poi installi. Le skill dei plugin hanno un namespace, ad esempio /commit-commands:commit.

Aggiungi un marketplace, installa un plugin​

Il marketplace ufficiale di Anthropic (claude-plugins-official) è disponibile automaticamente. Altri due vale la pena aggiungerli manualmente: il marketplace community curato e il marketplace demo di plugin di esempio.

Guided walkthrough1 of 4
  1. Esegui /plugin marketplace add anthropics/claude-plugins-community e /plugin marketplace add anthropics/claude-code. Aggiungere un marketplace registra soltanto il catalogo — non viene installato ancora nulla.
  2. Esegui /plugin per aprire il gestore (schede: Discover, Installed, Marketplaces, Errors). Il pannello di dettaglio mostra una stima del costo in contesto e un elenco 'Will install' di ogni comando, skill, agente, hook e server — controllalo prima di installare.
  3. Installa dalla CLI con /plugin install <name>@<marketplace>, oppure scegli uno scope (User / Project / Local) nell'interfaccia. Lo scope Project condivide il plugin con tutti nel repository tramite .claude/settings.json.
  4. Esegui /reload-plugins per recepire i nuovi plugin a metà sessione senza riavviare. Poi richiama la skill con il suo namespace, ad esempio /commit-commands:commit.
✦

Inizializza i tuoi marketplace (esegui dentro Claude Code)

/plugin marketplace add anthropics/claude-plugins-community
/plugin marketplace add anthropics/claude-code
/plugin
⚠️Watch out
  • •I plugin e i marketplace sono componenti altamente fidati — possono eseguire codice arbitrario con i tuoi privilegi. Aggiungi marketplace e installa plugin solo da fonti di cui ti fidi, e controlla prima l'elenco 'Will install'.
  • •Le skill dei plugin compaiono solo dopo /reload-plugins (o un riavvio). Se una skill continua a non comparire, svuota la cache: rm -rf ~/.claude/plugins/cache, poi reinstalla.

Plugin ufficiali da installare per primi​

Questi sono distribuiti nel marketplace ufficiale di Anthropic (claude-plugins-official) e sono il punto di partenza a più alto valore. Installa con /plugin install <name>@claude-plugins-official.

Intelligenza del codice (LSP) — fai vedere a Claude gli errori di tipo all'istante​

I plugin LSP collegano un language server a Claude Code, così dopo ogni modifica Claude ottiene diagnostiche automatiche (errori di tipo, import mancanti) e navigazione precisa (vai alla definizione, trova i riferimenti) invece di tirare a indovinare con grep. Richiedono il binario del language server corrispondente sulla tua macchina.

  • Quando usarli: qualsiasi sessione di programmazione non banale in un linguaggio tipizzato. Probabilmente è il singolo maggiore miglioramento di accuratezza per il lavoro di sviluppo.
  • Compromesso: server come rust-analyzer e pyright possono usare molta memoria su repository grandi; disattivali se diventa un problema.
✦

Installa l'LSP per il tuo stack

/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls

Flusso di lavoro Git e PR​

  • commit-commands — flussi git: mettere in stage, generare un messaggio di commit, fare push, aprire una PR. Usalo quando vuoi commit puliti e coerenti senza digitare tutta la cerimonia. Le skill hanno un namespace, ad esempio /commit-commands:commit.
  • pr-review-toolkit — agenti specializzati per la revisione delle pull request. Usalo prima del merge, o per avere un secondo paio d'occhi su un diff.
✦

Elementi essenziali Git/PR

/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins

Revisione di sicurezza automatica​

  • security-guidance — esamina ogni modifica che Claude fa alla ricerca di vulnerabilità comuni e istruisce Claude a correggerle nella stessa sessione. Usalo come rete di sicurezza permanente su qualsiasi progetto che scrive codice; integra (non sostituisce) una vera revisione.
✦

Controllo di sicurezza sempre attivo

/plugin install security-guidance@claude-plugins-official

Integrazioni esterne (MCP, preconfigurate)​

Queste includono un server MCP pronto all'uso, così salti la configurazione manuale. Ricorri a una di esse quando vuoi che Claude agisca direttamente in quel servizio.

  • Controllo di versione: github, gitlab
  • Gestione progetti: linear, notion, asana, atlassian (Jira/Confluence)
  • Infrastruttura: vercel, firebase, supabase
  • Design / comunicazione / monitoraggio: figma, slack, sentry
✦

Collega Claude ai tuoi strumenti

/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official

Toolkit fai-da-te e stili di output​

  • plugin-dev — un toolkit per creare i tuoi plugin. Usalo non appena la tua configurazione .claude/ vale la pena di essere condivisa.
  • agent-sdk-dev — strumenti per costruire sul Claude Agent SDK.
  • explanatory-output-style / learning-output-style — cambiano come risponde Claude: spiegazioni che insegnano man mano, o una modalità di apprendimento interattiva. Ottimi per l'onboarding o quando vuoi davvero capire le modifiche.

Skill ad alto valore per lavoro da svolgere​

Le skill brillano per attività ripetibili e cariche di contesto. Anthropic pubblica un set di riferimento pubblico nel repository anthropics/skills — gli stessi pattern che alimentano le capacità documentali e creative di Claude. Puoi copiare una cartella di skill in ~/.claude/skills/ (personale) o .claude/skills/ (progetto) e si caricherà alla sessione successiva.

LavoroSkill (da anthropics/skills)Quando ricorrervi
Documentidocx, pdf, pptx, xlsxGenera o modifica veri file Office/PDF — report, presentazioni, fogli di calcolo, moduli compilabili. (Implementazioni di riferimento con sorgente disponibile.)
Costruisci UIweb-artifacts-builder / artifacts-builderProduci artifact HTML/JS autonomi e mini-app.
Estendi Claudemcp-builderImposta lo scaffold di un nuovo server MCP quando ancora non esiste un'integrazione.
Brand e comunicazioneskill di stile brand-guidelines / internal-commsMantieni tono, naming e formattazione in linea col brand su tutti gli output.
✨Pro tip
  • →Non scaricare 30 skill tutte insieme in ~/.claude/skills. Le skill sono a divulgazione progressiva — finché non vengono attivate si caricano solo nome + descrizione — ma un mucchio gonfio e descritto in modo vago rende inaffidabile l'attivazione. Installane poche, scrivi descrizioni "Use when…" precise e cresci in modo deliberato.
  • →AILmanac offre pacchetti di skill già pronti che puoi inserire per provarli — vedi /docs/templates/skills.

Una skill che si auto-attiva che puoi incollare subito​

Per lavori intensivi di ricerca, una skill che àncora Claude a fonti aggiornate batte il dover rispiegare il tuo metodo ogni volta. Inserisci questo in ~/.claude/skills/research-brief/SKILL.md:

✦

~/.claude/skills/research-brief/SKILL.md

---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.

Dove vive la community (usa le liste curate, salta il rumore)​

Esistono decine di repository "awesome"; la maggior parte sono mirror generati automaticamente. Alcuni sono genuinamente curati e mantenuti — parti da quelli.

  • hesreallyhim/awesome-claude-code — l'indice curato a mano più conosciuto di skill, hook, slash-command, orchestratori di agenti, applicazioni e plugin. Decine di migliaia di stelle; riorganizzato attivamente.
  • ComposioHQ/awesome-claude-skills — un ampio catalogo curato di Claude Skills raggruppate per funzione (documenti, sviluppo, dati, marketing, sicurezza, automazione SaaS).
  • VoltAgent/awesome-agent-skills — oltre 1.400 skill della community + ufficiali (Anthropic, Vercel, Stripe, Cloudflare, Sentry, Figma e altre), portabili tra Claude Code, Cursor, Gemini CLI, Codex e altri.
  • anthropics/claude-plugins-community — il marketplace plugin ufficiale della community; plugin di terze parti che hanno superato la validazione automatica e lo screening di sicurezza di Anthropic.
⚠️Watch out
  • •I conteggi delle stelle e le rivendicazioni di essere 'i più completi' su questi repository sono volatili e a volte gonfiati da fork generati dall'IA. Tratta i numeri come approssimativi, giudica in base alla manutenzione e alla cura, e controlla sempre il contenuto effettivo di una skill o di un plugin prima di installarlo.
💡Key takeaways
  • ⭐Skill = una capacità su richiesta (SKILL.md); plugin = un pacchetto versionato (skill + agenti + hook + MCP/LSP) installato da un marketplace.
  • ⭐Aggiungi i marketplace con /plugin marketplace add <owner/repo>, installa con /plugin install <name>@<marketplace>, poi /reload-plugins.
  • ⭐Le vittorie più grandi del primo giorno: un plugin LSP per il tuo linguaggio, commit-commands, security-guidance e l'integrazione MCP per il servizio in cui vivi.
  • ⭐Le skill sono portabili e a divulgazione progressiva — cura un piccolo gruppo preciso invece di accumularne centinaia.
  • ⭐La fiducia non è facoltativa: i plugin e i marketplace eseguono codice con i tuoi privilegi. Controlla prima di installare.
