最实用的 Skills 与 Plugins

🟡进阶
🎯What you'll learn
  • ✅明白 skill 与 plugin 的区别，以及各自如何安装
  • ✅添加官方、社区和示例 marketplace，并从中安装 plugins
  • ✅为版本控制、安全、LSP 和 git 工作流挑选合适的 Anthropic 官方 plugins
  • ✅按要完成的任务来精选 skills：开发、写作、研究、数据、运维
  • ✅从知名社区清单中发现更多内容，又不被低质量的 fork 淹没

提升 Claude Code 最快的方式，就是别再事事手写，而是安装别人早已打磨好的能力。Skills 为 Claude 提供按需调用的专业知识；plugins 则把 skills、agents、commands、hooks 和 MCP 服务器打包成一个有版本、可安装的整体单元。本页是一份精选、可直接安装的精简清单——每样东西做什么、什么时候该用它，以及添加它的确切命令。

如果你对其中的机制还不太清楚，请先阅读 Skills 和 Plugins 与 Marketplaces——本页假设你已经知道 SKILL.md 是什么。

✅
最后验证时间：2026-06-24。

Marketplace 名称、官方 plugins 目录以及社区仓库的星标数会频繁变化。Plugins 会以你的权限运行代码——只从你信任的来源安装，并在依赖它们之前先在官方文档中确认当前命令。

请对照以下来源核实： 官方来源.

Skill 与 plugin：20 秒看懂的模型​

skill 是一个包含 SKILL.md（frontmatter 中的 description + 指令）的文件夹。Claude 只在相关时才加载它，或者你可以用 /skill-name 调用它。它存放在 ~/.claude/skills/（个人）、某个项目的 .claude/skills/ 中，或者在某个 plugin 内部。根据官方文档，自定义斜杠命令已经合并进了 skills——.claude/commands/deploy.md 和 .claude/skills/deploy/SKILL.md 都会创建出 /deploy。

plugin 是一个可分发的打包单元，它可以把 skills、agents、hooks、MCP 服务器和 LSP 服务器一起发布，并带有一个 .claude-plugin/plugin.json 清单文件。你从 marketplace 获取 plugins——marketplace 是一个目录，添加一次后即可从中安装。Plugin 的 skills 带有命名空间，例如 /commit-commands:commit。

Term shown.

添加一个 marketplace，安装一个 plugin​

Anthropic 官方 marketplace（claude-plugins-official）会自动可用。另外两个值得手动添加：精选的社区 marketplace 和示例 plugins 的示例 marketplace。

Guided walkthrough1 of 4
  1. 运行 /plugin marketplace add anthropics/claude-plugins-community 和 /plugin marketplace add anthropics/claude-code。添加 marketplace 只是注册目录——此时还什么都没安装。
  2. 运行 /plugin 打开管理器（选项卡：Discover、Installed、Marketplaces、Errors）。详情面板会显示一个上下文开销估算，以及一份「Will install」清单，列出每个 command、skill、agent、hook 和服务器——安装前先审查它。
  3. 在 CLI 中用 /plugin install <name>@<marketplace> 安装，或者在界面中选择一个作用域（User / Project / Local）。Project 作用域会通过 .claude/settings.json 把该 plugin 共享给仓库里的所有人。
  4. 运行 /reload-plugins，无需重启即可在会话中途加载新 plugins。然后调用带命名空间的 skill，例如 /commit-commands:commit。
✦

初始化你的 marketplace（在 Claude Code 内运行）

/plugin marketplace add anthropics/claude-plugins-community
/plugin marketplace add anthropics/claude-code
/plugin
⚠️Watch out
  • •Plugins 和 marketplaces 是高度受信任的组件——它们能以你的权限执行任意代码。只添加你信任来源的 marketplace 并从中安装 plugins，并先审查「Will install」清单。
  • •Plugin 的 skills 只有在 /reload-plugins（或重启）之后才会出现。如果某个 skill 仍然不显示，清理缓存：rm -rf ~/.claude/plugins/cache，然后重新安装。

值得最先安装的官方 plugins​

这些随 Anthropic 官方 marketplace（claude-plugins-official）一起发布，是信噪比最高的起点。用 /plugin install <name>@claude-plugins-official 安装。

代码智能（LSP）——让 Claude 即时看到类型错误​

LSP plugins 把一个语言服务器接入 Claude Code，于是每次编辑后 Claude 都能自动获得诊断信息（类型错误、缺失的导入）和精确的导航（跳转到定义、查找引用），而不是靠 grep 来猜。它们需要你的机器上装有对应的语言服务器二进制文件。

  • 何时使用： 在任何使用了类型化语言的非平凡编码会话中。这大概是开发工作中单项最大的准确性提升。
  • 取舍： 像 rust-analyzer 和 pyright 这样的服务器在大型仓库上可能占用大量内存；如果它拖慢了你，就禁用。
✦

为你的技术栈安装 LSP

/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls

Git 与 PR 工作流​

  • commit-commands —— git 工作流：暂存、生成提交信息、推送、打开 PR。当你想要干净、一致的提交而又不想敲那一套繁文缛节时使用。Skills 带命名空间，例如 /commit-commands:commit。
  • pr-review-toolkit —— 用于审查 pull request 的专门 agents。在合并之前使用，或者想给某个 diff 找第二双眼睛时使用。
✦

Git/PR 必备

/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins

自动安全审查​

  • security-guidance —— 对 Claude 做出的每一处改动审查常见漏洞，并指示 Claude 在同一会话中修复它们。把它当作任何写代码项目上的常驻安全网；它是对真正的审查的补充（而非替代）。
✦

始终开启的安全检查

/plugin install security-guidance@claude-plugins-official

外部集成（MCP，已预配置）​

这些打包了一个开箱即用的 MCP 服务器，让你省去手动配置。当你想让 Claude 直接在某个服务中操作时，选用其中一个。

  • 版本控制： github、gitlab
  • 项目管理： linear、notion、asana、atlassian（Jira/Confluence）
  • 基础设施： vercel、firebase、supabase
  • 设计 / 沟通 / 监控： figma、slack、sentry
✦

把 Claude 连接到你的工具

/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official

自建工具集与输出风格​

  • plugin-dev —— 一个用于编写你自己 plugins 的工具集。当你的 .claude/ 配置值得分享时就用它。
  • agent-sdk-dev —— 用于在 Claude Agent SDK 上构建的工具。
  • explanatory-output-style / learning-output-style —— 改变 Claude 如何回应：边做边讲的解释，或一种交互式学习模式。非常适合新手上手，或者当你真的想理解所做的改动时。

按要完成的任务划分的高价值 skills​

Skills 在可重复、需要大量上下文的任务上表现出色。Anthropic 在 anthropics/skills 仓库中发布了一套公开参考集——正是这些模式支撑着 Claude 的文档与创意能力。你可以把一个 skill 文件夹复制到 ~/.claude/skills/（个人）或 .claude/skills/（项目），它会在下次会话时加载。

任务Skill（来自 anthropics/skills）何时选用
文档docx、pdf、pptx、xlsx生成或编辑真实的 Office/PDF 文件——报告、演示文稿、电子表格、可填写表单。（源码可获取的参考实现。）
构建 UIweb-artifacts-builder / artifacts-builder生成自包含的 HTML/JS 制品和小型应用。
扩展 Claudemcp-builder当某个集成尚不存在时，搭建一个新的 MCP 服务器。
品牌与沟通brand-guidelines / internal-comms 风格类 skills让各类输出在语气、命名和格式上保持符合品牌。
✨Pro tip
  • →不要一次性把 30 个 skills 全堆进 ~/.claude/skills。Skills 采用渐进式披露——在被触发之前只加载名称 + 描述——但一堆臃肿、描述含糊的 skills 会让触发变得不可靠。先装几个，写出锐利的「Use when…」描述，再有意识地逐步扩充。
  • →AILmanac 提供了现成的 skill 包，你可以直接放进去试用——见 /docs/templates/skills。

一个你现在就能粘贴的自触发 skill​

对于研究密集型的工作，一个能让 Claude 立足于最新来源的 skill，胜过每次都重新解释你的方法。把下面这段放进 ~/.claude/skills/research-brief/SKILL.md：

✦

~/.claude/skills/research-brief/SKILL.md

---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.

社区在哪里（用精选清单，跳过噪音）​

有几十个「awesome」仓库；大多数是自动生成的镜像。少数是真正经过精选并维护的——从那里开始。

  • hesreallyhim/awesome-claude-code —— 最知名的人工精选索引，涵盖 skills、hooks、斜杠命令、agent 编排器、应用和 plugins。数万星标；持续在重新整理。
  • ComposioHQ/awesome-claude-skills —— 一个庞大的 Claude Skills 精选目录，按功能分组（文档、开发、数据、营销、安全、SaaS 自动化）。
  • VoltAgent/awesome-agent-skills —— 1,400+ 个社区 + 官方 skills（Anthropic、Vercel、Stripe、Cloudflare、Sentry、Figma 等等），可在 Claude Code、Cursor、Gemini CLI、Codex 等之间通用。
  • anthropics/claude-plugins-community —— 官方社区 plugin marketplace；通过了 Anthropic 自动化校验与安全筛查的第三方 plugins。
⚠️Watch out
  • •这些仓库上的星标数和「最全面」之类的说法都很不稳定，有时还被 AI 生成的 fork 夸大。把这些数字当作粗略参考，按维护与精选程度来判断，并且在安装前务必审查一个 skill 或 plugin 的实际内容。
💡Key takeaways
  • ⭐Skill = 一项按需调用的能力（SKILL.md）；plugin = 一个有版本的打包单元（skills + agents + hooks + MCP/LSP），从 marketplace 安装。
  • ⭐用 /plugin marketplace add <owner/repo> 添加 marketplace，用 /plugin install <name>@<marketplace> 安装，然后 /reload-plugins。
  • ⭐上手第一天收益最大的：为你的语言装一个 LSP plugin、commit-commands、security-guidance，以及你天天用的那个服务的 MCP 集成。
  • ⭐Skills 可移植且采用渐进式披露——精选少而精的几个，而不是囤积成百上千个。
  • ⭐信任不可妥协：plugins 和 marketplaces 会以你的权限运行代码。安装前先审查。
