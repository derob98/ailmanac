提示词模式库

所有级别

一套可复用的提示词模式工具箱。每张卡片都会告诉你它是什么、什么时候用，并给你一个可复制粘贴的模板。它们可以叠加使用——尽管自由组合。第一次来？先从提示词基础开始。

What you'll learn
  • 认识 11 个核心提示词模式以及每个模式各自的适用场景
  • 为任何任务抓取一个可复制粘贴的模板——零样本、少样本、思维链等等
  • 把多个模式（角色 + 示例 + 输出约束 + 允许放弃作答）组合成一条强有力的提示词
  • 用情境锚定（grounding）减少幻觉，用元提示（meta-prompting）修复薄弱的提示词

零样本（直接、清晰地提问）​

何时用： 简单、常见的任务。为什么： 最省力，往往就够了。

零样本摘要

Summarize the text below in 5 bullets for a busy manager.
TEXT: """{paste}"""

少样本（给出示例）​

何时用： 你需要特定的格式或风格。为什么： 示例比描述教得更快。参见把少样本用对。

少样本情感分类器

Classify the sentiment as positive/negative/neutral.
Review: "Loved it, shipped fast" → positive
Review: "Broke on day one" → negative
Review: "{new review}" →

思维链（一步一步地思考）​

何时用： 数学、逻辑、多步推理。为什么： 在作答前先推理能提高准确率。

思维链

Work through this step by step, then give the final answer on its own line.
PROBLEM: {problem}

任务拆解（把它分解）​

何时用： 庞大、模糊的任务。为什么： 更小的子任务更容易、也更可靠。

任务拆解

First, list the sub-tasks needed to {goal}. Then do them one at a time,
showing each result before moving on.

角色提示（赋予一个人设）​

何时用： 你想要某个特定的视角或标准。为什么： 能引导出相关的行为。

角色提示

You are a meticulous senior copy editor. Edit the text for clarity and concision
without changing meaning. Show a bullet list of the changes you made.

输出格式约束​

何时用： 结果要喂给另一个工具，或者你想要一致性。为什么： 可预测的形态才好用。

输出格式约束

Reply with ONLY valid JSON matching: {"title": string, "tags": string[]}.
No prose, no markdown fences.

自洽 / 自我批评​

何时用： 正确性很重要。为什么： 第二遍检查能抓住第一遍的错误。

自我批评

Answer the question. Then, in a section called "Check", critique your own answer
for errors or missing cases and revise if needed.

情境锚定（只根据提供的上下文作答）​

何时用： 事实很重要——针对你文档的问答、RAG，以及任何错误答案比没有答案更糟的场景。为什么： 对抗幻觉最有力的杠杆，就是禁止模型凭空编造，并明确允许它放弃作答。

基于上下文的问答

Answer the QUESTION using ONLY the CONTEXT below. Quote the exact sentence(s)
you relied on. If the context does not contain the answer, reply exactly:
"Not found in the provided context." Do not use outside knowledge.

CONTEXT: """{paste your sources}"""
QUESTION: {question}

元提示（让 AI 改进你的提示词）​

何时用： 你的提示词表现不佳。为什么： 模型很擅长发现自己缺了什么。

元提示

Here's my prompt: "{prompt}". Ask me the 3 questions that would most improve it,
then rewrite it incorporating sensible defaults.

迭代式打磨​

何时用： 几乎总是。为什么： 第一次的输出是草稿，不是终点。

迭代式打磨

{after a result} Shorter and more concrete. Keep the second paragraph. Add one
example. Drop the marketing tone.

:::tip 把它们组合起来 一条强有力的真实提示词通常是：一个角色 + 一个示例 + 一个输出约束 + 允许它说“我不知道。” 把这些卡片层叠起来。 :::

如何层叠这些卡片​

Guided walkthrough1 of 5
  1. 赋予你想要的人设或标准——例如一位一丝不苟的资深文字编辑——以引导出相关的行为。
  2. 给出一两个少样本示例，让模型比靠描述更快地学会你的格式或风格。
  3. 钉死形态：只要有效的 JSON、5 个要点、不要 markdown 代码围栏——下一步需要什么就约束什么。
  4. 让模型可以说“我不知道”或“在提供的上下文中找不到”，而不是凭空编造一个答案。
  5. 把第一次的输出当作草稿。要求更短、更具体、保留这一段、去掉那种腔调——反复打磨直到满意。

按 Enter 或空格键翻转卡片。使用左右方向键在卡片之间切换。已显示术语。

Key takeaways
  • 模式是可复用的构建块——按任务挑选：简单用零样本、要格式用少样本、要推理用思维链、大任务用任务拆解
  • 情境锚定（只根据上下文作答 + 允许放弃作答）是对抗幻觉最强的防线
  • 输出格式约束让结果可预测、可直接喂给工具
  • 卡片可以叠加：一条强有力的提示词通常是角色 + 示例 + 输出约束 + 允许说“我不知道”
  • 第一次的输出是草稿——迭代式打磨，并在提示词表现不佳时使用元提示
