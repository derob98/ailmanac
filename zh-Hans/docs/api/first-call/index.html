你的第一次 API 调用

🟢入门
✅
最后验证时间：2026-06-20。

下面的 API 形态是稳定的，但模型 ID 和 SDK 版本会变动。参见当前模型表。

请对照以下来源核实： 官方来源.

API 让 你的 程序与 Claude 通信。读完这一课，你将运行一次真实的请求，并准确理解其中每个部分的作用。

🎯What you'll learn
  • ✅创建一个 API 密钥，并把它安全地保存为环境变量
  • ✅为 Python 或 TypeScript 安装官方 SDK（或直接使用 cURL）
  • ✅发送你的第一个 messages 请求并读取 Claude 的回复
  • ✅理解四个核心字段：model、max_tokens、messages 和 system

三个步骤走完整段旅程​

下面的所有内容都归结为一个简单的流程。一边阅读一边把这张地图记在心里。

Guided walkthrough1 of 3
  1. 在 Anthropic Console 中创建一个密钥，并把它导出为环境变量，这样它就永远不会出现在你的代码里。
  2. 为 Python 添加 anthropic 包，或为 TypeScript 添加 @anthropic-ai/sdk。使用 cURL 则什么都不用安装。
  3. 把一个消息列表发给模型，并打印它返回的内容。

1. 获取 API 密钥​

在 Anthropic Console 中创建一个。然后将它设置为环境变量，这样它就永远不会出现在你的代码里：

✦

导出你的 API 密钥

export ANTHROPIC_API_KEY="sk-ant-..."
⚠️Watch out
  • •切勿提交你的密钥。把密钥保存在环境变量或密钥管理器中，永远不要放进源代码管理。参见安全页面 (/docs/security)。

2. 安装 SDK​

选择你的语言。SDK 会自动从你刚刚设置的环境变量中读取密钥。

  • Python
  • TypeScript
  • cURL
✦

安装（Python）

pip install anthropic
✦

安装（TypeScript）

npm install @anthropic-ai/sdk

无需安装任何东西——你只需要 curl。

3. 发起调用​

每个请求都是一个 messages 列表。模型以 content 回复。运行你所用语言的代码片段，你就会看到 Claude 回答这个问题。

  • Python
  • TypeScript
  • cURL
import anthropic

client = anthropic.Anthropic()  # reads ANTHROPIC_API_KEY from the environment

message = client.messages.create(
    model="claude-sonnet-5",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "In one sentence, what is an API?"}
    ],
)

print(message.content[0].text)
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic(); // reads ANTHROPIC_API_KEY from the environment

const message = await client.messages.create({
  model: "claude-sonnet-5",
  max_tokens: 1024,
  messages: [
    { role: "user", content: "In one sentence, what is an API?" },
  ],
});

console.log(message.content[0].text);
curl https://api.anthropic.com/v1/messages \
  --header "x-api-key: $ANTHROPIC_API_KEY" \
  --header "anthropic-version: 2023-06-01" \
  --header "content-type: application/json" \
  --data '{
    "model": "claude-sonnet-5",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "In one sentence, what is an API?"}
    ]
  }'

刚才发生了什么​

你发送了四条信息。下面是每一条所控制的内容：

  • model — 使用哪个 Claude。不要盲目硬编码；参见选择模型。
  • max_tokens — 回复长度的上限（以 token 计）。它 不 设置上下文窗口。
  • messages — 目前为止的对话。该 API 是 无状态的：要继续一次对话，每次都要把全部历史发回去。
  • system（可选）— 一条顶层指令，为本次调用设定 Claude 的角色。

  • →因为该 API 是无状态的，所以多轮对话其实就是一个不断增长的 messages 数组：追加每一条回复，并在下一次调用时重新发送完整列表。

💡Key takeaways
  • ⭐三个步骤：获取密钥、安装 SDK、发送一个 messages 请求。
  • ⭐把密钥保存在 ANTHROPIC_API_KEY 中，这样 SDK 会自动读取它，而它永远不会进入你的代码。
  • ⭐一个请求就是一个消息列表；回复会在 content 中返回。
  • ⭐该 API 是无状态的：要继续一次对话，需重新发送完整的对话历史。
  • ⭐model、max_tokens、messages 和 system 是你最先会用到的字段。
