学习路径（选择你的赛道）

⚪所有级别

挑一个最像你的赛道。每条都是有序的路径——按顺序完成各个步骤，已经会的就跳过。时间只是大致估算。

🎯What you'll learn
  • ✅找到唯一符合你目标的赛道——新手、知识工作者、构建者，或团队负责人
  • ✅沿着清晰、有序的阅读路径前进，而不是凭猜测从哪里开始
  • ✅大致了解每条赛道需要多少时间（约 45 分钟到约半天）
  • ✅如果没有一条赛道明显适合你，该从哪里入手

:::tip 不确定哪条赛道是你的？ 按你的目标来匹配，而不是按职位头衔。想要不写代码就拿到结果？好奇的新手。想让真正的工作更快？知识工作者。想定制 Claude Code 或 API？构建者。想把 AI 推广给其他人？团队负责人。 :::

🌱 好奇的新手 · 约 45 分钟​

你想把 AI 用好，但不想变成程序员。

Guided walkthrough1 of 5
  1. 入门级。五分钟内从 Claude 拿到一个真实结果。/docs/start-here/your-first-5-minutes
  2. 入门级。为你正在做的事挑选合适的模型。/docs/start-here/which-claude
  3. 入门级。让回答变得更好的核心动作。/docs/prompting/basics
  4. 入门级。用通俗语言建立一个关于 Claude 如何工作的心智模型。/docs/foundations
  5. 入门级。把它应用到每天的任务和小胜利中。/docs/playbooks/productivity

💼 知识工作者 · 约 2 小时​

你想让 AI 加速你真正的工作——写作、调研、分析。

Guided walkthrough1 of 5
  1. 先完成上面约 45 分钟的赛道，然后继续下面的内容。
  2. 适合所有级别。可以直接搬进真实工作中的可复用模式。/docs/prompting
  3. 进阶级。产出你真正能信任的结果。/docs/foundations
  4. 在写作、调研或数据分析中选择。/docs/playbooks
  5. 持久化的工作空间（/docs/claude-app）和可反复使用的模板（/docs/templates）。

🛠️ 构建者 / 开发者 · 约半天​

你写代码，想要定制 Claude Code 并基于 API 进行开发。

Guided walkthrough1 of 6
  1. 入门级。这个 CLI 是什么以及它为什么重要。/docs/claude-code/what-is-claude-code
  2. 入门级。把你的项目教给 Claude，并在它行动前先做计划。/docs/claude-code
  3. 进阶级。控制 Claude 能做什么以及它如何行事。/docs/claude-code
  4. 高级。从一次调用走到工具使用，再到完整的智能体。从 /docs/api/first-call 开始，然后是 /docs/api
  5. 进阶级。在一个真实项目上把这一切融会贯通。/docs/walkthroughs
  6. 高级。当你真正开始构建后，要安全地交付。/docs/security

👥 团队负责人 · 约 3 小时​

你正在向团队推广 AI，关心一致性、安全性和成本。

Guided walkthrough1 of 5
  1. 在继续这里之前，先覆盖上面的构建者基础。
  2. 高级。交付一套共享工具集，让团队以同样的方式工作。/docs/claude-code
  3. 进阶级。在速度、质量和花费之间取得平衡。/docs/foundations 和 /docs/api/choosing-a-model
  4. 高级。让自主运行在规模化时依然安全。/docs/security
  5. 适合所有级别。设定团队共同遵守的规范。/docs/security

:::note 赛道是层层递进的 知识工作者赛道以整条新手赛道作为起点。团队负责人赛道以构建者的核心（步骤 1–3）作为起点。如果你完成了一条赛道，那么上一层的赛道大部分其实已经做完了。 :::

💡Key takeaways
  • ⭐四条赛道，一个原则：按目标选择，然后按顺序完成各个步骤。
  • ⭐时间从约 45 分钟（新手）到约半天（构建者）不等。
  • ⭐赛道是嵌套的——知识工作者建立在新手之上；团队负责人建立在构建者之上。
  • ⭐拿不准时，就从「你的最初 5 分钟」开始——它在任何赛道上都有回报。

不确定哪个适合你？从 你的最初 5 分钟 开始——无论你最终走向哪里，它都用得上。另外读一读 难度等级与时效性的运作方式，这样徽章和日期就都说得通了。
