## The major players (by archetype, not ranking)

Think in **archetypes** — the rankings shuffle, the archetypes are stable:

- **Anthropic — Claude** · frontier *closed* model. Known for coding, agentic/tool use, long-context reasoning, and a safety-first posture. (The whole rest of this site goes deep here.)
- **OpenAI — ChatGPT / GPT** · frontier *closed* model with the largest consumer ecosystem and a very broad, multimodal feature set.
- **Google — Gemini** · frontier *closed* model, deeply wired into Google's products and data, with very large context windows.
- **Meta — Llama** · *open-weight*: you can download and self-host it. The backbone of much of the local/open ecosystem.
- **Mistral** · *open-weight* + API, European, efficiency-focused.
- **xAI — Grok** · *closed*, integrated with X, leaning on real-time signal.
- **DeepSeek / Qwen (Alibaba)** · strong *open-weight* models known for capability-per-cost; popular for self-hosting.

## How to choose a model for a task

<Steps items={[
  {title: "Write down the task + the hard constraints", body: "What must it do, and what's non-negotiable: data privacy (can data leave your network?), budget, latency, max context, regulated domain, on-device requirement."},
  {title: "Closed-frontier or open-weight?", body: "Need the absolute top capability with zero infra? A closed frontier model (Claude / GPT / Gemini). Need privacy, self-hosting, heavy customization, or lowest run-cost at scale? An open-weight model (Llama / Mistral / Qwen / DeepSeek) you host."},
  {title: "Shortlist 2-3 candidates", body: "Pick a couple that plausibly fit the constraints — don't agonize; you'll test them."},
  {title: "Build a tiny eval on YOUR data", body: "10-50 real cases with known-good answers. Run each candidate. This beats every public leaderboard for your use case."},
  {title: "Weigh cost & latency at your real volume", body: "Multiply per-call cost and speed by your expected traffic. The 'best' model can be the wrong one if it's 10x the price for a 2% quality gain you don't need."},
  {title: "Pick — and keep the eval", body: "Re-run it when a new model drops. Switching is cheap when you have an eval; it's a coin-flip when you don't."},
]} />

## What transfers across (almost) every model

The reason learning Claude well pays off everywhere — these skills are portable:

- **Prompting structure** — clear role, explicit task, examples, output format → [Prompting Basics](/docs/prompting/basics)
- **Context engineering** — deciding what goes in the window → [Foundations](/docs/foundations)
- **Tool use / function calling** — the pattern is near-identical across providers → [Tool Use](/docs/api/tool-use)
- **Retrieval (RAG)** — ground answers in your data → [RAG](/docs/foundations/rag)
- **Evals** — measure quality so you can compare models honestly → [Evals](/docs/power-user/evals)
- **Structured output** — make results machine-readable → [Structured Output](/docs/api/structured-output)

What does NOT transfer cleanly: exact API shapes, tool-calling JSON details, token limits, pricing, and the safety/refusal behavior — always re-check per provider.
