Building chat-like experiences on the API comes down to two practical realities: **stream** so users see output immediately, and **manage history yourself** because the API is stateless. Master both and your chat UX feels fast and remembers everything.

## Streaming

Without streaming, the user waits for the whole reply. With streaming, tokens arrive as they're generated — far better perceived speed.

<Callout type="tip" items={["Use the SDK's streaming helper instead of parsing raw events by hand — it manages the event lifecycle for you."]} />

<Tabs groupId="lang">
<TabItem value="python" label="Python">

```python
with client.messages.stream(
    model="claude-sonnet-5", max_tokens=1024,
    messages=[{"role": "user", "content": "Explain RAG in two sentences."}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
```

</TabItem>
<TabItem value="ts" label="TypeScript">

```ts
const stream = client.messages.stream({
  model: "claude-sonnet-5", max_tokens: 1024,
  messages: [{ role: "user", content: "Explain RAG in two sentences." }],
});
for await (const event of stream) {
  if (event.type === "content_block_delta") process.stdout.write(event.delta.text ?? "");
}
```

</TabItem>

## Multi-turn: you hold the history

The API has **no memory** between calls ([why](/docs/foundations/tokens-and-context)). To continue a conversation, send the **whole prior exchange** back each time.

<Steps items={[{title: "Start the conversation", body: "Send the first user message in the messages list."}, {title: "Capture the reply", body: "Read the assistant's response text from the result."}, {title: "Append both turns", body: "Add the assistant reply and the next user message to the same messages list."}, {title: "Resend the full list", body: "Send the complete messages list again — that is how Claude 'remembers' the exchange."}]} />

```python
messages = [{"role": "user", "content": "Hi, I'm planning a trip."}]
# ... get assistant reply, then append both turns:
messages.append({"role": "assistant", "content": assistant_text})
messages.append({"role": "user", "content": "Make it 3 days."})
# send the full `messages` list again
```

## Long conversations fill the window

As history grows it eats the [context window](/docs/foundations/tokens-and-context) and cost rises. Strategies to keep it in check:

- **Summarize/compact** older turns into a short recap you carry forward.
- **Trim** irrelevant earlier turns.
- Pair with **[prompt caching](/docs/api/prompt-caching)** to avoid re-paying for a stable prefix.

<Callout type="warning" items={["Every call resends the entire history — long conversations cost more and can eventually exceed the context window if you never compact or trim."]} />
