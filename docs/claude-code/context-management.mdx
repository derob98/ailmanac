## The big idea

Every session has a finite [context window](/docs/foundations/tokens-and-context). As it fills, older content gets squeezed and Claude can start to "drift." Managing context deliberately is what keeps long sessions sharp.

## The two essential commands

Two commands cover almost every situation. The trick is matching the command to your intent.

<Flashcards cards={[{front: "/clear", back: "Wipe the conversation and start fresh. Use it when you switch to an UNRELATED task. The cheapest way to regain a clean, focused context."}, {front: "/compact", back: "Summarize the conversation so far into a compact form — keeps the gist while freeing space. Use it when you want to CONTINUE the same task but the window is getting full."}]} />

Decide with one question: is this a new task or the same one?

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]
  A -->|Same task, context full| C[/compact]
```

## Auto-compaction

When context approaches the limit, Claude Code can **auto-compact** so work continues without a hard stop. It's a safety net — but proactively `/clear`-ing between tasks gives better results than relying on it.

<Callout type="tip" items={["Treat auto-compaction as a backstop, not a strategy. Clearing between unrelated tasks keeps context cleaner than letting the window fill until auto-compaction kicks in."]} />

## Techniques to stay effective

Beyond the two commands, four habits keep your context lean from the start.

<Steps items={[{title: "One task per session", body: "Don't let an afternoon of unrelated work pile into one context."}, {title: "Delegate heavy reading to subagents", body: "Hand large reads to subagents — they burn THEIR context and return only the conclusion. See /docs/claude-code/subagents."}, {title: "Reference, don't paste", body: "Point Claude at files with @path instead of pasting large blobs, and keep CLAUDE.md lean (see /docs/claude-code/claude-md)."}, {title: "Watch the budget", body: "A statusline showing context use tells you when to compact. See /docs/claude-code/statusline."}]} />

These map to live docs: [subagents](/docs/claude-code/subagents), [CLAUDE.md](/docs/claude-code/claude-md), and [statusline](/docs/claude-code/statusline).

:::tip Symptom → fix
If Claude starts forgetting an earlier instruction or repeating itself, the window is likely full. `/compact` (same task) or `/clear` (new task) usually fixes it instantly. More in [Troubleshooting](/docs/contribute/troubleshooting).
:::
