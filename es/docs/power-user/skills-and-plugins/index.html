Las Skills y Plugins Más Útiles

Intermedio
What you'll learn
  • Conocer la diferencia entre una skill y un plugin, y cómo se instala cada uno
  • Añadir los marketplaces oficial, de la comunidad y de demostración, e instalar plugins desde ellos
  • Elegir los plugins oficiales de Anthropic adecuados para control de código fuente, seguridad, LSP y flujos de trabajo de git
  • Curar skills por trabajo-a-realizar: desarrollo, escritura, investigación, datos, operaciones
  • Encontrar más en las famosas listas de la comunidad sin enterrarte en forks de baja señal

La forma más rápida de mejorar Claude Code es dejar de escribir todo a mano e instalar capacidades que otras personas ya perfeccionaron. Las skills le dan a Claude experiencia bajo demanda; los plugins agrupan skills, agentes, comandos, hooks y servidores MCP en una sola unidad versionada e instalable. Esta página es una lista corta curada y lista para instalar: qué hace cada cosa, cuándo recurrir a ella y el comando exacto para añadirla.

Si no tienes claros los mecanismos, lee primero Skills y Plugins y Marketplaces — esta página asume que sabes qué es un SKILL.md.

Última verificación: 2026-06-24.

Los nombres de los marketplaces, el catálogo de plugins oficiales y los recuentos de estrellas de los repositorios de la comunidad cambian con frecuencia. Los plugins ejecutan código con tus privilegios — instala solo desde fuentes en las que confíes y confirma los comandos actuales en la documentación oficial antes de depender de ellos.

Confirma con la fuente oficial.

Skill vs plugin: el modelo en 20 segundos​

Una skill es una carpeta con un SKILL.md (frontmatter description + instrucciones). Claude la carga solo cuando es relevante, o tú la invocas con /skill-name. Vive en ~/.claude/skills/ (personal), en el .claude/skills/ de un proyecto, o dentro de un plugin. Según la documentación oficial, los comandos de barra personalizados se han fusionado con las skills — tanto un .claude/commands/deploy.md como un .claude/skills/deploy/SKILL.md crean /deploy.

Un plugin es un paquete distribuible que puede incluir skills, agentes, hooks, servidores MCP y servidores LSP juntos, con un manifiesto .claude-plugin/plugin.json. Obtienes los plugins desde un marketplace — un catálogo que añades una vez y luego usas para instalar. Las skills de los plugins llevan espacio de nombres, por ejemplo /commit-commands:commit.

Añade un marketplace, instala un plugin​

El marketplace oficial de Anthropic (claude-plugins-official) está disponible automáticamente. Vale la pena añadir manualmente otros dos: el marketplace curado de la comunidad y el marketplace de demostración con plugins de ejemplo.

Guided walkthrough1 of 4
  1. Ejecuta /plugin marketplace add anthropics/claude-plugins-community y /plugin marketplace add anthropics/claude-code. Añadir un marketplace solo registra el catálogo — todavía no se instala nada.
  2. Ejecuta /plugin para abrir el gestor (pestañas: Discover, Installed, Marketplaces, Errors). El panel de detalle muestra una estimación del coste de contexto y una lista 'Will install' de cada comando, skill, agente, hook y servidor — revísala antes de instalar.
  3. Instala desde la CLI con /plugin install <name>@<marketplace>, o elige un ámbito (User / Project / Local) en la interfaz. El ámbito Project comparte el plugin con todo el equipo del repositorio a través de .claude/settings.json.
  4. Ejecuta /reload-plugins para incorporar nuevos plugins a mitad de sesión sin reiniciar. Luego llama a la skill con su espacio de nombres, por ejemplo /commit-commands:commit.

Inicializa tus marketplaces (ejecutar dentro de Claude Code)

/plugin marketplace add anthropics/claude-plugins-community
/plugin marketplace add anthropics/claude-code
/plugin
Watch out
  • Los plugins y marketplaces son componentes de alta confianza — pueden ejecutar código arbitrario con tus privilegios. Añade marketplaces e instala plugins solo desde fuentes en las que confíes, y revisa primero la lista 'Will install'.
  • Las skills de los plugins solo aparecen después de /reload-plugins (o un reinicio). Si una skill aún no aparece, limpia la caché: rm -rf ~/.claude/plugins/cache, y luego reinstala.

Plugins oficiales que vale la pena instalar primero​

Estos vienen en el marketplace oficial de Anthropic (claude-plugins-official) y son el punto de partida con mayor señal. Instálalos con /plugin install <name>@claude-plugins-official.

Inteligencia de código (LSP) — haz que Claude vea los errores de tipo al instante​

Los plugins LSP conectan un servidor de lenguaje a Claude Code, de modo que tras cada edición Claude recibe diagnósticos automáticos (errores de tipo, importaciones faltantes) y navegación precisa (ir a la definición, buscar referencias) en lugar de conjeturas con grep. Requieren el binario del servidor de lenguaje correspondiente en tu máquina.

  • Cuándo usarlo: cualquier sesión de codificación no trivial en un lenguaje tipado. Probablemente sea la mayor mejora de precisión para el trabajo de desarrollo.
  • Compromiso: servidores como rust-analyzer y pyright pueden consumir mucha memoria en repositorios grandes; desactívalos si te molesta.

Instala LSP para tu stack

/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls

Flujo de trabajo de Git y PR​

  • commit-commands — flujos de trabajo de git: preparar el área, generar un mensaje de commit, hacer push, abrir un PR. Úsalo cuando quieras commits limpios y consistentes sin escribir toda la ceremonia. Las skills llevan espacio de nombres, por ejemplo /commit-commands:commit.
  • pr-review-toolkit — agentes especializados para revisar pull requests. Úsalo antes de fusionar, o para conseguir un segundo par de ojos sobre un diff.

Lo esencial de Git/PR

/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins

Revisión de seguridad automática​

  • security-guidance — revisa cada cambio que hace Claude en busca de vulnerabilidades comunes e indica a Claude que las corrija en la misma sesión. Úsalo como red de seguridad permanente en cualquier proyecto donde escribas código; complementa (no reemplaza) una revisión real.

Pasada de seguridad siempre activa

/plugin install security-guidance@claude-plugins-official

Integraciones externas (MCP, preconfiguradas)​

Estas incluyen un servidor MCP listo para usar, así que te saltas la configuración manual. Recurre a una cuando quieras que Claude actúe directamente en ese servicio.

  • Control de código fuente: github, gitlab
  • Gestión de proyectos: linear, notion, asana, atlassian (Jira/Confluence)
  • Infraestructura: vercel, firebase, supabase
  • Diseño / comunicación / monitoreo: figma, slack, sentry

Conecta Claude a tus herramientas

/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official

Toolkits propios y estilos de salida​

  • plugin-dev — un toolkit para crear tus propios plugins. Úsalo en cuanto tu configuración de .claude/ valga la pena compartir.
  • agent-sdk-dev — herramientas para desarrollar sobre el Claude Agent SDK.
  • explanatory-output-style / learning-output-style — cambian cómo responde Claude: explicaciones que enseñan sobre la marcha, o un modo de aprendizaje interactivo. Estupendos para la incorporación o cuando quieres entender de verdad los cambios.

Skills de alto valor por trabajo-a-realizar​

Las skills brillan en tareas repetibles y cargadas de contexto. Anthropic publica un conjunto de referencia público en el repositorio anthropics/skills — los mismos patrones que impulsan las capacidades de documentos y creativas de Claude. Puedes copiar una carpeta de skill en ~/.claude/skills/ (personal) o .claude/skills/ (proyecto) y se cargará en la siguiente sesión.

TrabajoSkill (de anthropics/skills)Cuándo recurrir a ella
Documentosdocx, pdf, pptx, xlsxGenerar o editar archivos reales de Office/PDF — informes, presentaciones, hojas de cálculo, formularios rellenables. (Implementaciones de referencia con código disponible.)
Crear interfacesweb-artifacts-builder / artifacts-builderProducir artefactos HTML/JS autónomos y mini-apps.
Extender Claudemcp-builderGenerar el andamiaje de un nuevo servidor MCP cuando aún no existe ninguna integración.
Marca y comunicaciónskills de estilo brand-guidelines / internal-commsMantener el tono, la nomenclatura y el formato acordes a la marca en todas las salidas.
Pro tip
  • No vuelques 30 skills en ~/.claude/skills de golpe. Las skills son de divulgación progresiva — solo se cargan el nombre + la descripción hasta que se disparan — pero un montón inflado y vagamente descrito hace que el disparo sea poco fiable. Instala unas pocas, escribe descripciones nítidas de "Úsala cuando…" y crece de forma deliberada.
  • AILmanac incluye paquetes de skills listos para que los pruebes — consulta /docs/templates/skills.

Una skill que se dispara sola y puedes pegar ahora mismo​

Para el trabajo intensivo en investigación, una skill que ancle a Claude en fuentes frescas supera el tener que reexplicar tu método cada vez. Coloca esto en ~/.claude/skills/research-brief/SKILL.md:

~/.claude/skills/research-brief/SKILL.md

---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.

Dónde vive la comunidad (usa las listas curadas, evita el ruido)​

Hay docenas de repositorios "awesome"; la mayoría son réplicas autogeneradas. Unos pocos están genuinamente curados y mantenidos — empieza por ahí.

  • hesreallyhim/awesome-claude-code — el índice curado a mano más conocido de skills, hooks, comandos de barra, orquestadores de agentes, aplicaciones y plugins. Decenas de miles de estrellas; reorganizado activamente.
  • ComposioHQ/awesome-claude-skills — un amplio catálogo curado de Claude Skills agrupadas por función (documentos, desarrollo, datos, marketing, seguridad, automatización SaaS).
  • VoltAgent/awesome-agent-skills — más de 1.400 skills comunitarias + oficiales (Anthropic, Vercel, Stripe, Cloudflare, Sentry, Figma y más), portables entre Claude Code, Cursor, Gemini CLI, Codex y otros.
  • anthropics/claude-plugins-community — el marketplace oficial de plugins de la comunidad; plugins de terceros que pasaron la validación automatizada y el cribado de seguridad de Anthropic.
Watch out
  • Los recuentos de estrellas y las afirmaciones de 'el más completo' en estos repositorios son volátiles y a veces están inflados por forks generados por IA. Trata los números como aproximados, juzga por el mantenimiento y la curación, y revisa siempre el contenido real de una skill o plugin antes de instalarlo.
Key takeaways
  • Skill = una capacidad bajo demanda (SKILL.md); plugin = un paquete versionado (skills + agentes + hooks + MCP/LSP) instalado desde un marketplace.
  • Añade marketplaces con /plugin marketplace add <owner/repo>, instala con /plugin install <name>@<marketplace>, luego /reload-plugins.
  • Las mayores victorias del primer día: un plugin LSP para tu lenguaje, commit-commands, security-guidance y la integración MCP del servicio en el que vives.
  • Las skills son portables y de divulgación progresiva — cura unas pocas nítidas en lugar de acumular cientos.
  • La confianza no es opcional: los plugins y marketplaces ejecutan código con tus privilegios. Revisa antes de instalar.
